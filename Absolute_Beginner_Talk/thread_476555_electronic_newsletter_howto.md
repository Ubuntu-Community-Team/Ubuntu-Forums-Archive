### Post by arvevans on 2007-06-17
I'm not sure exactly what you mean by "publish an electronic newsletter"?  There are a number of possible ways to "publish" via the Internet:
[LIST]
[*]Make an email group by adding a group name to your email contacts list and them periodically sending your publication to this list.
[*]Publishing your content on a web page and then notifying your subscribers via an email that includes the web page URL.
[*]Using a Weg Log (Blog) to add sequential content to a private or public blog server.
[/LIST]

There are also probably purpose-written software applications that do much the same as the methods listed above, but in some proprietary manner.

If your interest is in Blog site updates, but want to do it directly on your own computer, then there are several Ubuntu programs that can help.  One that comes to mind is "drivel".  This is listed in the Synaptic Package Manager grouping.  Either scroll down to it or just do a search in the package manager for "drivel".

If this doesn't help, then please ask again on this thread and give a bit more detail about what you want to do.

### Post by arvevans on 2007-06-17
After re-reading your initial post, I see that you did specify email as the delivery mechanism.  

To further answer questions about formatting and delivery of an email newsletter:
[INDENT]For email you have the choice of using a web-based email handler, or use one that allows you to format and send email directly from your own computer.  Sending email directly from your own computer involves configuration of any of several email programs (evolution and Mozilla-Thunderbird come to mind).

You have the option of sending email in text or web/HTML formats.  If you send in HTML format only, you need to insure that your subscribers can all receive emails in that format.

Using HTML formatted email content is interesting because it allows you to use any of the available HTML type content editors, and makes it easy to include images and special text fonts & sizes.   There are a number of HTML based editors available that could be used to lay out and generate your publication.  Some have fairly easy learning curves, and some are quite involved (read powerful) and take a bit of effort to learn to use effectively.  Sodipodi claims to be the up-and-coming content layout and editing tool but I still use the old Mozilla Web page Composer <www.mozilla.com> (it is the "composer" window in the Mozilla web browser).   There are a number of newer and possibly better WYSIWYG web page editors available. [/INDENT]
