### Post by floke on 2007-06-14
Why doesn't it boot? Do you mean it doesn't boot into a GUI? You could try booting up in the recovery mode option - this will take you to a terminal (ouch!) and you can use the 'sudo dpkg-reconfigure xserver-xorg' from there (you can also try 'sudo dpkg-reconfigure -phigh xserver-xorg' (not sure what the difference is, if any). Once done, you might have to enter 'startx' - you can also try 'sudo dpkg-reconfigure gdm' - not sure if this will help, but worth a shot.

Some quick advantages of the terminal...

(1) Win XP - what's going on - chuntering away...what's it doing?? Who knows...
(1b) - Linux - what's going on - type 'top' and all is revealed (q to exit)

(2) Win XP - USB drive can't be removed (is being used rubbish etc... - used to happen to me *all* the time).
(2b) Linux (doesn't really happen - very rarely) - type 'umount /dev/disk' - et voila!

(3) XP - Window/application hangs - damn - stuck with it for rest of session until I reboot.
(3b) Linux - either 'xkill' and zap the offender; or 'top' to get the PID number (eg 5562) of the offending app - and then 'sudo kill -9 5562' - OR - eg if firefox hangs - 'sudo killall firefox firefox-bin'. Job done.

The terminal is your friend - you just don't know it yet ;)

### Post by w4ett on 2007-06-14
the terminal command to hand modify your xorg.conf file is: sudo gedit /etc/X11/xorg.conf

The command to use the Semi-Graphical interface to modify this file is:  

sudo dpkg-reconfigure xserver-xorg

ext3 is the filesystem that Ubuntu and most linux distros use and gparted, which is preloaded in Ubuntu will use this automatically

The resolution issues we can address when we get your GFX up and running....the short answer is the resolutions are controlled by your driver installed and your xorg.conf in combination
