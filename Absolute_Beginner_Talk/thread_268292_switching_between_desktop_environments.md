### Post by qyot27 on 2006-09-29
Alternatively, you can install Xnest and be able to run the other environment in a window while still inside GNOME, KDE, or whatever you're running at the time.  Since Fluxbox is, as already mentioned, very minimalistic, it's nice to be able to customize it without throwing yourself completely into it with no clue.  I personally love the *box approach, so much so that I've even went and gotten xoblite for the times I feel like using it in Windows, but that's beside the point.

Running another session with Fluxbox using Xnest allows you to focus on customizing it seperately from doing familiar tasks that you might want to perform at the same time.

When you run Xnest, the window will pop up with the login screen you're accustommed to.  Then just follow the instructions given in the post above mine (although when using certain login screen themes, it actually will be a seperate button that says 'Sessions' much like the default situation on 5.10).
