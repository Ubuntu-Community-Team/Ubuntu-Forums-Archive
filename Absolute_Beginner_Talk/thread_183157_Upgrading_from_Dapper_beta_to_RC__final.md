### Post by Dr. Nick on 2006-05-27
[quote=chmmr]
- I've recieved numerous "updates are available" notifications in the past few weeks, and I've applied them as they come.  Is this bringing me up to date with the release candidate?  Or is the beta on a different development track that will ultimately be abandoned once the final release?[/quote] 
The daily updates you are getting are the latest and greatest with dapper, at this point they are mainly bug fixes that are being found and corrected befrore the final release. It is all on the same developement track so to speak. No matter what version of dapper you started on or how you got their all the software is the same version as long as you recently updated

>  - If the beta is a separate animal from the final, what do I need to do to upgrade to the final?  Is this similar to upgrading from a previous version, ie Breezy?  If so, what sort of issues can I expect?  How sensible would a clean reinstall of Dapper final be? 
Since the beta isnt a seperate animal all you have to do to upgrade to the final  is just wait for the final to be released then apply all the updates. that should do it. Once the final is released that little update bubble will be alot less active :) After the final release it will only show any security fixes or updates to correct bugs not discovered before the final release. As long as you are not expierencing issues thus far in dapper the final should be no different, Between now and final you wont notice much change as all the "revoulitionary" changes that have the potential to cause problems have already been complete. I would only do a clean reinstall if your system is pretty messed up and it is to big a pain to fix, If alot of programs crash at random in the final and you cant tell why it may be necessary to reinstall instead of trying to fix every little problem.

>  - If an upgraded beta isn't much different from the final, what differences are there?  Are there still any good reasons to do a clean reinstall? 
A beta upgraded to the final will not appear any different then a clean install of the final unless you have done visual customizaton in your beta. The differneces will be mainly appearance if you are not using the default theme etc. Of course if you want it can be changed to the default then it will appear exactly like the final, Also a updated beta may have a few extra files installed, eg extra kernels, But that is not a big problem and can be easily fixed using a package manager and should not cause problems if ignored. 

So basically I would only reccomend a clean install if your system is trashed beyond repair. I went from the first version of Ubuntu (warty) all the way through breezy without a reinstall, but all the little bugs I developed over time were annoyoing me so I reinstalled the dapper beta and updated it all along and am very happy.
