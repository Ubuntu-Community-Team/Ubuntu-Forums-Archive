### Post by PeterJS on 2008-01-04
Yeah... that's going to be a problem. $http_proxy is a global setting for what proxy to use, so apt defaults to trying to connect to a proxy at localhost:4001, but what ever proxy you had their you've since uninstalled. To fix the problem in the short term run:
```

export http_proxy=""

```
In the more longterm you're going to want to look around at your config files and find out where that's being set and the entry that's setting it, and then remove it.

### Post by jre on 2008-01-05
Although you seem to already have found the core of your problem:
> **thefoolisme said:**
> Is it possible that moblock changed something about where to connect to the repos?
 No, moblock doesn't change anything with repos. It may only block them if they are on the blocklist and moblock is running and getting traffic via iptables rules. So when "iptables -L" doesn't show you anything with "moblock" it's not doing any harm.
