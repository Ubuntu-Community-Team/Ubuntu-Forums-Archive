### Post by bigboy_pdb on 2007-08-25
I agree that you should set up a dual boot system (at least initially).

There are some applications in Windows that you might prefer to use initially (or even indefinitely) over their alternatives in Linux (if alternatives exist). You can keep an eye on the development of WINE because those applications may be supported in WINE, or if they aren't then they may become supported in future releases of WINE.

Your decision of which system you are going to use should be based on your wants/needs. If you want/need to use a system where the internals aren't hidden from you, where there is free information that allows you to educate yourself about the system, where there are a good number of free compilers for developing software, or that has a powerful scripting environment then Linux is the better system to use. If you want/need to use a form of proprietary software that isn't available on Linux or that Linux doesn't support then you might prefer to stick with Windows (or to boot into it when you need that software). There are other reasons why you might want to use either but I'm giving a few that I can immediately think of.

If you feel uncomfortable with some of the Linux programs, a number of them (such as Firefox, Inkscape, GIMP, and Open Office) can be installed in Windows. By installing those programs in Windows you can get used to using software that you'll use in Linux in an environment that you're more familiar with. You can also check to see how well documents that are commonly used will be supported in Linux by using those applications. For example, you can open your Word documents in both Word and Open Office to check that Open Office displays the Word document formats properly (but remember that you'll need to install Microsoft fonts in Linux to display those documents properly in Linux).

When I wanted to start using Linux over Windows, I started off by installing open source software and then I set up a dual boot system. I ended up using the dual boot system mainly to help to get my wireless internet connection working in Linux (and for a couple of other things such as checking that a website that I had created could be viewed in Internet Explorer and to see if a PDF document that I had generated could be viewed properly by someone who I was sending it to). Basically, I didn't really use the dual boot system as a method for running Windows applications on a regular basis because I felt comfortable using the Linux software that was available for Windows from when I had installed it and used it in Windows.

### Post by bigboy_pdb on 2007-08-25
If it wasn't a laptop computer, I would have suggested adding another hard drive. However, since it is, the response is dependent on whether or not you need Windows XP for what you do.

The requirements for Windows XP can be found at the following link and the minimum hard drive space that's required is about 1.5GB (depending on your computer):
[http://www.microsoft.com/windowsxp/pro/upgrading/sysreqs.mspx](http://www.microsoft.com/windowsxp/pro/upgrading/sysreqs.mspx)
However, bear in mind that the 1.5GB most likely is the space that the operating system requires to be installed and you will need more space for your applications, system updates, and files.

If you use Windows XP for work, college, University, or for anything where it's necessary, your Windows XP applications don't take up too much hard drive space, and you've tried using Ubuntu's alternatives and either they don't work for everything or you're not experienced enough to make them work for everything then I'd recommend dual booting with a small partition for Windows XP. However, if you don't know how much space you'll need, you could do an installation of Windows XP on the full hard drive with full updates and any applications that you will need and then you can estimate how much additional space you'd need (for things such as future updates, additional software, or file space) and then you can defrag Windows XP and use GParted to resize your hard drive partition. If you need to share some files between Ubuntu and Windows XP then you could have an additional partition that has a FAT32 file system (since both Ubuntu and Windows XP can read from and write to FAT32 partitions).

If you need Windows XP and you need a lot of hard drive space then you could try allocating about half of the hard drive to each operating system (though you'll have more restrictions when working with both systems).

If you don't need Windows XP at all and you feel confident with your abilities to use Ubuntu in order to get things done or there's no hurry in you learning to use Ubuntu then you might want to remove it from the hard drive and only install Ubuntu.

You can always change your hard drive setup in the future (but you should back up your hard drive contents that you want to keep before doing it).
