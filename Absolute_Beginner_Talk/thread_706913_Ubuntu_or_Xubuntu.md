### Post by Jerry N on 2008-02-24
There have been some other threads in the last few days about running on slow machines.  If you normally use a fast computer, you will be very frustrated trying to use Linux on your old slow machine.  I tried LinuxMint Fluxbox on a 300mhz AMD with 256MB RAM and found it to be unsatisfactory.  Right now I am experimenting with PCFLuxBoxOS TinyFLux 1.0 on that machine.  It works fairly well browsing the internet but any configuration activity is like swimming in molasses.  Right now I am trying to set it up to access a printer on a WinXP machine - It has been downloading something or other related to printers for about 20 minutes.  

You might try DamnSmallLinux, depending on what you want to accomplish.  320mb might be enough for the PuppyLinux too.  

As I have mentioned on a couple of other threads, Windows 98SE (without networking or antivirus) works quite well on this old machine.

Fortunately, I have modern computers to run WinXP, Ubuntu, and LinuxMint.

### Post by johnn1949 on 2008-02-24
I'm sure xubuntu would run fine, assuming no hardware problems. 

 The Xubuntu uses the XFE desktop, where Ubuntu uses the gnome desktop. 

 If you are planning on working up to the current Ubuntu Gutsy on #2 or #1 computer you might want to consider trying to install Ubuntu Dapper Drake 6.06 first, since it uses the gnome desktop and will install on older models than Gutsy.  

I have Dapper running on an old hp with 700MHz and 256 mb ram and it works just fine.  The current Xubuntu 7.10 might install with fewer hardware problems,like getting wireless to work.  

If you aren't dual booting it's really easy to just try it and see if you like it and the try the other one. 

Then again, migrating from XFE to gnome wouldn't be that hard either.

Just have fun,  and try whatever appeals to you.  I started on #3 just like you are and in a few months I'm dualbooting gutsy on my newest machine and love it.
