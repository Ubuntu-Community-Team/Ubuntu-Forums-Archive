### Post by Cypher on 2007-12-31
A core file is usually generated by an application and can be used to debug it using GDB. The Kernel would generate a kcore file. The core files can be overly large, and there is a command to limit this size
```

ulimit -c <size>

```
If you set <size> to be 0, then no core file is created and this is probably a good size unless you're hankering to do some application debugging. :)

In most "user" cases, you can safely remove this file and continue. If you find that some application is misbehaving and notice this core file afterward in your home directory, it's entirely likely that that's the application causing issues and you might wnat to report that to the Ubuntu folks since you're testing the new version of Ubuntu..

### Post by Cypher on 2007-12-31
Yeah a Kernel wouldn't create any files in your home directory. That's just not what a Kernel would/should do. If you do go into /proc, you might find a kcore file, if anything, that's where the Kernel would dump it's crash info.

And I'm 100% positive that only applications create CORE files, so you're Kernel is perfectly fine..
