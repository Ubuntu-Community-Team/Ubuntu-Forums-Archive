### Post by Paqman on 2008-03-28
> **dapps2k said:**
> another friend told me that it probably be safer for me to just run it as a virtual OS.  So what do you guys think?

I don't know about "safer". There's a slight risk involved in partitioning for a dual-boot, so standard practice is to back up before starting. But after that I don't see any risk to a dual-boot setup.

Installing in a VM will mean a performance penalty, but has the advantage that the virtualised hardware is known to work well.

I'd say try the LiveCD. If your hardware has no issues then i'd definitely advise you to go down the dual-boot road. If you do have hardware issues, virtualise.

### Post by Red Shift on 2008-03-28
If your computer has more than 1 GB of RAM, I would virtualize because, to me, it's more convenient to start VirtualBox, VMware, etc and start Ubuntu than it is to shut down Windows and boot Ubuntu.
