### Post by m8zz on 2008-02-06
I installed Ubuntu 7.10 on an old beat up P3 750 with only 256MB. I assumed the freeze up at 15% was due to the lack of RAM (384MB is the recommended minimum). So during the install, I chose to manually configure the linux-swap to about 900MB. This seemed to work, as the installation zipped along to the end. I did a reinstall with the same linux-swap and the installation worked again.

Re PC recommendations: Ubuntu 7.10 installed with HDD space usage of 1.5GB on /, 2GB on /home, 200MB on /boot, and - of course - 900MB on /linux-swap. So you really need a mere 5GB HDD to use Gutsy Gibbon. Obviously, I was able to install with less than the recommended minimums, and I have a great Ubuntu super desktop/server (for WiFi-networked file sharing, printer sharing, and even remote scanner sharing). Plus, my old laptop uses very little energy and runs quietly and continuously, with a 50" LCD HDTV for a multi-media monitor.

Hope this helps.
