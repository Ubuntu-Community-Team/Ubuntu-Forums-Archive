David, yes if you boot from a "LiveCD" you can test and see what Ubuntu looks like and how it works. No changes will be made to your hard drive. (Unless you click on the "install" icon, in which case it will ask if you want to install to your hard drive.) Basically in a "LiveCD" session the whole operating system is running from the CD drive.

Note that running a LiveCD session gives you a good idea of what Linux is like, but a LiveCD is always *very slow* compared to running a real installation (CD-ROMs are not as fast as hard drives). So keep this in mind when trying it out.

A LiveCD is a great way to try Linux without any risk to your system. I recommend you give it a try. Just download the standard Ubuntu disk and burn the ISO image, and then boot from your CD drive.

### Post by old_geekster on 2007-01-23
I am a newbie to Linux myself.  I used SuSE 10 for a short time last year, but the hardware compatibility was not good.  So, I decided to try the latest version of Ubuntu this year.

There are some great things about using LiveCD.  It not only allows you to test the OS, but it also will install all of your components to see if they work.  It even recognized my DSL with no tweaking.  Of course, as mentioned earlier, when you decided to do a permanent install, the icon is on your desktop.  Simply click on it and the installation begins.  The most difficult part for me was figuring out how to setup the partitions.  Once that was done the remainder was easy.

Here are some things that I found to help me.

First, Firefox 2.0.0.1 was extremely slow.  So, I downloaded and installed Swiftfox 2.0.0.1.  It is a stripped down version of FF2 and is far faster.  At least for me.

Second, here is a link to the best "HowTo" site I have found for commands:

[http://ubuntuguide.org/wiki/Ubuntu_Edgy](http://ubuntuguide.org/wiki/Ubuntu_Edgy)

By using copy and paste, I have managed to install numerous programs to give them a try.  It has helped me learn the commands, also.

Next, I downloaded the Official Ubuntu Book.  This has helped me understand the OS much better.  I had to figure out why I couldn't read it.  The reason, it required a CHM reader.  This sent me on a search for the reader.

Last, this forum is definitely one of the best resources for Linux/Ubuntu on the web.

Good luck.

p.s. I have seen some complaints about having the bar (panel) across the top of the page, instead of at the bottom as in Windows.  All you have to do is drag it to the bottom of the page and it will stay.

### Post by 3rdalbum on 2007-01-24
I believe the reason why the packages aren't installing is because you don't have the Universe and Multiverse repositories enabled. Go into System > Administration > Software Sources and enable them. When you close the program it should ask you to reload the package list - do it. Now you will be able to install the programs.

Some people do say that Firefox is slower than on an XP box - this is probably because IPv6 is enabled on Linux and not on Windows. Have a look at this simple guide to turn IPv6 off: [http://en.opensuse.org/Disable_IPv6_for_Firefox]("http://en.opensuse.org/Disable_IPv6_for_Firefox")
