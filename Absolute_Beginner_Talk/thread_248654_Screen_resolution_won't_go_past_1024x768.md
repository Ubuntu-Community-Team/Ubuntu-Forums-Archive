How to fix (100% CLI my boy):

1. From windows, copy xorg.conf from your first post. Save it as a text file (xorg.conf.txt) to ? Floppy, USB, whatever.

2. Boot Ubuntu.

3. Log in -> mount your floppy/usb, whatever.
Example:
```
sudo mkdir /media/usb
sudo mount /dev/sda1 /media/usb
```

4. Restore your old xorg.conf:
```
sudo rm /etc/X11/xorg.conf
sudo cat path_to_xorg.conf.txt > /etc/X11/xorg.conf

Example (assuming usb with mount point/media/usb):
**sudo cat /media/usb/xorg.conf.txt > /etc/X11/xorg.conf**
```

5. Back up your working xorg.conf
```
sudo cp /etc/X11/xorg.conf /etc/X11/xorg.conf.org #(.org for origional; this is a BACKUP).
```

6. Now, open a terminal and type 100 times:
```
I will always back up system configuration files **before** I change them.
```

**To fix your resolution problem I need to know the monitor/model you are using, or more speicically, your monitors HorizSync and VertRefresh. (use google if you do not know what they are.)**

1. Now, restart gdm:
```
sudo gdm
```

2. Edit xorg.conf:
```
Section "Monitor"
	Identifier	"Generic Monitor"
	Option		"DPMS"
	HorizSync	**PUT IN YOUR MONITORS HorizSync**
	VertRefresh	**PUT IN YOUR MONITORS VertRefresh**
EndSection
```

### Post by Brightbelt on 2007-05-30
I had similar problems:

Here's what worked for me....
-I rebooted, went into Recovery Mode
-ran this: dpkg-reconfigure xserver-xorg
- Chose 'VESA' for my driver (it seemed counter-intuitive since I have an Ati and Ati is on the list but I was told Vesa is a common driver and sure enough for me, it worked)
- Chose Default settings whenever possible
- enabled all X.Org Server Modules except Glx
- Entered my exact ('advanced option') horizontal freq. rate and vertical refresh rate; you can find these in your monitor manual

Once I finished, I did 'sudo reboot' and my GUI was up and running. I've had to do this 2 or 3 times and it has worked every time.  I hope this helps, Frank Bright
