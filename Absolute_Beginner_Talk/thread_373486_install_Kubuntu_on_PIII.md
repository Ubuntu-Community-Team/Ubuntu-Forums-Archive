### Post by Bartender on 2007-03-01
Yeah, I'm wondering if the alternate install CD was burned correctly.

Another idea - download and burn a copy of PCLOS.  It seems to have better hardware detection than Ubuntu/Kubuntu.  If you have access to broadband this would be a relatively easy way to check and see if the problem is the PC or the Kubuntu CD you made or possibly Kubuntu's inability to find drivers for some piece of hardware.
Any way you can get your hands on some more RAM?  If you could double the RAM from 256 to 512 that would allow you to try out LiveCD's instead of alternate install CD's.  I don't have anything against alternate CD's but it's harder to verify that the CD is good.  The LiveCD's come with an error check function, right there on the first splash screen.  I've read that the error check is not 100% reliable, but it's certainly a good indicator.
