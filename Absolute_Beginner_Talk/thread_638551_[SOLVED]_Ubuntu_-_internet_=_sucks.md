### Post by ruibernardo on 2007-12-13
Hi,

a agree that using Ubuntu without internet is really complicated. Searching and downloading all packages and their dependencies from [http://packages.ubuntu.com](http://packages.ubuntu.com) is boring and some package can be missing when trying to install in the offline computer.

I'm trying to setup a website to ease the "search all needed packages" from [http://packages.ubuntu.com](http://packages.ubuntu.com). It's [http://nonetdebs.homeip.net](http://nonetdebs.homeip.net). 

It works by running a chroot where the user uploads the status file (/var/lib/dpkg/status) of the offline ubuntu. Apt-get is executed on the chroot with the repositories, kernel type (generic, amd64...), release (dapper, edgy, feisty, gutsy), with the package name he wants and the --print-uris option too list the packages files.

The webpage interface [isnt't very good]("http://ubuntuforums.org/showpost.php?p=3742162&postcount=11"), but in the end it will list links to the packages to be downloaded. 

APTonCD should be the first to download and install with "dpkg -i" so the other packages can be installed too.
