The problem with installing without the kernel is that you have nothing to boot up with :)
[http://en.wikipedia.org/wiki/Kernel_%28computer_science%29](http://en.wikipedia.org/wiki/Kernel_%28computer_science%29)

So you need it to fire up the system. The most likely reason you ran into the first problem is a bad cd. So when you boot up the with the cd there should be an option to verify media. Use that and see if the CD is bad. If it is make first use md5 to verify the downloaded ISO against the .asc file also available on the downlaod page. To do so on windows see this: [http://www.pc-tools.net/win32/md5sums/](http://www.pc-tools.net/win32/md5sums/)
