### Post by hammo on 2007-06-25
Because Ubuntu, is a form of Linux, which is an unix-LIKE system. Unix is inheriantly a secure system due to the way it has been developed. When Linux was developed it was made as a GOOD operating system. Windows is just a sloppily coded OS that serves a purpose. Also, most hackers that write viruses and bugs for Windows use Linux as their OS - why would they want to infect themselves?

[EDIT] Sorry, another thing I meant to add is that Windows has a lot of holes through which people can gain access to your system. Linux does have some holes, but not nearly enough to compete with Linux. *nix-like systems aren't perfect, but they're getting there :p. Also, another main reason that Linux is so secure is it is developed by a community that WANT a secure OS, whereas Windows is developed primarily to make money. Look at it this way. There have been like 7 main releases of windows in ~15 years. Linux is only up to version no 2.6. Developed with patience and detail in mind.

### Post by Golyadkin on 2007-06-25
It also depends on what you run on your Linux computer. A machine that runs a webserver, database server, SSH server, FTP server and mailserver has WAAAAAAAAAAAAAAYYYYY more security vulnerabilities than one that doesn't. Most Windows machines do not run those services, but they are vulnerable to a lot of operating system level exploits and security holes.

### Post by LaRoza on 2007-06-25
A variety of reasons, including the above.

People who write programs commercially, seem to forget Linux, why should virus programmers be different? 

A security flaw in Windows Vista, the animated cursor thing, took three months to acknowledge AFTER Microsoft knew about it.

Most people seem to use one account on there computer, the adminstrative, it is easier to kill a system if you are admin. Linux systems generally cannot run something as root (admin) by accident, it has to be deliberate. On this forum, the code "gksudo nautilus" is often given with very strict warnings, even Linux warns you, in Windows, you are always using the equivilent command, because of the account.

Linux is developed by a community and is fixed much faster than Windows, if a hacker can exploit a Windows security flaw for three months, why waste time with Linux which will be fixed much faster.

As the saying goes: "Locks are for honest people; thieves break them". If someone really wanted to hack into a Linux system, they probably could, but it wouldn't be worth it probably.
