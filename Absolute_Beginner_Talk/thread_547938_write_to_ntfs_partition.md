### Post by andreid on 2007-09-10
He probably already has ntfs-3g since he can write on the old partition...
To write on the newly created partition, you have to make sure that ntfs-3g is set as the filesystem type in /etc/fstab. That is:
```
$gksu gedit /etc/fstab
```
and make sure that you have a line like this for your newly created partition:
```
 UUID=24484D90484D6222 /media/sda5 ntfs-3g defaults,locale=en_US.UTF-8 0 1
```
Note: Do not copy paste this last line because you have a different UUID. The important thing is to replace the filesystem type (probably ntfs) with ntfs-3g

### Post by meierfra on 2007-09-10
Stop bumping, I was typing  as fast as  I can.


> btw, the partition i'm trying to read/write is 37gb. it's in the 120gig drive.

O.K  that should be  be hda5. Do this:

```
sudo mkdir  /media/name_of_your_choice 
```

(If you already have that folder in /media skip that step)


Open fstab via:

```
gksudo gedit /etc/fstab
```


Add this line at the end:

/dev/hda5  /media/name_of_your_choice  ntfs-3g defaults,locale=en_US.UTF-8 0 0

Save the file. Then 

```
sudo mount -a
```

And you should be able to access your partition.

If not it really should work after reboot.
