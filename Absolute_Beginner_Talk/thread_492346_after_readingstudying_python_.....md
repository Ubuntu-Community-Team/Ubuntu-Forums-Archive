### Post by annda on 2007-07-04
there are two good o'reilly books by mark lutz: learning python and programming python.

i have a mixed feeling about the collection of stuff on [www.python.org](www.python.org) - there is a lot, but it's not easy to comb through that.

but the advice i give to beginning programmers (myself included, i'm a beginner) is to set a task or a problem, or even a small application, and use all the available references, guides and tutorials to write it. google is not so bad after you start coding and know what questions you want to ask next.
