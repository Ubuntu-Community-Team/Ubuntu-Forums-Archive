### Post by gvgerman on 2006-09-27
Assuming you have been unable to download any mail (due to an invalid password) or assuming the mail has been left on the server, remove the "problem" account and start over.

If you have mail that you have previously pulled off the server associated with this account that you need to keep (and, thus, likely have deleted the server copy of the messages), you could add a new account with the appropriate settings giving the new account a different name from that given to the old account.  The old account's messages can be dragged & dropped into the new account.  Then  you can remove the first, or old, account that has the bad password.  Finally, once the first account is removed, you can rename the second (new) account however you'd like.

If you have a boat load of messages to drag & drop, the only issue you might experience is a corrupt *.msf file.  But these are not hard to fix and Thunderbird has a help file on their website that states how to do this. 

There is a hidden folder likely located your Home Folder that contains the password information, but I do not know how to edit it nor do I suggest doing so.
