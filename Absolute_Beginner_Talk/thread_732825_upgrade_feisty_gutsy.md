### Post by MichaelSM on 2008-03-28
To Michl.

Re, Autocad 2006+Wine, In Hardy.

Becos Wine is still stuck as far as I can tell in W98 ACad 2006 is probably NOT ON!

With the help of Linux gamers I was able to get ACad 2000 working up to a point with Wine but it was very ordinary indeed. 

I installed Innotek Virtualbox and becos it's XP (you need an XP install disc with codes) ACad 2000 runs perfectly.

In other words I've never found a hack and there are two major stalling points.

1) You must have an installable XP program with user codes. That's not too hard these days.

2) Your ACad 2006 is unuseable unless you have the keys to it which must be entered during the install.

If you have both of the above, install Innotek Virtualbox for your Ubuntu flavour, follow the prompts for XP install then go from there. 

After that install your ACad on XP/Virtualbox as if it was a Windows install.

That's the only way to do it. But you need the 4X4 codes for both XP and ACad.

Mike.

By installing XP in Virtualbox I mean that's what you have to do. It's just an hour plus of mindless boredom as W goes through its mind-numbing crap telling you how brilliant it is. 
However, once XP is up and running on VBox, the ACad install takes only a few minutes depending upon your processor and ram. Make sure when prompted that you allocate at least 192 meg ram to your XP/Vbox install. If you have more available with your 'host' Ubuntu then give it the works for VBox which will make the XP install quicker -so long as you're not trying to run collateral stuff on Ubuntu at the same time. In other words if you have 1 gig of ram in Ubuntu host, which needs little to idle on, assign as much as you feel happy with to VBox during the XP install. You can always change it back later. It's just a slider after all.
