### Post by itsjustarumour on 2008-05-01
> **patchido said:**
> i meant to use it in a widget layer and not in the desktop

I think what patchido perhaps means is that he wants to make the gDesklets "widgets" behave in the same way as Screenlets "widgets".  

To explain - If you have (for example) four desktops, by default gDesklets places a particular "widget" in the same place on each of your four desktops. Screenlets doesn't do this, and has on option in Properties/Options to "Treat as Widget" which when selected allows you to (for example) have just one instance of a widget running and to choose which of your four desktops you want to position it on - and it sticks there.  However with gDesklets, when you select a particular "widget" and it appears in the same place on all four desktops, there appears to be no way to configure this further - its "all or nothing".

If anyone knows a solution to this, I'd also be very interested to know!
