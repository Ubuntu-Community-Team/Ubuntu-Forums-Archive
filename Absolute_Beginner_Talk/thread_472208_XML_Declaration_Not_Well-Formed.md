### Post by Skrynesaver on 2007-06-13
It means that the first line of the file .fonts.conf is not a well formed XML declaration ;)
When XML is used it must start out by stating which xml version, DOCTYPE and dtd file to use to parse it.
The first line of .fonts.conf should be
```

<?xml version="1.0"?><!DOCTYPE fontconfig SYSTEM "fonts.dtd">

```

Compare file to /etc/fonts/fonts.conf and make sure the file /etc/fonts/fonts.dtd exists

### Post by Skrynesaver on 2007-06-13
OK here's my .fonts.conf  with the smooth rendering turned on, wouldn't have looked into it without your question and yes it does look better now ;)
```
<?xml version="1.0"?><!DOCTYPE fontconfig SYSTEM "fonts.dtd">
<fontconfig>
 <match target=”font” >
  <edit name=”autohint” mode=”assign” >
   <bool>true</bool>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="rgba" >
   <const>none</const>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="hinting" >
   <bool>true</bool>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="hintstyle" >
   <const>hintmedium</const>
  </edit>
 </match>
 <match target="font" >
  <edit mode="assign" name="antialias" >
   <bool>true</bool>
  </edit>
 </match>
</fontconfig>

```
All the other files should be left as they are by default, log out, log in and take a look see.
