### Post by bob12564 on 2008-01-23
I'm a Windows refuge trying to learn Ubuntu Linux.  

Media players and multimedia keyboard setup are well covered on these forums.  However, newbies like me need a plain English conceptual answers to help us understand why things are they way they are in Linux.

In Windows there are common dialog panels that are used by all Windows software.  These common dialogs cover things like printing and opening and saving files.  And keyboard drivers ensure that all the keyboard keys work the same in all software packages.  So, for example, printing in MS Word brings up a similar printer dialog panel as selecting "print" in Firefox.  And the keyboard works the same in Windows Media Player as in Real Player.

I used KeyTouch to enable the multimedia keys on my keyboard for use with Ubuntu.  It was an easy process.  My keyboard was undefined so I first used the KeyTouch editor and then imported the file into KeyTouch.  However, I am now realizing that there is no standardization among the various media players so having gone through the trouble of enabling the keys doesn't mean they have any practical use.  Fort example, if I insert an audio cd, Sound Juicer opens and plays the disk.  My needs are very simple and Sound Juicer will be fine for me.  Xine seems to be playing my movie clips well and also plays audio CDs, so it's a keeper.  VLC plays DVDs well.  But my multimedia keys don't work.  My volume controls work just fine, but the play/pause, stop, next/previous track keys do not.  

When I configured the keys with KeyTouch, I selected "XMMS" from the dropdown because I didn't see anything else that I recognized.  So here are my questions:

1) Why isn't there more standardization in Linux so that the keyboard definition works on all the media players?

2) What should I do in KeyTouch to enable the keys with Sound Juicer, Xine and VLC?  Is there anything that works for all 3?  I looked at Remoot but there seem to be too many quirks for me to want to use it at this point.

3) Since I'm using KeyTouch, should something be disabled with the Gnome keyboard setup facility?  Do they conflict?  The Gnome utility doesn't seem to see what I've done in KeyTouch.

Thanks for any education you can provide.  I'm sure I'm not the only beginner who is very confused.
