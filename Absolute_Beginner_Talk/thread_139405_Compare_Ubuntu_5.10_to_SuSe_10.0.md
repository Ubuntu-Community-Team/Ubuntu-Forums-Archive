### Post by faithman2k on 2006-03-04
Hi joeca0304,

I had both running for about 2 months on my laptop and used them exclusively for 1 month each.

SuSE 10.0
CONS
-I found that it was difficult to get SuSE to play multimedia (.wmv, .avi, DVD). 
-I also found that YAST was not very intuitive. It took ages to figure out and it was always difficult to work with.
-Takes up lots of disk space
- RPM software package system (I'm a big fan of .DEB... it seems to be better, although, if you rely on Debian repositories they will tend to be VERY out of date, but VERY stable. I like stablity.)
-I gave up trying to configure it to my taste... it was just too hard (ie. video playback for DVD and other formats. I mainly use my system to watch video, listen to music, surf the web, word processing, etc. nothing very special).

PROS
-Eye candy... it really is a very nice desktop (recognised my video card perfectly)
-Very good hardware recognition
-Has everything you want already installed (see CONS - takes up disk space). Except multimedia/video.
-It seemed faster than Ubuntu on my Dell Inspiron 8100 Pentium 3 1.2GHz

Ubuntu Breezy 5.10
CONS
-Desktop not as nice (I can live with that, brown grows on you).
-Less packages installed (for those that want more features out of the box)
-Didn't like my laptop's NVidia GeForce 2 on board card... needed to configure it "manually" with the NVidia script.
-Totem sucks. I replaced it with Xine

PROS
-Documentation is better. Wiki, ubuntuforums and also Debian docs are useful.
-.DEB packages. Synaptic is great (yes it is available for RPM packages, but it felt clunky, like it was an afterthought in Suse)
-Community is much more pleasant and friendly (Suse community was nice, Ubuntu has been nicer... NOTE: there are bad apples in every bunch. I looked at which community was consistent in their approach to newbies and questions)
-Based on Debian. I have tried lots of different distros (Slax, Slackware, Suse, Kanotix, Knoppix, Mepis, Mandrake, Linspire, Xandros, Red Hat, and others), basically I found that FOR ME (I stress, this is MY opinion, the debian based distros were best, they suited my style of usage and the I way like to do things).
-I found more development is happening at Ubuntu than SuSE. It seems more active.
-I found it easier to find software repositories for debian based distros such as Ubuntu.
-Less space taken up on drives.

In the end I chose Ubuntu because I felt more comfortable with it. I seem to get more done with Ubuntu. 

To be fair, Suse should be compared to Kubuntu as it is has more similarities with the the KDE GUI.

Basically, the best advice I can give you is to use both equally for a period of time, and continually ask yourself, "Can I live with this everyday?" You will know which one is for you by the end of that time period.

I hope that helps and I hope it makes sense (I had a long day today),

faithman2k

EDIT:
PS. Regarding differences. the biggest is the RPM vs DEB packaging. I couldn't see much more than that.

Just to be fair:

1. 5GB is not 'bloated' by any definition I would use.

2. SuSE has a _much_ better package selection feature (actually a full blown GUI module) than Ubuntu. When you access this YaST module during installation it will allow you to place on the disk only those packages that you desire. You could easily parse down the install size to a much smaller footprint.

[quote=]Compare the speed of YAST with the speed of SYNAPTIC.[/quote]

And YaST is not only a package manager. It is an entire system management user access point. Thus, this 'benchmark' is not really applicable.
