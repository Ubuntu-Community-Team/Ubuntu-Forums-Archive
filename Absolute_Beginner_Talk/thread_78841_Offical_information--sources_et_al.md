### Post by stuporglue on 2005-10-19
> 
1. Is there somewhere we can go to see what we should know before we proceed further? i.e., before we begin following outdated instructions.

Ubuntu is released every 6 months. Check the instructions for the version number. If the version matches, then the instructions will be current. Documentation such as tutorials and HOWTOs usually lags a little because it's user contributed, so it gets updated as people do the same things again on their new systems. 

> 
2. Is there a new "official" sources list we can use? Preferably with a date on it, so we'll know when it's outdated?
The sources that came installed are official. Anything else is not officially supported (with exception of locale changes, ie. us.ubuntu --> ca.ubuntu)

> 3. With outside projects such as the Unofficial Ubuntu Guide and Easy Ubuntu, I know Ubuntu is not responsible for them. But, what's the normal protocol for this type of thing? Do they routinely get updated as Ubuntu changes? Or should newbies be beware of them, because they may be outdated?
They're kept up by volunteers running those projects. Anyone using them is taking a calculated risk: is the potential for ease of use greater than the potential for them being wrong? I'm guessing they'll get major updates every 6 months (at release time) and then small changes for corrections between updates.


> 4. I've read over the Ubuntu.com site and a heck of a lot of the Wiki, the HowTos, and posts in many of these forums. I know KDE, Gnome, OpenOffice, etc., all have their own websites where much is explained that is not in the help files, and of course there's Google. But what about for Ubuntu itself? Is there someplace I'm missing?
What do you mean by "for Ubuntu itself"? I'm not exactly sure what you're looking for. 

Hope that helps some,
