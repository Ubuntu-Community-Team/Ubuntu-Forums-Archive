### Post by Mustard on 2005-12-13
After reading your other post in another section of the forum, I think you should proceed with caution.  The install disk won't mess with your bios settings.  You can only do that manually yourself.  The issue is going to be the files you have on your 3GB partition.  You would need a free partition to install on and currently its seems you have XP installed on a 1GB extended partition and some files sitting on your 3GB primary partition that you don't want to lose.  This is not leaving you with any free partition that the ubuntu install disk can format and install itself on.

Additionally I would search the forums(using the search function) and/or google up some stuff on issues that people have had with installing ubuntu where XP existed on an extended partition, rather than the primary partition.  I have some vague recollection of someone haveing some issues with this once.

Using your XP operating system you would probably need to create a seperate free partition of at least 2GB to try Ubuntu on.  Ubuntu will use about 1.4 GBs of this and that will only leave you with 600mb to play with and practically nothing left over for XP.

If you want to look at Ubuntu, you would be better advised to use the LiveCD version, which can allow you to preview Ubuntu without actually installing anything on your hard drives at all.

Basically I think you need to do a bit more reading around the forums to find threads on people who have installed dual boot XP/linux installations, to become familiar with what is going to occur, and additionally you probably need to look into getting a bigger hard drive.
