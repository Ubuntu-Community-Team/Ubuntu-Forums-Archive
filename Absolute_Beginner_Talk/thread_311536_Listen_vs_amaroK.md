### Post by aysiu on 2006-12-03
Well, I just installed Listen, and I can list a few things already:

1. The inability to edit ID3 tags inline (you need to open a separate dialogue for that)

2. The lack of support for global keyboard shortcuts--in AmaroK you can define your own.

3. Limited available view options for songs (track number, album, duration, artist)

4. No lyrics fetching

There are probably other features AmaroK has that Listen doesn't, but does it really matter? If Listen does everything you want it to do, that's great.

### Post by aysiu on 2006-12-03
> **nibi said:**
> since lyrics is also one of the features i need, i guess its amaroK in that case. One last question, does amaroK run just as good on gnome as it does on kde or are there any disadvantages of using amaroK on gnome instead of KDE?
It depends on what you mean by "just as good" (or *just as well*, I guess).

It will function correctly, however...

1. It will need to load up a bunch of KDE libraries. If you're a bit shy on memory (say, 256 MB of RAM), then this could slow down your system a little.

2. It won't integrate well visually with the rest of your Gnome experience.

3. Drag and drop to Nautilus may not work the way you expect it to (I haven't tested this in Edgy yet, but in older versions, dragging from AmaroK to Gnome's desktop would *move* the song instead of *copying* it).

### Post by adamkane on 2006-12-03
Big disadvantage to Listen is that it uses gstreamer, which doesn't work that well.

amaroK allows you to use the xine engine.

I use amaroK to listen to LOTS of podcasts.

amaroK has scripts which allow you to record streaming audio.

amaroK has a MySQL option, so you can manage massive libraries, if you are so inclined. 

Therefore amaroK is more robust.
