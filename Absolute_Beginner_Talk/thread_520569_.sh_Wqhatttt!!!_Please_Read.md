### Post by igknighted on 2007-08-08
Right click it, go to the permissions tab and check the "make executable" box.  Then, open a terminal and type ```
sudo Desktop/filename.sh
```

### Post by KIAaze on 2007-08-08
You can also make it executable from the terminal by doing:
```
chmod +x filename.sh
```

And you can run it as specified above or by double-clicking on it once it is executable.
It's always better to run scripts from a terminal though to see their output.

Then execute the script

sudo <script>

[SIZE="5"]**/!\ NOOO! /!\**[/SIZE]
Why sudo?
**_Only use sudo when necessary, i.e: when the script requires root permissions!_**

Normally, a well written script will tell you if it needs to be run with sudo.
Or it will automatically ask you for your password when needed.

_**You should never use sudo unless it is really needed.**_
And using sudo on a script from an unknown source can be very dangerous!

P.S:
And in case anybody was wondering why there is chmod 755 and chmod +x:

chmod +x just adds an executable flag.
chmod 755 gives the script very precise permissions:
rwx for the user
r-x for the group
r-x for other

See "man chmod" for more info. (or [http://catcode.com/teachmod/]("http://catcode.com/teachmod/") )

Personally, I also use 755. ;)
That way I know exactly what permissions it will have.
