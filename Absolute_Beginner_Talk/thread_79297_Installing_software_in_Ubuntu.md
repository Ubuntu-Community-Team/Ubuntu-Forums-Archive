### Post by aysiu on 2005-10-19
I didn't have any dependency problems installing Opera. Are you sure you got the right .deb? They make one specifically for Ubuntu (Breezy, Hoary, and Warty).

Software installation is dead-easy in Linux. It's easier than Windows even. See the second link in my sig for more details--it compares installing Blender in Windows to installing Blender in Linux.

For more software availability, you can also [enable extra repositories](http://www.psychocats.net/sources.html). Ask questions before you start making judgments, and you may learn something.

By the way, I just saw your post on installing Flash for Firefox. Once you enable extra repositories (a one-time deal--it doesn't have to be done for each new piece of software), it's in Synaptic--again, see the second link in my sig and the attached image.

### Post by Qrk on 2005-10-20
Opera uses qt, so it requires KDE base and some other stuff to run. But the deb you can download from the website doesn't install these by default, so its listed as a 'broken package.' 

All you need to do to install it is to use synaptic to get the basic KDE core, and your good to go. I always install K3b first, then opera. K3b gets all the stuff opera needs to run, and its the best cd burner for Linux anyway. (Although Gnomebaker has improved tremendously)

And as for Limewire... I find Gtk-gnutella to be much faster, and its in the repositories. 

Moral of the story --- there is an easy and a hard way to do things. You'll look back on your early problems and smile. Such is life
