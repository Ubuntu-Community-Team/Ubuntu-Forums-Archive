### Post by rkd on 2008-02-12
If you mean it won't let you enable writing on your Windows partition, that sounds like you still have the old read-only NTFS support. Installing the packages oedha mentioned will install read/write NTFS support, and the sudo mount command will mount the partition using the new read/write support.

You might have to unmount the partition before you can mount it again: ```
sudo umount /dev/sda1
```Note, there is no "n" in umount.

I've only ever seen mount commands with the -t ntfs-3g immediately after the word mount, so if you get a complaint about the mount command the way oedha wrote it, move the -t ntfs-3g to between the mount and the /dev/sda1.
