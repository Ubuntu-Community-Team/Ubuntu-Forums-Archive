### Post by carlosjuero on 2007-11-02
As the link posted a page back points out, many people tend to get into the mindset of comparing Linux -> Windows.. the thing is that you cannot do any system justice like that. Linux is a different beast from Windows - it does many things better (native support for reconfiguring the actual kernel) but some things are still not so mature (gaming).

It isn't about the 'power' you **want** to see in Linux, it is about taking a step back and comparing simple little things. Take startup time: Which loads faster? How about customizing the UI without a bulky addon program? By customizing I mean complete changes without any app running in the background - Linux has windows managers with different 'flairs', Windows has the Windows GUI and some skinning apps (there are some explorer replacements, but they don't retain the same flexibility of the original like the Linux GUI changes do).

How about task automation? With Windows you have to use their lovely 'Scheduled Tasks', with Linux a simple cron job setup and you are good to go (simple being relative) with less overhead.

Take a look at your system tray in Windows sometime, how many icons are there by default? How many are in Linux system trays by default?

There is a whole lot of comparison, but in the end it is the end user's view of the 'power' that matters - if you have doubts, then run Linux for a month and switch to Vista for a month and see which one suits you. Nothing anyone here can say will be able to sway you if you don't see the point.

I still dual boot into Windows for some things, and I notice the difference in startup speed every single time; I also notice the lack of a quick command line giving me more power over where things go (for Windows the only way to get true symlink ability is to download a seperate utility from MS - Linux has it built in).

No 'in the wild' viruses, more secure protection schemes (try changing a program file without root priveliges enabled, then attempt the same on a Windows program file [non kernel]).

Its all up to you in the end OP - if you aren't seeing the 'power' that others are talking about, then you likely will either never see it or it will take you a while to realize what exactly they mean. Nothing wrong with that, there are 2 types of PC users.. those who can Linux, and those who can't. Nothing wrong with either camp.
