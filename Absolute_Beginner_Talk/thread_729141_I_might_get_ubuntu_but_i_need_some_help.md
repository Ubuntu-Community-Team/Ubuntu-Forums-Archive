### Post by donnyblaze1 on 2008-03-19
Welcome!

1) Which iMac do you have?  Intel or PPC?  I would say you will definitely encounter some performance issues with an emulator like VMWare.  (OK, technically it isn't really an emulator, but there will still be significant overhead.)  Have you looked into a dual-boot setup, or even just running Ubuntu from a live cd?

2) Sourceforge and freshmeat.net are great places to find open source software.  Ubuntu's own software repositories have plenty of great software and will be very easy for a Linux beginner to use.

3)** Need** to learn command line?  Probably not in most cases.  Will you want to learn command line?  Yes.  Ubuntu and its derivatives are user-friendly enough to have GUI tools to accomplish most tasks, including basic and intermediate administration.  However, the more you use the system, you will encounter tasks that will need at least some interaction with a command line.  Don't fret though, man files, these forums and google are your friend.  Just take things a step at a time and learn as you go.

4) The default  Ubuntu desktop environment is gnome.  Have a look-sie at [www.gnome.org]("www.gnome.org") for some info.  Most Mac users I've talked with (myself included) have a pretty easy transition to the gnome or KDE environments.

If I were you, I'd definitely download an Ubuntu live CD and give it a go on your system before installing.  This will help you see and feel what you are getting into.  Good luck, glad to have you here!

### Post by kidders on 2008-03-19
Hi there & welcome,

> Q.1 I will use vmware since i have an iMac. Is there any performance issues that i should know about?Anything running in a virtualised environment will naturally perform less well than it otherwise would. Additionally, nothing you run in a virtual machine can directly access any of your hardware, which can be frustrating. Incidentally, is there some sort of iMac-specific compatibility problem with Ubuntu?

> Q.2 Currently i'm using sorceforge as a software catalogue can you recommend any other site?I would not recommend installing software onto an Ubuntu box from anywhere other than trusted software repositories ... unless of course you know enough about what you're doing to side-step any compatibility problems that may arise.

> Q.3 Very Important Do i need to learn Ternimal code to use ubuntu cause i don't know anything about programming.Using a terminal has very little to do with programming, really. Both Linux and OS X have very well-evolved terminal interfaces that are well worth learning to use.

> Q.4 How would you describe how applications are sorted and accessed? Is it like Finder?I'm not completely sure what you mean here, but if you're asking whether there is a Finder-like equivalent in the Linux world, the answer is yes ... several of them in fact. Most Ubuntu users use Nautilus, but alternatives include Dolphin & Thunar.

In OS X, applications are a bit all over the place ... parts of them live in /Library, ~/Library, /Applications and ~. Linux is very similar.

Anyhow, I hope that helps.

### Post by bro on 2008-03-19
Q.1 I will use vmware since i have an iMac. Is there any performance issues that i should know about?

R.1 Try and you know. By the time you consider to use Ubuntu as your main system you will want to have a install outside of a virtual machine. It will run fine but of course much slower in a VM than with a real install. 

Q.2 Currently i'm using sorceforge as a software catalogue can you recommend any other site?

R.2 You don't need a site. Ubuntu has two programs built in to search software with an just click to download and install. These are called repositories. You will want to add a repository occasionally. Mainly the medibuntu repository  so you can install much used proprietary software like flash, adobe reader, skype and such. 

Q.3 Very Important Do i need to learn Ternimal code to use ubuntu cause i don't know anything about programming.

R.3. Contrary to what some of the other comments here might say the answer is No. It is not that hard ones you do and even easier than graphic ones you get the hang of it. However, it is also a disease if you don't know it. It is a disease that stops new users from using linux. Fortunately Ubuntu has cured this disease, everything you need to do as an end user can be done with graphical interface. 
As you will find out, many help info on forums and blogs contain 'quotes' which you can copy past on the command line. This is easier and faster then pointing out graphical instructions. See for example ubuntuguide.org.
What you will also find out is that many enthousiastic users are evangelical about the command line. 

Q.4 How would you describe how applications are sorted and accessed? Is it like Finder?

Try and you will like it. Finder I don't really know. You can acces via a menu structure. You can acces via a quick shortcut bar with icons. You can install a dock like AWN (more or less like a mac has). You can use Gnome Do which I believe is inspired by Quicksilver on a Mac.

Just download and install and you will soon find out through first hand experience!
