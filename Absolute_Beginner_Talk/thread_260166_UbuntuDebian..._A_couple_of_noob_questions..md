Ubuntu vs Debian, a couple of questions from a noob.

OK, I am a Linux noob. At home I’ve been playing around with Fedora, DSL, Knoppix, Ubuntu, and Debian. I should mention that I am relatively technical, I am a computer science and mathematics graduate, I did a PhD on user interfaces, and now I’m working as a .NET developer. All very Windows centric so far, but I want to get more into Linux and I’ve been having a blast doing so. 

Anyway…

I’ve finally settled for Debian. DSL and Knoppix were great as live CDs but not really what I wanted. I did do a hard disc install of DSL but as far as I could work out it was Debian Woodie based, which is mega old. 

From what I’ve seen so far I get the impression that Ubuntu is Debian with ease of use and bleeding edge as a priorities, as opposed to Debian’s obsessive but somehow reassuring testing process. At the same time that sort of stability is perhaps not needed on a home desktop... This brings me to my first question. 

I can see that Ubuntu provides lots more up to date packages when compared to Debian Sarge. However, assuming that Sarge stability is overkill for a home desktop, what would someone stand to gain from running Ubuntu as opposed to Debian Etch (testing)?  Is Ubuntu about the ease of transition from Windows to Linux?

My other question is, if Ubuntu is basicly Debian with Windows like ease of use. Then why can’t this be a Debian package? Apt-get install usability? Or something like that? 

I hope I’ve caused no offence, I honestly want to know why people choose to use Ubuntu over Debian. I’ve very new to the Linux world and I had no clue that so many distributions existed, I actually didn’t know that there were distributions based on other distributions either! I also suspect I am missing the point somewhere, so feel free to point out the error of my ways.

The main reason I opted for Debian as opposed to Ubuntu (which lots of people recommended) is that I’ve ended up really liking Fluxbox (I discovered it when using DSL), which is much faster on my old Celeron 700 that I am playing with. 

Anyway, trying Ubuntu has been useful. I’ve ended up booting into Ubuntu to show non-geeky friends that Linux is really quite usable and pretty when compared to XP. Perhaps it is just me with my old machine, but Gnome seems like a huge hit for it! As was KDE that I tried on other distributions.

### Post by matthew on 2006-09-18
> **cunawarit said:**
> However, assuming that Sarge stability is overkill for a home desktop, what would someone stand to gain from running Ubuntu as opposed to Debian Etch (testing)?  Is Ubuntu about the ease of transition from Windows to Linux?Polish and ease of installation/use. Debian is excellent at many things, but it has a tendency to try to do everything. Its main userbase tends to be more technical so polish on the desktop isn't a high priority...that's not to say you can't make a Debian and Ubuntu desktop equally nice, it's just that the desktop is Ubuntu's focus.

> **cunawarit said:**
>  My other question is, if Ubuntu is basicly Debian with Windows like ease of use. Then why can’t this be a Debian package? Apt-get install usability? Or something like that? The Ubuntu developers go for bleeding edge stuff from the Debian Sid repos, work out the bugs, and every six months make a stable release. It's quite impressive, actually. In the process, though, sometimes incompabilities arise so you really can't mix Debian repositories with Ubuntu ones...it's more than just adding some pretty features. This page explains it well: [http://www.ubuntu.com/ubuntu/relationship](http://www.ubuntu.com/ubuntu/relationship)

Oh, and you can use [Fluxbox on Ubuntu]("https://help.ubuntu.com/community/Fluxbox") if you want to. Either way, welcome to our forums!
