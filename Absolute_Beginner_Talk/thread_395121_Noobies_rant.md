I had this problem, and here is what fixed it for me.

Edit this file:
```
sudo nano /etc/hosts
```

You will see:
```

127.0.0.1     localhost
127.0.1.1     hostname

```

Edit the file so it looks like this:
```

127.0.0.1     localhost     hostname
127.0.1.1     hostname

```

This seems to speed up the initial launch of Gnome apps for me. Secondly, you need to get direct rendering working.

Type this:
```

glxinfo | grep rendering

```

You will see either this:
```

direct rendering: no

```

Or this:
```

direct rendering: yes

```

If direct rendering is enabled, reboot your computer and see if the first fix alone worked. If it isn't, then you need to install the drivers for your graphics card. Ubuntu makes this process very simple.

For an NVIDIA card: 
```

sudo apt-get install nvidia-glx

```

If it cannot find the nvidia-glx package you have not enabled the correct package in apt's sources.list. You need to enable the "Restricted" package. You can do this one of two ways, either by GUI or by Terminal. 

The GUI Way:
System -> Administration -> Software Sources

The Terminal Way:

First, open the sources file:
```

sudo nano /etc/apt/sources.list

```

Next, uncomment the lines which say Restricted. (Note: uncomment just means remove the # symbol at the beginning of the line)

Now you should be able to apt-get the nvidia-glx package.

When this package is done installing, you need to set it up:
```

sudo nvidia-glx-config enable

```

Now, either restart Gnome by doing:
```

sudo /etc/init.d/gdb restart

```

Or (because some people get a blank screen from doing that), simply reboot:
```

sudo reboot

```

Let me know if this helps at all!

### Post by igknighted on 2007-03-28
I'm curious why you dismiss Swiftfox.  It **is** firefox, just someone took the time to compile it with the specific cflags for your processor.  All firefox themes and extensions work as normal.  If you go to their site ([www.getswiftfox.com](www.getswiftfox.com)) and get the installer script, right click the file and go to permissions, and check the box to allow it to be executable (or use chmod +x <scriptname> in the terminal).  Then cd to that directory and sudo ./<installscript> (to install it as root).  It will install to /opt so it wont interfere with the FF already installed.  It also will import all plugins and themes and extensions you are already using, and put in a new menu entry.

You will probably see a performance upgrade due to this, it was fairly noticable to me.  I have since ditched the whole *fox line because they really are bloated (especially in windows where it takes 10+ seconds to load) and use epiphany in Gnome and Konqueror in KDE.  Both are great apps, but much more lightweight, so they may not be your thing.
