### Post by dfreer on 2007-04-25
I'm thinking it's because you are using a live cd (which runs completely in RAM), on a machine with only 256 MB's of RAM. Ubuntu will run fine on this machine (once it is installed), just the live cd can be slow with that amount of RAM. Try using the alternate install cd, that should help you out.

### Post by russell.h on 2007-04-25
You might also try an alternate CD, which has a sort of a "text based graphical user interface". A lot of people freak out at the sight of "text based", but it isn't really text based, just a console version of the GUI. 

Don't worry about boot times on the Live CD, those are always absurdly slow. Once you actually get it installed you hear different things from different people, but I think most people (I do on all 3 of my computers) find that Ubuntu takes longer to get from being off to the login screen than Windows (XP), but one of the first things Windows starts is its interface, then it continues to load stuff in the background which is why a windows installation, particularly an old one (a new XP installation is surprisingly quick) tends to sit there churning for a few minutes after you turn it on, whereas in Ubuntu it loads the interface last (or close, I don't know), so when Ubuntu goes to the login screen its really ready to go.

But like I say, all of that can come later, get it installed first, until then expect *very* slow load times.
