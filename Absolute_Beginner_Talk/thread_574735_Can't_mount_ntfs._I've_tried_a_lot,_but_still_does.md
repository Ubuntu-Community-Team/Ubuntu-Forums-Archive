### Post by steve.horsley on 2007-10-13
Your entry in fstab for sdb5 is wrong.

 Column 1 is the partition identifier, either a device name like /dev/sdb5 or a logical partition label line UUID=blah... I don't know if you have the right UUID here, but you can check with the command **blkid**. Personally, I find using /dev/sdb5 easier, but the idea of UUID is that if you swapo the disks around, it still finds the right one.

Column 2 is your problem. It says where the partition is to be mounted - which directory its contents will appear in. This would normally be /media/someName or /mnt/someName. It shoudl NOT be /dev/sdb5 which is of course the device driver node for the partition. 

Still, you're doing well for a beginner. Have fun.

### Post by andguent on 2007-10-13
Agreed with above. The second column in /etc/fstab should usually be an empty directory ready for new files to be attached to it. Try changing /dev/sdb5 to /media/sdb5, and make sure that directory exists and is empty (mkdir /media/sdb5).
