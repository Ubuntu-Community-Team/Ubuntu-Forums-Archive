### Post by luptinpitman on 2007-09-17
cknight - A distribution, as far as I understand it, is simply a compilation of freely available packages that are put together and tested and then released with branding to make it discernible from the other distro's.

As you mention, anyone can just install a kernel, a desktop manager, and what ever applications he/she wants, but a distribution comes with a certain amount of testing and compatibility that you wouldn't get from a 'roll-your-own' type installation.

Distro's also are also categorized by the package management system that is used.  For instance, Ubuntu is based on Debian therefore it uses the Debian package management system.  The fact that one distro is based on another is quite the mind-bender on top of it when you are trying to get a handle on "what is ubuntu".  :)

I am still learning the ins and outs myself but this is the best way I can explain it...

### Post by mocoloco on 2007-09-18
So really your question is "what is a distribution?"  Like the word implies, it's a set of software that a person or group decided to distribute.  It's the kernel and modules, software (packages), and settings they liked or considered most useful.  Anyone could make one, and really anyone could build their own Ubuntu distro from scratch and end up making it exactly the same, although of course it would be a lot of useless work since it already exists :).

One of the differences I like with Ubuntu's approach is having separate distros for each DE (Desktop Environment).  Most distros like Suse, Red Hat, etc have you choose one on install, but consider it all part of one distribution, which would be confusing for users in Ubuntu's target demographic.

Along with the artwork as mentioned by gn2 maybe some of the documentation? (at the very least the parts that mention Ubuntu, right?).  As I mentioned above, default settings and default installed apps are the big differences between distros.  Here are some that are unique compared to most distros:
[LIST]
[*]Very simple installer with guided partitioning, Windows settings & files importer, and few questions
[*]No separate root account, user does root stuff using sudo only
[*]For the most part only one app for one function (many distros have two browsers, 4 irc clients, 3 media players, etc)
[*]Restricted driver's manager, prompts to install mp3 and other codes.  There are distros that just include that stuff, but for a distro that strives to stay "free" out of the box they compromise on making it easy to get non-free packages for the sake of functionality
[*]Cool sounding name :) (I know, that's opinion not fact)
[/LIST]
Sure you could take almost any distro and set it up about the same way.  That's one thing I've learned, in the end Linux is Linux.  You seem like you're searching for the secret ingredient that makes it different; there isn't one. I could tell my friend  "take this Gentoo CD and try it out, except you'll have to make sure you create your partitions just so, because it doesn't guide you, and you'll want to edit the sudoers list ... blah blah blah .... use emerge to remove some of these packages .... blah blah ... manually install binary video driver and edit xorg.conf ... blah blah blah .....download this cool human theme from gnome-look ... blah blah etc etc".  

Or I could say "pop in this Ubuntu CD.  I know you've never seen Linux.  You'll figure it out."

If there's anything that answers your question I would say that Ubuntu's uniqueness is the intangibles, like the community, and the cool name that cleverly captures that community spirit and the free software philosophy.  I think I'm beating a dead horse now, Ubuntu's fun.  Linux is fun.  It's just a bunch of stuff people did and they share it.  That's all, but it's enough.
