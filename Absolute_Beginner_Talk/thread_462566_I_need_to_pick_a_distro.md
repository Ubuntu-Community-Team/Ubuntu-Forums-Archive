### Post by pauligit on 2007-06-02
Hi fellows!

I started using Linux back in 2001. My first distro was SuSe, I guess 7.x. I remember I could never get my onboard cmi8330 chip to work using YaST. SuSE configuration software (Yast and Sax) was pretty buggy by that time. So I switched to Slackware. And I liked it. It was far easier for me to configure all my hardware under Slack. However, one of the main weaknesses of it is the lack of official packages for some software, and the lack of dependency management also. The former leads to strong limitations when it comes to installing lots of non supported software and trying to get stable... In this case you have to update one by one that software, which becomes an impossible task. I've always envied apt :)

Now I don't have as much time as I used to have during those highschool days. And I think I must move from Slackware.

So I need to find a distro with a good packages system, a somewhat easy to configure but not so dependent on specific software one (such as SuSE, for instance). One of the things I appreciate from Slackware is simplicity. Having a good, end-user configuration software should not be an obstacle for command line configuration.
Other important points are:
a) Rationality when it comes to hardware resources and neat loading
b) Good documentation online
c) An open, non-sectarian community.

Slackware used to have those three. Its community is particularly open-minded and willing to help. I don't want to start a distro war here, but one of the reasons why I've always seen Debian with reluctance is its community tendency to look down on newbies (I used to be one) and with people not belonging to their cult following.

Today I tried OpenSuSE LiveDVD, which I disliked. In fact it didn't configure my audio card properly and while trying to record audio it sucked. Besides, CD-based live distros such as Slax or ZenLive were really more appealing in terms of esthetics and user friendliness. 

So my question is: which distro should I use? As I said above, I'm a sort of experienced user in search for a desktop distro. Ubuntu, Debian or what else?

Note: I have an AMD 64 processor.

Cheers,

Paul

### Post by pauligit on 2007-06-03
You said right! I'm in search of an off the rack distro, without much things to configure.
Getting Slackware to work is no big deal though. I'd say it's indeed easier than setting up Debian. I usually copy&paste my xorg.conf from my past distros and as it comes to my audio card, alsaconf does the job just fine without me having to manually load the kernel module for it and setting irq's as I did in my old computer. Again, configuration is not the problem of my second and most loved distro, Slackware. It is keeping up stable; it is being able to chose from a wide variety of packages without having to compile or rely on some unsafe package of your own (an .rpm or .deb turned into a .tgz using some [shell script]("http://kitenet.net/~joey/code/alien/")) when it comes to installing some daily use software (ie, there are not official Slackware packages of most p2p software). And the same happens as regards dependency management, which Slackware lacks. This turns ordinary computing into a very messy thing. Slackware may remain good for servers, in which being _minimalistic_ counts.

The main reasons why I'm willing to switch to some .deb based distro (and I'm seriously considering Ubuntu and Debian) may be similar to those of a starter who seeks for a "transitional distro" without messing with dark screens and so forth. However, judging for what I could see in Ubuntu Live, it can perform all the things any other distro can: you can switch to the shell terminal if you want and you can even compile from scratch everything. At the same time, it offers a rational, non compulsory set of graphical tools for doing the admin chores easily.

In fact, I couldn't really see many differences between it and Debian, except for "aesthetic" ones.
