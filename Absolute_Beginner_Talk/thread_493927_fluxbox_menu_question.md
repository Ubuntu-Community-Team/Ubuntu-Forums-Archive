The menu file should work fine. It sounds to me like you have created a permissions issue.

What is in the right-click menu when you run as an ordinary user? Anything?

The ~/.fluxbox/menu file 

```
-rw-r--r-- 1 user user 3.8K 2007-07-03 19:55 menu
```should have this in it:

```
 [begin] (fluxbox)
[include] (/etc/X11/fluxbox/fluxbox-menu)
[end]

```Here are the files in my /etc/X11/fluxbox:

```
-rw-r--r-- 1 root root 6.5K 2007-07-03 09:36 fluxbox-menu
-rw-r--r-- 1 root root   66 2006-09-28 11:21 fluxbox.menu-user
-rw-r--r-- 1 root root 1.4K 2006-09-28 11:21 init
-rw-r--r-- 1 root root  306 2006-09-28 11:21 keys
-rw-r--r-- 1 root root 6.0K 2007-07-03 09:36 menudefs.hook
-rw-r--r-- 1 root root  535 2006-09-28 11:21 system.fluxbox-menu

```Check the permissions.

The fluxbox-menu file should have 'read' permissions for the user. It is generated by running:

```
sudo update-menus
```

EDIT: Just out of curiosity, is there a .fluxbox directory in /root?

not using a window manager always gives problems in ubuntu. my advise, install gdm, delete the ~/.fluxbox folder, it will create a new one. if you think not using a login manager saves you resources, your wrong it uses a extra process and more ram, the only benefit is space saving. ubuntu so much so depends on gdm that when you don't use it, you get all these little problems that eventually add up. your probably not noticing it now, but you will see the signs like several xserverauth, xsession-errors filling up, etc...

### Post by RedSquirrel on 2007-07-07
> **blastthisinferno said:**
> When I try 

ls -l /home/username/.fluxbox/

I get Permission denied.  So since I can't even read that directory, do you think thats the problem?

It sounds like that .fluxbox directory is only accessible by root and not your regular user.

Try this:

Exit Fluxbox. You should now be on the console logged in as your regular user.

Rename that .fluxbox directory:

```
sudo mv ~/.fluxbox ~/old.fluxbox
```Run startx as your regular user. 

If you are starting fluxbox with /usr/bin/startfluxbox, the startfluxbox script will recreate the .fluxbox directory for you and it should have the correct permissions. (Most people put /usr/bin/startfluxbox in their ~/.xinitrc, which is probably what you have, right?)

@kerry_s: I think the problem in this case has to do with blastthisinferno logging in as root and creating files and directories as root (and hence, with root permissions). Installing gdm does make some things a bit easier, especially since it is setup to run /usr/bin/startfluxbox, which is what one should use. A lot people think you start Fluxbox with /usr/bin/fluxbox, but that is not what the developers had in mind. ;)
