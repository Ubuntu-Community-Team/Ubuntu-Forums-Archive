### Post by pytheas22 on 2008-02-08
Yes, you do have to override a BIOS mode.  You have to write your configuration to a file in order to get it to work and specify which mode to override, and with which resolution.  In a terminal type

```
sudo gedit /etc/default/915resolution
```

and add the settings you need based on the resolution you want.  You have to choose a mode number to overwrite--choose a mode that you would never use, in other words choose a really low resolution--and you set the resolution that you want to overwrite that mode number to.  For instance, here is my 915resolution file.  I overwrote mode # 41, which by default is 600x800 pixels, to 1440x900 pixels:

```
#
# 915resolution default
#
# find free modes by  /usr/sbin/915resolution -l
# and set it to MODE or set to 'MODE=auto'
#
# With 'auto' detection, the panel-size will be fetched from the VBE
# BIOS if possible and the highest-numbered mode in each bit-depth
# will be overwritten with the detected panel-size.
MODE=41
#
# and set resolutions for the mode.
# e.g. use XRESO=1024 and YRESO=768
XRESO=1440
YRESO=900
#
# We can also set the pixel mode.
# e.g. use BIT=32
# Please note that this is optional,
# you can also leave this value blank.
BIT=
```

Then restart your computer and you should be able to use the resolution specified in your 915resolution file.  I know that this can be confusing if you've never done it before, so feel free to post if you have problems.

### Post by pytheas22 on 2008-02-09
glad you found a solution.  To apply the xrand command each time you start X, open up gedit (Applications>>Accessories>>Text Editor) and create a file with the following lines:
```

xrandr --newmode "1280x1024_60.00"  108.88  1280 1360 1496 1712  1024 1025 1028 1060  -HSync +Vsync
xrandr --addmode TMDS-1 "1280x1024_60.00"
xrandr --output TMDS-1 --mode "1280x1024_60.00"
```

Save it to your /home directory with a name like "fix-resolution.sh."  To make sure it's executable, navigate to the file in Nautilus (the Gnome file browser), right click on it, select Properties, click on the Permissions tab, and make sure the "Allow executing as program" box is checked.

What you've done is create a very dirty BASH script.  It's not a proper script, but in Ubuntu it should be close enough to work.

Now, so that this script gets run every time X starts, go to System>>Preferences>>Sessions.  In the Sessions dialog box, click Add, and on the line labelled Command, enter
```

sh ~/fix-resolution.sh
```

You can name this task whatever you want.  It will be run now every time you log in to Gnome.

As with most things in Linux, there are plenty of other ways to do this, but this is the simplest solution I think.  Note that if you have multiple user accounts, you will need to repeat the steps above for each one, because "Sessions" is user-specific.  If this doesn't work let me know.
