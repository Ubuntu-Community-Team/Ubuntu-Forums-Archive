### Post by aysiu on 2007-02-18
When you want to add Kubuntu to Ubuntu, you are not upgrading... you're adding. Upgrading means changing from an older version (say, 6.06) to a newer version (say, 6.10).

Both older and newer versions have Kubuntu and Ubuntu (and Xubuntu and Edubuntu). So you could have Ubuntu 6.10, but you can also have Kubuntu 6.10. They are the same version--one is not an upgrade of the other. But Xubuntu 6.10 *is* an upgrade of Xubuntu 6.06.

All the *buntu "distros" are really the same Linux distribution with different default programs and environments. Xubuntu is Ubuntu with a default XFCE desktop environment. Ubuntu is Ubuntu with default Gnome. Kubuntu is Ubuntu with default KDE. Edubuntu is Ubuntu with Gnome and some educational applications.

Other distros like Debian and Red Hat are actually **other distros**. They are not Ubuntu with a different set of default applications. They are different in structure, installation, maintenance, release cycle, and software sources.

### Post by aysiu on 2007-02-18
> **Patrick K said:**
> aysiu, how different is Debian? Is it majorly different or minorly different? I understand that Ubuntu is based on Debian.
Well, that kind of begs the question: How different are *any* of the Linux distros from each other?

As far as I know, distros differ only in these respect:

Funding
Release cycle
Distribution
Cost
Package management
Default packages
Installation choices
CD availability
Folder and config file organization
Support
Community

Even though Ubuntu and Debian share the same package management system, they don't share the same repository set and may not have exactly the same config file organization. I believe Ubuntu is a modified version of Debian Etch ("unstable"), but I don't know all the technical details, and I'm not a programmer.
