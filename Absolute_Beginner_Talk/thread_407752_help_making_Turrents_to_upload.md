### Post by tonyr1988 on 2007-04-12
First, welcome to the forums! I hope that you are happy with Ubuntu once you get the initial things worked out (most people are :D).

**Torrents**

You have a couple of options with torrent files. Three popular torrent programs that all allow you to make torrents (I believe) are:

- Azureus. It has a ton of features, but it also uses up quite a bit of memory. If you have a slow computer, I wouldn't recommend it. You can install it from Applications >> Add / Remove

- KTorrent. A lot of people like KTorrent because it's got a pretty simple layout. It is also available in Applications >> Add / Remove. However, since it's KDE (and you use Gnome), you'll have to download some extra files (it's really not a huge deal, though - just go grab a coffee and come back to your computer in a little bit :D).

- uTorrent + wine. Unless you used uTorrent under Windows, I would avoid this. It could just cause you more pain than it's worth.

They are free, so try them. I personally use Azureus, but you may want to try KTorrent first. If it doesn't have enough features for you, try Azureus. You can always uninstall them through Applications >> Add / Remove. If neither of those meet your needs, just say so and we can give you a few more. Those just happen to be the most popular ones, and will work for most people.

**Burning**

Go to Applications >> Accessories >> Terminal and type (or copy / paste):

> sudo apt-get install k3b

Type in your user password when it asks, and accept it when it asks you to download extra packages. It will be worth it. Many people (myself included) feel that K3B is the best CD / DVD Burning Application for any operating system. I used to *love* Nero, and now I think it's pale in comparison to K3B.

Of course, if you find that you don't like it, let us know and we can tell you how to uninstall it and give you more alternatives.

**Opening .exe**

You can do this under Linux, but they often will not work. Like CJay said, a program called wine can help. But it can be a pain to get certain programs working. When possible, try to get applications *native* to Linux (aka "built for Linux").
