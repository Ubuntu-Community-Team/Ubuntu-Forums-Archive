### Post by benanzo on 2007-08-04
I really like Sound Juicer for ripping CDs.

Have a look at [BasKet]("http://www.asktheadmin.com/2007/07/microsoft-onenote-for-linux.html") for note taking.  It's a KDE app.  Otherwise you could use Google Notebook.  They've got a neat [Firefox plugin]("http://www.google.com/notebook/download") so it integrates really well.

For podcasts and managing your iPod you can use Banshee as kind of a one-stop-shop.  Or you can use GTKPod (managing iPod music/video/podcasts) + gPodder/iPodder to download podcasts.

### Post by badactress on 2007-08-04
Hey! Yep.. Soundjuicer is fantastic. Yr right that by default it doesn't rip to mp3, but it's very easy to set it up so it does.

The latest version that comes with fiesty already has the settings in there ready to go. You just need to install lame mp3 encoder to activate it. Open console & type sudo apt-get install lame

if you have an older version, this thread tells you how to add mp3 encoding to the options:

[http://ubuntuforums.org/showthread.php?t=295698&highlight=soundjuicer+mp3]("http://ubuntuforums.org/showthread.php?t=295698&highlight=soundjuicer+mp3")

Hope this helps! :)

### Post by drobvice on 2007-08-12
Video podcasts won't be as simple as iTunes.  I have this problem and I haven't found a good solution for it.  My needs are: download and keep the 2 most recent podcasts and sync to iPod.  Most software cannot keep 2 most recent and delete the rest and if they can, the naming schemes/description will not be as nice as iTunes as it will be mainly the filename and may not be in the 'Podcasts' section on the player.  Amarok is pretty good about auto downloading but I had trouble with the iPod database and never was able to sync.  Floola is a good option and is fairly automatic in downloading, consistent file placement and naming.  It is constantly updated and getting fairly stable.  Might take a look at gpodder as well.
