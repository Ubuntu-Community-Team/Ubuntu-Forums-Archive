### Post by emarkd on 2008-02-14
[Wine]("http://www.winehq.org") is completely legal.  It's a project that started from scratch reimplementing the Win32 API in Linux and it's licensed using LGPL.  There's no MS code there since MS doesn't want to participate.  As for the software you run using Wine, each piece is subject to it's own EULA and you'd have to read them all to see what they say ;)

Most of the restricted drivers are not an issue of legality but one of freedom.  Ubuntu doesn't ship with non-free (non open-source) drivers enabled because if there's a bug in the driver Ubuntu can't fix it or provide support for it.

### Post by jaytek13 on 2008-02-14
There aren't any applications or codecs that are illegal themselves. Most of the time it's actually their usage that is illegal in some countries. Wine is totally legit. MP3 and DVD playback codecs are really the only thing that would be called into question, and in either of these cases it depends on the country in which you live, and for both of these cases there isn't really a suitable alternative. You could use .ogg instead of .mp3 and convert any mp3 files you have to ogg, which would make it "legal" regardless of where you lived. But as far as DVD codecs, you either install them or you don't, and if you don't you can't play DVD's.
