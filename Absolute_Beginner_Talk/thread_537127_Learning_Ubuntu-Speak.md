### Post by dabugas on 2007-08-28
Every console command has a manual which can be accessed by typing "man <command-name>" in the console. For example, to find out what all that stuff after grep is type "man grep".

The reason why no one tells you what the commands do is because a) you don't need it if you just want the damn thing to work, and b) some of it is fairly complex.

I've been using Linux for years and I probably know only 10% of the switches of the dozen or so commands that I use frequently. In other words, you don't *have* to know :) But it's good that you do, because the more you learn about the OS, the more you will be able to control it -- the console is a very powerful tool.

For example typing "grep "hello" * -ir" at your home directory in the console will list all the files in your home directory that contain the word "hello".
