### Post by dwblas on 2007-04-09
There's not much information there.  Check /etc/fstab and see if that's in there somewhere.  If it occurs on boot, then it probably is an error in the fstab file.  This command will tell you what you have and anything not listed should not be in fstab.
ls /dev/disk/by-uuid/ -alh
If you can't figure it out, post back with a listing of /etc/fstab as well as your disk configuration, hda1, hda2, sda1, etc.

### Post by Austin_KW on 2007-04-10
Your uuids dont match 3240F33940F30301 is now 92680E456BC846A3.They are just links to the real device partition /dev/sda6. I think it hapens when you repartion, resize partitions etc. 
Fix it in ftsab by either changing to the correct id or replacing UUID=3240F33940F30301 with /dev/sda6

edit, you then do not need to reboot "sudo mount -a" to test, will mount the filesystems in fstab

### Post by Austin_KW on 2007-04-10
/etc/fstab is just a text file that is read by the mount program to mount the filesystem in linux. Just about everything in linux and unix based operating systems is controled by config files. You just edit them to make changes.

Start a terminal (menu applications->accessories->terminal)

in the terminal type 
```
 gksudo gedit /etc/fstab
```
change **UUID=3240F33940F30301** to **/dev/sda6**
save
quit
then in the terminal test that the mount works (you should now not see an error for dev/sda6)
```
sudo mount -a
```
