### Post by HermanAB on 2007-10-04
It is important that you choose something appropriate for the amount of use that it will get.

GnuCash is similar to QuickBooks and can be used for a single user up to a small business, but neither of these are good for a medium size business with multiple offices, because they don't work over a VPN.  Also, the data file will eventually become very large and the system will slow down.

For a medium to large business, you should use SQL Ledger, since it has a database back-end.

I've tried GNUcash and KMyMoney. (I wouldn't touch Moneydance with a barge pole because it is a ghastly Java application.)  Neither is anywhere near as good as Quicken so I now have my old Quicken Deluxe 98 running on Wine.  Essential functions work OK except transaction splits. (A workaround is to enter 2-way splits as 2 transactions, etc.)  There are a few quirks about starting Quicken. When you run the application it hangs on the splash screen but you can dismiss it with the Esc key. You then have to use File to select and open your data file. You cannot backup to floppy using the Quicken menu or buttons. However, all you need to do is backup these 4 files which should be in the /home/yourusername/wine/drive_c/quickenw folder :

Q3.DIR  QDATA.QDF  QDATA.QEL  QDATA.QSD

These are the files copied in a normal Windows backup. **I suggest you copy these into your /quickenw folder before first using Quicken and then all your data from the Windows installation will be in place.**  Finally, don't try to open Preferences>Reminders because it will crash Quicken!  Just live with the default setting.

If you decide to go for GNUCash or KMyMoney I would definitely recommend KMyMoney. I am currently running it in parallel with Quicken in the hope that it will improve over time.
