### Post by ddrichardson on 2007-08-21
Just my 0.05p.

I recently installed Damn Small Linux on a similar machine: Pentium 200 MMX, 32 Mb RAM and 2 Gb Hard disk.

There were a number of problems I encountered. First off, boot DSL with the```
dsl noicons
``` option - it significantly reduces memory usage.

Second, you will need a custom [boot]("http://www.damnsmalllinux.org/wiki/index.php/Boot_Floppies") disk to point to the CD.

Thirdly, you will need a significantly large swap space. I put a 256 Mb disk on as soon as I had partitioned. Remember that with low memory you need to be using fdisk. To get swap from a root terminal up:```
mkswap /dev/hda2
swapon
```

Install to hard drive and off you go. I installed the XFree86 packages, but am dubious as to whether there is a benefit - the memory footprint with the vesa framebuffer driver is lower than the increased speed from XFree86.

Also bear in mind that if you use XFree86 you need to know the video card chipset, memory and the monitor's horizontal and vertical refresh rates.

Well mine is up and running. I don't know what to do with it. It's fast enough to browse the web with Firefox though. Abiword also runs reasonably well.

Lastly, although this is probably not too relevant - wireless support in DSL is poor. Ndiswrapper is terrible in DSL.
