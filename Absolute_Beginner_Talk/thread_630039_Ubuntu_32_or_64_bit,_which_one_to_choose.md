### Post by Rhubarb on 2007-12-02
64bit certainly does out perform 32bit especially with regards to encoding video + audio.
64bit can run 32bit programs, 64bit can play flash animations in firefox easily.
64bit also has just as many apps in the repositories as 32bit does.

Really, there is no reason to run 32bit if you have a 64bit capable PC.

### Post by Paqman on 2007-12-02
> **mdpalow said:**
> I read somewhere there is no flash support, and less software currently in the repositories - I decided to go with the 32-bit.


That's not actually true.

There used to be no Flash support for 64-bit. The solution was to use the 32-bit version of Firefox. Now however you can simply install flashplugin-nonfree from the repos and it automatically wraps it up nicely for 64-bit use.

Personally, i've only very rarely bumped into a package that was available for 32-bit but not 64-bit. If you've got a 64-bit chip you might as well use it to its full potential.

### Post by Rhubarb on 2007-12-02
> **willie_wang said:**
> key applications such as deluge, and some firefox addons don't support 64bit OSs. it was really annoying when deluge would keep crashing so that was the main reason for my switch anyway.
I've been running Deluge here on my 64bit install with no problems at all.
If you can, always use the latest version of Deluge from here --> [http://www.deluge-torrent.org/](http://www.deluge-torrent.org/)

There is a lot of FUD going around with 64bit not being good / stable enough.

I have a few firefox plugins that work perfectly, I have installed google earth, I can play games such as quake wars, I can run a virtual machine running xp or vista, I can install wine to play windows games too.
In fact there is no difference between 32bit and 64bit except that 64bit runs faster and can address more than 4GB of RAM.

There are a few open source games (like sauerbraten and assault cube) that don't offer 64bit executables on sourceforge, compiling these games is very easy to do and presents no problems.
