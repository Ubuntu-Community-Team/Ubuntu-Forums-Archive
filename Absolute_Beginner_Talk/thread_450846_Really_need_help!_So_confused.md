### Post by snakedr on 2007-05-21
Gnome is a graphical user interface. There is also a graphical user interface called KDE. Those are the two interfaces most use. Without these user interfaces you would be stuck with a command line interface. The graphical user interface allows the user to interact without having to use the command line. 

Those files opening in the archive manager are compressed archive files like a .zip file that you might be more familiar with. Source code is usually packaged that way.

If you are confused about compiling programs I would stay away from doing that until you become more familiar with linux. Programs that you compile or build from source code often require other dependencies (other programs) that are required to build the program. You will become discouraged quickly if your build fails for some unknown reason if you are not sure what you are doing.

### Post by celticbhoy on 2007-05-21
I think that you basically have to take a small step back. I first started to use linux in Feb of this year, and I now use it almost exclusively on a dual boot system with vista. You say that you are pretty smart with computers, but I am guessing that what you really mean is that you are an experienced user, with the operating systems you have come across up until now.

It has been my experience that the biggest difference between linux and (windows, OSX ) is that linux does not try and make decisions for you. You can make your computer whatever you want it to be. But because of this freedom, you have to put in a little more work. Take things slowly at first. go through your computers hardware bit by bit searching the forums making sure that everything is installed the way is should be. Then start to search for software through the repositories that lets you do what you want.

By this time you will have learned the basic principles of Linux, and how to overcome minor problems youself, and use the forums for more complicated issues.
