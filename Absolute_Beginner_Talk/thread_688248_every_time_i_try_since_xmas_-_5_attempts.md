### Post by rkd on 2008-02-07
VMware creates a simulated PC whose simulated hardware is completely unrelated to the actual hardware of the PC you are running it on.  And that simulated PC is the same, no matter what PC you run VMware on.  Many people run Linux in VMware, so you will be able to run Ubuntu in VMware. Running Ubuntu in VMware is not exactly like running Ubuntu on the real hardware, so it may not be trouble-free, but the problems will be different than the problems you are having now.

HP's suggestion about win.ini (are you sure it wasn't boot.ini?) has nothing to do with booting from a Live CD.  And although it is one way to set up dual boots, there are other ways that don't require changing anything in Vista to set up dual-booting.

Be careful about trying to wipe out Vista and replace it with XP, unless HP can provide XP versions of drivers for the hardware on that PC.  There have been cases in which some new components have Vista drivers, but no XP drivers.

The claim that Vista doesn't allow dual boot is just, plain wrong. Many people dual boot Vista with XP and with various Linux distros. Whoever told you it can't be done is wrong.  Perhaps HP Support told you that HP doesn't support you if you do that -- that's a completely different matter.

The problems you had with the display when running the Live CD would not be changed if you replaced Vista by XP.

I don't know why you are having so much trouble. It might just be that you need someone familiar with Linux to help you get started, and Linux Users Groups do a lot of that. If you want to contact a Linux Users Group and don't know how to find one, tell us what city you are in and we'll see whether there is one nearby.
