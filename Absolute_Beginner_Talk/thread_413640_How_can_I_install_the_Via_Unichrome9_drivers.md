### Post by nasos on 2007-04-22
I got a couple of hours of scratching to get this working. Here are some info that my beuseful to you:
0. I installed the drivers, while having enabled root password: this can be dangerous, but googling a bit will help. links:
[http://www.debianadmin.com/enable-and-disable-ubuntu-root-password.html]("http://www.debianadmin.com/enable-and-disable-ubuntu-root-password.html")
1. The only thing that looked smoothed was the openchrome how to guide:  [https://help.ubuntu.com/community/OpenChrome]("https://help.ubuntu.com/community/OpenChrome")
2. Make sure which of the 2 drivers you need (in my case it was the K8M890)
3. After installing the driver and having setup the xorg.conf file, do a system reboot (prefferred) or a gdm restart (ctrl-alt-backspace) <--- this will restart your desktop, so have everything closed.
4. open up a terminal and type:   
```
lspci  
```
If via driver is running you will see  lots of via related stuff.
5. If you have gone through 3D installation in the above guide, execute in the terminal: 
```
glxinfo | grep render
```
(you should see direct rendering: YES) - in my case glxinfo has a problem, but everything else works ok.
and then:
```
glxgears
```
(you should see SMOOTH rolling gears)

6. IN ANY CASE IF SOMETHING GOES WRONG AND YOU SEE NO GRAPHICAL INTERFACE:
open up xorg.conf and edit the  Driver "via" directive to Driver "vesa", by doing:

sudo nano /etc/X11/xorg.conf
go
```
Section "Device"
```

change driver "via" to Driver "vesa"

7. Have in mind that if 3D is not working for you, or causes problems to the system you have to fire out DRI (Troubleshooting in Openchrome how-to).


hope those scattered hints will help you out there. Good luck.

### Post by nasos on 2007-04-23
The empty space is tad weird. DO the following:

1. Go into the /etc/X11 directory
2. ls the files and see what xorg.conf...... files exist. There must be at least some backups. If xorg.conf exists then it probably contains something, look at the end of this post.
3. restore one of the older ones by doing:
```
sudo mv /etc/X11/xorg.conf.270420071145  /etc/X11/xorg.conf
```
(the 27042.... is just an example)
4. If no xorg.conf file exists you will need to reconfigure x-server by:
```
sudo dpkg-reconfigure xserver-xorg
```
this will get you into a menu that you will need to do various choices. Keep all as default but:
CHOOSE VESA AS VIDEO CARD DIRVER.
5. When you restart ubuntu you get your X back. Restart the process as desribed in openchrome for K8M890.

Another reason that xorg.conf is empty is that you don't sudo. Try to make sure that you open the file as a super user or with sudo command.
