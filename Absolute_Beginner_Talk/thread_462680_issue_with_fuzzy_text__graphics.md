absc,

The following section  of xorg.conf needs to be edited in my opinion.

[B][I]Section "Monitor"
Identifier "AS4637"
Option "DPMS"
EndSection[/I][/B]

At least the following lines in bold and italic should be added:

	***DisplaySize	h  v***
Where h=horizontal size and v=vertical size in millimeters
	***HorizSync	n-x***
Where n=the minimum horizontal sync rate in hertz and x=the maximum
	***VertRefresh	60-75***
We know those values, so put that in as is. You should be able to choose the vertical scan rate when you change resolutions. Choose a resolution with 60 hz vertical scan.

This is basically how I changed my xorg.conf file.

You have already copied it to your home directory as xorg.conf.txt or something like that. Edit that file in the text editor (Gedit), not a word processor, and make the suggested changes there. Exhaust all possibilities to find the values for h, v, n, and x. Make the changes and save under the same name in your home directory. As an example here is the same section from my xorg.conf file. This was edited by myself to fully identify my monitor.

[B][I]Section "Monitor"
	DisplaySize	596 335
	HorizSync	30-80
	Identifier	"N2751w"
	Option		"DPMS"
	VendorName	"ViewSonic"
	VertRefresh	50-75
EndSection[/I][/B]

After you have saved the file, strike the key combination "Alt-F2" which will open a dialog box to enter a command line. We're going to run Nautilus as super user by entering the following:

gksu "nautilus"

You can also open the terminal and do the same thing.

You'll be prompted for your password, and you will be allowed write privileges in the /etc/X11 folder. Copy your edited file into that folder. Rename the existing xorg.conf to something like oldxorg.txt. Rename the new file you just copied to xorg.conf and close Nautilus, and reboot which is the easy way to restart x-windows.

If somebody knows someway easier, jump in, because this way is not absolutely foolproof.

If for some reason the xorg.conf file is corrupted or unrecognizable, x-windows won't start. You'll only be able to log-in at the command line prompt. Go ahead and login there, then you can change directory to /etc/X11. After getting su or root, you'll need to rename the new xorg.conf file to something else or delete it, then rename the old one (oldxorg.txt) to xorg.conf. KIll all running processes and reboot. You should get back to the graphical log-in, but it will be fuzzy again.

I have to admit, I cheated and gained access to the root account to move my xorg.conf file, but the basic methodology is the same. SU should allow you to do the same thing. If you feel anxious about this at all, don't try this, and by all means, don't create a root account. This is not recommended by Canonical. And being root in the GUI is never recommended by anybody.

Good luck,
D. J. Chandler
