### Post by taurus on 2007-05-07
If you look at the output of "sudo fdisk -l", you should have known that your / partition is /dev/sda1 and it is already mount to / so you can't mount it again to /mnt/External.  On the other hand, your external harddrive is known as /dev/sdb1.

Therefore, try to mount it with

```
sudo mount -t vfat /dev/sdb1 /mnt/External -o iocharset=utf8,umask=000
df -h
```

### Post by rcmullins on 2007-05-08
CONCLUSION:
Finally.   I am posting my end-results hoping this will help others.

[LIST=1]
[*]Problem Definition:
External hard-drive would not mount after changing properties to 'NTFS'.  Found that there were some corrupt files (images) that seemed to be preventing this drive from mounting properly.  I may be understanding the full nature/scope of the problem, however this is what I experienced for the most part.
[*]Backup:
I wanted to be sure to have as much of my info as possible.  I opened up a command-line interface (Applications--->Accessories--->Terminal) and changed my directory to 'media';
cd /media
Once you are there create a directory to mount to, also called a 'mount point'. This is done with the following command.;
sudo mkdir usbdisk
What this does is allows you to create a mountpoint called 'usbdisk'.
Now you mount the disk (provided that the disk is not mounted yet), using the following command;
sudo mount -t vfat /dev/sdb1 /mnt/usbdisk -o iocharset=utf8,umask=000
This will allow you to mount the drive, and you can 'copy/paste' the files you would like to keep or back up.  If you have corrupted files that cannot be kept, Ubuntu will let you know this with a nice little 'I/O' error telling you that those files cannot be kept.
[*]Reformat:
I completely reformatted on a Windows XP as a NTFS file system.  There are many options for this, and alot on the Ubuntu forums on how to do this.  You can reformat the drive with NTFS or Fat32.  Fat32 is possible better if you have a Linux machine *and* a Windows machine that you wish to use the external on.  However there is a great little chunk of code that I am using to manage my NTFS drive on Ubuntu called ntfs-3g that allows you to have read/write support for the NTFS.  So I reformatted my drive on XP with NTFS.
[*]Install ntfs-3g:
Ubuntu Feisty does not come with ntfs-3g pre-installed or configured. So you will need to install it.  You do this from the Synaptic Package Manager.  Select, System-->Administration--->Synaptic Package Manager.  You will need to type in your password for root access and the package manager will load.  Click 'search' at the top and type in ntfs-3g.  Click the check-box for ntfs-3g and be sure that libntfs-3g0 and ntfs-config are also selected.  click 'Apply' at the top and proceed with the installation.
[*]Configure ntfs-3g:
This part is still a bit hazy to me, but this is what I did.  Select Applications-->System Tools-->NTFS Configuration Tool and check the box that says 'Enable Write Support'.  Then close that dialogue.
[*]Work with the Drive:
Now plug your drive in.  Since reformating Ubuntu will most likely be very happy to load the drive.  However you will notice that you will only be in 'Read Only' mode.  The reason this is, is because Ubuntu is not using the ntfs-3g driver to work with the disk.  So right-click on the drive icon on your desktop (when Ubuntu auto-mounts the drive it will put an icon on your desktop by default) and select 'unmount'.
[*]Re-mount your drive:
Open up another command line dialogue and assuming you did not delete or remove 'usbdisk' you can still use that as a mount-point for your drive.  Navigate to your 'media' directory once again and mount the drive using the following command;
sudo mount -t ntfs-3g /dev/sdb1 /media/usbdisk
This will mount the drive using the ntfs-3g driver and give access to all users.  An Icon will appear on your desktop, and now you can write, delete, pretty much anything you want to do with the drive.
[*]Un-mount your drive:
You will not be able to unmount from the desktop by right-clicking and selecting 'unmount'.  Ubuntu will give you an error mentioning that you probably mounted the volume from command line, which is correct.  So you will have to go back to command line and navigate to your media folder to unmount the volume/drive.  Unmount using the following command;
sudo unmount /dev/sdb1 /media/usbdisk
You will recieve 2 messages saying the volume is unmounted.  I am not sure, but you may be able to unmount the drive with either/or /dev/sdb2 /media/usbdisk  which is the reason why you recieve 2 messages with regard to the drive being unmounted.
***IMPORTANT:***
Be sure you unmount the drive EVERY time you are done using it.  Simply unplugging the drive can cause serious problems to your information and content, and could result in you having to read this again and starting over with reformatting.
[/LIST]

Good luck!
