### Post by 3rdalbum on 2007-06-05
Unless you've got over 4 gigabytes of RAM, you probably shouldn't be running the 64-bit edition.

The reason why Java doesn't really work under 64-bit is because, until recently, it was proprietry software and therefore the open-source community could not compile it for a 64-bit machine like yours. The good news is, the 32-bit edition will run perfectly well on your computer, and make everything easier.

If you're relying on proprietry software that does not have a Linux version, and there is no open-source equivilant, then you're not a great candidate for a perminant switch to Linux. Do you really need to run this poker software? If you do, then you should probably install Windows into a virtualiser program like Virtualbox.

I have a Linux program that does not appear to have a Windows equivilant - K3B. I don't know of any Windows programs that can decrypt a DVD on-the-fly without transcoding. So, it goes both ways. I have to reboot from Windows into Linux whenever I want to copy a DVD to play at work.

You do not need to run the installer each time you want to play the game. I don't have Enemy Territory (I've got Return To Castle Wolfenstein), but there should be two programs put into your filesystem; one is for single player, the other for multiplayer. You don't need to go to the terminal to run these.

The analogy of cars is well-worn, and completely worn out. In reply, I'll give you an article to check out: [http://linux.oneandoneis2.org/LNW.htm](http://linux.oneandoneis2.org/LNW.htm).  You're used to the Windows way of doing things, and so you find it difficult to use Linux. I'm used to the Linux way, and so I find it difficult to use Windows whenever I'm forced to. It goes both ways - the operating systems are just different, and that's the way we like it!

### Post by Madpilot on 2007-06-05
Regarding your problems with Enemy Territory, you certainly don't have to re-install it every time you want to run it, and it should appear in your menus, too, so you don't even have to touch the terminal to start it.

[https://help.ubuntu.com/community/EnemyTerritory](https://help.ubuntu.com/community/EnemyTerritory) will walk you through a proper install of ET.

Just as a general practices note, 99% of programs should never have to run with 'sudo' when all you're doing is running the thing. Installation, yes, but not basic running. Overusing sudo to run non-essential programs is a minor (but real) and avoidable security risk.
