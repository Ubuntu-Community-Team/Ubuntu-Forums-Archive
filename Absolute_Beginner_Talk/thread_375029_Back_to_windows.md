BACKING UP DVDs

### Post by Dylnuge on 2007-03-03
Don't give up yet!

Linux is not Windows. Its not harder, just different. Try this site: [http://www.linuxcommand.org]("http://www.linuxcommand.org"). You will learn the command line in days! (If you move on to scripting, be aware it is much harder, and unnessicary).

Also try this site: [http://www.linuxrsp.ru/win-lin-soft/table-eng.html]("http://www.linuxrsp.ru/win-lin-soft/table-eng.html"). Its a full list of Linux equivilents to Windows programs. Its a bit outdated, but shouldn't matter that much.

Finally, experimanet. Linux is about choices. GNOME not your style? Try KDE. Don't like single clicking to open files? Change it. Want Vista-like eye candy? Try Beryl, which coexists with GNOME or KDE now. (Warning: Do not try Beryl until you have a full grasp of Linux. Beryl is beta software. It should work fine, and I have never had it damage anything, but it may not work right away.)

If you remember when you first used a computer, how much did you understand? The keyboard, the mouse (if there was one), it was all rather confusing-right? Windows was like that too-wasn't it?

Well, Linux is the same. It is not difficult, just different. Considering the tasks you are doing, I assume you are an intermediate computer user. You will have the most trouble-an experianced user can figure things out fast, where as a beginner is not tied into Windows enough to need to figure things out. However, you can do it.

Hope you stay, but since the world of Linux is about choices, the freedom is yours. I have more respect for a Windows user who tried Linux then I do for some Linux users-its not about using Linux, its about making a choice-and trying Linux and choosing Windows is a good thing. You decided Linux was not for you-most Windows users don't ever see a Linux desktop in their life.

The best of luck, wherever you go
-Dylan

PS: The full list of commands is not too helpful, since there are over 1200 usually. However, man pages are very useful. So is typing a command followed by --help, it usually gives you some info, and if it does not, it commonly produces an error which tells you what you can type.
