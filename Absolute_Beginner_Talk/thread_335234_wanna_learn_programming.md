### Post by benuski on 2007-01-10
Well, out of the ones you listed at least, python is definately the easiest to learn.  Its an up and coming language with a lot of new things being written in it.  So if you want to start with something a little easier, its a good one to start with.  I really like it cause its a lot cleaner and minimalistic than other languages.  
However, if you want to really undertand the guts of Linux, especially the kernal,, and be able to help change it, you would want to go with C.  Things written that directly interact with hardware are often written in C, as it allows for more pinpoint memory management and control, among other things.

### Post by Seine on 2007-01-10
There's some much to programming that its hard to advise where to start.

Python is a good choice. You'll get a rich set of functions from IO to UI and (importantly) an exposure to OO ... all without the learning overhead of compiled languages. Many Linux apps are written in Python, so you can take it forward and make it your forté.

The other language I'd recommend is Java as it gives you magnificent potential to work as a programmer in business.
