### Post by Hendrixski on 2007-06-26
license:
Most of the software in Linux is licensed under the General Public License (GPL).  It is actually the single most popular software license, period.

If you want proprietary programs, those are available too.  A lot of software is cross platform, so will work on any Unix platform and on WIndows.  If not... there are programs such as Wine, CrossOver, and Cedega which will allow you to run Windows programs on Linux.  If those licenses require you to pay for the rights to that program then you probably should do so.  We're all very much against software piracy.

### Post by Foxmike on 2007-06-26
Maybe a little of history might not hurt here.  KDE began to be developped a bit before GNOME over QT libraries (written in C++).  Those libs where closed source.  Because of that, another project came out a bit later, called GNOME, for GNU Network Object Model Environment.  GNOME is built over the GTK (Gimp Tool Kit) libs that are open source (and written completly in C, for the technality).  After GNOME took more and more place, and with pressions incoming from the open source mouvement, Trolltech, the company that develops the QT libs released the QT libs under a dual licence, with GPL.

Both project were pushed with different philosophies.  At a certain point, there was a need for them to find some way to increase compatibility.  So the freedesktop.org project came in determing standards to follow to ensure interoperability of softwares.

Basically, KDE and GNOME are Desktop Environments.  You can use GNOME applications under KDE as well you can use KDE applications under GNOME.  The only drawback you will get by doing this is that you will carry more libs on you machine, and the loading time will be generally a little longer for the machine will have to load some libraries that are not already loaded.  So yes, there are mechanical differences (kioslaves for KDE, I don't know what is the equivalent for GNOME, for an example) and yes, the choice you will do will depend more on your taste than anything else.

I began with GNOME, and I am now mainly using KDE because 80% of my preffered applications are KDE.

Matter of taste, really, so enjoy!:)
