### Post by Rocket2DMn on 2007-12-23
While all linux distributions are built around the linux kernel, they can vary quite a lot.  The structure of the system remains similar over different distros, but each distro of family of distros uses different programs for installing/configuring everything.  Installs can vary greatly, some distros come with certain programs preconfigured, the size of the install varies, the support for them varies, etc, etc, etc.
GNOME and KDE are simply desktop managers built on top of X and are available for basically every distribution.  You can pick your favorite and put different window managers on top of that, too (like Metacity or Compiz Fusion and so on).  The GUIs are (for the most part) independent of the distro.
Have a look at [http://polishlinux.org/choose/comparison/?distro1=Ubuntu&distro2=openSUSE](http://polishlinux.org/choose/comparison/?distro1=Ubuntu&distro2=openSUSE) for a start.

### Post by mmb1 on 2007-12-23
The common point for all Linux distros is that they all contain the Linux kernel in some form.

Everything else, software, the gui, everything but the kernel, is selected by the distro developers for inclusion, creating differences.

Example: Ubuntu uses the restricted driver manager, Linux Mint uses envy.

### Post by jeffus_il on 2007-12-24
Linux is always Linux, A distro is mainly packaging, which can be very important. A kernel version is packaged, together with all the different sofware packages, including the desktops. Each distro has it policy, its reason for being there, it may be to have all the latest stuff, it may be stability, or maybe ease of installation. Since most of the software provided in distros is open source, each one can potentially have the same packages. The real differences are in the base distros, such as debian, red hat, gentoo etc. they have different methods af installing packages, and different utilities to do that, and also are different mostly in the content of the /etc directory ...

### Post by dnvikram on 2007-12-24
This is a very interesting question even though a simple one to reply to.

Where do I start with this?

:-)

First and foremost,remember two things before deciding on a particular distro/

a)What is the hardware you currently have at hand or which you want to use or isntall on?
b)For what purpose will you be using this machine for?(Primary Purpose)

Once you have the answers for the above questions in place,it becomes easy to choose a distro.

For example,

a)If you have a old and antique 486 or hardware on which your windows 95 also crawls or where you only have 64mb ram or 32 mb ram(worst case scenario) and want to use it for casual internet surfing,listen to songs,edit documents and have an decent GUI/desktop manager ,then it makes sense to go with Damn Small Linux or Puppy Linux which will run like breeze on these antique machines and you literally have to see to believe it.

b)Imagine,you have the latest hardware with cutting edge or bleeding specs and want to use that for full blown 3D effects filled compiz,KDE desktop manager,dvd  playback,ripping audio dvds,editing videos ,use every possible multimedia based application,watch movies,blah blah ..Then it makes sense to go with Suse 10.x or Ubuntu Gutsy Gibbon or Ubuntu Studio or Mandriva or PCLinuxOS.

c)Now if you want to do some serious stuff,like hosting a database or some intensive and massive animation or rendering or application or webserver with zero downtime,then it makes sense to go with RHEL4/CentOS/SUSE which are enterprise class distros for servers.For example,the blockbuster movie SHREK was rendered using a farm of servers running on Red Hat Enterprise Linux.

In My case;I have a Desktop running PCLinux 2007 in which I run a Windows XP VM.Its almost 4years old hardware with 768Mb ram and PCLOS runs truly out of the box on it when compared to other distros including Gutsy Gibbon or Fedora 8 or Linux Mint.

Its the opposite on My Dell Inspiron 1420 notebook.The only distro which is truly out of the box on this hardware is Ubuntu 7.x and 70% of the other distros I tried installing on this machine failed during the live cd phase only.

Similarly,I have a 3 year old Acer Travelmate series laptop with 40gb drive and mere 256MB ram and it runs Fedora 7 flawlessly and with blistering speed after disabling unwanted services.

Then I have this CentOS 4.5(x86-64bit) running on My 64bit Dual Core processor based Dell Optiplex 745 like a charm and have been using it as a ftp server and a db server running Oracle 10g R2 with almost 0 downtime in the last 2months.


Every distro has its own unique + points.Suse for example,is famous for its package management YAST ,its stunning looks and if I am not wrong it was the first distro to introduce the SEARCH feature.
