### Post by Kvark on 2006-05-11
I second the ratpoison reccomendation. It's 100% keyboard controlled.

On the other hand I personally hated the default seemingly random keyboard shortcuts. But after changing all the shortcuts to follow the strict easy to remember rules I wanted it's wonderful to start any program and switch to any open program with keyboard shortcuts.

Ratpoison has another usual trait as well. It uses split screen (also called frames or tiles) instead of windows to split the screen between several programs. It's good when you type in one program and read information or preview the result in another program. Just press one keyboard shortcut and they get half the screen each. To achieve that with windows you need to move around and resize them.

On the other hand starting Gimp, splitting the screen in 3 and resizing the frames so the one with the image gets as much space as possible is more work then just letting the 3 windows pop up. I had to drag everything from Gimp's second window into it's first window so it only has one exta window besides the image. Then make a bash script that made the frame resizing and started Gimp. But after that it was just one quick keyboard shortcut to get Gimp started just as I wanted it.

### Post by kabus on 2006-05-11
A slightly less minimal alternative to ratpoison would be wmii-3, which has some unique features like window stacking, a floating layer (for using Gimp and other broken apps),and deals with the incremental resizing of terminals more gracefully than ratpoison. 
An additional bonus for me is that it uses vi-keybindings by default (ratpoison uses emacs-style).

[http://www.wmii.de](http://www.wmii.de)
