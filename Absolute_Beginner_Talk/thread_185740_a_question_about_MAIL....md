Hello,

A friend of me does share the email-box with his girlfriend. On different computers/accounts, when one opens the inbox with any mail client, the mail gets downloaded to the (local) box and is read there. But when the other opens the mailbox _later_ on the other computer, the *mail isn't on the server anymore and cannot be read*. The only solution is that one forwards the mail again to the own address so that it's again on the server... :-?  pretty annoying.
The mail is on a pop.server.

My question: is it possible to let the mail on the server (or anywhere) where *both* can access it at different times and computers. Only when deleting it from there, it should also be deleted.
If som how to achieve this?

Thanks

### Post by rdd on 2006-06-01
In most e-mail clients there is the option 'leave mail on server'. Look for it in the eMail account settings. That should solve the problem.
