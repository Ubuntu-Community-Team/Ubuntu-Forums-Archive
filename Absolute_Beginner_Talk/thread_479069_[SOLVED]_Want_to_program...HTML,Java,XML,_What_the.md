### Post by Modred on 2007-06-20
Maybe someone can satisfy this curiousity of mine: someone asks to learn to program, and you recommend they learn HTML?  Just because it has "language" in its acronym doesn't mean it's programming.

Anyway, to give a little cleaner difference between these "languages":

Markup languages are used to store and/or present data.  For example, if you have a document titled "Learning HTML", you might use specific markup to denote that as the title.
```
<title>Learning HTML</title>
```
There's really no *programming* involved, just structuring your data in a way that a web browser can comprehend it.  Now there are some variants of XML, such as XSL, that have some programming constructs, but you probably won't be getting into that for a long while.  HTML is more relevant to *design* than programming.

A programming language, on the other hand, provides a set of instructions that are translated such that the computer executes a certain operation in response to the instructions.  For example, you write a few commands, which cause the computer to print "Hello World!" to the command line in a terminal. (Python example below)
```
print "Hello World!"
```
If you want to make *programs*, you'll need to get into a language like Python, Ruby, Perl, C, C++, Java, etc.  Even if you're making web based programs, you'll need something like Python, Perl, Ruby, or Javascript to provide the application logic, since HTML doesn't do logic, just structure.

If you're doing HTML, then yes, a good editor (hence, probably not Frontpage) can help make your life easier, but it's better to type most of the stuff yourself.  That way you actually understand what you're doing.

If you want to learn programming, I'd suggest you start off with Python or Ruby.
