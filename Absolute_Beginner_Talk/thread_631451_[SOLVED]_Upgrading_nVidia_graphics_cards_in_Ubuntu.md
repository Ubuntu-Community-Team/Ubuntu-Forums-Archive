### Post by Hospadar on 2007-12-04
I think that you will be switching from the "nvidia-glx" drivers to the "nvidia-glx-new"

I would first do a "sudo dpkg-reconfigure xserver-xorg" and change your graphics driver to "nv" (instead of "nvidia"), do ctrl-alt-backspace to restart X, make sure it still works, then uninstall "nvidia-glx" in synaptic, then put in the new card and use the restricted driver manager to set up the new driver.

you might just be able to pop in the new card and have it work right away with the new bullet-proof X setup and the restricted driver manager, so you can always try that, but I think you will have to do a little more phaneagling due to having to switch drivers.

(Also, If it turns out that you're already using the "nvidia-glx-new", check synaptic, you shouldn't have to do anything, but i'm pretty sure your card uses the older package)

### Post by _algol_ on 2007-12-08
Ok, figured it out....

...I was using the 'nvidia-glx-new' driver (or something like that), from a repository (i.e.:  installed via Synaptic).  Turns out that particular driver doesn't support the 8800GT (!).

Had to go to nVidia's website, download the beta linux driver (169.4, I think), and then install that.

Now it works OK. :)

Detailed directions are in another post out here:[http://ubuntuforums.org/showthread.php?t=623180&highlight=8800GT]("http://ubuntuforums.org/showthread.php?t=623180&highlight=8800GT")
