i recommend dual booting to everyone new to linux, even if it is ubuntu. but as the previous posters said, 4 GB is really very/too modest to run an OS. also, with older hardware it's not much fun running wine or virtual machines (slow).

### Post by Patrick K on 2007-02-10
W98 natively supports drives up to 137GB (or, IIRC, 127GiB). Whether your BIOS does is another question (check your MoBo manual). An 80 or 120gb drive should be plenty for what you want. I would partition it as 20gb for /, 50gb for /home, give 10GB to windows for an 80GB drive (adjust the numbers as you see fit and what size drive you use). Then dual boot. 

I would use an internal drive, they are much faster than external. W98SE may support USB2, W98 (which is what I have) supports 1.1. Either way, USB is much slower than IDE of any flavor. Externals are fine for backing up but not so great for running an OS from.

EDIT: Your system board may not support USB 2.0. If not, you'll need an adapter board and drivers to run it. This will complicate trying to run an external as a bootable drive.

If you are running 100 Windows programs, chances are you won't be happy dumping Windows entirely.  Win9x or ME can be installed on as little as 2 GB, but realistically 8 GB is the bare minimum for any GUI OS these days.  But Win9x simply isn't safe to use on the Internet any longer.

Considering all the apps and data you have you need more space,  You should never let a Windows drive get over 90% full as it will fragment your files with geometrically increasing frequency until the drive fills or fails.

I'd recommend getting another EIDE HD for Linux and/or backup purposes.  You can buy a white label (unbranded) 80 GB IDE drive for about $40 from 3BTech.net or other online reseller.  Then, you can control your boot priority easily from your BIOS setup and keep Linux entirely separate from Windows. You could dual-boot from the GRUB or LiLO Linux bootloader but you need another drive anyway.  Two drives is the simplest, most foolproof way to go.

Of course, you should install one of the utilities to read Linux ext2 & ext3 file systems from Windows such as Explore2fs or Ext2fsd and Ubuntu should see your Windows drive and include the ability to mount, read, and write to it from Linux.  If you install the Reisert file system used by SuSe you may not be able to see it from Windows at all.

Before you order your HD, look inside the case.  If your CD and HD are on the same cable, you will need to get a second IDE ribbon cable (80 pin) and plug your new drive into the end of the cable with the jumper set as Master (M) or Cable Select (CS) and the other end goes into the second IDE slot on the motherboard.  If the CD and current HD are on different cables, set your new drive's jumper to CS or Slave (S) and plug it in the middle of a cable.  If the cable doesn't have a second plug, you'll need to order a dual plug cable with your new drive.  

Some of the latest SATA motherboards have only one IDE controller.  In that case, buy an SATA drive (also available as white label for the same price) and be sure you have/order a SATA power tap adapter for older 20-pin power supplies or available SATA power tap (on newer 24-pin power supplies).
