### Post by Lord Illidan on 2007-01-04
> **wannaBeHacker said:**
> well what about gentoo? i heard that its the god of all linux os
Don't believe everything you hear. While it does offer a performance increase, it is at the expense of heavy compilation. Which means that an upgrade of a program can take hours instead of minutes.

And it's not for the linux newbie either. First get your hands on an easier distro like Ubuntu, then slowly work your way up the mountain. You can't run before you know how to walk.

Regarding Fedora, it's not a bad distro, but I find the package management to be lacking in comparison with apt-get and synaptic. Also, the repos were very slow for me in comparison to Ubuntu's repositories. And the community is better in Ubuntu.

Still, you can always dualboot.

### Post by darrenm on 2007-02-28
I prefer the Debian way of organising packages so updates don't break things. e.g.

Ubuntu's package of BIND:
named.conf with all the main non-changing config options
named.conf.options with the main changeable options
named.conf.local with all your local zones defined. This means an update should only ever touch the main config file that you don't need to configure for your own domain.

Fedoras BIND:
named.conf : everything in the one file and updates save your config file to named.conf.rpmsave
