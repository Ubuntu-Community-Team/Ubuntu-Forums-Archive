### Post by spamking2000 on 2007-10-26
Obviously there are a lot of threads set up on which is better, and I think its really comparing apples to oranges. Thunderbird is a very nice mail client and I'm a big mozilla fan.

That said, Evolution really has a wider audience for usage than Thunderbird. For many, that makes Thunderbird a nice program, but not a practical program from an integration standpoint.

I use Evolution for its PIM support. There is no built in PIM support in Thunderbird, nor will it sync with my Palm. Evolution has the built in, well integrated PIM application set as well as good support for IMAP, Groupwise, and Exchange, VCF, and iCAL. I don't believe there is any other FL/OSS software that quite measures up as a directly competing software. There are many other applications that do some of the same things, but not all.

I do however believe that the user interface in Evolution could be spiffed up quite a bit. I think the screen layouts could be revised, some more color added - basically just pretty the thing up a bit! Also, I'd like to see some changes to the layout of the Address Book and improvement to the message composer when the message being replied to is an attachment. Mail signature HTML support could use some improvement as well, especially the editor.

So there are ups and downs, there may be apps that might fit your needs better than Thunderbird or Evolution. They both have their strong points and features and they both have their limitations. I would ask that you simply take a look at your own requirements and then choose the app that most closely matches your requirements.

For me, that's Evolution. It has its yucky points at times, but it does a great job of being integrated and feature rich.

Thunderbird has several hundred extensions, including ones for a calendar and Palm support. It doesn't have as much built-in support as Evolution. 

I suggest you try both for a while and see which one you like. You should be able to configure them to peacefully coexist (if you're using POP tell it to leave a copy on the mail server).
