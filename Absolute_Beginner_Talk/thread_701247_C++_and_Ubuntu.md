### Post by mimada on 2008-02-19
Check out this [FAQ]("http://ubuntuforums.org/showthread.php?t=606009"). I have been messing around with Anjuta and Eclipse. They both have their strengths and weakness's. Eclipse is a very robust environment but it's written in Java and feels a bit sluggish. Anjuta feels quick but it takes a while to tune it to your liking. Unless you're pressed for hard drive space, you should install both and see which suits you the best.

C/C++ is a fairly difficult language to start with. Its primary strengths are speed of execution (which is critical for 3D intensive games and applications) and the ability to access low level functions (helpful for OS's and drivers). The biggest pitfalls are usually around memory management, weird pointer bugs, and general goofiness when you try to be too clever in your coding.

A popular alternative to C/C++ is Python. It's easier to learn and pretty fast for most applications.

You ought to decide what you want to create or work on before you chose a language. Good luck!

### Post by TuxImpersonator on 2008-02-19
And as for your question about C++ being different in Linux, from your point of view it's not. The C++ code you write will be exactly the same if you where on window or linux. The only difference is if your on a linux machine your computer will make a program that runs on linux and if your on a windows machine your computer will make a program (that does the same thing) that runs on linux.

There are one or two high end differences between the two but you won't come across those for a long while and when you do you'll probably be at the stage where you can figure out what to do!

Happy coding
