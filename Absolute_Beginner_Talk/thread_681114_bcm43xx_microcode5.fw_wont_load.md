### Post by pytheas22 on 2008-01-28
sounds like maybe there's a problem with your graphics card, and your system is actually booting, but X (the graphical interface) is failing to start.

How far does your system get when you try to boot?  If you press control-alt-f2 at the screen where you get the message about the bcm43xx microcode..., what happens?

Beyond that, you could take a look at the system log, located in the /var/log directory.  If your system is indeed booting but merely failing to start X, you can access the logs from the command-line and view them with nano or vi.  If your system really isn't booting at all, you would have to boot to the live CD, mount your Ubuntu partition and access it that way.

As for the message about "bcm43xx microcode5.fw wont load ," it's telling you that there's a problem with the firmware for your Broadcom wireless card.  Was wireless working alright before, or did you not have it set up?  That warning may have nothing to do with your main problem; it could be that it always complains about the Broadcom firmware, but you never saw it before because you were always running X.

### Post by pytheas22 on 2008-01-28
I can think of two options:

1. try logging in to X as root: press ctrl-alt-f2, login in there as your regular user, and then use "sudo -s" to become root.  Then type "startx"  I'm not positive that this will allow you to log in as root graphically, but it should.  If the problem is due to corrupted gnome settings, then presumably you would be able to log in as root, because your root account would have all of the default gnome settings.

2. as a last resort, you could remove all of your gnome settings by deleting all of the directories in your /home folder that contain gnome settings.  "rm -r /home/YOURUSERNAME/.gnome* " should do that.  Then the default settings would be restored.  But I would use this as a last resort, because I can't guarantee that it won't just cause more problems (the post [http://ubuntuforums.org/showthread.php?t=89250](http://ubuntuforums.org/showthread.php?t=89250) says it won't, but you never know), plus you'd lose all of your settings, which would be annoying.

### Post by pytheas22 on 2008-01-28
An interesting thing happened:

I wanted to see if I could indeed run startx as root from the console in order to log in to X that way. ### Post by pytheas22 on 2008-01-29
in your xorg.conf file, there is a section called "Device."  Within that section there should be a line similar to "Driver 'intel'"  Replacing "intel" with "i810" should solve the problem (at least this one, which may or may not be the root cause of the larger problem...), which was caused because there are two versions of drivers for Intel graphics cards, and you apparently don't have the "intel" one installed.

Alternatively, you could just install the "intel" driver with the command
```

sudo apt-get install xserver-xorg-video-intel
```

which would also presumably make X start.

Try either of those things, and then doing the startx thing as root, and I think that there's a decent chance that X will start.

### Post by pytheas22 on 2008-01-29
The only explanation for that behavior is that you might not have the repository for intel-video set up.  There must be a way to set up the repository from the command line, but I have no idea how; I can only do it graphically through Synaptic.

To workaround that, you should be able to download the intel video driver from [http://packages.ubuntu.com/gutsy/x11/xserver-xorg-video-intel](http://packages.ubuntu.com/gutsy/x11/xserver-xorg-video-intel) and put it somewhere on your file system.  Then you can change to the directory in which it's located and install it with
```

sudo dpkg -i xserver-xorg-video-intel
```

[note that the name of the package might be a little different than listed above]

I'm not sure if it will be able to resolve dependencies automatically or not--it should, but who knows.  If it doesn't, you will have to see which packages it needs as a precondition for installing intel-video, and then get them from [http://packages.ubuntu.com/](http://packages.ubuntu.com/) and install them.  In particular, I think that you might need libdrm2 installed as a dependency, which you can get from [http://packages.ubuntu.com/gutsy/libs/libdrm2](http://packages.ubuntu.com/gutsy/libs/libdrm2) .

Or, as an alternative to all this, you could also just replace "intel" with "i810" in your xorg.conf, which would in principle solve the problem just as well.
