### Post by oedipuss on 2006-09-12
Xgl is something like a second X server run on top of the current X server, xorg. If I understand it correctly, in the future, when Xegl is done, it will be able to completely replace xorg, and run without it. Currently , I believe Xgl is meant to be little more than a demo of Xegl.
AIGLX on the other hand is an extension of xorg, not a separate thing. It works fully already, provided the drivers of your gfx card support some things. Unfortunately the nvidia and ati proprietary drivers dont support a necessary extension yet.
There generally is debate as to what is better, which in my opinion shows they're both good :P
