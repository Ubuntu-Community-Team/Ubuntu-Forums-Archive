### Post by shawnjones on 2008-04-06
Basically it means that make went to go and build utsrelease.h and could not find the instructions to do that in the makefile. A system call is an interface between an application and the kernel itself it's important, there are quite a few of them as well, here are a few links that will explain in detail for you. 

[[COLOR="Blue"]syscalls(2) - Linux man page[/COLOR]]("http://linux.die.net/man/2/syscalls")

[[COLOR="Blue"]Errors Generated by Make[/COLOR]]("http://www.gnu.org/software/automake/manual/make/Error-Messages.html#Error-Messages")

Basically, although rebuilding a kernel is fairly easy, there are a few details that may very from system to system and distro to distro. You should try to "Keep it Ubuntu" whenever you can, but especially if you have never done it before and are not familiar with the process.
