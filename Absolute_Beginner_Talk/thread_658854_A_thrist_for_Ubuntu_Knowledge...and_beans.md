### Post by Schalken on 2008-01-05
Manpages are not a very good tool for learning a new command let alone OS, an alphabetic list of options to a specific command is not very helpful and is %90 information you will never use. Suprisingly, Wikipedia often provides a useful summary of command options and usages, amongst other linux things. (for eg. [http://en.wikipedia.org/wiki/Ls](http://en.wikipedia.org/wiki/Ls) )

If you want to learn how to configure things like GRUB, Apache or Samba, for example, your best resource is the documentation made by the developers themselves. For example, there are thousands of howtos and tuts on configuruing GRUB, all saying different things, but just look how concise and accurate the official [GRUB Manual ]("http://www.gnu.org/software/grub/manual/grub.html")is - and its garunteed to work.

As far as books go, I agree with Xavieran in that the O'Reilly books are a well respected resource for any open source hacker.
