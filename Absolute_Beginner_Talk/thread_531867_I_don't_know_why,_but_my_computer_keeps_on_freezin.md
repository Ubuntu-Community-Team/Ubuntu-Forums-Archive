### Post by HermanAB on 2007-08-22
A frequent source of trouble is the advanced power savings and interrupt features - therefore booting with the options 'acpi off', 'noapic', and 'nolapic' can frequently make an obstreperous machine work reliably.  Otherwise, if the system board and memory is simply out of specification, then turning the board clock speed down in the BIOS settings may help.

### Post by Anzan on 2007-08-22
Were you by any chance running Firefox when the freeze occured? 

In my own experience and that of several colleagues Firefox 2's caching feature can cause Gnome sessions to freeze. CTL+SHIFT+DELETE empties the cache and if used frequently freezes don't occur. 

If your session has frozen then ALT+SysReq+k will end the session and you can log back in.
