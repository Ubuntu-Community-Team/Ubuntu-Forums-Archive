### Post by logos34 on 2008-01-08
> **cjv998 said:**
> So I'm thinking I should just replace all references to "hda2" in mtab and fstab with hda4...would this fix my problem?

You also need to edit /boot/grub/menu.lst.

If you used **cp -a** or cpio to copy root, you have to change the UUID to the new one.  If you cloned/imaged it with partimage or dd command, the uuid did not change. 

The easiest way would be to take out the uuids:

fstab
> /dev/hda4 / ext3 defaults,errors=remount-ro 0 1

For menu.lst, change 'groot' line and ubuntu 'root' lines to (hd0,3):

> # groot=(hd0,**3**)

> root  (hd0,**3**)
kernel  /boot/vmlinuz-2.6.22-14-generic root=**/dev/hda4** ro quiet splash

change kernel option line:
> # kopt=root=**/dev/hda4** ro
