### Post by taurus on 2007-02-07
Apparently, there is still some left over from the previous installed version.  Therefore, see if you can remove it completely from a terminal with

Applications -> Accessories -> Terminal
```
sudo aptitude --purge remove vmware
sudo aptitude update
```
Now, see if you can install it this time.

>  Also, how do I boot in as root? I cant drag files to specifc fodlers without going thru the terminal...you already have vmware installed because xorg depends upon it.  fire up synaptic and do a search on name only for "vmare". you will see that you already have xserver-xorg-video-vmware.

 NEVER try to log in as root. thats not a good idea. what you can do is to go to the terminal and type in 'gksudo nautilus'. this will give you root privilages whilst using nautilus and will allow you to drag items into places outside of your home directory. NOTE: to run graphical applications with root privileges, use gksudo. when running a non-graphical script with root privileges, use sudo.

### Post by m.musashi on 2007-02-17
If you are still trying to solve this...

To reinstall vmware, uninstall and remove the /etc/vmware folder
```
sudo /usr/bin/vmware-uninstall.pl
sudo rm -Rf /etc/vmware
```
and then delete the entire /vmware-server-distrib/ folder from your /home (if that is where you put it. Re-extract the archive or just download a new copy. Then run the install as usual. It fixed the problem for me.
