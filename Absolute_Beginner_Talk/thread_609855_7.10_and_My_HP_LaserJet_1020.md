### Post by drs305 on 2007-11-19
One of the problems I've experienced with my HP usb laserjet was that even though the printer was found and recognized,  the Device URI was never correct. I don't recall the exact address, but it included "serial" even though it was a usb printer. 

When I changed the URI to the correct usb address it immediately started working. On my computer with a HP laserjet 1000,  the Device URI is:  usb://HP/LaserJet%201000  
With a LJ 1020, I would guess it  might be      usb://HP/LaserJet%201020

### Post by lancest on 2007-11-22
Thanks. Just got it working by following: [http://foo2zjs.rkkda.com/INSTALL](http://foo2zjs.rkkda.com/INSTALL)
Something was different from before maybe the last line
------------ HP 1020
    Install build-essential FIRST:
	$ sudo apt-get install build-essential
	$ wget -O foo2zjs.tar.gz [http://foo2zjs.rkkda.com/foo2zjs.tar.gz](http://foo2zjs.rkkda.com/foo2zjs.tar.gz)
	$ tar zxf foo2zjs.tar.gz
	$ cd foo2zjs
	$ sudo make uninstall
	$ make
	$ ./getweb 1020
	    OR other printer
	$ sudo make install install-hotplug cups
	$ sudo gnome-cups-manager
	[configure ColorMode = Color if a color printer]
	$ sudo make cups                
make install-hotplug

### Post by jeays on 2007-11-23
My Gutsy Gibbon installation worked fine with an HP 1020 until today, when I turned it (the printer) off and on to put in more paper.

I eventually found that the firmware file sihp1020.dl had disappeared from /usr/share/foo2zjs/firmware.  I have no idea how it got deleted.  The printer has been turned off more than once since the Gutsy installation.  Replacing the file with a fresh copy got the printer to work again.

Clue - look in /var/log/messages to see the complaint about the missing file.
