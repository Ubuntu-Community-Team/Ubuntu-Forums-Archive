### Post by Songwind on 2007-03-05
I am a fairly new Ubuntu user who decided to go 64-bit.

There are good points and bad points.  IMO there is an appreciable performance increase in anything that uses large amounts of RAM at once or does a lot of calculations.  Graphics programs and that sort of things.

The bad points basically all come down to the same issue that plagues every 64-bit system right now: not everything is available in 64 bit.  If you can compile from source you'll usually be okay, and you CAN run 32-bit applications under the x86_64 kernel.  You just need the right libraries and support files - the guys in the 64bit forum are great for helping out with this sort of thing.

The only bad problem I have had was due to the very proprietary nature of my printer.  I have a Dell 720 photoprinter, which is a rebranded Lexmark z615.  It isn't compatible with any printing standards like Postscript or PCL, and there was no 64-bit driver available.  I did get it working but it involved quite a bit of work on my part to track down a workable solution.

I'd say try it for a few days - it's not like you can't back up your /home directory and reinstall the 32-bit version if it's just too much hassle for you.
