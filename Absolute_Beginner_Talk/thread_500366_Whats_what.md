### Post by Rocket2DMn on 2007-07-13
Linux is a whole set of open source operating systems based on fundamental operating system code called a kernel.  Linux Torvalds created the linux kernel, hence the name.
Ubuntu is one of the linux distributions (based on the linux kernel, but with it's own modifications).  Ubuntu is actually based on another, much older distro called Debian.
Gnome is a desktop environment - it is essentially the GUI you see and is built off the X-server which is the fundamental GUI code that different desktop environments use (Gnome, KDE, Fluxbox just to name a few).
Beryl is an addition to Gnome that has lots of great flashy extras and increased functionality.  Compiz Fusion is the future of Beryl - it is a combination of Compiz and Beryl.  Metacity is the default desktop manager in Gnome.
Enjoy!

### Post by sab0403 on 2007-07-13
Linux... while there is some debate over this, Linux is the OS. Technically Linux is just a kernel, and the GNU project provided libraries to make it an all-around OS, but in short, yes, Linux is the OS.

Ubuntu is a *distribution* of Linux. Since Linux can be modified to your liking, some people took the original kernel and started changing stuff to fit their needs. Ubuntu is such a modification, derived from Debian, one of the original distributions (hence, a modification from a modification).

Gnome is a Graphical User Interface, or GUI for short. Think of it as a graphical front-end to the "core" OS; without Gnome, you'd see a Command Line Interface (CLI for short), like the one in the terminal. Windows originally was conceived as a GUI for DOS, for instance.

Beryl is a window manager. It controls the actual aspect of the windows, desktops, etc. The difference between Gnome and Beryl is that Gnome provides a graphical interface out of *nothing* (more correctly, from nothing graphical); Beryl depends on Gnome (or another window manager, like KDE) to handle the basics of windows, what the mouse does, etc., and modifies aspects on top of that.

### Post by sab0403 on 2007-07-13
Beryl, Compiz and Metacity all are Window Managers. What that means is that they control the look and feel of the windows, panels, and in general the desktop. They take the basic look that the GUI gives you (whether it's Gnome, KDE, whatever) and modify it to your liking. Think of them as "theme managers", though the comparison is rather weak (window manager are much, much more powerful than a simple theme handler).

You *could* uninstall Beryl and install Compiz. You don't have to, though. The thing with FOSS (Free and Open Source Software) is that it's constantly under development, and so every couple of weeks you might find a new version. Compiz was the "original" project for an advance window manager; some time after they started, some guys decided they wanted to do something similar but by themselves, and so they "forked" the project and created Beryl. But now they decided to join forces once again. So... yeah, Compiz is the one that's getting developed nowadays, but I've had Beryl for a while and haven't had any problems. If you want the very latest, though, go with Compiz. Beryl is not going to be developed further.

### Post by RomeReactor on 2007-07-13
Hi. Just a few comments:

As has been said, [Linux]("http://en.wikipedia.org/wiki/Linux") is the kernel: created in 1991, it provides communication between the hardware and the applications, and resource management. The kernel by itself would be very difficult to use by ordinary users. Also, it's creator's name is [Linus Torvalds]("http://en.wikipedia.org/wiki/Linus_Torvalds"), not Linux Torvalds as previously said.

The [GNU project]("http://en.wikipedia.org/wiki/GNU"), founded by [Richard Stallman]("http://en.wikipedia.org/wiki/Richard_Stallman"), created an almost complete [Unix-like]("http://en.wikipedia.org/wiki/Unix-like") operating system: development started in 1984 (7 years before the Linux kernel was first released), and by 1991 all that was needed was the kernel. Both projects benefited from each other, as the GNU project provided several much needed applications and the kernel provided them with a way to communicate with the hardware.

Gnome, KDE, Xfce, etc, are [desktop environments]("http://en.wikipedia.org/wiki/Desktop_environment"): [graphical user interfaces]("http://en.wikipedia.org/wiki/Graphical_user_interface") that allow the user to interact with the operating system from a visual standpoint.
