### Post by p_quarles on 2007-08-02
Most ISPs will *say *they don't support Linux. This is because the typical service agent doesn't know anything, and has a script that they read to people depending on their OS (e.g., "click the start button, go to 'system'"  etc.).

That said, the only features of internet service that are OS-dependent are network interface drivers and client software. Linux has all that covered.

The thing to do is make sure that you have/get a modem which uses a web configuration interface. That way, you can ask for instructions on using that, and pretend that you're using Windows. Most modems do this, as far as I know, but make sure before you buy.

### Post by ramjet_1953 on 2007-08-02
If you are using an ADSL router, Ubuntu should just see it as a network connection and you should be able to connect without any additional configuration.

Also, most ADSL routers are themselves configured using a web browser.

The config information needed for the router itself is identical, regardless of the OS that you have installed.

### Post by irish_flu on 2007-08-02
Be forewarned:  Some ISPs will fudge a little and instead of saying "we don't know how to support Linux" they'll say "our service doesn't work with Linux".

My friend who uses Ubuntu (he's no geek; I gave him a computer with Ubuntu, his first computer) recently got an Insight cable connection.  He told them he uses Linux, and they wouldn't even sell him service.  I had him call back and say he used Windows, and they happily sold him service.

When the installer got there, he (who was a contractor hired by Insight)  was more than happy to let me walk him through checking the settings (set DHCP, set gateway address, etc) in Gnome over the phone so that he could make sure my friend's computer worked before he left.
