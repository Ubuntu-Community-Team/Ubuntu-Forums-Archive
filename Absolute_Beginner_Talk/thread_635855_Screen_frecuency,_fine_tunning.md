### Post by Don Gatto on 2007-12-18
Hello,

In case you haven't already find it out, I can give you a possible solution. Use this command in the terminal: *sudo dpkg-reconfigure xserver-xorg*. (Your password will be asked.)
This command lets you configuring many things, including frequencies. Though it's easy to ruin your system with it, it looks like you will know what to choose in every option. Now, among the last steps (I believe) there will be one where you can choose how do you want to config your monitor. There are three levels, you'll need the last one (*expert*). You have to set there manually the highest resolution, but you can set the frequencies there as well.
I don't know if there is an easier solution but this one should work.

### Post by tgalati4 on 2007-12-18
If that doesn't work, then load SUSE Desktop 10 (SLED10) and grab the modelines from xorg.conf.  It uses a different xorg configurator that seems to get it right.  

For instance to get 1280 by 1024 at 99 Hz on a Sony Trinitron/Dell Monitor with an old Dell:

Section "Monitor"
  DisplaySize  350 260
  HorizSync    30-107
  Identifier   "Generic Monitor"
  ModelName    "P992"
  Option       "DPMS"
  VendorName   "DELL"
  VertRefresh  48-170
  UseModes     "Modes[0]"
EndSection

Section "Modes"
  Identifier   "Modes[0]"
  Modeline      "1280x1024" 188.88 1280 1376 1520 1760 1024 1025 1028 1084
  Modeline      "1024x768" 150.39 1024 1104 1216 1408 768 769 772 828
  Modeline      "1024x768" 149.05 1024 1104 1216 1408 768 769 772 827
  Modeline      "1024x768" 147.88 1024 1104 1216 1408 768 769 772 827
EndSection
