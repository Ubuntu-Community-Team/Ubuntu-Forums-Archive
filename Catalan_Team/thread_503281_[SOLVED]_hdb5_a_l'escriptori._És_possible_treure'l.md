### Post by Arnau_MusicaLliure on 2007-07-17
Bones!
Estava fent una renovació del meu escriptori (utilitzo gnome, ubuntu 7.04 i compiz fusion) i he trobat un magnífic invent que no coneixia.
Se'n diu avant window navigator. D'aquesta manera he fet un buidatge total de icones a l'escriptori. 
El "problema" (sé que és una "pijadeta") és que la icona d'un disc dur(hdb5) que sempre està allà per defecte desde el dia que vaig instalar Ubuntu no el puc treure.
Si vaig allà i pico suprimir em diu:
"No podeu moure el volum «hdb5» a la paperera.
Si voleu desmuntar el volum, utilitzeu «Desmunta el volum» del menú emergent del volum."
Això ja no em fa gràcia, perquè llavors em vec obligat (segons sembla) a desmontar el vòlum per tal de treure l'icona de l'escriptori i llavors cada vegada que volguès obrir l'hdb5 l'hauria de montar i apareixeria aquella icona a l'escriptori.
Tot i així, si em dirigeixo a la icona i pico "Desmunta el vòlum" em diu:
No es pot desmuntar el volum.
unmount: El muntatge de /media/hdb5 no concorda amb el fstab.

Aquí us deixo el meu fstab:
# /etc/fstab: static file system information.
#
#  -- This file has been automaticly generated by ntfs-config -- 
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>

proc /proc proc defaults 0 0
# Entry for /dev/sdb2 :
UUID=f5b88e5e-aaa5-4c81-9f23-1ca3db17ef61 / ext3 defaults,errors=remount-ro 0 1
# Entry for /dev/sda5 :
UUID=44FC-5A89 /media/hdb5 vfat defaults,utf8,umask=007,gid=46 0 1
# Entry for /dev/sdb1 :
UUID=A4542A2D542A02A2 /media/sda1 ntfs-3g defaults,locale=es_ES.UTF-8 0 1
# Entry for /dev/sdb3 :
UUID=7709b63b-7a34-4c46-a56f-ce801aa4f35a none swap sw 0 0
/dev/hda /media/cdrom0 udf,iso9660 user,noauto 0 0

També he de dir, que no sé perquè, un dia per un altre em va desapareixe (bé, que desde Linux no la puc veure) la partició on tinc instalat windows xp (la que posa Entry for /dev/sdb1) i mai n'he tornat a veure rastre.

Gràcies ;)

### Post by CarlesOriol on 2007-07-18
Anem a pams:

1) L'avant-window-navigator està bé, també tens altres alternatives com el gdesklets (en repositoris), el kiba-dock (per compiz / fusion / beryl) i naturalment el superkaramba per kde.

2) El nom de l'unitat: 
UUID=44FC-5A89 /media/**hdb5** vfat defaults,utf8,umask=007,gid=46 0 1
correspon a l'etiqueta del disc dur que vas posar al formatejar-la (o reanomernar-la) en msdos/windows, en aquest cas hdb5.  

3) Si vols que no es vegin les icones de les unitats muntades a l'escriptori pots fer-ho amb:

gconf-editor

anant a la branca: 

  apps->nautilus->desktop 

         i desmarcant: volumes_visible

 [I]A mi m'agrada tenir-les invisibles a l'escriptori i veure-les amb l'applet d'unitats en una barra d'eïnes/menus.
[/I]
4) L'unitat en windows xp el més segur és que tingui algun error i no te la deixi muntar. El ntfs-3g és molt sever amb aquestes coses. Inicia l'ordinador amb xp i realitza un xequeig del disc. (et caldrà reiniciar un parell de cops ja que no ho farà amb l'unitat d'inici funcionant). Un cop fet tornes a l'ubuntu i veuras com ja la tens.
