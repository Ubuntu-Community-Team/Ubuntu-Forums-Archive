### Post by undolaure on 2008-07-22
En aquest aspecte que parlàveu de si formatejar els discs durs els malmet o no, val la pena comentar que hi ha diverses maneres de formatejar un disc dur. Les que aquí majoritàriament es mencionaven -la de marcar les dades com a "esborrables"- són les que, efectivament, triguen uns pocs segons a realitzar-se.

Els formatejos de disc dur orientats a resoldre els errors del disc, per la meva experiència consisteixen a reinicialitzar-los (posar-los com "sortits de fàbrica") i realment poden requerir un ús molt intensiu del dispositiu. Consisteixen a donar valor 0 a cada unitat d'informació, i això en els discos durs actuals de + de 100 GB és un procés que pot durar algunes hores. Per fer això no n'hi acostuma a haver prou amb les utilitats de formateig dels sistemes operatius sinó que s'acostuma a necessitar el software del fabricant del disc dur.

Això és una cosa que acostumo a fer sempre en qualsevol disc dur a partir d'un temps raonable com per exemple a partir d'un any d'ús, i quan es vol canviar o alterar-ne del tot l'estructura de particions o passar d'un sistema windows a un gnu/linux. Jo no sóc cap tècnic d'aquest tema, però fer això m'ha servit com a garantia que durant la instal·lació (de windows, de gnu/linux) aquesta no trobava coses rares al disc que li impedissin procedir.

Sobre el tema que ens ocupa, podeu veure una entrada al meu [**wiki**]("http://www.imatgedart.com/wiki/enllacos:gnulinux:hardware:desgast_hdd_loadunloadcycles").
ala, salut.

### Post by papapep on 2008-07-22
De [http://en.wikipedia.org/wiki/Segmentation_fault](http://en.wikipedia.org/wiki/Segmentation_fault) :

> A segmentation fault (often shortened to segfault) is a particular error condition that can occur during the operation of computer software. A segmentation fault occurs when a program attempts to access a memory location that it is not allowed to access, or attempts to access a memory location in a way that is not allowed (for example, attempting to write to a read-only location, or to overwrite part of the operating system). 

Traducció lliure (i ràpida):

"Un error de segmentació (sovint referit com a segfault) és una condició d'error que pot ocòrrer durant el funcionament de programari. Un error de segmentació succeeix quan un programa intenta accedir a una posició de memòria d'una manera no permesa (per exemple, intentant escriure a un lloc on només es permet llegir, o quan s'intenta sobreescriure una part dels sistema operatiu)."
