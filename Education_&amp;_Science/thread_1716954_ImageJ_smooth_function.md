I am slowly transferring my work activities to Ubuntu but, at the moment I have a few problems with ImageJ. I have filed a bug report but I am also curious if other ubuntu user experience similar problems.

First some technical information:

- I am using ImageJ version 1.45d on
- a Ubuntu 10.04.2 LTS (32 bit) computer with 
- java version 1.6.0_20. 
- I have allocated 2000Mb memory to ImageJ

I experienced 2 problems.

[LIST=1]
[*] I cannot apply the smooth function to a (8-bit greyscale) stack. After I confirm that I want to apply the smooth function to all frames, nothing happens and the histogram of the images do not change, which should happen
[*] When I resize a 8-bit stack using the rectangle tool and the crop function (shift+s) the resulting window refuses to close and I have to run the sytem monitor to end ImageJ
[/LIST]

I need these function for the analysis I am doing, so solutions are very welcome! But, if more people confirm the same problem it might get fixed faster.

### Post by RonaldP3 on 2011-03-30
I have solved the problem. Downloading and installing ImageJ from their website [http://rsb.info.nih.gov/ij/download.html]("http://rsb.info.nih.gov/ij/download.html") cured all of my problems.

Apparently there is something wrong when you install imagej using the ubuntu software centre.

I hope this information is of use to somebody!

Ronald
