### Post by gunksta on 2009-03-18
[FONT=Times New Roman][SIZE=2]You're going to have a busy senior year!

>A) Construct a computer from scratch, all hardware bought separately

[/SIZE][/FONT][INDENT][Hardware & Laptops]("http://ubuntuforums.org/forumdisplay.php?f=332")

The Hardware & Laptops forum may be your best bet for hardware related questions. If you look around there are several guides to buying hardware that is compatible with Linux. Linux (and thus Ubuntu) has _the_ best hardware compatibility of any operating system ever built. But, not all hardware performs equally under Linux and some things are simply a PITA to get working.

I suggest you keep things simple. My 10 cents - build an Intel box. I don't know how powerful you want to make this thing, but the Atom processor costs way less than $100 per chip and has graphics capability. I know it can run Linux. There will be dual core Atom chips sometime this fall (if you believe all the advertising hype). But, I'm not sure you anyone is producing mother-boards for full sized machines for the Atom processor.

But, Intel makes plenty of other products too. You don't need to spend a bundle to get an adequately fast system, unless you are a gamer. The advantage of an Intel based system is that you can get a mother board with an on-board graphics system. Intel graphics aren't the best, but they are more than adequate for most of us. As an added advantage, the drivers for Intel' video hardware is all open-source and is well supported by Intel. If you use ATI or NVidia graphics, it _may_ be a little harder to set up. Some cards are a piece of cake. Some are rather tricky. Some are supported only the proprietary drivers. Some are really only supported by the open-source drivers. Some can use both. As they say, YMMV. That's why I would recommend using Intel. There's no reason to waste a bunch of time learning the intricacies of the X Server when you really just want to learn how Linux works (there is more to Linux than the X Server!).

Hard drives - they should all work equally well. (Give or take).

Networking - You don't need wireless in a desktop (I assume). Basic networking cards should basically all work easily under linux. Some wifi stuff is a little trickier, but that keeps getting easier and easier.

Printers - I'm biased towards HP. But, others work well too. Just look on the Internet before you buy.
[/INDENT][FONT=Times New Roman][SIZE=2]
>B) Use a Linux Distro ( Probably Ubuntu, just to get my feet wet)

[/SIZE][/FONT][INDENT]K/Ubuntu is a terrific distro to learn on. Another good option is Fedora, produced by Red Hat. Obviously, this being the Ubuntu Forums, I am biased in favor of Ubuntu and it's various flavors (Kubuntu, Xubuntu, Mythubuntu, etc.)

But, you don't need to say, "just to get my feet wet". I've been using Linux since 1999 as my primary desktop / laptop OS. I use Ubuntu / Kubuntu because they are easy to set up and are very powerful / flexible. I _could_ use a distro like Slackware, but my boss pays me to do my job, not configure my distro. Unless you have a fairly specialized need, or just an interest in learning how to do everything by hand (which is perfectly valid) the main distros (Ubuntu, Fedora, Suse, etc.) are more than adequate for most of us.
[/INDENT][FONT=Times New Roman][SIZE=2] 
>C) Develop a practical, but simple application.

[/SIZE][/FONT][INDENT]Lots of choices there. Do you have any programming experience? There are forums here that are more focused on programming than this forum. If you need to learn how to program, I would probably recommend Python. It is well supported by Linux (used internally by Ubuntu for most system configuration stuff). A basic install has 95% of what you need to do your development (python, text editor, command-line). Because python is an interpreted language, you can ignore a lot of the more complicated parts of programming and focus on building a program that actually does something. My first language was Java, a wound I am still slowly recovering from. My therapist says I may some day forgive my professors, but not today.  :-)

[/INDENT]HAVE FUN!

### Post by Greslore on 2009-04-25
Cool project!

Seeing how you are focusing on Linux, I would suggest modifying the last part of coding an application.  Instead of this, I would do this focusing on the OS aspect such as building a LAMP (Linux, Apache, MySQL, PHP/Perl/Python) server using Ubuntu.  IE - Linux System Administration.

You could build the hardware from scratch, install Ubuntu server with Apache.  Throw on an install of Wordpress and you now have a fully functional web server.  If you have highspeed internet, maybe register with DynDNS and bam - your new server is now accessible from the internet to show to your project coordinator or class.  The last component of this project could be focused on the operational maintenance of this server, such as a backup plan (disaster recovery plan).  Write a shell script that backups the MySQL and www directory to an external harddrive or maybe a remote drive like Dropbox (free 2gb) or something. 

 
You could document/photograph your project each step of the way.  Definitely lots of options!!  And after this, you could maybe go on and look into LPI or Linux+ certs.  Not to mention you now have a test bed to develop all your web code for your future endeavors in computer science! =P
