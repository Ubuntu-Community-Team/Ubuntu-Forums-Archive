I had the same problem and found out that the modified file BibTexCiteKeyOnly.js in the zotero translators folder does not work with the new version. So, I used the Bibtex.js that comes with the new version and modified it again in order to do the easy copy_paste_thingy.
This is what I did with the new (and working with version 2.11) BibTex.js:


First, in order to get suitable keywords in your Zotero-library:
var citeKeyFormat = "%a_%t_%y";  -->  var citeKeyFormat = "%a%y"; //or something like that

..and if you want only double-digit years:
return date.year;   -->    return date.year.toString().substring(2);

becomes::::::::

function doExport() {
    //Zotero.write("% BibTeX export generated by Zotero "+Zotero.Utilities.getVersion());
    // to make sure the BOM gets ignored
    var first = true;
    var citekeys = new Object();
    var item;
    while(item = Zotero.nextItem()) {
        // determine type
        var type = zotero2bibtexTypeMap[item.itemType];
        if (typeof(type) == "function") { type = type(item); }
        if(!type) type = "misc";
        
        // create a unique citation key
        var citekey = buildCiteKey(item, citekeys);
        
        // write citation key
        Zotero.write("\\cite{"+citekey);
        first = false;
        Zotero.write("}");
    }
}

Now tell zotero to use this file in the settings/export and everything should work again!

### Post by ajlyon on 2011-03-25
You need to change the UUID (just change a couple of hexadecimal digits) in the JSON header at the beginning of the file, or else Zotero will see them as the same translator, and show you only one the two versions.

And only a few translators are shown in the export menus because only export translators are shown; most translators are web translators.
