### Post by tkjacobsen on 2007-05-14
Many scientists use LaTeX to make their projects. LaTeX is not like OpenOfficeWriter or Word, it is a typesetting system. It produses really nice output and is very stable because all the text is saved as unformatted text. Instead of choosing Headline font or something like that you code it like

\section{A section}
text goes here

\subsection{A subsection}
text goes here

then the table of contents os autogenerated with the command \tableofcontents (everything starting with "\" are commands

once you are getting used to it it is very efficint.

You can find more information here
[http://en.wikipedia.org/wiki/LaTeX](http://en.wikipedia.org/wiki/LaTeX)
[http://www.latex-project.org/](http://www.latex-project.org/)

To use it in ubuntu install texlive-full and texmaker. (kile instead of texmaker in kubuntu)

EDIT:
You can also make documents from multiple files using the \input commando. E.g. you can have each chapter in a new file, I find that very useful, especially when collaborating with others.

EDIT #2: I can recommend this introduction. "The not so short introduction to latex"
[http://www.ctan.org/tex-archive/info/lshort/english/lshort.pdf](http://www.ctan.org/tex-archive/info/lshort/english/lshort.pdf)

### Post by tkjacobsen on 2007-05-15
You can easily include figures in your text like this:

```
\begin{figure}
\centering
\includegraphics[angle=90,
                 width=0.5\textwidth]{test}
\caption{This is a test.}
\label{fig1}
\end{figure}

```

Then another place in your document you can reference the figure with \ref{fig1}, this way references is always right. You can also label sections and tables etc.

If you use pdflatex to bulid your sources it supports png/jpg and pdf image formats.

If you use an editor like Texmaker you can use that to insert the commands as above, it has lots of them to paste for you.

### Post by Typhoon on 2007-05-16
> **Eagleon said:**
> hmm.. how does Lyx compare to LaTex? thank you for the suggesion elfstone214 :)

LyX acts as a front end to LaTeX. More precisely, it uses LaTeX for its typesetting. LyX is very easy to use, yet quite powerful. I used it to typeset and self-publish a cookbook written by my wife. You can see some of the sample pages here: [http://web.aanet.com.au/sage](http://web.aanet.com.au/sage).

However, raw LaTeX is more powerful. I write legal texts where I need at least three indexes: a table of statutes, a table of cases and an ordinary index. Using LaTeX with the memoir class makes this easy. I use it for writing (along with Emacs + Auctex + Reftex) even though my 19th century publisher insists on "Word" documents! the program latex2rtf provides the translation.

HTH,
Typhoon

### Post by Typhoon on 2007-11-13
Re: LaTeX vs Docbook

The problem with Docbook is that the markup is so intrusive. It is so complicated that it gets in the way of writing. LaTeX can do that, but there seem to be better front end editors.

IF (and it is a big IF) your writing is not too complex (I'll explain that in a moment), you might try one of the "lightweight" markup languages. I have been writing article length documents using Emacs and Muse. The writing looks like plain text, but it follows a few simple rules. It can output a lot of formats, but in particular, LaTeX and Docbook. It also generates good, clean html.

Writing not too complex: I don't mean the ideas here, but rather the structure of your document. If you need to put in indexes, lots of citations, lots of cross-references, then I would recommend going with LaTeX. Even then, it might be worth doing a lot of the writing with the simple markup, then adding the other things later.

Typhoon

I think it is important to distinguish between the writing process and the publishing process. For writing, we want software that provides a lot of support for the writing process. That process may depend on the kind of writing that you do.

Whatever support you need, you certainly DON'T need the intrusion of markup language. It is merely a distraction from the writing process. I tend to like plain text with a first class text editor - emacs in my case, but this is no place for an editor war. Lots of good ones out there.

On the other hand, some academic writing benefits from some markup. You can do some indexing as you write, citations and cross-references require some kind of markup. I find LaTeX the best here, particularly since I can hide most of the markup in emacs. It looks almost like plain text, but with all the extra functionality.

LyX provides this same functionality -- almost. In my writing, I need to have multiple indexes. Easy with LaTeX (use the memoir class), but not so easy with LyX.

Cheers,
Typhoon
