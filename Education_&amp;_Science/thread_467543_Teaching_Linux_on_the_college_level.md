### Post by bigboy_pdb on 2007-06-12
"Running Linux" and "Beginning Ubuntu Linux" are worth taking a look at.

[http://www.oreilly.com/catalog/runlinux5/](http://www.oreilly.com/catalog/runlinux5/)
[http://www.apress.com/book/bookDisplay.html?bID=10259](http://www.apress.com/book/bookDisplay.html?bID=10259)

If there are any bookstores near you that carry them then you can skim through them there.

There are also "Linux Bible" and "Ubuntu Linux Bible", but I don't care too much for the "Bible" books. I find that they waste some sections explaining details for things that people either know from using the software or that people could learn more easily by using the software.

Also, there are many online resources that might have some things that you're looking for if you find that some books don't provide everything that you need. A few links that might have something worth while are:

[http://www.oreilly.com/openbook/](http://www.oreilly.com/openbook/)
[http://tldp.org/guides.html](http://tldp.org/guides.html)
[http://snap.nlc.dcccd.edu/reference/](http://snap.nlc.dcccd.edu/reference/)

Of the books that I've read, I've been pleased with quite a few of the books from O'Reilly.

In addition, you might find some books/manuals using 'Synaptic Package Manager'. I did a search on "book" when using it and I found a packages for "The Advanced Bash-Scripting Guide", "Debian bookmark collection", "Extensive tutorial and documentation about C++", "Common Lisp the Language, second edition", "Derivations of Applied Mathematics", "Dive Into Python", "DocBook: The Definitive Guide", "Recommendations of the W3", "Advanced Linux Programming", "GTK+/Gnome Application Development", "KDE 2.0 Development", "The Hardware Book", "Documentation by and for newbies",  "Common Lisp programming techniques", "PNG: The Definitive Guide", "Programming Ruby", "Linux: Rute User's Tutorial and Exposition", and "Zope Open Content Book". There's also documentation for different projects.
