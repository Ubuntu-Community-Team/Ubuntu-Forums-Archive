### Post by Prospero2006 on 2008-06-07
Here's the monitor shut down script for starters:

This is dependent upon a couple of things:
      -First, The machines in the room have to have
       static ip addresses. Mine are sequential. They start
       at 10.43.231.1 and end at 10.43.231.30
       (Sequential ip addys make all kinds of scripting
        really easy.)

      -Second, all of my kids log in using the username: student
       That's also important. 

```

ipaddy=1
until [ $ipaddy -eq 30 ]
do
  ssh 10.43.231.$ipaddy 'rmmod usbhid; export XAUTHORITY=/home/student/.Xauthority DISPLAY=:0; xset dpms force standby;' &
ipaddy=$(($ipaddy +1 ))
done
ssh 10.43.231.27 -p 24 'rmmod usbhid; export XAUTHORITY=/home/student/.Xauthority DISPLAY=:0; xset dpms force standby;'

ssh 10.43.231.10 -p 24 'rmmod usbhid; export XAUTHORITY=/home/student/.Xauthority DISPLAY=:0; xset dpms force standby;'

```

That script is executed as root. First disables USB support. Without it, the kids can move the mouse and the screen comes back. Then the export command gives root access to the student desktop. The the 'xset dpms force standby;' cuts the monitor.

You could replace the 'xset' command with any progam name, for example Firefox, and every computer in the room would pop a firefox browser.

I have scripts like this which install programs, remove programs, power down the lab, reboot the lab, change the wallpaper, change the gdm, you name it. It's pretty flexible. 

Let me know if it works!
