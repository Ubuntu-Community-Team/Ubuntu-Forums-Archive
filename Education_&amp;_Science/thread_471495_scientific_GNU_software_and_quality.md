Computational biology is rapidly switching to open source programs.  You mention GNU, and I'm not totally sure what fraction is GNU licensed.  Regardless, there has been an explosion of bio research made free and posted to projects like R (via. bioconductor.org), bioperl (and other bio-lang's), and sourceforge.  

Bio-Informatics journals such as PLoS, BMC-Bionformatics, and Bioinformatics seem to be granting preference to projects with source code that has been made available.  Please note, this is just a recent observation of mine.   Submitting source code with a publication sends a message to the publishers that you welcome peer-review and that you have created quality software that will stand up to public scrutiny.

### Post by in_flu_ence on 2007-06-12
I don't think that it is a matter of what software you are using for journals. For example, though the norms always go with Matlab (because there are generally academic license available), Octave are generally accepted among scientist. My department basically use Octave rather than Matlab unless there are some tool package that is needed and not available otherwise.

GNU software are generally new (except for computer scientist). Many of them require time to practice and use and this is always not possible given that there are more important focus to look into in research projects. I had wanted to pick up R but have been postponing till my vacation.

There are also scenario where the reliability of the results from software are not well verified (comment if I am wrong). For this, I quote the example of CFD. Fluent and CFX are well tested and they are well trusted by companies and schools. OpenFOAM might do similar thing but it is relatively new comparatively. No research group will want to diverge their effort to test a package unless it can provide the extra benefit (site license is generally available so cost might not be an impt factor to consider).

Nevertheless, there are also as many of the above have quoted. Certain areas are using more and more open source software especially bio-informatics because it gives the flexibility to improve on the code for more sophisticated research.

Just wait and see. I m sure there will be more GNU software published in the next 5 years.

### Post by DrOlaf on 2007-06-13
Most of the software I use for DNA sequence analysis is FLOSS in some regard, even if it isn't GPLed. Programs like ClustalW, Staden (RIP), SplitsTree, R/BioConductor and countless little utilities are all well-regarded in my field and no-one bats an eyelid when I put them in publications. When the commercial apps cost $3,000 a shot it fosters quite a widespread desire to provide FLOSS alternatives! If I had the programming skills I'd be happy to produce and distribute my work-related code.
