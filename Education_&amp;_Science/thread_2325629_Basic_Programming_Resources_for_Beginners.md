### Post by Joe_Willmann on 2016-05-29
Languages are cool, but languages do not teach you how to program.  To program you need to understand what a computer is and how it works.  I bet everyone on this forum think that they know all of this but to fundamentally understand the basic concepts of if() then, else, subroutines, branch control, uniary operations, logical expressions, libraries, and the list just grows.

I like to teach newcomers top down development where you start simple and add complexity.

I would add several concepts that I would think are fundamental.
Let's start with the fundamental concepts of software development.  Methodologies such as OOADP, Functional Analysis

Also teach people to beware of religons, as in Java, C++, OO and others.  There are no perfect languages or perfect methodologies that will solve every problem.  Each of these things are tools and some tools are better at solving some kinds of problems.  Understanding which tool applies to which kinds of problems is really handy.


Version control - The importance of keeping things under version control, preventing loss, maintaining history, maintaining 
Defect tracking - understanding the role of Bugs.  Believe it or not they add value, it is how users tell us what is wrong with the software and push development forward.
