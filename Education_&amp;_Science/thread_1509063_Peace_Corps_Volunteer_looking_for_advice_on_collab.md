### Post by imda2001 on 2010-06-13
Hello, I am currently a [Peace Corps Volunteer in Mexico]("http://mexico.peacecorps.gov/"), and we are trying to find an open-source tool to help us and the Mexican agencies we work with share information / collaborate.  We are not familiar with the popular open-source offerings, and the various Wikipedia pages on [collaborative software]("http://en.wikipedia.org/wiki/List_of_collaborative_software") and [content management software]("http://en.wikipedia.org/wiki/List_of_content_management_systems") list so many options that we don't even know where to start!  So, if anyone with experience with these kinds of packages and thinks they could apply to our use (see below), we would appreciate recommendations / guidance on popular, well-supported open-source packages!!  We think the key is finding something that is flexible enough that we could modify it, and is popular enough that we could get support or others are also making modules.

The idea is that we want to be able to share information across volunteers / agencies (and eventually in rural communities, too)and provide a way to collaborate in real-time (or almost real-time).  For example, this is how you navigate Immigration / Customs at the Mexico City airport.  Or, alternatively, I'm working on a project for wildlife camera traps with a volunteer hundreds of miles away and we need to share information.  Also, we want to encourage our partner institutions to use the same system.  For them, this could be to track camping applications and permits (yes / no / waiting on this person's signature).  Or creating a website for professional networks / collaborations with outside entities, showcasing special technology or wildlife of the institution.  

We do not have a specific project / use in mind and are trying to keep the end goal flexible so that different users can customize the system to suit their needs (hence our requirements are a bit general).

We are trying to use a [Pugh matrix / decision matrix]("http://en.wikipedia.org/wiki/Decision_Matrix") to figure out which system would meet our requirements the best, but are having a problem narrowing down the list of technologies to evaluate against our criteria.  From a content management perspective, Drupal, Joomla, Moodle, Plone, and Liferay have been recommended to us as being popular--on target?  What about other options, like collaboration software?

So far we have done a pairwise comparison matrix of 14 requirements to come up with the 7 requirements that we think are most critical for the software.  They are (in descending order):

1) A forum should be allowed for public discussions
2) Open-source
3) Different project sites can be isolated from each other within the same system, via permissions
4) A search feature shall allow users to search for files, discussions, tasks, etc., throughout a project site
5) E-mail notification for any new items can be configured by the user
6) Users can upload any file type and assign version control to it
7) Users shall be able to reset their own passwords via the system.


Thanks in advance for any tips!!!
