### Post by ronniet on 2007-09-25
[I]Are you a keen podcaster?
Think you can create an interesting and friendly Ubuntu podcast?[/I]

Then why not submit a proposal to be the _official_ ***full circle* podcast**?

[IMG]http://fullcirclemagazine.org/wp-content/logo_final.png[/IMG]

> 
**[SIZE="3"][COLOR="Red"]Before you submit a proposal, read these terms![/COLOR][/SIZE]**

You would be creating a podcast using the ***full circle*** name so you must agree to:
[LIST]
[*]the podcast should be created using only free software (the magazine is!)
[*]the podcast should contain absolutely _no_ swearing
[*]the podcast should be aimed at beginner to intermediate *buntu users
[*]the podcast should be in both MP3 and OGG format
[*]the podcast will be mentioned in the magazine so the podcast should mention the magazine
[*]the podcast should be submitted for review before release. We will never censor the podcast but we do want to make sure the presenters aren't blackening the ***full circle*** name! 
[/LIST]

_**Your proposal should contain:**_

**How** do you plan on producing it?
**What** length would the podcast be? And what will it contain?
**Why** should there be another linux podcast?
**Where** are your presenters located? (use Skype, or something similar, for a worldwide podcast group?)
**When** would you release it? Which day? How often?

And anything else you think will impress us. :)



**[COLOR="RoyalBlue"]The presenters of the podcast decide on the content, not us.[/COLOR]**

[COLOR="Red"]Closing date for proposals: Wed. 31st October.[/COLOR]

Approximately one week later we will announce the team who will be making the **official full circle podcast**!  :)

If you have any questions about the terms or your proposal, feel free to email me, [Ronnie]("mailto:ronnie@fullcirclemagazine.org") (Editor).

[LIST]

### Post by popey on 2007-10-08
This is certainly an interesting idea. One point I'd like to raise. You have requested that the podcast should be made using only free tools. This is something I am keen on (the ubuntu screencasts are created using only free tools), I like the "eat your own dogfood" approach. I assume you are talking about "Free" (as in speech) and not "free" as in beer?

However there is a conflict. You have requested that the podcast should be available in Ogg and MP3 format. Ogg is clearly not going to be a problem, MP3 is however. For some jurisdictions (UK included) it's not possible to make MP3s using completely Free software because the MP3 format is encumbered by patents.

Technically I suspect that one could upload a WAV or Ogg to a server in some other region (such as Russia) and re-encode there, then upload to the server, but it's somewhat dodgy ground.

Can you clarify what you mean by "Free".

Note: I am most definitely not a lawyer ;)
