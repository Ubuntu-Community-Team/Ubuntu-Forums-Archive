### Post by SLA_leandrin on 2009-11-06
OK, si te detecta el módem y el problema es con el network-manager-gnome (que me pasa a mi también), lo podés utilizar al módem con la línea de comandos todo con programas ya preinstalados en Ubuntu.

Abri  la consola y escribí:

```
sudo pppconfig
```

Te van a salir una serie de pantallas con instrucciones, lo que tenés que hacer es una conexión nueva:

[[IMG]http://imagedough.com/id/244/173/582/347/107/thumb/Pantallazo-leandro_leandro-laptop_-Downloads.png[/IMG]](http://imagedough.com/imgs/244173582347107)

A partir de ahi los seguí las indicaciones. Te crea un archivo de texto en

```
/etc/ppp/peers
```

con el nombre de la conexión que pusiste, por ejemplo te adjunto el mío para que veas la configuración;

```
# This optionfile was generated by pppconfig 2.3.18. 
# 
#
hide-password 
noauth
connect "/usr/sbin/chat -v -f /etc/chatscripts/Claro"
debug
/dev/ttyUSB0
115200
defaultroute
noipdefault 
user "ctigprs"
password "ctigprs999"
remotename Claro
ipparam Claro
```

Ese archivo es el que llamas luego con el comando pon:

```
pon Claro
```

Fijate si te funciona, espero haber sido mas explícito ahora.

Saludos.

PD: sacale la extensión .txt al archivo cuando lo copies.
