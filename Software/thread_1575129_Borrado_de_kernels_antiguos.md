### Post by miguelwill on 2010-09-17
creo que la mejor opcion que tienes es hacer 3 pasos :

1.- Identificar el kernel instalado y funcionando:

```
root@miguel-desktop:~# uname -r
2.6.35-22-generic
```

2.- listar los kernel's instalados

```

root@miguel-desktop:~# dpkg -l|grep linux-image|awk {'print $2'}

linux-image-2.6.35-19-generic
linux-image-2.6.35-20-generic
linux-image-2.6.35-21-generic
linux-image-2.6.35-22-generic
linux-image-generic

```

el que dice "linux-image-generic" debes obviarlo, ya que es un paquete generico para el kernel.

3.- De esta lista, omites la version que tienes instalada y el paquete generico, y proceder a desinstalarlos:

```

root@miguel-desktop:~# dpkg -l|grep linux-image|awk {'print $2'}|egrep -v "linux-image-generic|`uname -r`"

linux-image-2.6.35-19-generic
linux-image-2.6.35-20-generic
linux-image-2.6.35-21-generic

```

```
root@miguel-desktop:~# dpkg -l|grep linux-image|awk {'print $2'}|egrep -v "linux-image-generic|`uname -r`" | xargs apt-get remove -y


Leyendo lista de paquetes... Hecho
Creando árbol de dependencias       
Leyendo la información de estado... Hecho
Los siguientes paquetes se ELIMINARÁN:
  linux-image-2.6.35-19-generic linux-image-2.6.35-20-generic linux-image-2.6.35-21-generic
0 actualizados, 0 se instalarán, 3 para eliminar y 2 no actualizados.
Se liberarán 322MB después de esta operación.
¿Desea continuar [S/n]?
```

con eso estaras desinstalando los paquetes de los kernel que no estes ocupando (dejando el que este en ejecucion), y con eso, automaticamente, se debe actualizar el menu de grub en cada kernel a desinstalar (lo acabo de ejecutar en mi instalacion, sacando las versiones que ya no estaba usando por los updates de ubuntu 10.10 beta :D )

saludos
