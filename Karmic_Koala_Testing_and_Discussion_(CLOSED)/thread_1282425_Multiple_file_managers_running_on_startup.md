### Post by drs305 on 2009-10-04
I've resigned myself to showing the Desktop as well, since this is the easiest and most permanent fix. The bug has been around at least since I started testing Karmic at A2.  

If you really HATE showing the Desktop, there is another workaround. You can edit /usr/share/applications/nautilus.desktop and change the following line:
> 
X-GNOME-AutoRestart=true
to
X-GNOME-AutoRestart=false


This will prevent the spawning of multiple Nautilus windows on startup. The problem is that updates often overwrite the file and you have to repeat the edit when the behavior returns. (Note you can stop the spawning once it starts by running "killall nautilus" multiple times until the terminal reports "no process found").
