If something is more intimidating at first glance, how can it be more intuitive?

This idea looks like it is a more efficient type of scrollbar, but it is much less intuitive than what we use today. I can see my (not so technically-inclined) friends, parents and grandparents all struggling with this concept, and the few that become accustomed to this concept will become irritated when using other operating systems. 

To apply this concept in Ubuntu by default seems somewhat akin to forcing everybody to use a DVORAK layout keyboard for one specific operating system, and then ignoring the consequences (that 99% of the world's computers use QWERTY)...

I appreciate the author's originality, but it seems like an unnecessarily confusing concept. One way to reduce this confusion would be to separate the page location indicator from the scrollbar area, as most people would instinctively try to click and drag on it.

### Post by 23meg on 2009-06-27
> **ELD said:**
> I still don't get what the difference is to be honest, since the site itself says gnome shell will replace parts of gnome? To quote:
"The plan is to have it be an optional feature in GNOME 2.28, and to replace the existing components for GNOME 3.0."

"The existing components" in this context refers to Metacity + GNOME Panel. GNOME Shell is not the whole of the projected GNOME 3.x platform, but a component of it, in the same sense that Metacity and GNOME Panel are not the whole of GNOME 2.2x, obviously.

People tend to use the terms GNOME Shell and GNOME 3 interchangeably, perhaps due to the fact that GNOME Shell is one of the two most user-visible and already-functional components of the GNOME 3 plans at present (the other one being GNOME Zeitgeist). I intended to point out that factually that's not any more correct than using the terms Metacity and GNOME 2.2x interchangeably.
