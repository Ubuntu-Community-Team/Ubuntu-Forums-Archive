### Post by sim-value on 2009-06-10
Hi i just had this idea i would like to hear your Thoughts on:

Many things in Ubuntu can be done Easily with the commandline if you know the commands ...

Right now if a User wants something lets say "Play piano.mp3 at Startup" he will either:

1. Think it doesnt work
2. Search for commands and be frustrated it doesnt work VIA Gui ...

Here is where the Script center come into Play.

In the script center (SC) users can add there Terminal Scripts (think of Macros) which do usefull things then no longer at the Copy and paste of a Line but at the Click of a Button. SC offers an Full featured Search & Tagging Function.

SC is the Start place for anything the User wants to do SC Scripts can switch settings (Do not ask for confirmation at shutdown) Automate and easy tasks (Add repositories & Keys) help in File manipulation (Drag and drop a file at the script to change its creation date) and much more .. in short it gives an intuitive to all the Power of the Terminal.

Lets demonstrate this in two examples: 

User A wants to turn dontzap off .. So he goes to SC and enter "Activate Ctrl+Alt+Backspace" or "dontzap" into the search function ...
Then clicks the result "Bring Ctrl+Alt+Backspace back" eventually enters his PW and is Done !

User B wants to let his PC turn off at 5 PM so he goes to SC enters "Shutdown automatic time" (Because he is used to tagsearch) selects the Appropriate Result Clicks is Prompted for the time and DONE !

SC is designed (would be anyway) to be a All-In-One Solution,Hotfix,Help and Assistance center using the Power of the Communitys Terminal Knowlegde ...

So this is my idea only problem im seeing is that all commands added would need to be checked ...

/me
PS: if you want a shortway of imagining this think of Ubiquity or GNOME do extended whith an AI so that you only enter what you want and get it ...
