### Post by ranch hand on 2009-10-21
I have upgraded a number of 9.04 variations to 9.10.  Most of them have quit that behavior as they have been updated and the usplash stuff from 9.04 is replaced.

I am still having trouble with one of them but it is with the GDM.  You do not appear to have that problem or you souldn't be getting the first image that you show.  That image, if you are interested, is in /usr/share/images/xsplash and GDM gets the image for the login screen form there too using the bg_2560x1600.jpg.

I would try hitting "e" when you get your menu and removing the first 2 lines, the ones about record fail, and see if that helps first.

If that doesn't work, do the smae thing but replace the whole entry with;
```

        set root=(hd0,1)
        linux /vmlinuz root=/dev/sda1 so quiet splash
        initrd /initrd.img

```
Two things;
1. the above is assuming SATA, you may need to edit line two
2. I only have one pae kernel install and I do get a few (less than 6) lines of text before the usplash with the above type entry for it.

### Post by gdonwallace on 2009-10-21
When you did the upgrade, it should have tried to install new grub information for booting up.  It it did, it should have asked if you wanted to keep the one that it found, or install the newest.  If the install did this, then it found a menu.lst file that was modified to bypass the xsplash and run boot up process showing what was happening as the system boots up.

in the /boot/grub directory you should see menu.lst & menu.lst~.  The second is the backup that the system makes.  Check that one to ensure that it boots to the latest kernal.  If it does, just copy it to menu.lst and it will fix the problem

If you are not familiar with the copy command try this:

**sudo cp menu.lst~ menu.lst**
