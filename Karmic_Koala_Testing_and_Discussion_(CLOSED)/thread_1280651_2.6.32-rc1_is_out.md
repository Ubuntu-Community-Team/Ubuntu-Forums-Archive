### Post by MacUntu on 2009-10-02
Actually it's out a couple of days:

[QUOTE=Linus Torvalds]

It's been two weeks (and then some - but with last week being LinuxCon and 
Plumbers conf I extended it by a few days), and as usual that means that 
the merge window is all over and done with. 2.6.32-rc1 is out, so give it 
a whirl.

What can I say? 67% drivers (the bulk of which is from 'staging', but 
there's driver changes all over), 10% firmware, 10% arch updates 
(dominated by arm, but MIPS, POWER, SH and x86 updates are there too, 
along with the new 'SCore' architecture), 5% Documentation, and a random 
smattering of other things (ie the normal filesystem, kernel, networking 
etc updates).

For a change, I don't think we have a single new filesystem this time 
around, but we do have updates to existing ones (ocfs2, btrfs, nfs, nilfs, 
xfs, gfs2, ext4 - you name it).

Some of the more interesting changes (but perhaps that's just me) are some 
of the VM updates (ZERO_PAGE is back!) and the writeback work by Jens and 
others to spread out writeback by backing store. 

Go wild, test it out, and let us know about any regressions you find,

		Linus
[/QUOTE]

[http://lkml.org/lkml/2009/9/27/166](http://lkml.org/lkml/2009/9/27/166)

Running fine here so far.

For sreadahead to work, those changes are required: [http://kernel.ubuntu.com/git?p=ubuntu/ubuntu-karmic.git;a=commitdiff;h=8cced4c6dcdcc6ba4a679dcf7ebb3000045a581f](http://kernel.ubuntu.com/git?p=ubuntu/ubuntu-karmic.git;a=commitdiff;h=8cced4c6dcdcc6ba4a679dcf7ebb3000045a581f)

### Post by zevans on 2009-10-03
X repos I use:

```

# deb http://ppa.launchpad.net/ubuntu-x-swat/x-freeze-test/ubuntu/ karmic main # disabled on upgrade to karmic
# deb-src http://ppa.launchpad.net/ubuntu-x-swat/x-freeze-test/ubuntu/ karmic main # disabled on upgrade to karmic

# Bryce's X updates
# deb http://ppa.launchpad.net/ubuntu-x-swat/x-updates/ubuntu/ karmic main # disabled on upgrade to karmic
# deb-src http://ppa.launchpad.net/ubuntu-x-swat/x-updates/ubuntu/ karmic main # disabled on upgrade to karmic

# Martin Pitt's X debug stuff
# deb http://ppa.launchpad.net/pitti/ppa/ubuntu/ karmic main # disabled on upgrade to karmic
# deb-src http://ppa.launchpad.net/pitti/ppa/ubuntu/ karmic main # disabled on upgrade to karmic

### Tormold Volden's X.org On The Edge
# deb http://ppa.launchpad.net/tormodvolden/ppa/ubuntu/ jaunty main
# deb-src http://ppa.launchpad.net/tormodvolden/ppa/ubuntu/ jaunty main
# deb http://ppa.launchpad.net/xorg-edgers/ppa/ubuntu/ karmic main # disabled on upgrade to karmic
# deb-src http://ppa.launchpad.net/xorg-edgers/ppa/ubuntu/ karmic main # disabled on upgrade to karmic

```

I found out about most of these by following bug discussions on Launchpad, and from this thread, which is about Intel but most of it applies to X development in general...
[http://ubuntuforums.org/showthread.php?t=1130582](http://ubuntuforums.org/showthread.php?t=1130582)

Since some of these are nightly snapshots they WILL sometimes break, so worth making sure you have a copy of the debs of your current version before you let an update go through, and also that you know how to shut down the box cleanly if X hangs. Pleased to say I haven't had a hang for some months now though!
