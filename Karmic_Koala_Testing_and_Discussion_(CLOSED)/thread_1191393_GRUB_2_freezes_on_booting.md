The problem is: once GRUB 2 menu appears on screen (it's chainloaded from Legacy), he cannot use his keyboard arrows to move accros the different entries of the boot menu. The selection stays on the first entry, and this guy cannot go to another one (i.e. for booting into Windows... ok, you might say that's not a problem because Ubuntu's better ;)).

Here are some things we did:
[LIST]
[*] We first tought that because his Windows version has an "É" in its name ("Windows XP **É**dition Familiale"), it might have been the cause of the problem. [*] We found his GRUB 2 entries contained an illegal boot option for Linux kernels (the comment *#GRUB_DISABLE_LINUX_UUID=true* was added as a boot option in kernel entries :-k). We edited */etc/default/grub* to correct the boot options and we executed *update-grub* to regenerate *grub.cfg*. Now, the menu entries should be alright, but the problem stays ;
[*] We tried disabling the use of UUID, by uncommenting the option *GRUB_DISABLE_LINUX_UUID* in */etc/default/grub* and again executing *update-grub*. It did not solve the problem. Recommenting this option did not solve the problem too.
[/LIST]
