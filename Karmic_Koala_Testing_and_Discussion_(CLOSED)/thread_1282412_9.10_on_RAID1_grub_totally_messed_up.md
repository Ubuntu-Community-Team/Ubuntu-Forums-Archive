I'm unable to boot to my system and I don't know what else to try without reinstalling the system, so I'll try asking here.:confused:

I've just installed Ubuntu 9.10 beta on the partitions that previously contained Ubuntu 9.04 (I did a fresh install, not an upgrade). Since I'm using two drives connected through RAID1, I used the alternate CD for the installation.

This is my complete previous configuration:
```
3 drives: sda and sdc were identical, sdb was separate and contained Windows XP.
As for the partitions:

/dev/sda1 + /dev/sdc1 = md0 : this was the root partition (bootable)
/dev/sda2 + /dev/sdc2 = md1 : the home folder
/dev/sda3 + /dev/sdc3 = md2 : a storage partition
/dev/sda4 + /dev/sdc4 = md3 : swap space
/dev/sdb1 : not on RAID1, this contained Windows (bootable)
/dev/sdb2 : not on RAID1, another storage partition
```

When I installed the new system, I kept the partitioning untouched except for: **sda1,sdc1,sda2 and sdc2** (which I formatted). I also had to delete the **md0 and md1** virtual drives and create two new ones, which the partitioner decided to call md4 and md5. So after the installation things looked like this:

```

/dev/sda1 + /dev/sdc1 = [COLOR="Red"]md4 [/COLOR]: this is the root partition (bootable)
/dev/sda2 + /dev/sdc2 = [COLOR="Red"]md5 [/COLOR]: the home folder

/dev/sdb1 : not on RAID1, this contains Windows (bootable)
/dev/sdb2 : not on RAID1, another storage partition
```

the installation worked perfectly, except for the startup, during which grub gave an error (sorry, I can't find it now) and then loaded a list whith the Ubuntu system but not the Windows one.

Probably my big mistake was to try the procedure anyway, running ```
GRUB> find /boot/grub/stage1

I've also tried both methods after changing the hard disk order in BIOS (and after reverting it back). The root/setup procedure always completes without errors, but nothing changes at startup. My main concern is that there is no menu.lst on my drives.
