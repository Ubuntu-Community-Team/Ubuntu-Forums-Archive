### Post by negativ on 2009-10-22
Because frankly, GRUB2 is 900 steps in the Microsoft direction.

They have gone back to the same mechanics of lilo in a lot of ways.  The config files are generated by scripts so you aren't supposed to edit them.

They have tried to make it portable, which isn't always a bad thing but a boot loader is a very specific beast.

They rewrote it from scratch, not always good because grub had finally got a lot of the edge cases down (bios oddities etc).

They wrote a whole vga console emulator, added gettext and all the other **** that goes along with gettext. WHYYYY???  I call that bloatware.

Addeed scripting support (because I really want loops, variables, functions and conditional statements in my boot loader).

Memory manager (why? just load my kernel into highmemory dammit).

It's gone from being a simple and working bootloader to mostly being a kernel and micro OS itself.

They are wasting time getting it to work on powerpc and ultrasparc, which already have bootloaders. I don't belittle this extra work, I just question it usefulness.

I guess my real biggest gripe is the config files are not editable. If you want to remove the windows boot entry from your boot menu, you have to edit /etc/grub.d/ scripts to not call the script that looks for the windows parititon and adds it to the config file.  

Thats just total **** in my opinion.

Grub2 feels like they have been trying to re-create a kind of open firmware for the pc. Soon we'll need a neo-grub just to boot grub to boot the linux kernel.

But that's what I get for foolishly coming back to a distro that makes a point out of removing choices during installation.

### Post by kansasnoob on 2009-10-23
> **CoreyB. said:**
> Is it possible to install grub legacy instead of grub 2 at first installation?

Well, sort of. But it can be quite tricky unless you know what you're doing.

The package 'grub-pc' (which is grub2) is installed by default, but if you choose "advanced" on the final review of installation changes when installing, you can choose not to "install" grub to the mbr (which is the default).

The tricky part is that after installation you'll have no way of booting the new Karmic install unless:

(a)You have a multi-boot and know how to add Karmic to an existing grub.  

(b)Instead of rebooting after the install choose to keep working from the Live CD and mount and chroot the newly installed Karmic and install legacy-grub or the boot-loader of your choice before rebooting.

Regardless you must still do some package changes or 'grub-pc' will want to "update" during such things as a kernel update and you'll end up hosed when you least want to be.

I can't really advise you without knowing your unique situation. 

Why do you want to?

What is your partition/drive layout?

How many operating systems and what are they?

I actually recommend using grub2 in most cases.

I've submitted this for tutorial status (with a few very minor changes) and am awaiting approval:

[http://ubuntuforums.org/showpost.php?p=8071880&postcount=18](http://ubuntuforums.org/showpost.php?p=8071880&postcount=18)

Edit: It's possible that option exists using the Alternate CD, but I'll have to check it out and see. It's likely to be a few days before I get around to it though.
