It's an evolution, provided you have the dependencies it can run in 2.28. I doubt that will be recommendable though, it uses whole new intrastructure (the introspection stuff as well as the javascript bridge to GObject), it's also still early in development. I doubt it will be ready for us to bestove on innocent users as anything but a preview, aside that the design isn't fully hashed out yet there are many cases the shell people has not yet figured out how to deal with.

As for what will be in 2.28, I think we will have a clearer idea once the cycle is up and going for a while. We still haven't even reached the module proposal stage yet. Webkit-gtk has been making a lot of progress in accessibility so now it should finally be able to be made the default in Epiphany as well as a blessed dependency for GNOME. That is one thing I look forward to personally.

Aside that the Tomboy syncronization service sounds like an idea that is made of complete win, I hope that gets to a point where we have a prototype soon.
