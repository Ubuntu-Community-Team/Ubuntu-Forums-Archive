### Post by infamousrev on 2009-09-10
Ive done this quite a few times, 

Try this (added new dir mount to prevent conflicts):

sudo mkdir /mountpoint 
sudo mount /dev/sda3 /mountpoint
sudo mkdir -p /mountpoint/dev
sudo mount --bind /dev /mountpoint/dev
sudo cp /etc/resolv.conf /mountpoint/etc/resolv.conf
sudo chroot /mountpoint
sudo mkdir -p /proc
sudo mount -t proc proc /proc
sudo mkdir -p /sys
sudo mount -t sysfs sysfs /sys



Note the mkdir -p, commands are mainly to make it failsafe

### Post by philinux on 2009-09-10
I got this mount/point thing.

They mean it as an example, In my case I used /mnt/karmic

e.g mkdir /mnt/karmic

This is my bash history for this.
sudo mkdir /mnt/karmic
sudo mount /dev/sdb1 /mnt/karmic/
sudo mount -o bind /proc /mnt/karmic/proc
sudo mount -o bind /dev /mnt/karmic/dev
sudo mount -o bind /dev/pts /mnt/karmic/dev/pts
sudo mount -o bind /sys /mnt/karmic/sys
sudo cp /etc/resolv.conf /mnt/karmic/etc/resolve.conf
sudo chroot /mnt/karmic /bin/bash

Once in then you can do sudo apt-get update etc etc.
