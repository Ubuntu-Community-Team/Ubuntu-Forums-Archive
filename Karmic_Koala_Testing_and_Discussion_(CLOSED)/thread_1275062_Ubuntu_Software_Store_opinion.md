### Post by alvevind on 2009-09-25
I think the idea of Ubuntu Software Store is really great.

But I agree there are significant usability issues that ought to be ironed out before releasing it.

I think your points are all valid. 

First step is a menu navigation to the left, the next is a icon based grid, the next is a confusing path-that-looks-like-buttons based navigation at the top and a single-click-does-nothing list. 

This happened to me the first time I used the application:
After navigating to the piece of software I wanted to install I found myself clicking the "Get Free Software" button. This of course brought me two steps back and did not install anything. On the second attempt (after several more clicks) I scrolled down half a page and found a very well concealed install button. If the application had been usability tested this would not be hidden in this way. When you get to the application the install button should be right there and not hidden. I also do not see any good reason why the application window itself is so small by default, forcing the user to scroll more than needed. 

Programmers are (unfortunately) programmers and often love to show off all the different GUI possibilities for no particular reason. But for actual everyday users who just want a job done it is much better to stick to a simple and obvious navigation scheme. 

Hopefully the Ubuntu usability team will review this application soon. It is not big changes that need to be done, but the usability flaws are quite obvious. 

Until then we can stick to Synaptic. :-)

### Post by The Cog on 2009-09-25
I agree it is non-intuitive to use. The item list has a little arrow all the way over on the right hand side, so far away from the text it's not al all obvious, and its purpose is far from obvious. Having clicked it, you discover that a new tab appears on the top and you can then understand what the square on the right was for.

The series of tabs along the top doesn't behave like tabs should - clicking the leftmost tab closes all the tabs to the right of it. WTF??? The usual way to do this is with left and right pointing forward and back arrows like a browser or a wizard, not mysteriously interlinked tabs.

And when it's downloading something there's a progress bar that doesn't progress, and a button next to it that I guess might be a cancel button. But it's not labelled and doesn't have a tool-tip when you hover over it, so the only way to find out if it really is a cancel button is to try it while doing a download and see what actually happens. 

The name software store is suggestive of having to pay - I half expected to see a series of commercials when I first clicked it - software emporium or software repository might be a better name. The first impression with all the different "department" icons looks really nice. Then all the usability issues show up.

For some reason the whole thing seemed to be non-intuitive from start to finish. It works, but nothing works quite how you would expect - like a house where the light switches are on the hinge side of the door and the hot and cold taps are swapped etc. You can't just use it, you have to experiment with it to figure out how it works. Last time I found a UI that difficult to use was one of the stranger skins for the xine media player.
