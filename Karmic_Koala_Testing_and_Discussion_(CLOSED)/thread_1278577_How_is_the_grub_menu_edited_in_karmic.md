### Post by drs305 on 2009-09-29
barisurum,

Grub 2 takes a completely different approach to the menu. The file that controls the menu you see is /boot/grub/grub.cfg, but it is not meant to be edited.

Rather, changes are made to /etc/default/grub and the files in /etc/grub.d
When "sudo update-grub" is run, the scripts in these files make the changes to grub.cfg.

Here is a basic introduction to Grub 2. There is a link at the end to the Ubuntu Grub 2 wiki as well:
[Grub 2 Basics]("http://ubuntuforums.org/showthread.php?t=1195275")

You should not edit the /boot/grub/grub.cfg file.  It is read only for root.  Yes you can change the permissions, edit the file.  This will last just as long as update grub is not run.  When update grub is run that file will be over written and the permissions back to read only.

What you want to do is go to /etc/grub.d to create custom entries and to /etc/default/grub to edit your timeout and hidden menu options.

The grub.cfg file is generated by update-grub from those two locations.

There are a lot of threads on this testing forum about this and how to get several problem OS' to work (Fedora, Gentoo, Mandriva).

There is a problem with os-prober in relation to those OS'.  It is coming along.

You can also edit the menu when it comes up by pressing "e" as in grub-legacy.  The editing, I think, is a little easier in grub2.  You do not hit "b" to boot (as in grub-legacy) but Ctrl+x.

It is important to know that the way things are counted is a little different.  The drive is numbered as in grub legacy but the partition is now numbered as in gparted.  An entry in grub-legacy that called for (hd0,0) would, in grub2 be (hd0,1).

I was able, for the first time to boot into Mandriva2009-1 64bit today with an edit to the generated menu entry.  Happy.
