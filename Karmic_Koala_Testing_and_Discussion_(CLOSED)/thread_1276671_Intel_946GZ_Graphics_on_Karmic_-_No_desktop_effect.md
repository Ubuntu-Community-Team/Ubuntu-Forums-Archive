edit: I found a better workaround. As quoted from Arnaud on [Launchpad]("https://bugs.launchpad.net/ubuntu/+source/libdrm/+bug/430876"):

> This lead me to the following workaround:

which always fixed the problem.

----
Alright, I found a workaround. The issue was with the /dev/dri/ folder not existing, and /dev/dri/card0 being placed in /dev/.

If you're having this issue too, add these lines in /etc/init/gdm.conf after the line 'script':
```
    ### Fix missing /dev/dri/card0
    # Make sure that /dev/dri/ exists
    if [ ! -e /dev/dri/ ]; then
        mkdir /dev/dri/
    fi
    # Create symbolic link
    if [ -e /dev/card0 ] && [ ! -e /dev/dri/card0 ]; then
        ln -s /dev/card0 /dev/dri/card0
    fi
```

That seemed to fix it for me after a reboot. If you want to enable effects now and reboot later, you can run the following commands in the terminal. 
Note: The symbolic link will get removed after a reboot and disable your desktop effects unless you complete the first workaround.
```
$ sudo mkdir /dev/dri/
$ sudo ln -s /dev/card0 /dev/dri/card0
$ sudo pkill X
```

Hopefully some of you will find this useful. I was helped in finding this solution by [yofel]("http://ubuntuforums.org/member.php?u=870338") and [ActionParsnip]("http://brainstorm.ubuntu.com/knetworkmanager/contributor/ActionParsnip") in the IRC channel.
