### Post by oldfred on 2009-10-17
Do not edit grub.cfg, It is to only be created by combining all the other bits & pieces by update-grub.

from arpanaut first link, the quick answer to the specific question is #7 in the link.
To prevent one of the /etc/init.d files from running, remove the "executable" bit.
          o Example: If you don't want to see the "Memtest86+" entries, run this command:
            sudo chmod -x /etc/grub.d/20_memtest86+
          o Run the update-grub command to allow the changes to be incorporated in grub.cfg
