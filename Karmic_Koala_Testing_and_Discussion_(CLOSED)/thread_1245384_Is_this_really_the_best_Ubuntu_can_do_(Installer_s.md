The package manager (which is what implements the Add/Remove) in Ubuntu exists because the software it manages all have the same cost (free).

For Windows to have something like that, they would need to essentially create a Windows App Store. The problem is that people have sold software independent of MS for so long, it could be hard to migrate them towards such a system.

The only reason it's worked so far on any other platform is because the creators of the stores tend to have control over the software that is allowed to run on the platform/hardware (ie apple or palm apps must be signed by apple or palm if they are to run on anything other than developer registered devices).

The other option would be to "jailbreak" the device and distribute apps another way, but who really envisions that someone is going to spend time creating such a system for Windows based pcs (which are essentially already jailbroken). Though really. . . there is probably good money in that.

I've spent over four years helping new users on these forums, and I can assure you there are plenty of new users savvy enough to install their own OSes and more knowledgable about computers than I am (e.g., have built their own computers, know how to program) who do **not** know to install programs using Add/Remove.

They will still, as their first instinct when looking for programs, open up a web browser and go to the program's homepage to download a .tar.gz file--thinking that's the Linux equivalent of a setup.exe file.

If the slideshow does nothing else, it should acquaint ex-Windows users with package management.

And it is no more annoying to veteran users than the regular progress bar.

**Scenario 1:**
*Veteran user clicks install and walks away.* With the old installer, you don't see anything, because you just walked away. With the new installer, you also see nothing, as you just walked away.

**Scenario 2:**
*Veteran user clicks install and does something else.* With the old installer, you don't see anything, because your new application (Gnometris or Firefox) is blocking the installer progress bar. With the new installer, you also see nothing, as your new application is blocking the slideshow tutorial.

**Scenario 3**
*Veteran user watches the progress bar*. I don't know why you would do this, but if you do then all you see is a bar moving from left to right over the course of 20 to 30 minutes. With the new installer, you'll see a few pretty pictures and some text to read.

> From ./README:

* Write in the present (no "Ubuntu will" or "Ubuntu would"). Not just
present tense, but with an awareness of what is going on around the
slideshow. We can assume the user is running a default Ubuntu session as
he installs it, and also that he has chosen to install Ubuntu willingly;
that person is now an Ubuntu user. The sales-person voice should not
exist. If you start to hear that voice, file a bug immediately.

* Avoid the car instruction manual voice where possible. That is, try not
to tell people where specific features are, but what they can expect to
find. This is because the slideshow won't follow our users everywhere,
so we have to answer a different question: "What next?" If we fixate
energy on pressing buttons, it will not stick :)

* Write from the perspective of us (people, probably) explaining
Ubuntu and free software to the immediate end user; the person behind
the screen. Think of it as enthusing with a friend about this great new
thing they have started to explore, making sure they know what they need
to know to really enjoy it.
