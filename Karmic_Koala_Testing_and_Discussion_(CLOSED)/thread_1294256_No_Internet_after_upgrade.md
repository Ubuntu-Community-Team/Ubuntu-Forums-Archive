### Post by shafin on 2009-10-18
Hi,
I've been running Karmic since alpha 3 in a virtual machine without any problems. Yesterday I decided to bite the bullet and upgraded. My machine has been upgraded from Hardy>Intrepid>Jaunty, with breakage occurring in each of them. However, all of them was concerning xorg. This time, something else broke. 
I have no internet connection after upgrade. Network Manager Applet do not appear in the notification area. If I remove and re-add notification area applet, it reappears. Then there are no connections. I had a wired connection, which did not carry over the upgrade. Now a new connection is there, with name Ifupdown(eth0) which is not even editable. I can add a new connection, but if I tick "apply to all users", the connection instantly disappears. I can add a connection without this box ticked with all my settings, but that do not connect. It just stays there. "sudo dhclient eth0" conects to LAN, but as I am on a manual connection and not a DHCP one, it still do not connect to internet.  Is there a way to remove this ifupdown connection? I suspect it's conflicting with my newly created connection.

Please help me find a solution for this. Is there any way I can connect via terminal with my IP, netmask and gateway?

``````
cat /etc/resolv.conf
# Generated by NetworkManager
search www.google.com
nameserver 208.67.220.220
nameserver 208.67.222.222
``````
$cat /etc/NetworkManager/nm-system-settings.conf
[main]
plugins=ifupdown,keyfile

no-auto-default=MAC address ... put Yours ...

managed=false
```With a setup like this You should be able to edit Your connection in NM. Good luck.
