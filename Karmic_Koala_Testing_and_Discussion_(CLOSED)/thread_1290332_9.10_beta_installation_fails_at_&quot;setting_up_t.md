### Post by anewman on 2009-10-13
The upgrade didn't go to well so I thought a clean install would be a good idea. I am using latest Beta CD downloaded via ubuntu website. It runs the live cd fine, but fails to install. It gets as far as "setting up the partitioner" with the window displaying a percentage that does not increase, usually 2%. The window then disappears (I presume it is crashing out for some reason though no error is displayed). This leaves the console displaying the following 3 lines of text 
> Starting early crypto disks... ok
Starting remaining crypto disks... ok
Starting init crypto disks... ok
No further text is displayed and eventually a crippled live version is loaded up with the help icon missing, accompanied by a 0 bytes left on the disk warning. Have searched on the net but not managed to find anyone else with the same symptoms.

Original installation was from an 8.10 Intrepid Ibex CD and this was upgraded to 9.04 Jaunty Jackalope which all worked fine.

Machine is somewhat antequated. AMD Athlon xp 2200+, 1gb DDR Ram, Abit NF7-S v2.0 (Nvidia nforce2 chipset). GFX Nvidia GeForce4 Ti4200.

Attempts at remedying the problem have included disconnecting all hard drives other than the one being installed to. Disabling floppy drive in Bios. Also tried using a separate CD drive, and attempted the Windows-based installer. Also tried deleting the partitions on the hard disk and formatting it.

Thanks in advance for any input or suggestions :)
