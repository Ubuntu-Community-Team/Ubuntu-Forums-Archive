### Post by Merk42 on 2009-06-04
Old thread [here](http://ubuntuforums.org/showthread.php?t=1140246)

Themes and/or a new look is always a popular topic in the development and art subforums.  A new thread was bound to be made, so why not by the same user who made the last one which as of this writing has over 18K views as in the second most popular thread in the Testing and Discussion subforum.

As with the last one, I'd ideally like this thread to be a discussion of the official changes to Karmic Koala.  I plan to update this first post with any important information regarding the UI of Karmic.

So here's a bit of a recap from the last thread, which may also serve as  a FAQ.

**THE FIRST ARTWORK DEADLINE HAS COME**


**THE SECOND ARTWORK DEADLINE HAS COME AND GONE AND YOU DIDN'T MISS ANYTHING**

We did get the final(?) xsplash in Alpha 6 though
Click to watch a video 
[[IMG]http://www.markecurtis.com/etc/boot.png[/IMG]]("http://www.youtube.com/watch?v=cOJUY35N-pU")

**THE FINAL ARTWORK DEADLINE HAS COME**
[img]http://www.markecurtis.com/etc/appearance.png[/img]

***So we're finally getting a new theme this time, right?***
Sorta.  [quote=Kenneth Wimer (kwwii)]there will not be a new human theme {for Karmic}[/quote]
Yet see above


While Mark Shuttleworth did speak about it early on:
> The desktop will have a designer's fingerprints all over it - we're now beginning the serious push to a new look. Brown has served us well but the Koala is considering other options. Come to UDS for a preview of the whole new look.
He later said this:
> it has taken a long, long time to pull together a design team i had hoped to have that team in place six months ago, but it's still forming i think we will make good progress in the next cycle you can already see a few things that have borne fruit from that team notifications, time zone selector in installer, etc but it's fragmentary i'm pretty darn confident we'll have a new look for 10.04 but i think only pieces of that will emerge for 9.10
Basically the thing to take away from this is if anyone says anything about a change or new feature, even Mark Shuttleworth himself, take it with a grain of salt.

***Great, so no theme AGAIN?***
Well it's still "Human" but darker, more "chocolate" and has the Humanity Icon set
Also, we're getting a new GDM (login screen). 

For anything else, here is the schedule for the Artwork deadlines:
[ArtworkFirstDrop](https://wiki.ubuntu.com/ArtworkFirstDrop) is August 27th
[ArtworkSecondDrop](https://wiki.ubuntu.com/ArtworkSecondDrop)  is September 10th
[ArtworkDeadline](https://wiki.ubuntu.com/ArtworkDeadline) is September 24th
In the meantime, the best place to look in the blueprints.  As of right now the entry for [Karmic Desktop Artwork](https://blueprints.edge.launchpad.net/ubuntu/+spec/desktop-karmic-desktop-artwork) is blank aside from "also probably themes and stuff".  I originally though that was because  UDS had just ended, and there still need to be a bit more time to organize a lot of what happened there.  At this point though, I'm not sure what is going on...
On the plus side the [community](http://www.nabble.com/ubuntu-art-f12728.html) is always active.  Most popular now are an alternative icon set (Breathe) and the New Boot Experience. Also, the (arguably) popular GNOME-Colors, Shiki-Colors, and Arc-Colors will be in the Karmic repositories. It's not default, but it is something.

***Why won't they ever listen to us? We keep saying we want a new theme!***
Where are you saying it though?  You should contribute to [brainstorm](http://brainstorm.ubuntu.com/) if you have an interface idea, or the [art team](https://wiki.ubuntu.com/Artwork) if you're more graphically inclined.  I'm sure some of you are surprised to read that coming from me.  This thread is how I'm contributing, getting discussion across, being "the" thread to view for the average user, and maybe convincing some of them to take their ideas a step further into the art projects or brainstorm.

***So if I want a change, I have to do it myself?***
Some changes are scheduled to happen. If you really want change, [build the GNOME shell now](http://live.gnome.org/GnomeShell), [Karmic will have **GNOME 3**](https://wiki.ubuntu.com/DesktopTeam/Specs/Karmic/GNOME3) in the universe (not installed by default).  It's still really early at this point.  It hasn't even been decided what is going to happen to minimized windows.
**Plymouth** may or may not be in Karmic.  The idea is to focus on such a quick boot for Karmic+1, supposed 10seconds, that it would be too quick to even notice the change to Plymouth.  
There should be a **Boot Experience** discussing the full graphic transitions [from BIOS/EFI to Desktop](https://wiki.ubuntu.com/DesktopExperienceTeam/KarmicBootExperienceDesignSpec)
While it seems there is no new theme, there is some sort of framework being worked on.
[quote=Kenneth Wimer (kwwii)] we are working towards client-side window decorations and stuff...well, the windows don't have an orange bar at the top
that alone is a pretty big change[/quote]
 
***I don't care anyway, I wish they wouldn't waste time on a new theme, get rid of the bugs first!***
The people working on theme, aren't the people working on the bugs. This is a good thing, since overall a person who knows programming doesn't know graphic/interface design and vice versa.  So we should be able to have our cake and eat it too. Can't we all just get along?

***It's so easy to change, what's the point?***
That may be true, but we shouldn't need to. If we want to fix bug #1, I strongly feel it has to look appealing. If Ubuntu is in stores, it will be shown with the default theme.  First impressions count, Mark Shuttleworth said so himself. 


Hopefully this thread can be used for a discussion on what sort of "new look" we'd like in Karmic Koala. I myself would like for whatever the look is, to be also in the other *buntus. Distros like Fedora and openSUSE have a similar look across their DEs.

This should be the same for all effects and theme choices. So a basic system unable to support emerald with low memory would run the basic GTK theme and background, but a newer more powerfull beast would install with all the bells and whistles from the start - showning the user what Ubuntu is capable of looking like.

I know a lot of people will hate this idea, but I think that is the problem when it comes to discussing default themes - as linux is so configurable then there are a great many choices to be made when deciding on a default look/feel.

Download to your desktop.

Leave it as an archive (tar.gz)

Right click anywhere on desktop.

Select 'Change desktop background'

Click 'theme' tab

Click 'install' button on bottom right.

Point it to your tar.gz and you're good.

On that same screen you can switch back and forth between themes.

PS you can delete the tar.gz once it has been installed.
