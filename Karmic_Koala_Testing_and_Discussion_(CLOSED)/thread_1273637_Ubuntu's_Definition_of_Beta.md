### Post by Keyper7 on 2009-09-23
Technically, yes, the beta is supposed to be feature complete and developers only concentrate on fixing bugs at this point.

However, the beta release is also a period when there is a massive influx of new bug reports from users who didn't dare to test the alphas. So while most of the updates will probably be minor and won't bring any new features, it's reasonable to expect *a lot* of them to come.

### Post by Keyper7 on 2009-09-23
> **novafluxx said:**
> Whats the real difference between the Beta and the Release Candidate then?

It's more or less like this:

Alpha = Major upgrades, addition of features and api changes. Bugs *certainly* exist.

Beta = Feature and api freeze but no polish. Bugs *probably* exist.

RC = Polished for release. Bugs *shouldn't* exist.

Final = In an ideal world, an exact copy of RC.

The interval between Beta and RC is bug fixing period. The interval between RC and Final is something like "This should NOT be a bug fixing period, but the real world is cruel and Murphy is a bitch. So let's be prepared for the worse."
