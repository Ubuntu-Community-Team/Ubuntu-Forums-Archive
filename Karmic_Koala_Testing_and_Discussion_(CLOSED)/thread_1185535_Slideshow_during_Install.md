Right now we have thrown in the documentation type stuff we believe is necessary, and the next step for content is going to be trimming that down and adding some fun stuff; what's new and *exciting*. ### Post by Mr. Picklesworth on 2009-06-13
Oh, two more!

I am hesitant to have launchers, simply because it would encourage people to run a ton of stuff on the live cd during a fairly delicate operation. Nothing bad has sprung up from this happening, to my knowledge, but I don't want to poke at it. However, if you check out slides/MenuTest.html you'll see my ongoing attempt at introducing applications more clearly.

The images at the bottom of each slide are how they are for a few reasons. First of all, there isn't much space to store this in. Secondly, there isn't much physical space in the 800x600 resolution target. (In retrospect, X is doing a really good job nowadays with getting the right resolution out of the box, so maybe netbook size is a better target in the future; 1024x700). Thirdly, a single scaled screenshot just doesn't work.
The plan is to hopefully show people what they need to see to think "hey, that looks interesting!" and then to find the rest on their own because the applications are so easy they don't *need* documentation!


There, I think that introduces the plan fairly smoothly. Go ahead and join the team, put up new bzr branches, file bugs (feature requests, please) and have fun :)

### Post by Mr. Picklesworth on 2009-06-13
Size of everything in bzr presently is around 900 kb. With the awesome pruning that just took place today (Thanks, Andrew S!) that size can probably be split in half. Just a bunch of unused screenshots hovering around.

The written content for each slide is split up in different HTML files, each hovering at about 1 kb each. I agree, some screenshots could definitely do with being translated, but in a lot of cases the written content is unimportant. (A "language neutral" localization, maybe using "Lorem Ipsum..." type text, would be pretty handy for this. Anyone know of such a thing?).
Since this is using web technologies, it would be *reasonably* easy to have alternative language screenshots hosted somewhere over the Internet, with the English ones on disk as a fallback.

Olskar: Yep, I think Blender deserves a mention somehow, if 2.50 makes it. That's one of the puzzles to solve. It definitely can't have a full 50 second slide, so maybe a slide with a 2x3 grid of pictures to point out that Universe does have real apps (different picks for each release?) would be a nice touch if there is space to spare.

That's sort of what I mean when I say that I would love this to *inspire* people to explore and get the most out of their computers, rather than just show them how to use (or why they should use) the OS. Ubuntu (and free operating systems in general) is about a lot more than the desktop; it's about the well maintained library of free software. Hence my absolute glee about that pruning, because it opens a bit of room for the cool stuff. (Although Andrew S also makes the important point, which I think I feel in action now he mentions it, that if we have anything more than 7 slides we can't expect people to remember them all).

Still loving that OpenSUSE screenshot.
I should point out that the html here is tidy, and this has the awesome capacity to contain all the brand new CSS3 goodies we want, like argb. It's also quite possible to pull a complete 180 for design at any time, if need be :)

### Post by Mr. Picklesworth on 2009-07-31
Oh, yes, feedback on this one will be REALLY appreciated. I, for one, am definitely in agreement that it looks awesome, but I want to make sure there is a rough concensus (and check if anyone else has some ideas) before pushing it to trunk :)
Keep in mind, if you do have alternative mockups to throw in, that the selection of building blocks has been trimmed to icons and text; no screenshots. (At least not in this iteration).


I want to avoid launching apps from the slideshow. I would prefer that people learn to launch apps the same way they would on the installed desktop, and I don't think the slideshow should actively *do* anything other than talk. As for links, there are currently real links for Launchpad Answers and Ubuntu Forums, but let's not expect them to work. (Don't say "click here :P).
The reason for that is simply because working hyperlinks require another step in whatever renders the page, so this could slip by with them not working.


As for getting the slideshow in, there is a really solid chance of it happening. Evan D is being very helpful, of course. Pywebkitgtk is in main, so that's one thing down. The content is proposed for inclusion (though there are some licensing details to tidy up).

It needs to be all tidy and good to go within a few weeks. The content is, today, a bit shaky but I think that could happen without too much trouble.

With regards to making it happen: I will see if I can come up with a todo list. (It would be a radical improvement from my previous, err, rather unfortunate management). Off the top of my head, it would be nice to have a smart makefile that doesn't rebuild things when there are no changes, and (as mentioned) licensing for icons needs to be cleared up. Also, the localization infrastructure is only half implemented; index.html needs some way to learn what locale it needs to display, then load slides from that locale's slides directory (eg: slides.fr).
If anyone considers himself a good writer, do feel free to conjur up your own spin on the slide text. I've started collecting these; they are all decent, but good writing usually comes from a single voice so we'll need to decide on one in a couple weeks.
