### Post by gnomeuser on 2009-09-06
I tried it last night, it's far behind the openSUSE Goblin release in terms of usability and polish. It seems in fact to be a very plain Moblin2 release with nothing added of value so one might as well run the official release from the Linux Foundation (true it don't yet come with office tools but since it is based on Fedora surely that is a fixable problem) or the nice openSUSE Goblin release.

Neither option is production quality yet but they are getting closer. There are matters of instability to deal with.

I would still like to see Moblin integrate some light wordprocessing I doubt however that the tactic taken by Novell and Ubuntu is productive. OpenOffice is simply to heavy, it still doesn't fit into the new interface. OLPC's Sugar interface has a specialized AbiWord instance included with collaborative editing via telepathy, this style of integration seems very well suited for Moblin so I would wish for that be the direction they would go to.

The inclusion of the Anjal email client instead of reliance upon webmail is a baffling choice in it's nonsensicalness. It's slow, requires storage space, wastes battery for filtering. I just don't feel it buys the user anything special for the obscene cost it carries.

I am also not sold on the built in media library, it seems incomplete, unstable and lacks library handling. I think it would have been a better bet to build on top of Banshee.

Finally carrying around Firefox for a netbook is a bit on the heavy side. You aren't using the main (insecure) selling point of Firefox namely plugins. It clobbers performance and has none of the stability/security features available to us as offered by a design like Chromium has. If you are building a system that is in essence a portal for the online experience I think the trade off in memory usage that sandboxing and process separation is well worth the investment. The stupid choice here is to stick to a known unstable platfrom with none of these features.. sure you get a downloader for free but really, would you rather have stability and the requirement to write your own downloader or get it for free and have one bad tab be able to take down everything.. yes that was what I thought as well.

I am impressed by the empathy work, I still wish they would realize that they desperately need to invest in making video/voip work for the major protocols. This is a killer feature for an on the go netbook experience and so far it only works for jabber style protocols. The chat functionality also needs to be separated out into a better integrated application, the current method of working tends to get chat windows lost.

All this being said, my EeePC 1002HA runs Goblin and I am fairly pleased with the experience so far.
