### Post by rockin_goliath on 2009-07-21
Instead of the default panel setup for Ubuntu Netbook remix, I have been using the plain old Window List and uni4dfx's Window Button Applet ([http://gnome-look.org/content/show.php/Window+Applets?content=103732]("http://gnome-look.org/content/show.php/Window+Applets?content=103732"))

I found this to be much better than the default Window Picker, since it allows me at actually read the names of the windows on the panel. It is also similar enough to using just the regular Metacity window border that I hardly notice that the buttons are actually sitting in the panel and are not part of the window itself. It is also requires fewer clicks to unmaximize windows (two with the default method and one with uni4dfx's method).

I personally think this a much better approach to window management on netbooks and small screens, since it makes window management a more familiar experience and saves the same amount of vertical space. There is actually no loss of horizontal space either, since under the default method, windows were displayed only as icons. Using the window list and uni4dfx's Window Button Applet, windows fill the available space and then start to compress their width while displaying as much of the name as possible.

The one problem with the Window Button Applet is that it does not seem to be bound to the state of the current window. For example, instead of only displaying itself if the current window is maximized, it will display itself if there are *any* maximized windows, and disappear only if there are no maximized windows. This behavior could probably easily be corrected.
