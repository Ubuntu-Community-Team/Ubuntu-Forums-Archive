### Post by dino99 on 2009-08-27
As you have several kernel release, grub2 might pop up menu to choose the kernel you want to boot with. Last upgrades have set Esc & shift as hot keys if the menu is hidden.

If your config have been customized, let run 
- sudo update-grub
- sudo os-prober ( you might have installed it)
- sudo grub-mkconfig

here is my /etc/default/grub:

# This file is sourced by update-grub, and its variables are propagated
# to its children in /etc/grub.d/
GRUB_DEFAULT=0
GRUB_TIMEOUT=5
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

# Uncomment to disable graphical terminal (grub-pc only)
#GRUB_TERMINAL=console

# Uncomment if you don't want GRUB to pass "root=UUID=xxx" parameter to Linux
GRUB_CMDLINE_LINUX="gfxpayload=true"

as you can see, there is no hidden command line. That is a non customized file.

### Post by alphacrucis2 on 2009-08-27
> **Jim March said:**
> OK...that answer was...not real understandable.  Sorry.

Can somebody please explain how to make sure grub2 always pops up a kernel choice menu?

Comment out the lines in /etc/default/grub

That say:

```
GRUB_HIDDEN_TIMEOUT=0
GRUB_HIDDEN_TIMEOUT_QUIET=true
```

ie put a # in front of them (or remove those lines). Save the file then:
```

sudo update-grub
```
