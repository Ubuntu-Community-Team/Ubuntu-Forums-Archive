Are you sure it's the 64-bit plugin? If it is, why is it using nspluginwrapper (which is necessary only for 32-bit plugins)? You performed an ldd of the wrapper library generated by nspluginwrapper, which is why it is described as 64-bit.

Nevertheless, whether it's 32-bit or 64-bit:

Install PulseAudio Volume Control (pavucontrol), launch it, and monitor the Playback tab. Run Rhythmbox on its own, and then Flash on its own, and see which application shows an entry. 

If it's the 64-bit plugin, I suspect that Flash isn't passing audio to the PulseAudio server (and if an application plays sound directly to the ALSA sound device, PulseAudio gets blocked until that application releases control of the sound card).

If it is the 32-bit plugin (which I'm fairly sure), then the problem is what I originally described.

P.S. The "padsp" wrapper passes OSS output to PulseAudio, so it won't work in this case; Flash only support ALSA output by default.
