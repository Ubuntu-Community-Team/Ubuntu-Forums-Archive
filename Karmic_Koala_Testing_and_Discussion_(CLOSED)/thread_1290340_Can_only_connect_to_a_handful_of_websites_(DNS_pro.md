### Post by manuelb on 2009-10-13
Since i think you are using the "Network Manager" applet, you'll probably will end up with it overwriting your /etc/resolv.conf file everytime the NM service get started/restarted.
I would try right-click NM, "Edit connections", choose your interface, "Edit", "IPv4 settings", choose "Automatic DHCP addresses only" and set your DNS servers as:

```

208.67.220.220, 208.67.222.222

```

This way you are configuring OpenDNS to resolve domain names.
If you have other machines on your local LAN i would recommend you to set "local" as the "Search domain". "Apply" the changes.
Now NM should restart itself and should write your changes in "/etc/resolv.conf", cat it until you see the changes happens.
If nothing happens in say 1 or 2 minutes, right-click the NM icon, disable it and then re-enable it (the first checkbox that says "Enable networking").
Now you should have your resolv.conf updated.
Also, please tell the output of this:

```

cat /etc/nsswitch.conf

```
