### Post by garvinrick4 on 2009-08-28
WUBi dual boot with Vista- 9.10 Karmic Alpha 4 64 bit. Printer not recognized at all in USB port. Have deleted rebooted and started new process, no recognition at all of printer.
 Vista side printer is working fine.  Anyone can help will appreciate.   
       Found Fix below    Sorry for so many posts. Will not happen aqain. 

Desire: To make any added component see USB connection.   

terminal Type: lsusb   (will see dev #'s and bus #'s for computer)

sudo rmmod usblp    
sudo aa-complain cupsd
sudo chgrp lp/dev/bus/usb/3 digit bus #/3 digit device #
sudo chmod 664 /dev/bus/usb/3digit bus#/3 digit device #

Go to graphic interface to add printer manually   System>Administration>Printing> Click New
                                                                                                                or CTRL +N
System will then find USB device and driver if necassary.


Linux instructions  Ubuntu 9.10 Karmic Koala Alpha 4   (intent to make CUPS see USB Printer)
                                                                                        and driver

### Post by drs305 on 2009-08-28
It will help if you identify your printer.

I am using a normal Karmic Alpha 4, not Wubi, but since others may see this post:

My LaserJet 1000 printer was not recognized either. Ubuntu saw it with the command "lsusb". 

UPDATE: 1 Sep 09  The following no longer works after yesterday's updates. System > Admin > Printing is not available. Adding the printer via hp-setup also does not work.  

To get it working, I did the following:
Uninstall all CUPS and HP referenced packages in Synaptic.
Install HPLIP (which also installed many CUPS dependencies).
System, Printers still would not find it.
Opened a browser with an addres of [http://localhost:631/](http://localhost:631/) which opened CUPS.
Selected Administration tab, Add Printer.
My HPLJ was identified and recommended "HP Laserjet 1000 hpcups 3.9.8".

Installed and printed normally again. I may have had to recycle the printer power.
