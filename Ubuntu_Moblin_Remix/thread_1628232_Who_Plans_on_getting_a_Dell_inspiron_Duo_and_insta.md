Rotate left:
```
#!/bin/sh

xrandr -o left
```

Rotate right:
```
#!/bin/sh

xrandr -o right
```

Invert:
```
#!/bin/sh

xrandr -o inverted
```

Return to normal: 
```
#!/bin/sh

xrandr -o normal
```

### Post by saverio.miroddi on 2010-12-24
Got the laptop as well.

Overall, it's a mixed experience.

* COMPATIBILITY/DRIVERS *

At the date of today, 2010/Dec/24, the Dell support site doesn't have any drivers.
I entirely wiped the hd and reinstalled windows, and had to do an  annoying research to find compatible drivers. I couldn't make the  auto-rotation work.

The basic hardware works out of the box in Ubuntu NE 10.10, including the touch.
 
I had two occasions where Ubuntu exhibited a bug where it didn't accept  any click (touch/mouse) anymore less than a minute before starting using  the touch screen.
This doesn't happen in windows, so I exclude hardware faults. I also used the installation just a few times.

* SCREEN *

Disappointing. Although it's "usable" (keeping in mind that as long as one sees something on a screen, it is "usable" by definition), the vertical range is very limited.
This prevents the screen to be used in portrait mode, because the right part is darker than the left one, unless you keep the left farther than the right when you read.
The 10' is also quite narrow, although this is a subjective opinion.

As a matter of fact, when I use it to read on the bed, I use it in landscape mode.

* CPU *

I don't find the N550 system sluggish, actually even on Windows 7 I felt it as responsive and didn't experience lag.
Ubuntu NE 10.10 [Desktop interface] is responsive as well; I also tried some 480p videos in youtube and they were smooth. As reference, the same Ubuntu version was a *complete* disaster, essentially unusable, with an Atom 280 (Windows instead wasn't) system.

Keep in mind that most of the reviews on the internet refer to the  pre-installed Windows 7, which I believe is full of junk and, certainly, has a  terribly programmed custom interface.

* CONSTRUCTION *

The construction is cheap.
The fan is noisy.
There is too much space around the screen itself.

The laptop is a bit heavy for "demanding tablet usages" (ie. bed reading), although I'm not informed on the weigths of comparable machines.

I feel that the lid however is robust enough not to break with the usage.
The keyboard is pleasant.

* OVERALL *

The laptop makes sense for what is being sold, that is, a cheap tablet with the premium price of an innovation.
I can guess that it would be an amazing machine if it would be produced as a Lenovo high end or Apple model - but in both cases it would surely cost twice as much.

I specifically purchased it because I needed a "powerful" tablet without spending too much. Wrapping up, I got what I wanted - it may satisfy your use cases as well.

Saverio

Looks like some other people have the same issue with the eGalax Touch Screen and ubuntu.   There is a document on how to get it work here.. I am still playing with it

Got it to work!   

Login and open termina
Type sudo gedit /etc/default/grub
find the line that says 
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
Replace with: GRUB_CMDLINE_LINUX_DEFAULT="quiet splash i8042.noloop=1 usbhid.quirks=0x0eef:0x725e:0x40"

Save and close geditor. 
Run sudo update-grub
Reboot..  Everything is working :D
