### Post by fct on 2011-01-26
> **Praveen30 said:**
> ok!!! i am getting something but can you describe this thing more deeply???plz....

Not sure about the specific problem of floats in Turbo C, but basically:

a char (8 bit) goes from 0 to 255 when unsigned. -127 to 128 signed.

16 bit goes from 0 to 65535 unsigned, -32567 to 32568 signed.

It's basically 2^bits. For 32 it's much much bigger.

So that seems to be the problem with the Turbo C compiler and floats. Try using doubles.
