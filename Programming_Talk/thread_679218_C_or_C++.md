### Post by LaRoza on 2008-01-26
A flame war! We haven't had one of those in a while.

C and C++ are two very different languages, despite a common syntax (for the most part) and C++ being inspired by C.

They require different ways of programming, and are as different as Java and JavaScript. (Not in the same way as them though)

If you want to learn C (which I recommend), C++ won't help.

If you want to learn C++, C won't help.

Opinions are like sphincteral orfices, everyone has one, and opinions are the only thing you are going to get :) I gave you mine.

As a note, both languages have a large following and are standardized with a sizable standard library, and with good GUI and other libraries available. See my wiki.

Rarely do you see a "vetern C/C++" programmer, as I think one would naturally chose one over the other for most programming tasks.

A quick overview, so you can make up your own mind:

**C**
* Originally designed for programming operating systems in the 70's. It is used on every platform, and the building of a C compiler is essential for a new platform. Most apps you use are written in C or use C in some way. Python can use functions written in C, which gives it the ability to use the speed and low level features of C, and still have the speed and flexibility it is known for.

* It is procedurally designed. You can write OO code in C, but it doesn't have classes or the ability to have functions in abstract data types.

* It is low level, and if you ever learn assembly (which I recommend, at least to some degree), the language will make a lot of sense.

* C is a language for programmers. It gives you power...and I am not going to quote *Spideman* again this week.

**C++**
* Designed as an increment of C, tries to keep it backwards compatible, which leads to weird C++ rules sometimes. 

* It is OO designed, and is class based.

* It is sort of low level, and can use all of the C standard functions, but if you are going to use it this way, learn C.

* It is developed and has a good standard library

### Post by Jessehk on 2008-01-27
They are both different languages designed for different purposes.

Here's almost the same code in both languages. I cheated a little here because the <list> header is actually a doubly-linked list and I implemented a singly-linked list.
Also, C **does** have some data-structure support in libraries like glib, but since they aren't "official", I didn't consider them. Plus, I used Boost.Foreach in the C++ version, simply because I can. ;)

**C Version**
```

#include <stdlib.h>
#include <stdio.h>

void *xmalloc( size_t bytes ) {
    void *mem = malloc( bytes );
    
    if ( mem == NULL ) {
        fprintf( stderr, "Unable to allocate memory. Quitting" );
        exit( EXIT_FAILURE );
    }
    
    return mem;
}

typedef struct slnode {
    void *data;
    struct slnode *next;
} slnode;

typedef struct {
    slnode *head;
    void (*del)( void * );
} slist;

slnode *slnext( slnode *node ) {
    return node->next;
}

void *sldata( slnode *node ) {
    return node->data;
}

slist *slist_new( void (*del)( void *) ) {
    slist *n = xmalloc( sizeof *n );
    
    n->head = NULL;
    n->del = del;
    
    return n;
}

slnode *slbegin( slist *list ) {
    return list->head;
}

slnode *slend( slist *list ) {
    return NULL;
}

void slist_print( slist *l, void (*print_data)( void * ) ) {
    slnode *iter = slbegin( l );
    for (; iter != slend( l ); iter = slnext( iter ) )
        print_data( sldata( iter ) );
}

static slnode *make_node( void *data, slnode *next ) {
    slnode *n = xmalloc( sizeof *n );
    
    n->data = data;
    n->next = next;
    
    return n;
}

static slnode *destroy_node( slnode *node, void (*del)( void *) ) {
    slnode *next = slnext( node );
    
    if ( del != NULL )
        del( sldata( node ) );
    
    free( node );
    
    return next;
}

void slist_append( slist *list, void *data ) {
    slnode *iter = NULL;
    
    if ( slbegin( list ) == slend( list ) )
        list->head = make_node( data, NULL );
    else {
        iter = slbegin( list );
        
        while ( slnext( iter ) != slend( list ) )
            iter = slnext( iter );
        
        iter->next = make_node( data, NULL );
   }
}

void slist_delete( void *l ) {
    slist *list = l;
    slnode *iter = slbegin( list );
    
    while ( iter != slend( list ) ) {
        iter = destroy_node( iter, list->del );
    }
    
    free( list );
}

int *make_data( int x ) {
    int *result = xmalloc( sizeof *result );
    
    *result = x;
    return result;
}

void print_data( void *d ) {
    int *data = d;
    
    printf( "%d\n", *data );
}

int main( void ) {
    slist *list = slist_new( free );
    int i;
    
    for ( i = 0; i != 10; ++i )
        slist_append( list, make_data( i ) );
        
    slist_print( list, print_data );
    
    slist_delete( list );
    return 0;
}

```
