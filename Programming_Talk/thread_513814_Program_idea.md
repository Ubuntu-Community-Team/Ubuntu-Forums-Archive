### Post by Modred on 2007-07-31
> **Quikee said:**
> Create the program, patent the idea, give open source community the promise not to sue. ;)

Or take a cue from Microsoft and promise a fragment of the open source community not to sue.

There are two basic criteria in a license that you will want to consider: freedom and copyleft.

Freedom should be rather obvious -- the user is entitled to the same rights to view, use, modify, distribute, &tc the code.  Note that "open source" isn't always libre (free as in speech), although it may be gratis (free as in beer).  An example would be a license that allows others to view the code and submit patches back to the main developer, but restricts the user from redistributing their modified version.  Such a license would not be free, but it would be open-source.

[Copyleft](http://www.gnu.org/copyleft/copyleft.html) is a concept of preserving the rights of downstream users by requiring subsequent modifications made by third parties to use the same license as the parent distribution.  While it would be possible to have a non-free copyleft, I've never heard of such, although many free licenses are not copyleft.

Here's a breakdown of a few popular licenses:

MIT -- free & not copyleft
BSD -- free & not copyleft
GPL -- free & copyleft
LGPL -- free & not copyleft

If you don't want others to have the possibility of making money, you cannot make your program free software, although it could still be open source, as I described above.  But your best bet for avoiding *commercial* use of your code, which I'm guessing is what you meant all along, would be to use GPL.  Of course, under the GPL, I am legally entitled to take your code and sell it for whatever price I can get, but as has already been said, if you are giving the code away for free then I shouldn't have much of a market.
