### Post by cguy on 2009-02-08
Yes, Project Euler is awesome for learning Python!

Read the tutorial on python.org, search on google for "learn python in 10 minutes" if you have previous programming experience (follow the first link you get) or search any other tutorial that you can understand.

After getting a grasp of the basic concepts, go to Project Euler and try to solve the easy problems. You're going to make use of everything you've read.

Use google a lot for handling the error messages.

Develop your programs in GEdit where you can use a bottom pane terminal & python console. Use the python console intensively to see how different operations affect some variables.

In a few days you'll master python. (almost :D )

### Post by CptPicard on 2009-02-09
Euler is interesting in its own right but the problems are language-independent puzzles that do not really teach you much about the language once you figure out what you need to do to solve the actual problem, as the code for the correct solution is often very short.

You'd be better off just taking up some hobby project of your own... maybe a little text-based game or whatever. That way you are actively programming all the time, which is the only way to get a handle on the language.

### Post by Greyed on 2009-02-10
My suggestion is a tad different.  I'm presuming you'll get books that work for you and want to use the language.  Try finding simple things which bug you and try writing a Python script to handle it.  Nothing major, just small automation here and there.  If you're anything like me writing code to solve a problem that has no bearing to me means code that will never be written.  As examples of code I have written in the past to learn Turbo Pascal, Perl and Python...


[LIST]
[*]A random number generator that takes RPG dice notation (xdy+/-z).
[*]A random password generator that takes a pattern and spits out passwords in that patter..  EG bbaaBB11 might result in fjieLX85 or pqoeMW73.
[*]Determining if I'm booting on the metal or in a VM and copying the requisite configuration files into place for each.
[*]My current project, the second time around, a script to help me feed spam/ham into Spamassassin's Bayesian classifier.
[/LIST]

Each of those were small projects, hours to days at most, to code they were easy for me to tackle while learning.  Since each project was something that I used at the time, and still do in most cases, I had incentive to finish them.  Learning to code, like learning to write, is easy compared to learning the discipline to finish the project; be it a program or a book.
