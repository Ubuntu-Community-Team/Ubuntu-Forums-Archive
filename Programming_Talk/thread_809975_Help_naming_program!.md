### Post by EXCiD3 on 2008-05-27
So ive started a project similar to APTonCD that you are able to download packages from the repositories with and save them onto a USB drive so that you can update Ubuntu computers at home that have dialup or no internet connection. This will also let you install new applications and make sure that ALL dependencies are downloaded also. Currently I have been using the name APTonUSB but I would like to get a new more meaningful name especially since it is confusing be so close to APTonCD and really has nothing to do with the project in any way or function. 

I was thinking of a name that has to do with either "gathering" or "scavenging" because the application is going to gather all the dependencies for the applications selected and updates available. I also dont want to have it related to Debian, Ubuntu, or aptitude because I have plans to branch to other distros via plugins that would allow the same functionality on other package managers of Fedora, Suse, Arch, etc.

So far the best I have come up with is Jackal, as they are scavengers. Let me know if you have any ideas!! :D

Oh and a link to the sourceforge project page is in my signature if you would like to take a look at the project so far (its just getting started).

**UPDATE: **So we have come down to thinking about using the name "keryx" for the application. Keryx is a messenger between states in ancient Greece. Thought this was fitting because the application is going to be working between "states" or rather operating systems. Lengthier description here if you want to take a look: [http://www.britannica.com/EBchecked/topic/315587/keryx](http://www.britannica.com/EBchecked/topic/315587/keryx)

### Post by mssever on 2008-05-30
In my opinion, Keryx is simply way too obscure. How many potential users have ever heard of a keryx before--or even know much about ancient Greek history? Granted, names that are meaningless to all but the author and those who happen to read the right bit of documentation are in vogue, but I don't think they're helpful.

Take, for example, baobab. Now, if you've been to Africa and seen a baobab tree, you can understand where the name came from--and it's even quite descriptive. But how many Linux users have seen a baobab tree? So when baobab shows up on your menu on Ubuntu systems, it has the name "Disk Usage Analyzer," which actually exlpains what it does. Of course, Disk Usage Analyzer might be a bit too boring, but at least it isn't obscure.
