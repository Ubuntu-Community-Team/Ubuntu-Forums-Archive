### Post by kaens on 2008-04-22
My advice would be to find an open source project that you use a lot, or that you really like, and start reading the code for it. Figure out how it's organized, and follow the development mailing list for it if one exists.

It's even better if it's a project that you have an idea for adding some functionality to - you'll have some incentive to finding out where the places you need to make changes in are, and how to make those changes.

Also, you'll probably either want to find a project written in a language that you're familiar with, or set aside time to familiarize yourself with the language that your project of choice is written in.

Really, developing the skill to read and understand other peoples code is the important thing here, and there's really no good way to do that other than just jumping in and figuring out a codebase.
