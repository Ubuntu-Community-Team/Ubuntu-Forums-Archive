### Post by totalbeginner on 2010-03-10
I started learning JNI recently by writing a HelloWorld toy example. I followed the instructions to create the java file defining the native method and the header file generated by javah. However, when it comes to compile my c++ file, which is listed as follows:

```

#include "Hello.h"
#include <iostream>

using namespace std;

JNIEXPORT void JNICALL Java_Hello_DispHello (JNIEnv *env, jobject obj) ()
{
    cout<<"Hello"<<endl;
    return;
}

```using the following command:
```

gcc --share -o libhello.so -I /usr/lib/jvm/java-6-sun-1.6.0.17/include -I /usr/lib/jvm/java-6-sun-1.6.0.17/include/linux Helloimpl.cpp

```or
```

g++ -o libhello.so -shared -I /usr/lib/jvm/java-6-sun-1.6.0.17/include -I /usr/lib/jvm/java-6-sun-1.6.0.17/include/linux Helloimpl.cpp

```They both give me the following error messages:
```

Helloimpl.cpp:6: error: ‘Java_Hello_DispHello’ declared as function returning a function
Helloimpl.cpp: In function ‘int Java_Hello_DispHello(JNIEnv*, _jobject*)’:
Helloimpl.cpp:6: error: new declaration ‘int Java_Hello_DispHello(JNIEnv*, _jobject*)’
Hello.h:16: error: ambiguates old declaration ‘void Java_Hello_DispHello(JNIEnv*, _jobject*)’
Helloimpl.cpp:9: error: return-statement with no value, in function returning ‘int’

```If I wrote the same thing using the c instead of c++. It would work perfectly. I don't really have any clue where this went wrong.

Thanks very much,

### Post by azagaros on 2010-03-12
My thought what happened is with scope.

The "using namespace std;" might have screwed up because of the export call.

The Using Namespace keyword is never recommended for use.

Make your code look like:
[PHP]
#include "Hello.h"
#include <iostream>

JNIEXPORT void JNICALL Java_Hello_DispHello (JNIEnv *env, jobject obj) ()
{
    std::cout << "Hello" << std::endl;
//    return;  // the function is defined as void, not really needed
}
[/PHP]

It's basically the same code but the scope issue isn't there, which is the source of most undefined type errors.
