### Post by lloyd mcclendon on 2007-01-31
the language is easy, the standard api is pretty nice (not as nice as STL but still pretty cool).  but i'd imagine that most of those jobs would involve working with JEE or several open source frameworks - both of which have a considerable learning curve and are best left alone until you are very comfortable with the concepts of the language.  

you might want to google for java web development and see where it takes you.  until you get proficient with it, it can be very painstaking and frustrating.  eventually you'll get to a point where it will start to make a lot of sense and you'll see how flexible your code can really be.  it's pretty fun too - you'll often find yourself with many good options on how to solve a problem or improve something.

someone once told me that software development is all about decision making, and that the end quality of your software is a direct reflection into the quality of the decisions that you made.  he went on to say that this is exactly why PHP sucks and java is good - you have so many choices over here, bad programmers don't want choices because they can't handle them and see them as a distraction.
