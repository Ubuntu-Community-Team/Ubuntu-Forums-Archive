### Post by hod139 on 2006-03-16
A primitive triple is different from an integer triple. A primitive triple is one such that a,b,c are all coprime, if a^2 + b^2 = c^2.

Without going into the math, all primitive triples can be generated by the following parameterization:
a = m^2 - n^2
b = 2*m*n
c = m^2 + n^2
where m > n and GCD(m,n) = 1

If the GCD(m,n) > 1, then the generated triple is not primitive, it is a multiple of some other primitive triple. For 2 good pages on pythagorean triples, see 
[http://www.math.clemson.edu/~rsimms/neat/math/pyth/]("http://www.math.clemson.edu/%7Ersimms/neat/math/pyth/")
[http://www.mcs.surrey.ac.uk/Personal/R.Knott/Pythag/pythag.html]("http://www.mcs.surrey.ac.uk/Personal/R.Knott/Pythag/pythag.html")

Here is some code I whipped up real quick.  Since you wrote in C, I wrote this in C as well. 
```

#include <stdio.h>
#include <stdlib.h>
#include <math.h>

/* Taken from http://paul.rutgers.edu/~rhoads/Code/int_gcd.c
   binary GCD alg. Based on following facts.
   If N and M are even, then gcd(N,M) = 2 * gcd (N/2, M/2)
   If N even and M odd, then gcd(N,M) = gcd(N/2, M)
   If N,M odd, then |N-M|<max(N,M) thus guaranteeing that
             we can use gcd(N,M) = gcd(N-M,M) to reduce.
   fact: a&1 is 1 iff a is odd. Mult. and Div. by 2 more efficiently
   implemented as shift-left 1 and shift-right 1 resp.
*/   
long gcd( long a, long b)
{
  int t;

  if (a < 0) a = -a;
  if (!b) return a;
  if (b < 0) b = -b;
  if (!a) return b;
  t = 0;
  while (! ((a|b) & 1)) {a >>= 1; b >>= 1; ++t;}
  while (! (a&1)) a >>= 1;                     
  while (! (b&1)) b >>= 1;
  while (a != b)
     {
     if (a > b)
         {
         a -= b; 
         do {a >>= 1;} while (! (a&1));
         }
     else {
         b -= a;
         do {b >>= 1;} while (! (b&1));
         }
     }
  return(a<<t);
}

/**
 * Generates a pythagorean triple for the input m and n, where m > n
 * Returns garbage if m <= n or if the GCD(m, n) !=1
 */
void generatePrimitiveTriple(int m, int n, long *outA, long *outB, long *outC){
    if(m <= n)
        return;
    if(gcd(m, n) != 1)
        return;
    
    // Save computation by computing m^2 and n^2 once
    int m2 = m*m;
    int n2 = n*n;

    *outA = m2 - n2;
    //*outB = 2*m*n;
    *outB = (m*n) << [COLOR=#0000ff]1[/COLOR]; [COLOR=#808080]*// left shift by 1 is fast way to compute 2*m*n*[/COLOR]
    *outC = m2 + n2;
}

/**
 * Main entry point.
 */
int main(int argc, char**argv)
{
    long a=0, b=0, c=0;
    int m = 2;
    int n = 1;

    if(argc != 2){
       printf("Usage: pythag n \n");
       printf("Where n is the number to search for \n");
       exit(1);
    }
    long numberToFind = atol(argv[1]);

    // Search for a primitive triple with value = numberToFind
    for(m = 2; m < 100; ++m){
       for(n = 1; n < 100; ++n){
          generatePrimitiveTriple(m, n, &a, &b, &c);
          if(a == numberToFind || b == numberToFind || c == numberToFind){
             printf("Triple = [%d, %d, %d] generated by m=%d n=%d\n", a, b, c, m, n);
          }
          a=b=c=0; // Reset the values to find
       }
    }
    return 0;
}

``` 
To compile, ```
gcc -o pythag -O3 pythag.c
``` and to run
```

./pythag 14
Triple = [48, 14, 50]

``` 
**EDIT: **I should add that I only have m and n go to 100 in the main loop. I figured this was high enough for 14. If you want to search for larger numbers, you can increase the end conditions in the nested for loop.

### Post by hod139 on 2006-03-20
Just be careful that you don't exceed the max integer and cause an overflow. C will let you do this, and you will get unexpected behaviors. This is why I used unsigned longs and set a maxValue. Using an unsigned long datatype, the max value can be computed as:
```
unsigned long maxSearch = (unsigned long) floor(sqrt(ULONG_MAX / [COLOR=#800080]2.0[/COLOR])));
```
This will ensure that when you calculate m^2 + n^2, you don't get an overflow.  If you want to use ints, the same technique can be found to ensure you don't get an overflow:
 ```
int maxSearch = (int) floor(sqrt(INT_MAX / [COLOR=#800080]2.0[/COLOR])));
```
