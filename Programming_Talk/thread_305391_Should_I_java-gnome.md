### Post by RAOF on 2006-11-23
Mono is a part of the default Gnome desktop; Java isn't.  There's nothing wrong with mono, C#, or any of the other CIL languages.

However, I'd suggest Python.  Python is fast enough, and you'll (probably) be much more productive in it than Java.  You don't realise how useful a dynamically-typed language is until you use one.  Also, it has a solid unit-testing framework.  Test-driven-development is cool :).

Incidentally, I think the mono (or, more properly, the CIL) is the way things will be going.  You can use whatever language you want (C, C#, Python, VB.Net, Java, whatever) and have access to the same libraries.  Write a library in Python, subclass it in a C# library, and use it in a Java app.  **This**, not C#, is what I think the coolest feature of the CIL is.  Never again will you have to wait for bindings to be written for your language of choice.

Incidentally, Java-on-mono: [http://www.mono-project.com/Java](http://www.mono-project.com/Java)
