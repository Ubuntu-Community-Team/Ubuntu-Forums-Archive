### Post by themusicwave on 2008-03-11
I run into this writer's block sometimes too.

You want to code, but you have no ideas at the moment.  Having coded all day for work or school usually makes the problem worse.

Usually I find inspiration through necessity.  I'll need to do something so I'll write a program to solve it.  Often this is a task I could do manually, but write a script for instead(because I hate tedious manual tasks and coding is fun)  like renaming all my music files or something.

Other times I just get hit with a "Wouldn't it be cool to..." idea.

Sometimes the idea just has to come to you, you see a short coming in an existing program and can fix it or make a better one.  You need to perform some task that could be done programatically.

So just relax and the idea will come.  As others said till then there are always open source projects.

I started by noticing that I performed a task frequently, and writing a bash script to automate it. For example, after doing a [FONT=Courier New]cd[/FONT], I usually do an [FONT=Courier New]ls[/FONT]. So, I wrote a little bash function that automatically does an [FONT=Courier New]ls[/FONT] after a [FONT=Courier New]cd[/FONT]. (There's an interesting little gotcha there to explore.) Or, for different ways of listing files, I have an alias called [FONT=Courier New]ll[/FONT] that runs [FONT=Courier New]ls -l[/FONT]. Then, inspired by RedHat, I wanted a shortcut called [FONT=Courier New]l.[/FONT] that would only list the dotfiles. Turns out that a simple alias doesn't quite cut it; a function does nicely. If you don't already use the command line extensively, start doing so, and you'll quickly discover things you can do. Nowadays, if I sit down at another system, I try to bring my aliases and functions along, because they're so handy.
