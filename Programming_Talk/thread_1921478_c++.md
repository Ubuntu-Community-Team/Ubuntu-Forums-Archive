### Post by trent.josephsen on 2012-02-07
I didn't get Stroustrup until I was already well versed in software engineering, so I can't speak to how good it is for beginners, but I'll counter with this:  sure, maybe it's not an *ideal* book for learning C++, but *it is still a quality resource* -- and to be honest, there really aren't many of those, or at least they get drowned out by the thousands of bad ones.  For every *C++ Programming Language* there's at least a dozen *Jamsa's C/C++ Programmer's Bible*s (worst language-learning experience I ever had, do not want).  As a beginner, it's hard to know what books are worthwhile, and Stroustrup is the only one I can feel confident recommending because I know it teaches real, useful, portable C++.

Sure, it might be a bit overwhelming at first, but hey, if you start with C++ as a first language, you should probably know what you're getting yourself into.  Maybe the experience will convince you to pick a more manageable language like Python instead.  After all, that's roughly what happened to me with C.

### Post by xytron on 2012-02-07
I learned much of C++ from Stephen Prata's book _C++ Primer Plus_, there's now a new 6th edition that just came out and it covers the new C++11 standard.  The book has a friendly style and a good order to how different language topics are presented.  He keeps the program examples short, which I think is best for a beginner.  

I also read the Deitel's book _C++ How to Program_ and found their writing style too formal and some of the program examples too long.

Bjarne Stroustrup's _Programming: Principles and Practice Using C++ _looks like a good book, it's a very indepth book about learning to program that happens to use C++.


So my recommendation for starting out is Prata's book.
