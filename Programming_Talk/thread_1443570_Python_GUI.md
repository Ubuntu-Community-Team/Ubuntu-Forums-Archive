### Post by Namibnat on 2010-04-20
Python is so much easier to work with than C++, especially if you are just starting out.  Because you don't need to compile python, by far the best way to start is to simply use any simple text editor...Gedit or Nano both work great.  Then you can simply use the terminal to run your programs.  When I am playing around with scripts or learning something new, I usually have one bash terminal open on the side and use Emacs separately.  Then each time you save you can just hit the up key/enter in the terminal.

In that way you can do everything without touching the mouse...and without initially knowing a bunch of shortcut keys.  Fast, easy and all the tools that you need are already there.

I do like to try out different things, and I one Python IDE that I did like was [SPE]("http://pythonide.blogspot.com/")

### Post by MCVenom on 2010-04-20
Py3k guy here, so the IDEs I've used on Linux are all Py3k (and Py2 series) compatible. (Btw, learn Python 3, it's not yet in wide use yet, but I don't feel like it would be right learning Py2 now -- Py2 is going out of 'active development' with this year's 2.7 release, Py2 series is incompatible with Py3 series, basically this; things are gonna start moving to Py3, and you don't wanna be left behind and have to 'relearn' Python ;))

-Geany is an ok IDE for Python (you'll have to tweak one of it's config files to get it to execute with Python 3, but it's quite simple and I can post instructions if you need them)

-I tend to keep going back to regular old IDLE though, I haven't found that I really need anything more than IDLE as it is now. It's minimalist compared to traditional IDEs, but it's really very useful. Personally I'd use it over just coding in an editor like gedit or nano (don't know about Vim or Emacs, I'm a Vim guy myself but I haven't looked for any Python plugins for it).

And another tip; if you're wondering where to go after Python, I'd recommend trying just regular C (It's what Python is coded in), it's simpler than C++ from what I've heard and it can come in handy in many places. Good luck!

P.S.: If you're looking to share your Python programs with Windows users, cx_freeze is great for making Windows *.exe files :p
