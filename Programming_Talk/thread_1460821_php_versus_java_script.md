### Post by Reiger on 2010-04-23
I say, JavaScript/ECMAScript is the more interesting language of the two. Prototypes are a pretty cool feature. Functions as first class citizens.

PHP is, despite all efforts to the contrary, first and foremost a server-side language. You can build systems tools with it; you can build websites with it; but what you can't really do well with it is building desktop applications or similar. You would *have* to write your own bindings first.

By contrast JavaScript has actually successfully made the jump from `websites' to `general purpose' language.
