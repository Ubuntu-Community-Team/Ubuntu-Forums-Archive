### Post by andrewc6l on 2009-11-18
Usually, deciding whether to use an IDE or a text editor depends on the size of the project. If you think you'll have lots of files, an IDE makes sense. If you just have one or two, a text editor is fine.

If you want to use an IDE, there's almost always an Eclipse plugin for the language you want. 

If you want to use a text editor, here are a couple of things to think of:
 - vi is almost everywhere, but it's a pain to use
 - almost everything has Emacs key bindings, so if you know those you can switch among a number of editors (including Eclipse :-) )

When I want a text editor, I now use mg by preference, which is a micro-Emacs clone. If you want full syntax highlighting, you're more likely to get it on full Emacs. But every now and then you run into a system that doesn't have an Emacs editor installed, so it's also useful to pick up vi - at least enough to move up/down/left/right, delete a line, copy a line, and save.
