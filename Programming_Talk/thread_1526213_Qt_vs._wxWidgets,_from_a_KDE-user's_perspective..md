I also noticed something about the qtcurve style used to provide consistent theming for GTK applications in KDE. All the menus have flat edges.

So I have a few simple questions. Is there any way wxWidgets could create native Qt widgets rather than GTK? If not, what are some free developer tools to build Qt applications with Python bindings?

I may have to simply program two applications side by side, just for the sake of consistent UI. For me, design is paramount, and inconsistent menus really throw a knife in the spokes.

How do you guys get around it? How many of you Gnome users would be just fine running more qt applications in Gnome, like KDenLive, miniTube, Krita, Amarok, etc. for example? Do the round menus set you off?

Is there a possibility GTK 3 will have rounded menus, which could partially solve this issue? I just feel like I'd be missing out on a lot of modern capabilities if I stick to wxWidgets in the long run.
