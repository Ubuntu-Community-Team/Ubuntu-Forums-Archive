### Post by Reiger on 2008-09-25
> **kavon89 said:**

Not really. What you get is a system independent (well, so long the JVM is installed, that is) Python implementation + Python bindings with a Java app both ways.

Presumably the amount of people that have some JVM installed still exceeds (and presumably that will remain so in the forseeable future) the amount of people with (a) Python interpreter; and since many a comprehensive app will likely benefit from some kind of scripting support, and since Jython allows one to access the Java objects within Python; and since Jython allows one to acces the Python objects within Java ... how can it get any better?

Theoretically you can code everything in Python and have the output be a Java app; but that is less realistic than the converse:

Java is, at least with Java 6, certainly fast enough to cope with the workload. I don't know about Python, but I do know that Java allows you to work with pipes, on whatever the platform (given the right JVM is installed...)...

The corruption is this. In English you have the verb "to mess", so "to make a mess" is redundant in the system. Of course there are cases where you just can use them, but because you don't have any other more proper way to express yourself.

Other historical corruptions for example in Latin was the overuse (and misuse) of Accusative, which lead into the Nominative of most Romance languages...

Of course, in a highly colloquial context, you can use them... and even myself.
