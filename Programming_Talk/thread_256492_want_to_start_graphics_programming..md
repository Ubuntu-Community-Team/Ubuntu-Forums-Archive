### Post by amo-ej1 on 2006-09-13
If you want to do graphical programming you can choose to work with opengl (the best known tutorial site for opengl can be found at [http://nehe.gamedev.net](http://nehe.gamedev.net) which was originally windows/c++ only but has been ported to many different languages/platforms/api's). But OpenGL is graphics only so if you want to go behind that you should take a look at SDL which is more like DirectX, a combination of different subsystems, video, audio, input handling, ....

I don't know how much experience you've already got with programming but if you don't have any prior experience with C/C++ i'd suggest you dive into those first before diving into such api's.
