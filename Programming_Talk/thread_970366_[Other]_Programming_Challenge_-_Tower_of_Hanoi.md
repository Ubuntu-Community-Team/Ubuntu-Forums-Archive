An oldie but goldie, this challenge is actually "stolen" from a test my father took when he learning programming like um.. 15-20 years ago or something like that and I thought it could be a good challenge here. 

[SIZE="4"]**The Challenge:**[/SIZE]
**The concept**
Make a game (Tower of Hanoi) that contains of three simple sticks standing upright, on one of these sticks (the left one) are 5 blocks, starting with the biggest under and getting smaller for each one. (some of you might have seen these in puzzle collections or likewise)
The goal is to move the blocks to the right stick, but a bigger block may never go on top of a smaller block and only one disk can be moved each time and it has to be from the top of one of the three "towers".

**Language:** Any 

**What is needed:**
You need a start and a finish, the program has to recognize when the player has completed the task (Middle stick does NOT finish the game). 

You need a graphical representation be it in terminal or any other way.

A simple way of moving the blocks.

The program has to recognize an invalid move and stop it (preferably with a notification like - Move is not allowed, you can not put a bigger block on top of a smaller one/or you can only move one block at the time).


**Bonus:**

Time trial - the game measures the time it takes the player to complete.
Move counter - the game measures how many moves it takes the player.
time + moves - the combines the two above and produces a score based on it.

Highscore - Highscore list, either as long as the program is running or a saved one. 

Difficulty setting - use easy = 3 blocks, medium = 5, hard = 7. (or something like that)

Any other thing you would feel a game like this needs.
