### Post by sacherjj on 2007-10-03
10 minutes to a lifetime.

It will take about 10 minutes to understand a typical "Hello World" program in most languages.  It can take a lifetime to fully learn "programming" and even that is impossible.

To have a time, you need to have what point you are trying to reach.  Only then can an estimate be given.  When you are programming for enjoyment, you aren't really setting a time to learn something and there is always something new to learn.

### Post by pmasiar on 2007-10-03
It depends how much expert you want to become, and how much time you dedicate to that.

See wiki in my sig for training tasks and links. Start with command-line programs. PythonChallenge is fun way to learn Python, will take you couple months to solve all 33 puzzles. Then there are more libraries to learn...

Years ago, I read article that it takes about 10 years to become competent programmer. Codin big project in a team is different from coding just you alone. You need to learn also teamwork tools, like version control system, etc. After that, learning new language will take you weeks/months, and  all pain is learning new libraries and new environments. Ie debugging web-based app is rather different from debugging command-line script just parsing files, and it is different from a 3D GUI game.

But if you have dedication and drive, you will succeed.

### Post by scruff on 2007-10-03
I began teaching myself Perl around 5 years ago. I went on to Python, C#, and now C++. Getting started it feels slow, but one day with some practice you'll find that you all of a sudden 'just get it'. At that point, when things really make sense (took me almost a year I suppose) then you can move around to different languages pretty much with ease and learn at a pretty rapid rate. 

C & C++ is a fairly big jump from the interpreted languages (perl, python) but is well worth learning. If I had to start over, I think I would have done C++ first, then Python and eventually Perl. I was at Linuxworld in NYC when one of the Gentoo dev's recommended I learn Perl as my first language, so thats what I did :)

I personally wouldn't have even bothered learning C#, but the company I work for wanted a bunch of stuff written in it. It's an ok language (kinda like java), but it's still designed by Microsoft and most of the libraries are a black box. Stick with open source stuff if possible!
