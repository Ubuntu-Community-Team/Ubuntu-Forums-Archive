### Post by TheFu on 2014-10-12
I've never used any IDE in a decade of programming on Linux (longer on UNIX systems).

The normal tools are:
* Any editor - vi, vim, geany, or even emacs, if you are desperate. Syntax highlighting is handy, but I've seen every editor on every platform get confused with professional code. Must be 20 other editor options.
* Any terminal (need 2 usually)
** make (like VS-project manager to build the code in a smart way)
** gdb / ddd window (debugger)
If you set the mouse to follow focus, then a slight bump and up-arrow will run the last command (make/gdb debugger to debug the newly built program).

Of course the emacs folks will tell you to their editor is an IDE. I suppose it is better than any Java-based editor on bloat. Opinions vary.
