### Post by mssever on 2007-08-09
A word of caution: Don't get too hung up on GUIs. There's a reason that most tutorials start you off with command line programs. The most important part of your program is the logic that actually makes it work. That's where you should concentrate--especially as a new programmer. And the command line is the simplest type of input/output that you can use to test your core logic.

Once your core logic works and is well tested, you can focus on the user interface and make a GUI if you want. But don't jump the gun, or you'll likely end up with more than your fair share of bugs.

I'm going to say it depends on what your goals are. If you just want to see what programming is like, a loosely-typed or scripting language would be a good place to start. Python seems to be VERY popular, PHP is not difficult (very widely used on the web),  or maybe even JavaScript (although I have personal issues with JavaScript).

The problem is there are a million different programming languages, and they all have their uses. As mentioned earlier, C/C++ and Java are widely used at the enterprise level, but come with a more steep learning curve (C/C++ more-so then Java). I usually recommend to people that are new to programming and want to get into programming on an intimate level, that Java is a great place to start. C# is also a possibility, but it's a Microsoft technology (which has been ported to Linux via Mono), and has gained quite a bit of ground in the market of the pass few years.

While it's not advisable to do so, if you just want to jump straight in and build some GUIs for fun, NetBeans with Java is great, as is C# with SharpDevelop.

The fact of the matter is, a good programmer should be language independent; meaning that you should have a good grasp of programming theory, and whether you implement a software solution in Java or C++ or PHP doesn't matter. The syntax and libraries are different for each language, but the concepts are still pretty much the same.

Below are a few lines of code from various languages that do the exact same thing, so you could always just choose one based on how easy you find the syntax to be.

Python:
```
print "Hello World!"
```

Java:
```
System.out.println("Hello World!");
```

C++:
```
cout << "Hello World!" << endl;
```

C#:
```
System.Console.WriteLine("Hello World!");
```

PHP:
```
echo "Hello World!";
```

JavaScript:
```
Document.writeln("Hello World!");
```

Hope that helps.

IMO operator overloading is a powerful and useful tool for rapid design. I do a lot of graphical and 3d programming so I'm frequently having to deal with 2d and 3d points and vectors. In a language with operator overloading, I can produce the following code:

```

int time;
vec3 position, origin, velocity;

position = origin + (velocity * time);

```

Whereas without operator overloading I'm stuck using something like:

```

int time;
vec3 position, origin, velocity;

vec3_mov(position, velocity)
vec3_scale(position, time);
vec3_add(position, origin);

```

The second closely resembles assembly code to me. Obviously the first one is much cleaner and even a non-programmer can interpret it.

Not that operator overloads can't obfuscate code as well, but they usually have the effect of abstracting and making things feel more natural and allowing you to type much less.

Overloading in-general is a good thing IMO, not just with operators.

This statement declares a class named Foo, with a method called main(String[] args). The method is invoked when the program runs, and the arguments are the command line arguments that are passed to the program, and are stored in an array.

class Foo

    public static void main(String[] args)
    {
        for(int i=0;i<args.length;i++)
        {
            System.out.println(args[i] + "\n");
        }
    }
}

```
And you ran it with:
```

$ java Foo 1 2 3 

```
You would get the strings "1", "2", and "3" printed to the screen. 

In C:
```

int main(int args; char * argv[])
{

}

```
In C, the first argument is the number of arguments and the second is an array of strings. The first argument (argv[0]), is the program name.
