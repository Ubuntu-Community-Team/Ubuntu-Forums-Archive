### Post by splicerr on 2010-07-14
In my experience most projects are very happy to accept patches for bugs. So you could start by finding a project that you're interested in and start submitting patches to bug reports :) Eventually if they're satisfied with the quality of your code you can propose feautres that you'd like to implement yourself. If the project your interested in has an associated mailing list, just subscribe to it, maybe lurk a bit and introduce yourself.
