### Post by r-senior on 2010-12-18
Frustrating as it can be until you get your head round it, it is worth writing a few simple programs with just a text editor and the command line so that you get a fundamental understanding of classpaths, jar files and how they relate to package structure. Post help here if you get stuck.

I'd marginally recommend Netbeans after that, because it's slightly simpler to set up than Eclipse and has a simpler interface. But there's not that much in it.

The [Java Tutorials]("http://download.oracle.com/javase/tutorial/") are a good resource.

Basic console apps give you a grounding in the language but then GUI apps and web apps are really two separate branches. There isn't really a direct progression from GUI apps to web apps, so you could skip GUI if your goal is web and vice versa. Android is primarily a branch of GUI and I don't see any reason why you couldn't move directly from console apps to learning Android.

### Post by Wistful on 2010-12-18
For Java development I recommend [**[COLOR="Red"]IntelliJ IDEA[/COLOR]**]("http://www.jetbrains.com/idea/") &#8212; The Most Intelligent Java IDE, it has great [**support for Android latest version**]("http://www.jetbrains.com/idea/features/google_android.html") and various other features out of the box. If you're just getting started the IDE doesn't matter, choose one and get your hands dirty in code, right away. (I suggest to install your IDE of choice manually from the official site rather than Ubuntu repositories)

As for learning Java try reading a good book, watch YouTube tutorials, try various tutorials from the web or specialized video training from LiveLessons, VTC, and other vendors. Also see the sticky on [Java FAQ]("http://ubuntuforums.org/showthread.php?t=713622") on how to install Java on Ubuntu. (nowadays you can go to Ubuntu Software Center and on the left hand side Canonical Partners you have Java packages for the Oracle Java SDK)

If you already know another programming language it should be easy to get started.

### Post by idi0tf0wl on 2010-12-19
Learning to write applications for the command line in Java makes for a very natural transition to Android. Being able to exploit any data in Java hooks into Android capably and smoothly, because the Android sdk, minus some nifty "convenience classes," is very much a way to put Java data onto a screen.

Netbeans is my preferred Java development environment, but for Android you'll probably be best suited to go with eclipse for all the nice plugins.

Editing your code in a terminal is fun and worth learning, but graduate to an IDE as soon as you understand it - you will appreciate the time it will save you from concept to realisation. Always run it from a terminal, though.

Have fun!
