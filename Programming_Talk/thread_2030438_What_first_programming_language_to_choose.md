### Post by trent.josephsen on 2012-07-20
If you are a novice programmer, it's way too early to be thinking about contributing productively to any kind of project. It's even, IMO, too early to be asking questions about what such-and-such a language is capable of; most of the answers to those questions are not only vague but complex beyond what's relevant to you right now, as a learner.

Furthermore, what you think you want to do is likely to change as you gain experience. What's important to you as a new programmer is not learning the tools you'll use when you're a seasoned professional, just like when you learn English you don't start out by reading legal briefs. Learn to program now and get some broad experience; you can always specialize later. I used to want to write my own OS; I changed my mind pretty quick.

Which is all to say: Learn Python. :D It's not the "best" language, because there is no such thing. It's not the most widely used language either (although it is fairly popular). But Python will give you a good foundation you can use to build just about anything, and knowing one language well makes it drastically easier to learn more.
