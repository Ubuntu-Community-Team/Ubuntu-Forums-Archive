### Post by Tim Sharitt on 2009-01-04
Glade only lays out the interface to be loaded by a program using the libglade library. You will still need to write some actual code to carry out whatever you want to accomplish. There are several languages which are capable of loading the glade xml files, so you can use whatever language suits you.

### Post by ggaaron on 2009-01-04
I've tried to find some tutorials for GTK+ and Glade, but it seems that Glade is awfully undocumented... If you want GTK+ then maybe you'll find something there:
[http://www.pygtk.org/pygtk2tutorial/index.html](http://www.pygtk.org/pygtk2tutorial/index.html)
If you want Qt then this will help:
[http://www.riverbankcomputing.co.uk/static/Docs/PyQt4/pyqt4ref.html](http://www.riverbankcomputing.co.uk/static/Docs/PyQt4/pyqt4ref.html)
and Qt also has a designer, I've used it so I can tell you how to use xml files generated by it. My examples are about python, generally a scripted language will be easier than compiled language such as C++. You can also just write a script in Bash that will do the work - it will be easiest and fastest but won't have gui.
