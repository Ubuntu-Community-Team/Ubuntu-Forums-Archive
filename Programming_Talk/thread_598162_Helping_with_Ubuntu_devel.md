### Post by nhandler on 2007-11-03
I would start by getting familiar with packaging. From there, you can try and fix various bugs listed on launchpad. These things aren't always programming related, but they are an easy way to contribute. The knowledge you gain can also be applied to any program you might make. For example, you could make some app in python (and maybe add a GUI), then you can package it up. Finally, if it is found to be useful, it might make it into the repositories.
