### Post by fiddler616 on 2008-11-02
Hello,[URL="http://ubuntuforums.org/showthread.php?t=769320"]
There has been a movement[/URL] to have prefixes for Programming Talk, sort of like the [xfce], [all variants], [ubuntu] etc. ones in the main support threads.

However, these prefixes have not yet been implemented, so I would like to urge you to write them yourself. So instead of:
> cPickle stack underflow
you would put
> [Python]cPickle stack underflow
if it's not language specific, try [algorithms] or [structures].

Why?
Because I think it's in the same spirit as the want for descriptive titles. You've heard the examples--that it's a lot better to say:
> Grub Error 17 with Wubi
instead of
> Ubuntu brokennnnnnnnnnnn. HELP HELP HELP!!!

So having prefixes lets the people who *do* know the language in question jump in, and those who *don't* use their skills on another thread (which will be easy to find.)

Here's LaRoza's proposed list of prefixes to use:
> # Tools
# Languages
# Web
# Algorithms
# Structures
# Shell
# Assembly
# C
# C++
# C#
# D
# Java
# Lisp
# Perl
# PHP
# Python
# Ruby
# Tcl
# GTK+
# QT
# Tk
# wxWidgets
# Multimedia
# FAQ
# n/a

Cheers.
