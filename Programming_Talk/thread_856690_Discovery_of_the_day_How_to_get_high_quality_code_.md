### Post by Mr. Picklesworth on 2008-07-11
People often say that open source software and libraries have poor documentation; mainly stubs with function names and their arguments, cryptic sentences for what they do, and little else. I would agree that it is usually pretty limited. (Although it's also nice to all be in Devhelp, but that's beyond the point).

One thing that makes good documentation is having nice code examples in the docs. Just a while ago, I thought that was something rather lacking that could make this documentation better. I also thought it rather ironic, given that this is all open source, but there is a difference between finding a small relevant snippet regarding a function and finding the code for a project.
Then I realized that the amount of source code I have downloaded could be measured in tons! I also was reminded about that powerful search engine which has been politely plodding away in the background, waiting for my attention. (Beagle, in my case).

Hm...

So, I was wondering about g_list_next; it was confusing me by returning a GList, so I was wondering whether to create a new pointer or use the same one. Opened up Beagle, typed g_list_next and BOOM, I had 49 examples, all in C, of how best to use the command. Adding to that victory, the ones I looked at were all nice code and completely relevant (since I download code I am interested in, of course). These were all functional, deployed code, as well; very educational and a good opportunity to gradually explore the different programs that run on my desktop. In this case, I learned about the Sticky Notes applet.

Cool, eh?

The trouble with Google Code Search is that the top results are often from the libraries using the code themselves, which while cool is not always the nicest place to find out. In other cases, it's from completely different projects, often on different platforms. Further, it needs me to be connected to the Interwebs; often when I want to get some real work done, I intentionally cause my finnicky wireless driver to fail by entering and exiting sleep mode :P

Not that Google Code isn't extremely handy, of course -- especially to find funny comments, or bigger examples, or stranger things.
