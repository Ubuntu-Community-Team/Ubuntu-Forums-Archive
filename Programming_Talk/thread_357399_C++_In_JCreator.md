### Post by Naralas on 2007-02-09
JCreator is my favorite development environment, so on my C++ learning quest I have tried to enlist it as my primary soldier.

I followed a guide (which, in case anyone else wants to do this, I shall post) which was pretty simple, so simple in fact, I wanted to slap myself for not knowing what to do.

Now I have one final step, I have Dev-C++ installed, (which I hate, mostly because I am used to JCreator filling in the opposite brackets for me, but also because it cannot handle tab's well), and now I have two issues.

1) What file in Dev-C++ / bin is the actual compiler?
2) Why do my new .cpp and .c associated files not have Java's nice "ill add the other bracket for you" feature :( It was especially nice when I put {, and it put the opposing } and even jumped me down a line and indented!!

Here is my re-write, of the "good guide gone bad", for Pro 3.5
[list]
[*]Configure | Options | Documents
[*]After clicking documents click "New" in the side window that proceeds it
[*]In the following window associate .cpp and/or .c files with this new type
[*]Use the cpp.syn that Sun kindly provided
[*]That parts done, there might be other blanks but its all obvious stuff
[*]Still in options, go to Tools
[*]New | Program
[*]The new program is your C++ / C / C# compiler
[*]To compile you click Tools, then click the new tool... (CTRL + 1 also works)
[/list]

Now just to find my raw C++ compiler file and I am off to the races..
