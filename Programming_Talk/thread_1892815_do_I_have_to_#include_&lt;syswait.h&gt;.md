### Post by Bachstelze on 2011-12-08
> **Mr.Pytagoras said:**
> No I don't get warnings. 

Even with -Wall?

> So I use the wait like this:
```
if(wait(NULL) == -1)

```

I'm not passing and adress of a int that would reprezent the exit value, because I don't need it. So is this correct even without the header?

This is not correct without the header, because you're passing NULL, which may have type int or void*, when the function expects int*. You must do

```
if(wait((int*)NULL) == -1)
```

> If so how the compiler know about the wait function without the header?

The compiler does not know about the function, it will just pass whatever parameters you give. This is why you should always include the header, because it allows the compiler to check that what you are passing matches what the function expects.

*EDIT:* And in most cases convert NULL to the appropriate pointer type, though some argue that you should always explicitly cast NULL when passing it as a function argument.

Compiling with -Wall is just a good practice in general. C has a reputation for giving you "enough rope to hang yourself". A lot of that can be prevented by heeding the warnings generated by -Wall. If you were not calling a library function correctly you may not have found out until runtime, and even then, it may not have given you as nice of an error message as -Wall
