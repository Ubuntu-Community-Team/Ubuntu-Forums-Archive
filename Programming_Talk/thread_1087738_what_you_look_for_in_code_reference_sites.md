### Post by Firestom on 2009-03-05
I have been learning some languages XHTML, CSS, Javascript, Actionscript, C and C++ from a "code reference" website. This website showed how programming worked from the start and gave plenty of examples of code to make sure that you were on the right track.

This said, I don't (and I think you don't) expect newbies to be learning everything from scratch on your website. But explanations are golden in the art of programming. If you don't explain clearly a few things, let's say, for example, bitwise operators, users won't understand how these work unless you explain how numbers, bytes and bits work.

Also, some libraries specific codes would be useful (oh how I wish I could find the d* problem with my Save file function on Qt), because people usually don't stick with the standard library really long. When someone shows them to use a library, or they decide to learn one by themselves, usually they stick to it really long (it stuck with SDL a looooong time before going on with Qt).

As for the topics that will be discussed (Network, Crypting, GUIs), I don't know what visitors will want, but I would really like to have a plethora of codes and examples on I/O devices (Files, Sockets, Drives), but that's just me hey!

On thus, good luck with your website!
