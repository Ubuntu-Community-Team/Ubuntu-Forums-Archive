### Post by r-senior on 2012-05-27
Eclipse, Netbeans and Intellij IDEA are the most commonly used Java code editors. All have free as in beer versions and can be used for commercial and non-commercial projects. But they are IDEs rather than simple code editors.

What features did you want from your editor?

Code completion? Syntax highlighting? Auto-formatting? Project management? Build tool integration? Intelligent refactoring? Archive generation? Javadoc generation? Unit test support?

In my experience, even small but non-trivial Java projects soon end up with enough source files to justify one of the popular IDEs. A plain text editor and a build tool like Maven can do the job but once you start creating package structures, just having the package-tree navigation of an IDE makes life easier.
