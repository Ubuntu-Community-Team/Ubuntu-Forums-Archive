### Post by Delever on 2008-05-14
Right. My story first.

After moving to Linux, i decided to learn programming here. Specifically, C++. I had some previous experience with it, but that involved no more than two source files. However, I have some experience with other programming languages (like delphi/VB/python/php/Java/C#), so I thought, heck, that won't be too hard. Well.

The main problem was not programming language, but "correct" use of it. I saw those fancy "make" commands for other programs, so I started to learn about makefiles, how to use them in my own project. But there seemed something lacking, and I strugled to understand "correct" way of doing it. I googled alot for linux programming...

And that was main mistake. I should have googled for **GNU** programming.

That fancy name of kernel, "linux", flashing everywhere, but in development world, it means what it is: kernel. As newbie, I had no idea about it, searched in wrong places, and learned hard things (well, there will be use for them so no big deal).

This post is just request to add explanation to **sticky in this forum**, that programming for wide user base is programming with GNU, and possibly add link to relevant manual, like this one: [http://www.amath.washington.edu/~lf/tutorials/autoconf/toolsmanual_toc.html](http://www.amath.washington.edu/~lf/tutorials/autoconf/toolsmanual_toc.html).

To save capable newbies from quiting :)

Thanks :mrgreen:

That's alright, I also suggest to get familiar with makefiles to anyone, because these autotools generates them. And there is no need to use autotools when just learning programming.

However, information I was searching was related to this thing: 

*What should I do to not screw up someone else's OS instalation if I want them to install my program?*

So I found that using "Linux" keyword when searching for that is not good idea. GNU is basically what Linux distributions are based on.

And autotools is not the main thing. Such general information where programs should be installed, where header files go, what directory structure is prefered is invaluable to someone new.
