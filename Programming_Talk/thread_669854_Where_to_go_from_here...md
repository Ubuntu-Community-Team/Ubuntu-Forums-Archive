### Post by Wybiral on 2008-01-16
Learn them all. After you learn one language, learning new ones gets easier and easier. XML and UML aren't actually programming languages. XML is just markup and UML is just a way of describing object relations (more of a visual representation). You shouldn't have any problem retaining them, they don't actually DO anything.

So you're basically learning C++, Java, JavaScript, and ActionScript (and it sounds like you already know C++). Chances are, you aren't going to need to learn any of them too in-depth. Java shouldn't be too hard for you to grasp after learning C++. ActionScript and JavaScript are EXTREMELY similar languages (ActionScript basically just extends the ECMAScript standard with classes and explicit type declarations).

In fact, ActionScript is also very similar to Java in some ways. Look at these two examples: [here]("http://en.wikipedia.org/wiki/ActionScript#ActionScript_3.0") and [here]("http://en.wikipedia.org/wiki/Java_(programming_language)#Applet"). I bet after you start working with them, you'll learn what they have in common and where they differ and you shouldn't have too much of a problem.

### Post by mathisdermaler on 2008-01-16
It's my experience that usually system administrators learn a scripting language that already has a lot of momentum for the platform you use.  On windows this is batch/cmd files (yuck!) and on unix this is whatever shell you prefer.  

It's also very useful to be able to read other people's shell scripts; a sysadmin will not do much "write a new script from scratch" stuff but will do a lot of "slightly modify this 5 year old script so that it won't back up /home/arthur/really-big-file."  

If you want to get a feel for this, try reading
/etc/init.d/networking 
on your linux install.

For what it's worth: you said you did well in your C++ course.  If you *liked* it that may be a sign that you should be a programmer/developer instead.

### Post by Cappy on 2008-01-16
As a system admin you are going to want bash, perl, python, AND a bit of C++ knowledge. All are best at different kinds of tasks in linux so you will need all of them. How much you specialize in each and the order you learn them is of course up to you.

If I were going to try to get a good portfolio to become a system admin I (myself) would become active in an bug/exploit tracking, bug/exploit patching, computer defense oriented community. Also, the book "Stealing the Network: How to Own the Box" is rather insightful.

Obviously when I think of "system administrator" I think of keeping things running well and most of all secure. The best linux book I've ever read for administrators was Sybex Linux System Administration.

### Post by pmasiar on 2008-01-16
Don't be afraid to learn many different languages - it is good for you. You are not getting vocational traning as java code monkey, but understanding that even if syntax is different, language inside implement the same concepts.

In fact one of my favorites in college was "comparative study of programming languages", where we crammed basic concepts of language in a week or two, from COBOL to Lisp and ML. Just to see that the more they look different, the more they are same.
