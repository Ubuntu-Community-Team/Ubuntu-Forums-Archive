### Post by schmendrick on 2008-03-25
monodevelop seems very promising, however there is no auto complete (intellisense) feature for class instance members, which will be a thing that a VS developer will miss very much. 
i hope they include that soon allthough i am sure thats a tricky thing to do. 

when i develop C++ on ubuntu i normally use Eclipse with CDT 4.0. 
though i am not 100% satisfied it works for me. for making switching a bit more painless you can look at 

[http://www.ibm.com/developerworks/opensource/library/os-eclipse-visualstudio/](http://www.ibm.com/developerworks/opensource/library/os-eclipse-visualstudio/)

to get started with eclipse. 

When i tried out netbeans it seemed to bit more visual studio like, but syntax highlighting in is better in eclipse (and even better than when coding (unmanaged) in VS)

though on windows i still prefer VS Express, mainly because of the slicker debugger. 

hope this helps
