### Post by cmat on 2007-08-20
I started with VisualBASIC, C++, then Python. They are all quite easy but by far Python would have been the best for me to start off with. VB was just a cheap way of making GUI apps, you really lack the knowledge of non-OOP. Python gives you good knowledge of program structure which is a fantastic primer for C++. Not to mention it's simple and you don't need to declare variables. It's also my favorite programming language.

### Post by pmasiar on 2007-08-20
easiest entry for pretty decent platform games in (sadly, windows-only) Game Maker - GUI designed to create games without writing any code. You are still programming: you need to handle objects, collisions, events, timers etc. You just do it using very high level language with excellent visual representation.

But this language is not universal - is for games only. If you want to learn real mainstream usefull programming language, Python is what you need. C++ is way too complicated for what you need. C++ will generate faster code, but it will take you weeks or months more to learn it enought to write any code. Wiki in my sig  will get you started with free online books.
