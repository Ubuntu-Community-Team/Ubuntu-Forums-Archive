### Post by Tony Flury on 2009-11-20
I would strongly suggest that you get the idea first - before starting a project. 

Why not get the idea first (something you need doing) and develop an initial version, and then you can create a project around your initial version for people to extend/improve your original.

Starting a project without any code at all is unlikely to get any interest.

WHat might be better for you right now, would be to get involved in an existing project that uses the technologies that you know. Grab the code, look at the existing bug list - see if you can identify where the bug is and submit a fix.
Doing that will get you familiar with all the project type stuff - change control, bug reports, bug finding and testing etc.
