### Post by rbprogrammer on 2010-11-21
What kind of kernel development?  Linux kernel?  Or are you ambitious enough to develop your own kernel?

Regardless of what kind, my suggestion would be to get very familiar with operating system concepts.  That is, memory management algorithms, process scheduling algorithms, etc...

And in my technical experience, C will definitely help in "kernel development", however the crutch will be the OS concepts.  Knowing how to code something is important, but you first must know what to code.
