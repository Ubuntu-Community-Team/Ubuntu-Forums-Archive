### Post by LaRoza on 2007-10-01
I see that five threads with this same title came up when I typed the title in to make this thread, so it must be a popular subject.

I started a wiki for those programming in Ubuntu. It is meant to fulfill the purpose of the [http://ubuntuforums.org/showthread.php?t=554070|New to Programming in Linux? -- Read This!] sticky (formerly "Windows Programmers -- Read This!), but with more information in a better format.

I am currently working it, at the moment, so it is lacking content. I wish to have, for each languages listed:

* the steps required to get the tools for the language (usually the command to install the packages neccessary).
* A brief description of the language, i.e. what makes it different.
* A "Hello world" program, with a step-by-step guide to running it as a program.

The wiki is at [http://laroza.wikidot.com/one](http://laroza.wikidot.com/one)

Right now, I am creating the pages for several languages, and only giving a single line (the tool required), and I will fill it out later, hopefully, with help.

### Post by LaRoza on 2007-10-31
I have decided to make this wiki editable by anon users. I hope it provids a good resource to those beginning programming in Linux. 

The sticky, "New to Programming in Linux -- Read This", is not a good solution. I have seen common questions for C, including:

* stdio.h missing (the famous "sudo aptitude install build-essential")
* getch() missing (the famous "conio.h is not standard")
* what compiler (gcc)
* howto compile, link, etc
* socket programming in Linux

Of course, other languages have similar issues. The sticky can not cover this very well. I hope to have on the wiki are listed on this page: [http://laroza.wikidot.com/notes](http://laroza.wikidot.com/notes)

For the wiki, I want a good coverage of common issues with people coming over to Linux, either from Windows or a new learner.

It would be nice if everyone could contribute to it. 
[http://laroza.wikidot.com/one](http://laroza.wikidot.com/one)

I don't want coverage of the language, tutorials exist else where. If you are working on a page, and don't know of a tutorial worth listing, use my wiki if my wiki has that language on it.
