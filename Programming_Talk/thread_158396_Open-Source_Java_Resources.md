### Post by Keffin on 2006-04-12
[quote=thumper]Meaning - why without getting caught up in the Sun or Java vendor community.

What is your impression of being caught?

Why would that be a problem?  Is it a problem?[/quote] 
[http://www.gnu.org/philosophy/java-trap.html]("http://www.gnu.org/philosophy/java-trap.html")

There was lots of "heated debate" a while back because OpenOffice.org 2 was using Java for more of the components, and it looked like it might not be possible to use it with a free (as in speech) JRE. Potentially this meant that e.g. Ubuntu would not be able to distribute it along with the rest of the OS. You can look up OOo on wikipedia to read about that.

As for where to look, there's nothing wrong with Suns tutorials and documentation, after all the goal of gcj/classpath is to implement all of the java spec. You should just be aware that the free implementations have issues with the newer java 1.5 stuff, and swing. I found a comparison of jdk1.4 with classpath here [http://www.kaffe.org/~stuart/japi/htmlout/h-jdk14-classpath.html]("http://www.kaffe.org/%7Estuart/japi/htmlout/h-jdk14-classpath.html").
