### Post by Josh1 on 2007-12-29
I have seen this question alot recently, and have decided to list a few of my favourite editors for editing PHP,HTML & CSS since I do 99% of my development on my Ubuntu PC.
Some of these are WYSIWYG (they come with the option, I never do this but some of them have an option for it).
I wrote about this on my [blog]("http://joshuataylor.info/major-web-development-editors-in-ubuntu-part-1/") as well.

Here is the list, my review for it and my personal rating for the application. I have also added how to install the program as well!
If you would like to add to this list, PM me or add a reply.

Bluefish

Their description:
“Bluefish is a powerful editor targeted towards programmers and webdesigners, with many options to write websites, scripts and programming code. Bluefish supports many programming and markup languages, and it focuses on editing dynamic and interactive websites.”

Screenshots:
Bluefish has a few screenshots up at their Screenshots page. Go take a look.

My Rating:
4.5/5

My Comments:
I use Bluefish as my main editor. It has alot of inbuilt features that makes it shine, one of them being support for many different coding languages. Quite a nice, easy to use interface. Loses a point due to clutterness, but this can easily be fixed

Website:
[http://bluefish.openoffice.nl/](http://bluefish.openoffice.nl/)

Download:
sudo apt-get install bluefish
NVU

Their description:
“Finally! A complete Web Authoring System for Linux desktop users as well as Microsoft Windows and Macintosh users to rival programs like FrontPage and Dreamweaver. Nvu (which stands for “new view”) makes managing a web site a snap. Now anyone can create web pages and manage a website with no technical expertise or knowledge of HTML. ”

Screenshots:
NVU has a few screenshots up at their Screenshots page. Go take a look.

My Rating:
2.5/5

My Comments:
I really don’t like NVU. It’s great for beginners, sure, but it’s lacking alot of major features. Use this if you use Frontpage in Windows.

Website:
[http://www.nvu.com/](http://www.nvu.com/)

Download:
sudo apt-get install nvu
Quanta Plus

Their description:
“Quanta Plus is a highly stable and feature rich web development environment. The vision with Quanta has always been to start with the best architectural foundations, design for efficient and natural use and enable maximal user extensibility.”

Screenshots:

Quanta+ has a few screenshots up at their Screenshots page. Go take a look.

My Rating:
4/5

My Comments:
Quanta Plus is my favourite editor to use while using KDE. It is quite feature rich, but I sometimes feel it is just a glorified gedit as I don’t use many of it’s features.

Website:
[http://quanta.kdewebdev.org/](http://quanta.kdewebdev.org/)

Download:
sudo apt-get install quanta
Screem

Their description:
“SCREEM is a web development environment. It’s purpose is to increase productivity when constructing a site, by providing quick access to commonly used features. While it is written for use with the GNOME desktop environment in mind it does not specifically require you to be running it, just have the libraries installed.”

Screenshots:
Screem has a few screenshots up at their Screenshots page. Go take a look.

My Rating:
4/5

My Comments:

Screem was one of my first editors I used when I started to make the move to Ubuntu in January 2006. I didn’t stick around long using Screem mainly because I didn’t develop much in HTML, more in PHP and the syntax hightlighting wasn’t very exciting. It’s still one of my favourites though for doing HTML.

Website:
[http://www.screem.org/](http://www.screem.org/)

Download:
sudo apt-get install screem
gPHPEdit (Only does up to PHP 4.3.0)

Their description:
“SCREEM is a web development environment. It’s purpose is to increase productivity when constructing a site, by providing quick access to commonly used features. While it is written for use with the GNOME desktop environment in mind it does not specifically require you to be running it, just have the libraries installed.”

Screenshots:

gPHPEdit has a few screenshots up at their Screenshots page. Go take a look.

My Rating:
0/5 (NA)

My Comments:

I haven’t really used gPHPEdit, mainly because it only supports up to PHP 4.3.0 and with PHP5 as standard, and PHP6 on the horizon, itWebsite:
[http://www.screem.org/](http://www.screem.org/)

Download:
sudo apt-get install screem
