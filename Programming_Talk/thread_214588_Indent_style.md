### Post by jayemef on 2006-07-12
K&R all the way for me, which makes it easy to find support in various IDEs. My coding practices are pretty regimental. If any of you are sticklers for coding practices like I am, I would highly suggest O'Reilly's [Perl Best Practices]("http://www.amazon.com/gp/product/0596001738/ref=sr_11_1/104-6707665-7558332?ie=UTF8"). While it has perl in the name, at least 50 percent of the book applies to any language. It's a very top-notch book, and will give you good insight on how you can improve your coding style and/or reinforce the good techniques you already practice.
