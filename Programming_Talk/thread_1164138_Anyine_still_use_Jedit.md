### Post by cl333r on 2009-05-19
That's a good question I guess. I even contributed several patches to jEdit but then replaced it with NetBeans when version 6.0 came out. I dropped it because:
1) It's in Java and while NetBeans is also a Java app - NetBeans at least gives me everything I need with lots of serious plugins.
2) At least by default jEdit doesn't add automatically the closing brace.
3) It's not an IDE, rather the emacs of Java.

But in general it's a good one and if it didn't feature a slow startup (which is common to Java) I would prolly be using it more often, as a replacement for gedit, for example.

otoh I'm also considering to drop NetBeans some day because it's very heavy and slow, especially on Linux.
