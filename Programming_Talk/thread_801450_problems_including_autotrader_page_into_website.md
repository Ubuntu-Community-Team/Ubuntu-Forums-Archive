The generated page, available at [http://atsearch.autotrader.co.uk/www/dealer_services.asp??did=126582&pageid=1&originalid=&gid=nogroup&tid=2]("http://atsearch.autotrader.co.uk/www/dealer_services.asp??did=126582&pageid=1&originalid=&gid=nogroup&tid=2"), is not valid xhtml 1.0 strict, which is what I'm aiming for with the site I'm building.

### Post by WW on 2008-05-20
You could give [BeautifulSoup](http://www.crummy.com/software/BeautifulSoup/) a try (Ubuntu package: [python-beautifulsoup](http://packages.ubuntu.com/hardy/python-beautifulsoup)).

For example, I saved your link; the file name ended up as dealer_services.asp.  Here are a few lines of interactive python that muck about in the soup:
```

$ python
Python 2.5.1 (r251:54863, Mar  7 2008, 03:39:23) 
[GCC 4.1.3 20070929 (prerelease) (Ubuntu 4.1.2-16ubuntu2)] on linux2
Type "help", "copyright", "credits" or "license" for more information.
>>> from BeautifulSoup import BeautifulSoup
>>> doc = open('dealer_services.asp').read()
>>> soup = BeautifulSoup(doc)
>>> soup.contents[0].name
u'html'
>>> soup.contents[0].contents[0]
u'\n'
>>> soup.contents[0].contents[1]
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>Auto Trader Dealer Pages - Central Garage</title>
<link rel="STYLESHEET" type="text/css" href="http://dealer2.autotrader.co.uk/126582/126582_1.css" />
<script language="JavaScript" src="http://www.autotrader.co.uk/common/dhtml.js" type="text/javascript"></script>
</head>
>>> soup.contents[0].contents[3].name
u'body'
>>> soup.contents[0].contents[3].contents[0].name
u'span'
>>> soup.contents[0].contents[3].contents[0].contents[1].name
u'table'
>>> soup.contents[0].contents[3].contents[0].contents[1].contents[0]
u'\n'
>>> soup.contents[0].contents[3].contents[0].contents[1].contents[1]
<tr><td>
<div class="stockstrap"></div>
<div class="stock">
<div class="stocktitle"></div>
</div></td></tr>
>>> 

```
