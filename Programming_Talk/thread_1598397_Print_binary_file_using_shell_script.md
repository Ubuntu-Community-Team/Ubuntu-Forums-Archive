### Post by James78 on 2010-10-18
Well, this command here went reasonably well on the terminal with me. I have no idea why you'd want to print a binary file, but here ya go anyways.

I'm sure it could be done slightly better than this, but I'm sure this'll work regardless.
```
cat $f > /tmp/throwaway | lpr
(Remove file after end of loop is done)
rm /tmp/throwaway
```
If that doesn't work (like it doesn't print regular files correctly), you could try something like..
```
if [ -x $f ]; then
   cat $f > /tmp/throwaway | lpr
else
   lpr $f
fi
rm /tmp/throwaway
```
In the example above, it will only work if the binary file has the executable (-x) flag. So make sure it does if you use this.

No guarantees it'll work of course... It could be that lpr hates seeing the contents of executable files, with all the strange characters and encoding.

* If lpr hates the file being piped to it (" | lpr"), then just remove " | lpr", and on a newline, add "lpr /tmp/throwaway"

I came up with a solution myself as well just a minute a go like so:

```

#!/bin/bash
f=/path/printqueue
t=/path/tmpfile

shopt -s nullglob

tr -cd '\11\12\15\40-\176' < $f > $t

echo 'start'
lpr -r $t
rm  $f
echo 'done'

```
