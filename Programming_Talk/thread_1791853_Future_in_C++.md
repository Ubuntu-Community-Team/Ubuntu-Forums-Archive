### Post by Tony Flury on 2011-06-28
When you start programming as a hobby, then in my experience it is better to start with a project you care about, rather than something you just do for "practice".

That project maybe an application for you to make your life a bit simpler, or maybe contributing to another project that someone else runs that you want to be sucessful.

By embarking on a project which you have a personal stake in (i.e. you want to see it completed), you are more likely to work on it - solve the problems etc.

An example - my first major hobby project was to implement a PC based front end that allows me to upload images to flickr, and allows me to add Tags to them - create new sets and add images to them, change permissions etc - all without logging onto flickr. I worked hard on the project and now it does what i need when i need it.

Since completing it i have identified a number of improvments i could make - making the flickr interface a plugin (so that other plugin could be written for other functionality), making it catalogue the pictures on the HD and numerous other features. I have not started on any of them - because, right now, I don't need them, and therefore the motivation is not there for all of these improvments.
