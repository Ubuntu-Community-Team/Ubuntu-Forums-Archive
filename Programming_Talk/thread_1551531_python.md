### Post by oldfred on 2010-08-12
I went with gtk only because I liked the idea of the visual designs from glade and the separation of graphic code from working code.

### Post by HoKaze on 2010-08-12
I'd say it depends. wxpython is probably the best long-term choice in terms of what you can do with it, native looks across Windows, Linux and OS X but tkinter is simpler and included by default with a python install, making it the easiest for the end-user too as they don't have to install anything.
I personally use tkinter but I do admit that it looks awful (even with tile theming added into python 2.7 and python 3.x it's still pretty bleh) and can be a pain for larger, complex programs.

I'd probably advise against using QT or GTK and go with either wxpython or tkinter for the GUI. But hey, I'm not much more experienced than you so don't take my word for it :P

### Post by robots.jpg on 2010-08-13
If you develop on Linux, for Linux, I would recommend GTK.  For working  on multiple platforms (especially Windows), I'm definitely a fan of  wxPython.  It's not difficult to install, and it plays nicely with  py2exe if you use that utility.  I find them both pretty similar in  terms of documentation and ease of use.

Tkinter is on the bottom of my list just because it doesn't draw from  the native controls like the others, so it looks completely alien wherever  you use it.  Something about the programming interface also feels less intuitive to me, but maybe it's just because I've used wx so much more.  There's definitely an  advantage to having it built in to python, though.  You won't run into  some of those minor quirks and exceptions wx and GTK can show across  different platforms.
