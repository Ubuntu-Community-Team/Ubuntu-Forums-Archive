### Post by amo-ej1 on 2007-01-25
All,


My 2 cents on this ;-), i've always been a fan of online programming competitions. There are a lot of 'hacking' challenges, basicly they all start with a webpage with a password hidden in java script and  extend to some programming and logic puzzles. The big advantage is that they have a game like and challenge like interface which makes it great fun to do. 

Sample sites are: [http://www.bright-shadows.net/](http://www.bright-shadows.net/)  

But the idea is, there is a challenge, you submit input or a solution which gets validated and a ranking is coupled to that. I had much fun during exams (yeah i know :D) competing with a friend.

Another alternative is [http://acm.uva.es/problemset](http://acm.uva.es/problemset) it's a spanish university which offers an 'online judge' basicly they have a _huge_ problem set archive and you submit c/c++/java/pascal (eat that you python lovers :D) sourcecode which they unit test with several constraints.

Personally i'd really enjoy this last site, the only problem is the level of difficulty of these problems, many require you to use advanced algorithms to solve the problem within (CPU)-time. While this would scare beginning programmers off. 

So my point being a wiki is a nice idea, but by putting some challenge or 'raison d'etre' behind it you can make it more fun to follow up. Say you have a challenge, you solve it, fine, now what ? You learned to type in your code which you thought of yourself so probably in your own mind you're convinced you code is the best ever and run without flaws so what have you learned except typing ? Imo nothing.

Anyhow my 2 cents credit is finished, but I would be interested in any idea that pops out of this thread ;)

### Post by rjm1982 on 2007-01-29
Ok, so not to step on your toes...

This is similar to an idea I had not too long ago.  I have actually started work on it, but my time to work on it has fallen off due to a huge amount of 'real' work...

Basically, I was going to create an entire site devoted to programming challenges.  Challenge series' acutually.  My plan was/is to create timed challenges that build on one another...like this:

Challenge would be created, initially, for the basic layout of the program and the core/basic code.  Say the end product would be a syntax highlighting text editor.  The first challene would just be setting up the GUI and common code.

The entrants would be rated (by the community) and that code would become the code-base for the project from them on.  The next challenge might be (in this example) the syntax-highlighting function.  Same deal, it gets rated by the community, and the winners code is folded into the code base.  

The next challene may be file handling...so the previous code is downloaded as the base, and you build on that.  The best one wins there as well, and is added to the code base.

Eventually, you end up with a completely community-developed application with the best of the best in it.  It not only gives you an opportunity to learn and practice (really, how often is "i cant think of anything to write" your excuse for not messing with code, its mine quite often), but it teaches the importance of good, clean code and the basics of community development.  Not to mention, provides (through archiving of all entries) and huge base of examples of different solutions to the same problems.

I think this is more advanced than your idea, so its probably not your goal, but I thought that this was related enough to convey to you (and those reading but not posting) about something that may exist in the near future...
