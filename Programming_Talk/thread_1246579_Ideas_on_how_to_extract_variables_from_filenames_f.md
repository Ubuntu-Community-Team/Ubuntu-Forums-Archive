### Post by DaithiF on 2009-08-22
I'm not sure I've fully understood what you need, but it sounds like you want to extract the date and time parts of a file name like Jupiter0003 07-08-29 21-39-49#.bmp into a couple of variables?

assuming you have the file name in a variable already ($file say), then
```
dave@xxxx:~/tmp$ file="Jupiter0003 07-08-29 21-39-49#.bmp"
dave@xxxx:~/tmp$ datetime=($(echo $file | sed 's/^[^ ]* //; s/#.*$//'))
dave@xxxx:~/tmp$ echo ${datetime[0]} ${datetime[1]}
07-08-29 21-39-49
```

### Post by geirha on 2009-08-22
In the resulting filenames, what does the number correspond to (e.g. 0020 in R0020.bmp)? 

1. Is it the last four digits in the filename? Jupiterqq24 07-08-29 21-38-59[COLOR="Red"]0052[/COLOR].bmp
```
for file in *.bmp; do echo "R${file: -8}"; done
```

2. Is it the first digits in the filename? Jupiterqq[COLOR="Red"]24[/COLOR] 07-08-29 21-38-590052.bmp
```
for file in *.bmp; do tmp=${file%% *}; printf "R%04d.bmp\n" "${tmp##*[A-Za-z]}"; done
```

3. Is it not related to the filename, only a counter that increases along with the lexiographical order of the original files?
```
files=( *.bmp )
for ((i=0; i < ${#files[@]}; i++)); do printf "R%04d.bmp\n" "$i"; done
```

### Post by stroyan on 2009-08-25
You can use bash alone to parse strings.
Here is an example that uses read and ${Var:Offset:Length} to parse them.
```

#!/bin/bash
for f in *.bmp
do
    read j d t<<<"$f"
    t=${t:0:8}
    echo file "'$f'" date $d time $t
done

```
The read builtin separates fields using the same IFS delimiters value
that is used for shell input.  The IFS value can be changed if you want
to break up fields at different delimiters than the default whitespace
characters.  If you do that you need to be careful to put it back to
the default for other shell script parts that expect the default value.
(The defaults are fine for your task.)
