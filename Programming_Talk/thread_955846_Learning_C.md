The von Neumann architecture and one of the more prominent corollaries, the linear address space, can not safely be ignored when learning to program. The concept of a variable occupying space in the linear address space of storage is not intuitive. It has to be learned and understood. Let's not forget that even we, the more experienced members, had to learn such things at one time or other. It is just such a basic premise that we are not aware of it any more.

There have been questions about variables shared between pieces of code, of results being evaluated in-place or by using auxiliare variables and so on which can be answered either by a very advanced conceptual abstraction or by going step by step through the usage of the real estate of the RAM.

People who posed those questions were not obviously helped with the advanced abstraction.

A Pointer, BTW can be more than just an offset into the address space of an array. In some languages they are the only means to manage memory, to map data elements to graphs and many more applications.

### Post by pp. on 2008-10-24
> **CptPicard said:**
> Do they, really? For example, you can code in Python perfectly well without ever having the address of your variable.

I was speaking about a cognitive problem often encountered in beginning programmers, and the kind of questions asked was "why does the value of the variable ***here ***change when I change the value of the variable ***there***?" or "Why does the result of computing the values in a tuple differ from computing the values in discrete variables?".

My analysis of the reason why those questions are asked often arrives at the conclusion that people who ask such questions do not take into account (or are not aware of) the concept of storing variables in numbered storage cells which several parts of the same program can refer to.

It's not in any obvious way related to pointer arithmetic or to the actual addresses of any piece of data, although those are concepts which derive in a straight forward manner from the concept of numbered storage cells.

Of course, programming in Lisp or Prolog makes it possible to ignore the basic computer architecture much longer than programming in Algoloids or Assembloids does because the concept of the variable as a chunk of memory does not really apply to the architecture of those languages, as far as I can determine.
