### Post by epictetus on 2008-09-04
Starting to learn programming languages (though not actually *learning* them) is my forte!  I've only really done anything of length in Mathematica and one or two little things in Python.  (And LaTeX / HTML don't count, right?  I mean, when someone without programming experience thinks of capital-P Programming, they don't think of layout systems---they think of writing up something on a keyboard that makes wonderful clackity clack noises and then hitting enter a few times whereupon their program outputs lots of data and simultaneously breaks into the NSA and halts a Russian nuclear weapons launch.)   I've invested much time in starting to learn Java, C++, Python, Ruby, and Objective-C.  I'd say that the most daunting aspect is unintuitive syntax---learning a programming language is very much like learning a foreign language except that when you screw up, things crash.  With that, I'd recommend Python.  My computer science friend has referred to it as "practically executable pseudocode."  Stick with a decent IDE, too (maybe the Python IDE).  Syntax highlighting makes code so much more readable for beginners.

### Post by themusicwave on 2008-09-04
I'd say start with something higher level like python, ruby or PHP.

I don't recommend C++.  Many people don't find sorting out pointers and friend functions and .h and .c files and all that very fun.  Start with a very simple dynamic language that can accomplish her goals and go from there.
