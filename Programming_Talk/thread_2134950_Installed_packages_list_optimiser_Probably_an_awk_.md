Do you really need to parse out the depends from 'apt-cache show'? couldn't you just use 'apt-cache depends'?

```
$ apt-cache depends htop
htop
  Depends: libc6
  Depends: libncursesw5
  Depends: libtinfo5
  Suggests: strace
  Suggests: ltrace
$ 
$ apt-cache depends htop | awk '$1 ~ /Depends:/ {print $2}'
libc6
libncursesw5
libtinfo5
$ 
```

### Post by schragge on 2013-04-13
Try
```
aptitude --display-format=%p --disable-columns search '?installed ?not(?reverse-depends(?installed))'
```
or, using the shorthand notation
```
aptutude -F%p --disable-columns search '~i!~R~i'
``` or even
```
aptitude -F%p --disable-columns search '~i!~R~i!~Rrecommends:~i'
``` to also exclude targets of *Recommends:*

However, bear in mind that the *~R*/*?reverse-depends* search term of aptitude will also list packages providing a virtual package that given package depends on, e.g. if package **foo** *Depends: www-browser* then
```
aptitude search '!~Rfoo'
```
will exclude _all_ packages that provide www-browser, so you may want to explicitly include installed providers of virtual packages:
```
aptitude -F%p --disable-columns search '~i(!~R~i!~Rrecommends:~i|~P~v)'
```
