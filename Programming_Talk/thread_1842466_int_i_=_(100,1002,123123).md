### Post by Senesence on 2011-09-11
If you compile with the -Wall flag, you will get:

```

warning: left-hand operand of comma expression has no effect
warning: left-hand operand of comma expression has no effect

```

It's basically telling you that it's ignoring all but the last number in that comma delimited sequence.

So, all that's left is this: "int i = 4546;", or an equivalent statement when everything gets compiled.

... I'm not really sure why it works this way - I would just call that a syntax error, but I guess somewhere in history there was one reason or another.

In C, commas separate instructions, and the result of the group is set by the last instruction. The code in parentheses is only executing:

101;
233;
4546;

which are all valid instructions (even if they trigger some warnings with -Wall)

Edit (ninja'd):
This behavior is coherent with the propagation to the left of expressions in C. When one writes:
x=y=123;
x is set because "y=123" propagates 123 to the left... but one can also consider that just 
123;
Does nothing but propagate 123 to the left (there may be a more technical term for this..)

### Post by Bachstelze on 2011-09-11
> **Senesence said:**
> 
... I'm not really sure why it works this way

This is just the standard behavior of the comma operator. the expression (a,b) has the value of expression b. The expression a is evaluated for its side-effects, but its value is discarded.

For example with

```
int i = (t=1, t+2);
``` 

i has value 3. The comma operator is rarely useful in practice, generally people who use it are just beginners who think it's cool to cram as many statements as possible in the conditional part of a for loop, like:

```
int (a=0, b=2, c=1234; a < 10; a++, b+=210)
```
