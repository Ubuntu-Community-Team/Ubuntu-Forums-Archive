Ok..this might be a good thread if we treat it right. First, please the idea here is NOT to post tutorial websites, those can be found all over the place.

This is to place great sites that are indispensable in terms of the information they provide. I'm thinking like specs and interpretations of specs for various APIs, file formats, anything like that. Something that is truly an information gold mine. Keep in mind that the best references aren't always the original creator.

Some that would be good to see:
Linux System API Reference (I have a few bookmarked, but don't know how good any are)
Qt API reference (I know the Trolltech one, but not sure if there are better)
GTK+ Reference

This is the kind of thing I'm talking about.

Basically, [http://www.wotsit.org/](http://www.wotsit.org/) is a website that hosts specs for many file formats; even some really old ones.

The Java API: [http://java.sun.com/j2se/1.5.0/docs/api/](http://java.sun.com/j2se/1.5.0/docs/api/)


You probably meant
[http://java.sun.com/javase/6/docs/api/](http://java.sun.com/javase/6/docs/api/)
which is the current resource.

Also, [http://producingoss.com/](http://producingoss.com/) is a crucial free book for anyone willing to participate in a large OSS project (or perhaps start one). Discusses the practical and cultural side of things: first, what the language is (licenses, hackers, forks, etc), then what software you need (bug trackers, etc), and most importantly, what the communities are like and what interpersonal skills you need. Extremely useful on several levels.
