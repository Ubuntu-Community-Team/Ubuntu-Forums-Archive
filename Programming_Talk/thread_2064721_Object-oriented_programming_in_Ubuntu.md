### Post by xytron on 2012-09-30
> **titusjaka said:**
> It means I have to use Borland libs and Borland compiler :)

Or something very similar

You ought to be doing your development on a Windows computer.

I don't like the C++ Builder style of programming.  Sure it makes it simpler to build your application GUI, but in the end it will make your programs depend on all the C++ Builder libs. 

If you are going to program for Windows it's best to just use Microsoft's Visual C++ plus there's a free version of the compiler.

Apparently that is the compiler your instructors know and that's the one you'll have to use.

The best you can do is to try to keep your user interface code separate from everything else then it will be far simpler to port you code over to a different compiler when you are ready.
