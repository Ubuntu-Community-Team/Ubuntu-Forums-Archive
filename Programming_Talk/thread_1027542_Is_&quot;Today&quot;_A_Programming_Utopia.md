### Post by CptPicard on 2009-01-02
From the theoretical perspective, of course programming will not and can not disappear, because by definition there is no such thing as automated programming/debugging in the general case. Machines cannot program themselves, because in that case they could solve the halting problem, which is not possible.

Therefore, all these UML tools etc will have to work with some subset of Turing-completeness, and if they don't, you still end up with a language capable of infinite loops, in which case -- when doing "general case programming" -- we have better tools than drag and drop designers.

The really interesting bits in CS and programming are not where interns can be used as replaceable cogs... IMO the fun part, where human beings will be required, is at the intersection of some interesting real-life problem, its related theory, and its translation into a non-trivial implementation making use of interesting possibilities granted by the implementation language... you can never automate producing those solutions.
