### Post by lykwydchykyn on 2012-01-18
I guess I'll throw my opinion out here, for what it's worth:

1. You pretty much have to learn HTML first.  No matter what other languages you're coding in, when it's all said and done you're sending an HTML file to the browser.  And I disagree with the previous poster, learn HTML5.  It's where pretty much everything is headed now.

2. CSS is pretty essential to doing anything with modern HTML from a looks perspective.  You can't style HTML at all without CSS (or using deprecated tags, which you don't want to use).

3. I think third is a toss-up between Javascript or a server-side language.  I'd lean towards a server-side language like PHP, because not only can you make perfectly useful sites without javascript; it's arguably better practice to make a functional site without javascript, and then later enhance it with javascript.

I'd recommend PHP here, because it's got a ton of libraries for interfacing with just about anything (databases, graphics libraries, authentication, etc).  I agree that you need to learn the newer approaches to PHP coding, though not strictly OOP.

I've tried python for web development, but (while I LOVE python for just about any other job) it doesn't shine for web development IMO.  I dearly felt the lack of immediacy and convenience functions compared to PHP.

Can't comment on Perl, I've never managed to learn it adequately.  Frankly, though, if you have one server-side scripting language that works for you, I don't see a reason to run out and learn another.  Life's too short.

### Post by SeijiSensei on 2012-01-25
> **lykwydchykyn said:**

I endorse this statement.  I've been writing PHP scripts for well over a dozen years now.  I rarely use Javascript for anything other than the occasional "onload" or "onclick" type of handler.  I much prefer having all the code safely stored in carefully protected directories on the server than being exposed for all to view as Javascript.

PHP has an extensive and easy-to-use set of IMAP functions that made it a good choice for this task.  I've also written command-line PHP scripts to parse logs and store the entries in an SQL database with a PHP front-end to run queries.
