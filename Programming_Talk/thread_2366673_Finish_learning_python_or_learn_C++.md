### Post by tdawgf on 2017-08-07
Most people would probably tell you to start with C++ since that is what your university uses. However, I would advise you to stick with Python for the time being. Now, the syntax to python is a bit different from C++ but you will be getting a head start in something that I have found that universities have a hard time teaching. You will be learning how to think like a programmer. Python is a great langauge for learning Object Oriented Programming and, if you learn the concepts of OOP, then the learning curve for C++ will be a decent bit lower.

At the end of the day, learning either won't do you harm, and I always recommend that you keep at least 1 systems language and 1 scripting language in your toolbox combined with what ever other languages you want to learn.

### Post by ian-weisser on 2017-12-19
You will learn more by doing both.

By solving each of your problems in both languages, you will see how the structure and syntax guide your thinking, and how each solved low-level problems like memory management. You will more quickly grasp the advantages and disadvantages of each language and your skills will be greatly enhanced.
