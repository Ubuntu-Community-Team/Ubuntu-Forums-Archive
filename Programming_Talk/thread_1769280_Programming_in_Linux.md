### Post by Vaphell on 2011-05-27
it may something to do with the fact that linux is a system perfect for tinkerers and there are tons of programming languages and programming related tools readily available, while windows is more suited for ordinary consumers, where inner workings are usually hidden. Probably more a matter of perception than anything. That being said Visual Studio is an excellent environment, but it forces you into MS ecosystem, that's an obvious strategy to lock businesses in and secure profits down the road. If you want to give your users choice and go crossplatform, that's not the way to go.

As for C#, it is native in linux thanks to mono, though many people are cautious towards it, as it ties linux ecosystem based on freedom with microsoft tech and MS does not have a stellar track record of playing nice in the long run. There are tons of other languages available, more or less cross platform, so mono is by many considered a wasted effort that doesn't bring anything of value to the table.

### Post by nemilar on 2011-05-28
I can't really speak to anything about C#, but as far as Linux as a platform for learning how to program, there are a few strong arguments I think you can make:
[LIST]
[*]Free (as in beer) access to compilers for almost any language
[*]System files are not hidden, you can easily see what is going on at all levels of the OS
[*]Wide range of documentation and support available on the web
[*]Most Linux users are programmers or at least tinkerers; it is a helpful culture for a learning environment
[/LIST]
