[SIZE=2]But the special thing about Wallch is its function that allows you to use all of its Unity shortcut options without even the GUI to be open! This means that even if Wallch is closed, you can enable Live Earth Wallpaper or the normal Wallpaper Changing Process without opening the GUI (GUI = Graphical User Interface (this means the window of Wallch))!
For example, if you want simply to change wallpaper, you can click on the 'Change Wallpaper' option. Your wallpaper will change by picking randomly an image from the ones you had chose the last time you opened Wallch! Same thing with 'Start Changing Wallpapers', with the difference that this starts the normal Wallpaper Changing Process using the delay you had set the last time. Also the pause/start/next/previous options work too!

Whenever you want you can change from 'Live Earth Wallpaper' to 'Start Changing Wallpapers' and back!

Hello, sorry for not replying, I hadn't seen the other post :/ 
Well, I just tried it and seems to work just fine.
You need to have more than 1 picture inside the list of the images, it doesn't relate with the creation of Wallch Albums! You just need to have more than 1 picture inside the images' list of the program before closing it. The program will read from these images, so, if you don't have even one it won't change the pictures and it will show an error.

Well, on the other hand your problem should be different, I think that your problem has to do with this: [https://answers.launchpad.net/wallpaper-changer/+question/170414](https://answers.launchpad.net/wallpaper-changer/+question/170414)
Briefly, Wallch saves the list with the images only when closes, and it has to close properly. If you shut-down your pc without closing properly wallch prior to this, the list won't be saved. As a result, Wallch will not find any pictures to load on start-up.
As you inform me that the list is empty every time you start Wallch, this means that you don't close Wallch properly :)

As answered on the buddy in the above link, this has already been solved and it will be available on the next version of the Wallch.:P

Just make sure you're closing Wallch properly each time you reboot your PC for now, something you shouldn't take care of  in the future (while using the versions that are to come), because they will both save the list every 4 seconds after the last change made on the list and on the proper close of the program ;)

Wallch is running on stealth mode :) xD

Because of the fact that many users prefer their desktop background to change without Wallch to be visible anywhere, we have decided not to show up anywhere.
This stealth mode (OK, it's called non-GUI mode :P) co-operates well with the Unity shortcuts. When Wallch is in non-GUI mode you can select Stop, Next, Previous etc from the Wallch's unity shortcuts.

So, the fact that Wallch isn't visible anywhere is not a bug, is more of a feature so as not to become annoying.

In Version 2, when folder monitoring feature will be implemented, you won't have to open Wallch in GUI mode to add the picture manually.
You will move the picture(s) to the monitored folder and they will be added automatically in both GUI and non-GUI mode.

Ok, the tray now is a quite bad story, as you may know, trays weren't supported in 11.04, only indicators were. Now, the sni-qt package enables tray icons to show up. I am not sure that tray will be supported in 12.04 too (It should anyway), and that's why we are not going to code anything to alter the attitude of the tray. If we had an option for the user to show up to the tray on startup and start the corresponding action as well (start changing wallpapers or live earth),  and then tray isn't supported (and so it doesn't show up), this will be very confusing for the users, so we have to be careful.

It is OK to ask for features here or to discuss :)
