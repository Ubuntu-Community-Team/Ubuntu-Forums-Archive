Now, the question changes **entirely** when asking if BASIC is good for making program X or Y. The answer then depends on what you need to do, and most importantly, if the language is extensible. Being able to add C/++ modules, or even Java and Python modules, is important.

BUT, I don't know of any BASIC's that allow for certain things like void pointers or function pointers, iterators, or more complex loops.

For instance, the "STEP" keyword doesn't allow for certain things like exponential increase:

```

int i;
for(i=1; i<=1024; i*=2) printf("%i\n", i);

```

And definitely not:

```

float i;
for(i=1.05; i<=515.5; i*=i) printf("%f\n", i);

```

You would have to construct those conditions and loops on your own.

But mostly... Function pointers. A language without true classes absolutely must have function pointers. If there are dialects of BASIC with either classes or function pointers that I don't know about, then that's fine. But I've never heard of any...

```

#include <stdio.h>

// Function to point to
float func_add(float a, float b){ return a + b; }
float func_sub(float a, float b){ return a - b; }
float func_mul(float a, float b){ return a * b; }
float func_div(float a, float b){ return a / b; }

// Return pointer to correct function
float (*get_func(char op))(float, float)
{
    switch(op)
    {
        case '+': return func_add; break;
        case '-': return func_sub; break;
        case '*': return func_mul; break;
        case '/': return func_div; break;
    }
    return NULL;
}

int main()
{
    float a=100, b=200, (*operation)(float,float);

    printf("a + b  = %f\n", get_func('+')(a, b));

    operation = get_func('/');
    b = operation(b, 10);
    printf("b / 10 = %f\n", b);

    operation = get_func('-');
    printf("a - b  = %f\n", operation(a, b));

    return 0;
}

```

And what about iterators? I don't know of any BASIC's capable of creating custom iterators (since they don't have function pointers).

```

#include <stdio.h>

float func_print (float a){ printf("%f ", a); return a; }
float func_increm(float a){ return a+1; }
float func_double(float a){ return a*2; }
float func_square(float a){ return a*a; }

void iterate(float *data, int size, float (*op)(float))
{
    int i;
    for(i=0; i<size; ++i) data[i] = op(data[i]);
}

int main()
{
    float data[5] = {0, 1, 2, 3, 4};

    iterate(data, 5, func_print);
    printf("\n");

    iterate(data, 5, func_increm);
    iterate(data, 5, func_print);
    printf("\n");

    iterate(data, 5, func_double);
    iterate(data, 5, func_print);
    printf("\n");

    iterate(data, 5, func_square);
    iterate(data, 5, func_print);
    printf("\n");

    return 0;
}

```

The fact that I can manipulate that array in any way I want without needing to write a new "for" loop each time not only saves my time to type it out, but it also saves program size while adding a strong generic capability to your programs which allows you to reuse large portions of code.

This is one thing I remember about BASIC that I wasn't happy with. It was really hard to reuse code and write generic routines. Sure... Someone could add classes and pointers to BASIC... But then it wouldn't be "BASIC" anymore, it would be C or Python but with words instead of "{}" or indentation :)
