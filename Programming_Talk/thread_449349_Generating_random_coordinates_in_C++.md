### Post by WW on 2007-05-20
No, that won't work.  Instead, choose a random radius and a random angle, with the angle between 0 and 2*Pi.  Then convert from polar coordinates to (x,y) with x = radius*cos(angle), y = radius*sin(angle).

This will not give a *uniform* distribution; your points tend to be closer to the center.  To get a uniform distribution, choose a random number u between 0 and 1, and then let radius = R*sqrt(u).

### Post by hod139 on 2007-05-20
Uniformly distributed means that all regions of the shape are equally likely to be selected by your random number generator.  More formally, [SIZE=-1]the probability of a number falling in a particular region is proportional only to the area of the region.

For a square, this guarantee is simple, but for more complex shapes, like a circle, it gets harder and harder to construct uniform samplings.  The easiest way, (and this works for more complex shapes) is the so called accept-reject method.  You bound your shape with a larger shape that is easy to generate a uniform distribution (e.g. a square), and throw away the points that do not lie inside the actual shape.  For your circle example, you can surround it with a square, generate uniform random points inside the square, throw away points that do not lie inside the circle.  The resulting sample will be a uniform sampling of the circle.
[/SIZE]

Here is a program that choose a random radius between 0 and R, and a random angle between 0 and 2*pi, and then converts to (x,y) coordinates.

**circle.cpp**
```

//  circle.cpp

#include <iostream>
#include <cmath>
#include "mtrand.h"

using namespace std;

int main()
    {
    MTRand r(123);
    double R = 10.0;
    for (int i = 0; i < 10000; ++i)
        {
        double theta = 2*M_PI*r();
        double radius = R*r();
        double x = radius*cos(theta);
        double y = radius*sin(theta);
        cout << x << " " << y << endl;
        }
    }

```
The second attached plot shows the result. Note that all the points are inside the circle, but the distribution is not uniform--the points are denser near the origin.

The next version generates a uniform distribution:

**circle_uniform.cpp**
```

//  circle_uniform.cpp

#include <iostream>
#include <cmath>
#include "mtrand.h"

using namespace std;

int main()
    {
    MTRand r(123);
    double R = 10.0;
    for (int i = 0; i < 10000; ++i)
        {
        double theta = 2*M_PI*r();
        double r1 = sqrt(r());
        double radius = R*r1;
        double x = radius*cos(theta);
        double y = radius*sin(theta);
        cout << x << " " << y << endl;
        }
    }

```
The third attached plot shows that result.

Finally, this version uses the "accept-reject" method that hod139 described.

**circle_ar.cpp**
```

//  circle_ar.cpp

#include <iostream>
#include <cmath>
#include "mtrand.h"

using namespace std;

int main()
    {
    MTRand r(123);
    double R = 10.0;
    for (int i = 0; i < 10000; ++i)
        {
        double x,y,m;
        do
            {
            x = R*(2*r()-1);
            y = R*(2*r()-1);
            m = sqrt(x*x+y*y);
            }
        while (m > R);
        cout << x << " " << y << endl;
        }
    }

```
The fourth attached plot shows the result.

(To clearly show the difference between the uniform and nonuniform cases, I left the points as floating point.)

You should not declare variables inside the for loop and you also should not use the sqrt operator inside your circle test inner loop since sqrt is very slow.   You can make the  point inclusion test based on distance squared instead of distance, which will be a much faster test.  Here is the slightly modified code:

//  circle_ar.cpp

#include <iostream>
#include "mtrand.h"

using std::cout;
using std::endl;

int main()
{
    MTRand r(123);
    double R = 10.0;
    double Rsquared = R*R;
    double x,y,m; // Declare outside the loop
    for (int i = 0; i < 10000; ++i)
    { 
        do
        {
            x = R*(2*r()-1);
            y = R*(2*r()-1);
            m = x*x+y*y;
        }
        while (m > Rsquared); // Use distance squared, not distance since sqrt is very slow
        cout << x << " " << y << endl;
    }

   return 0;
}

### Post by hod139 on 2007-05-21
I should add that the accept/reject method I mentioned earlier is really known as Monte-Carlo estimation.  The circle in square example can actually be used to [estimate Pi]("http://www.modula2.org/projects/pi_by_montecarlo.php").  Modifying the above code to compute pi:

```

//  Uses probability to estimate pi

#include <iostream>
#include <cmath>
#include "mtrand.h"

int main()
{
    MTRand_closed r(time(NULL)); // closed interval rng, w/ random seed
    double x,y,m;
    unsigned long inside = 0; // Number of "hits" inside the circle 
    unsigned long N = 1000000; // Num of trials
    
    // Throw many "darts" are top right quadrant of the circle,  
    // recording the ones landing inside the circle
    for (unsigned long i = 0; i < N; ++i)
    {  
       x = r(); // double floating point numbers in the closed interval [0, 1]
       y = r(); // double floating point numbers in the closed interval [0, 1]
       m = x*x + y*y;

       if(m <= 1) // inside circle
         ++inside;
    }
   
    // Area of square is R^2 = 1, area of circle quadrent is Pi*R^2/4
    // Area of ratio is Pi over 4, so we can calculate pi as 4 * ratio
    std::cout << "Monte Carlo Pi = " << 4.0 * static_cast<double>(inside) / N << std::endl;
    std::cout << "Pi = " << M_PI << std::endl;
   
    return 0;
}

```and running:

Pi = 3.14159

This can by applied for any circle with radius R.  Each point is defined by (x, y).  The x-coordinate is the radius * cos of the angle counterclockwise starting at 0 [or 2pi].   The y-coordinate is the radius * sin of the angle counterclockwise starting at 0 [or 2pi].
[img]http://img107.imageshack.us/img107/9831/vectorrx9.png[/img]

PandaGoat's solution is wrong, but not for these reasons.  Let me restate the definition of uniformity: Uniformly distributed means that the probability that the point is in a region depends only on the area of the region, not its location on the circle.  In PandaGoat's solution, the radii are being randomly chosen from [0, R] under a uniform probability distribution.  Without loss of generality, let me set R equal to 1.  Now, we can expect half of all the radii to lie in [0, 1/2] and the other half to lie in [1/2, 1].  However, the areas of the inner circle of radius 1/2 and remaining outer region are not equal.  The area of the inner circle is:
pi*(1/2)^2 = pi/4
The area of the outer ring is:
pi*(1)^2 - pi/4 = (3*pi) / 4

Therefore, using PandaGoat's method, half of all randomly chosen points are being placed in a region 3 times smaller than the other region in which the other half of the points are being placed!

Now, if we look at WW's solution:
```

x = sqrt(r) * cos( phi );
y = sqrt(r) * sin( phi );

```and do the same area analysis, The area of the inner circle is:
pi*(1/sqrt(2))^2 = pi/2
and the area of the outer ring is
pi*(sqrt(1))^2 - pi/2 = pi/2
Here we see the areas are equal, and we have a uniform distribution.

I hope this clears up the confusion.
