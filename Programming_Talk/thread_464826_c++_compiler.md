### Post by LaRoza on 2007-06-05
My preferred method of compiling is through the command line.

My favourite editors are:

Windows:
1. Crimson Editor
2. Programmers Notepad

Linux:
1. GEdit
2. Kate

For me, an editor must have these features:
1. Syntax highlighting (all good editors have this, including GEdit, the default Ubuntu text editor)
2. Tabbed windows (I dislike having more than one window open for a single project)

-edit I am in school for most of the day, and am stuck using Windows XP on the schools computers, I can not install software, so I use a lot of portableapps, from [http://portableapps.com/](http://portableapps.com/), there are other portable apps I use that are not on this site. Most of the apps can be run in wine, including Firefox and Dev-C++ Portable. There is a Dev-C++ for Linux, but I don't use it. So I often use Dev-C++ Portable, which uses g++, and can also be run from the command line.

### Post by ankursethi on 2007-06-05
I personally prefer Scribes for light editing work. Saves a lot of trouble and overhead of loading a full blown IDE to edit each file.

Since currently I'm working on a Python app, I use SPE (Stani's Python Editor) as my IDE. It doesn't have as many features as, say, Eclipse, and I like that.

For C/C++ I'd recommend Eclipse, probably because I never could get the hang of Anjuta. In a KDE environment, nothing beats KDevelop. For Java, again Eclipse.

For C# there's MonoDevelop. Never tried it myself because I'm not a C# guy, but most people around here say that the version in the repos is buggy, so better get the official version from the website.

In terms of tools there's not much to say. It's all the same. GDB for debugging, the GCC for compilation, make for project management, CVS or Subversion for versioning etc. There was even a book about these tools, AFAIK.
