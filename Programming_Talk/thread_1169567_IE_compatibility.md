### Post by Reiger on 2009-05-26
Seconded: there are simply too many different implementations of how a table should look like (and too many hidden, dark and badly documented corners which tinker with default values and hence mess up your carefully crafted layout). And the onus isn't entirely on MSIE here, either.

It takes some time and brain power on your part but you can do all things layout with simple <div> elements and be (as far as is reasonably possible in the badly broken and inconsistent browser landscape) assured that if you *always* use a STRICT doctype (doctype triggering is the last resort in quirks mode) you gain a consistent enough layout to make it look intentional. (You'll still see low-level differences: for instance in Font rendering; Firefox implementation of "Justified" looks quite different from that of MSIE or that of Opera.)

You do not even need fake images either. With CSS you can position images (like top center or 50 50% or 300 150px) nicely, which means you can push all non-content images to the background of a nested div element (which is nice towards those people that use w3m or similar text-only browsers: they won't see as much junk).

### Post by Codemunkee on 2009-05-26
Suggestions about HTML/CSS conventions and best practices aside, I've gotten around this issue by applying a style within a SPAN in addition to writing the image inline within the SPAN.
 
For example, the following line defines a PNG background style for my class.
```

#myClass {display: inline-block;}
#myClass {filter:progid:DXImageTransform.Microsoft.AlphaImageLoader(src='YOURPNGPATH.png');

```
 
then, within the HTML:
```

<span id="myClass">
<img src="YOURPNGPATH.png">
</span>

```
 
This was the only way I could preserve alpha channels in my PNGs. Otherwise, the images would show up just as grey blocks.
 
Not sure if that's entirely applicable to your scenario but I looked at your source and noticed you didnt add the transforms, so maybe it will help.

### Post by tturrisi on 2009-05-26
To make the page render correctly in Internet Explorer and all other browsers do this: (so the content is centered)

<body style="text-align:center;">

and then change your existing div container to:

<div style="text align:center; width:780px; margin:0 auto;">

and then you MUST set text alignement for all elements inside the <div>, e.g. <p style="text-align:left;">.


Also, some of your code is incorrect. Use either absolute OR relative, not both in the same element.

```
<tr style="position:absolute; position: relative;" >
<a style="position:absolute; position: relative;>
```

Read up on CSS positioning:

I'm a bit confused on the proper syntax.
absolute positioning uses the window edges, relative positioning uses the parent edges.

Thus, if position something absolute top:10px left 10px, it's located 10px down from the top and 10px to the right of the left side of the window (down from top and in on left measuring from the window edge to the top and left sides of the element).

If position something absolute top:10px and right:10px it's located 10px down from the top and 10px to the left of the right side of the window (down from top and in on right measuring from the window edge to the top and right sides of the element).

Relative is the same except the position is measured from the parent element edges, not the window edges. Just start thinking with the idea that everything on the page is a box with dimensions, and a box can contain other boxes.

Using CSS for layout has many advantages, for one there is massive less source code duplication, which results in pages that are a hell of a lot easier to maintain. CSS also allows you to layer background images, enabling techniques such as sliding doors. This makes it relativity easy to create layouts which are completely dynamic, not only to browser size, but also allowing visually impaired usurers to zoom the text without breaking the layout and design.

Also in the field of usability, applying layout images using CSS allows users to easily change the look of a site, for example I cannot look at the common black text on a white background themes for long without getting a headache. To get around this I use stylish to replace all backgrounds with plain grey. this dousen't work with sites which use tables of images to create the design, which are normally dealt with by hitting the back button ASAP ;)

Also for users of accessibility technologies such as screen readers tables are a problem as they often require the navigation to be plased first in the html, resulting in the reader reading through all of the navigation before the content, which could result in hitting the back button and looking elsewhere.

> 
absolute positioning uses the window edges, relative positioning uses the parent edges.

not quite correct, absolute positioned divs are aligned absolutely to there containing relativity positioned div. this allows you to create a ``container'' div to centre the page, for example:

```

<html>
    <head>
        <title>example</title>
        <style type="text/css">
            * {
                padding: 0;
                margin: 0;
                border: 0;
            }

            #container {
                position: relative;
                width: 90%;
                height: 400px;
                margin: auto;
                background: #a00;
            }

            #subobj {
                position: absolute;
                left: 20px;
                top: 20px;
                color: #fff;
            }
        </style>
    </head> 
    <body>
        <div id="container">
            <h1 id="subobj">text</h1>
        </div>
    </body>
</html>

```

see open source web design and htmldog for lots more examples

Of course, this is the proper way to center a wrapper element with class "container":
```

body {
    /* 
       For IE6 compatibility.
       Optionally, put this in a separate stylesheet and use
       conditional comments
    */
    text-align: center;
}

.container {
    width: 70%; /* Or whatever */
    margin: 0 auto;
    text-align: left; /* Because we certainly do not want all of
                         the text to be centered. Again, this
                         can be in a separate stylesheet */
}

```
However, absolute positioning is needed to center an element both vertically and horizontally:
```

.container {
    width: 800px; /* Or whatever */
    height: 400px; /* Or whatever */
    position: absolute;
    top: 50%;
    left: 50%;
    margin-top: -200px; /* -Height / 2 */
    margin-left: -400px; /* -Width / 2 */
}

```
