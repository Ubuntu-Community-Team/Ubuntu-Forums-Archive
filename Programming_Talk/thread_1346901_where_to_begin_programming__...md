### Post by A_Fiachra on 2009-12-05
I am told, though I've never read any from cover to cover, that the Dive Into Python books are quite good -- there is a Python 3 book -- but most Python out in the world (including what is included with standard linux distros) is Python 2.x

I've looked at the differences and they are not too radical, also Python comes bundled with a 2-to-3 script that identifies 2.x features that will not work in 3.x.

I agree with Can, PHP is a mess -- don't waste your time.

Perl is fantastic but it is a very odd language that has a long learning curve.

C++ and Java are important, but it seems to take forever to get anything done in either -- lot of advanced topics come at you before you can write a line of code and one can easily spend months (or years) learning those languages before creating anything other than toy applications.

Ruby is popular because of Ruby on Rails, but Ruby itself is a bit odd as a first language because of its functional pedigree.

Personally I like Python.  It's clean, it's powerful, it plays well with others, it's useful across a wide variety of computing domains (web, gui, parsing, command line, scientific computing (NumPy and SciPy packages are great)), it's easy to get started with, it's easy to continue learning, it can be extended, etc etc etc
