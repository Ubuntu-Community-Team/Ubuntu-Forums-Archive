### Post by pmasiar on 2008-08-20
Every now and then yet another language flamewar erupts. Some are based on comparing languages, and we eliminated them partially by establishing "Why I love/hate language X" threads, so we don't feel obliged to repeat the same arguments again and again.

But wars continue, this time along lines when language seeker (LS) asks: "what is best first language to learn" or "what language to learn next".

I noticed there are four school of thought about this issue:

**historicist** mantra is: language A worked for me, try it. Obviously this has limited use: historicist most often does not have experience with other suggested languages (so cannot compare from experience, only from prejudice), or if old hand like me was historicist, everyone should learn Algol60. Good luck finding it!

**mercantilist**: says "language B is best to find a job". That might be true, but hardly relevant: (1) LS might want to use language for research, fun or contributing to FOSS, and (2) why not learn best language for beginner, then learn best language for the job? Then LS knows 2 languages, which is obviously better than one. 

**agnostic** schools says it does not matter, because all languages are Turing-complete, and LS is not able to distinguish between languages and pick best tool for the job until s/he learns couple of them, so is OK to start at any language. And every time someone suggest a language, they suggest another to give LS more "choice".

**intuitivist** says "language Z is the most intuitive for beginner, so learning anything else is wrong" (while substituting different languages for Z). 

Sometimes they are mixed: historicist who learned "business' language as first might be also mercantilist. Agnostics often suggest 'useful' language like a mercantilist to get a job.

So LS is facing all this cacophony of suggestions. What is going on?

Of course, as always, there are couple cognitive biases going on: 

[Bias blind spot](http://en.wikipedia.org/wiki/Bias_blind_spot) is of course main meta-bias. :-) and you may consider reading [List of cognitive biases](http://en.wikipedia.org/wiki/List_of_cognitive_biases) for inspiration :-)

[Confirmation bias](http://en.wikipedia.org/wiki/Confirmation_bias) and [selection bias](http://en.wikipedia.org/wiki/Selection_bias): historicist wants his original choice be confirmed by others making same choice. 

[Bandwagon effect](http://en.wikipedia.org/wiki/Bandwagon_effect) for all but agnostics

Another trick is [The Paradox of Choice](http://en.wikipedia.org/wiki/The_Paradox_of_Choice:_Why_More_Is_Less) which tricks agnostics to suggest more languages if false hope that with more language to chose, they are helping LS to make better choice and make her happier  (it's the opposite).

I'll let you discuss this, I'll just tell that I am intuitivist, consider all those biases harmful for optimal language selection, and the only valid metrics is "which language is intuitive and friendly to beginner".

<joke>
With obvious answer **Python** as confirmed by 95% of people know know it (rest is for Ruby, and slavik is for Perl :-) ).

So if you are agnostic, mercantilist or historicist, reconsider your bias, learn Python, and agree with me :-)
</joke>

**Edit**
I added joke tags above to make it more obvious (some people missed) and dig out for you couple more biases:

[desire of closure](http://en.wikipedia.org/wiki/Cognitive_closure_(psychology)) - "a desire for definite knowledge on some issue and the eschewal of confusion and ambiguity.

[Buyer's bias](http://en.wikipedia.org/wiki/Post-purchase_rationalization) (Post-purchase rationalization) - after people have invested a lot of time, money, or effort in something to convince themselves that it must have been worth it

[http://en.wikipedia.org/wiki/Reactance_(psychology](http://en.wikipedia.org/wiki/Reactance_(psychology)) - an emotional reaction in direct contradiction to rules or regulations that threaten or eliminate specific behavioral freedoms. It can occur when someone is heavily pressured to accept a certain view or attitude. Reactance can cause the person to adopt or strengthen a view or attitude that is contrary to what was intended and also increases resistance to persuasion.

[http://en.wikipedia.org/wiki/Status_quo_bias](http://en.wikipedia.org/wiki/Status_quo_bias) - people like things to stay relatively the same. - a combination of loss aversion and the endowment effect, 

[http://en.wikipedia.org/wiki/Projection_bias](http://en.wikipedia.org/wiki/Projection_bias) - a defense mechanism in which one attributes one’s own unacceptable or unwanted thoughts or/and emotions to others. Projection reduces anxiety by allowing the expression of the unwanted subconscious impulses/desires without letting the conscious mind recognize them

I found not one but **two** biases what I can be safely accused of having, but I will not make it too easy for you guys: If you want to know it, you will have to read through them all and think about it :-)

About the subject (obviously, no use in debating specific languages here, that has already been done). 

I found, by accident, the propaedeutic value of learning [Esperanto]("http://en.wikipedia.org/wiki/Propaedeutic_value_of_Esperanto") before learning other languages. That means, if a person learns Esperanto, a language that is characterised with simplicity and doesn't quite resemble more complicated languages, that same person will be able to learn another language swifter than if she started with the other language. I think that is amazing. 

I also think a similiar effect will be found in programming. If you take a language like Java or C, their syntax and "grammar" is a little bit arcane, compared to the simplicity of Python (in this case, Ruby and Perl also). For Python, if you want to print something, you print it. If you want to assign a variable, you assign it (and the interpreter handles the rest). If you want to define a block, you indent. Whereas in C you have to use a standard library function (or severl of them) and take care of the data type when outputting variables. You have to declare the data type of a variable and that datatype is not enforced, so you have to be careful. And you define blocks with braces. This is all just basic concepts, when you get to looping or data structures, it becomes even more contrasting.

So I think teaching a simple programming language, with a simple syntax and logical structure (meaning, logical even if you don't understand it, a person could get the gist of some Python code, most C code would baffle the average person). This language would not put great demands on the programmer either and allow the person to use concepts as they learned them, not have to ignore them until later or studying them in depth before using them. This language could be constructured purely for learning, but Python without using its standard libs or features that are less apparent (functional programming, list comprehensions, etc) fits the bill. 

So when someone wants to "learn to program", it is important to give them a base for learning any language as soon as possible, and it is only bonus if that language is useful. Instead of creating a language for such use (like BASIC from the olden days), using Python since it is already there makes sense.

If the person learns output, variables, conditional control, looping, functions and basic math functions in programming, and possibly basic OO, they will be set to learn any language. In Python, those concepts are extremely simple and clear to a beginner; Python can showcase concepts without people really knowing the language.

Learning that things doesn't really teach a lot (learning how to program hasn't be done yet) but the concepts are there in an easy to understand format. When I started with C++, I didn't get a lot of things and wrestled with C++, instead of programming. If I took a week to study Python (or in this case, almost any other language) I'd have gotten C++ much faster I think, even taking that extra week into account.

I think there is a study using Java and Python which is similiar to the Esperanto studies. I will look for it.
