### Post by Tony Flury on 2012-04-25
My recommendation : 

Start with python : [http://docs.python.org/index.html](http://docs.python.org/index.html)

Great support for loads of different libraries (including GUIs and console applications) very quick to start learning, and without many of the early learning pitfalls that a number of other languages have.

Don't dismiss python as "a scripting language" - it is entirely possible to write some very powerful applications using it and it's libraries.

Once you have mastered some of the concepts of programming with python - then look at other languages - depending on what type of application you intend to develop.

### Post by LemursDontExist on 2012-04-25
If you want to be up and running and writing apps in the shortest time, along with everyone else I endorse python.  It is both easy to use, and powerful enough to do just about anything.

That said, if you're interested in understanding how computers work, I would start with C.

Python is written in C, and I think that being a really good Python programmer requires understanding C.  Python hides a lot of the details from you, and it's hard to write the best code if you don't know what's going on under the hood.

You need to be able to read the source code when you're confused, or write an extension when you need functionality that no one else has implemented.  With the combination of Python and C you'll find yourself writing code in a small fraction of the time it would take in another language with no meaningful sacrifice in terms of what you can do or runtime.

### Post by LemursDontExist on 2012-04-25
> **xTheAngrySockx said:**
> So it looks like python is the way to go :D
Just one more question, how do I go about getting an IDE or something? I've seen eclipse for java and they seem really useful.

Serious IDEs have a substantial learning curve, and I would avoid them until you've got your bearings.  Until you're writing more than a few hundred lines of code at a time, an IDE will just get in your way.

I would recommend using an editor that does syntax highlighting though.  The built in text editor in ubuntu (gedit) works fine.  Geany is a slightly more feature-rich alternative.  Set your editor to use spaces instead of tabs, and make a tab 4 spaces wide.  That'll save you a lot of hassle in the long run.
