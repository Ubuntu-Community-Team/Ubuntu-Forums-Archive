### Post by 11jmb on 2012-06-14
Are you using GNU assembler? If so, both syntaxes are supported with the directive
```

.intel_syntax noprefix

```

Edit: I should point out that inline asm syntax is not standardized, so you may have to change more than just intel/at&t depending on your compiler

### Post by xb12x on 2012-06-16
> **11jmb said:**
> I must ask: what is your purpose here? Multiplying 2 16-bit ints is hardly a useful way to use assembly.

I had the same question. 

But whatever your purpose, you could transpose it from assembly to 'C' and avoid the issues with inline assembly, AT&T syntax vs. Intel syntax, etc. 

Or

You could create a separate file for any required assembly routines and link to it. This file could be written using whatever assembly syntax you're most comfortable with. I would choose the Nasm assembler which supports the Intel style. By far most of the assembly code in the world is Intel style.

The Nasm assembler is a Linux, open source assembler which supports Intel style syntax. If you are familiar with Microsoft's Masm you'll be right at home with Nasm.

Of course, assembly coding in general is obsolete now, and anyone learning to code in preparation for a career should not waste time on it. I suspect today's beginning coders will go their entire careers rarely, if ever, having to deal with  assembly code.

```

gcc -c assembly_source.s -o assembly_source.o
```
