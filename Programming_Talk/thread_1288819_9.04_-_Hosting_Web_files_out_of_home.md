### Post by tj111 on 2009-10-11
I'm trying to set up the ability to host my web development files out of my /home directory using virtual hosts, however every configuration I have tried so far results in a http 403 forbidden.  I have changed the permissions on all the files to 775, changed group ownership to 'www-data', and tried a plethora of different virtual host configs.  Here are my httpd.conf and my virtual hosts file, I left my apache2.conf file with default settings.

httpd.conf
```

DocumentRoot "/var/www/"

ServerName localhost:80

<Directory "/var/www">
	Options Indexes FollowSymLinks
	AllowOverride all

	Order Deny,Allow
	Deny from all
	Allow from 127.0.0.1
	Allow from 127.0.1.1
	Allow from localhost
</Directory>

<Directory "/home/tj/Empression/diagnostics/Web/">
	Options Indexes FollowSymLinks
	AllowOverride all

	Order Deny,Allow
	Deny from all
	Allow from 127.0.0.1
	Allow from 127.0.1.1
	Allow from localhost
</Directory>

```

Virtual hosts
```


#ServerName localhost
NameVirtualHost *:80


<VirtualHost *:80>
	ServerName localhost
	DocumentRoot "/var/www"
</VirtualHost>


#Visual Diagnostics
<VirtualHost *:80>
	ServerName diagnostic.localhost
	DocumentRoot "/home/tj/Empression/diagnostics/Web/VisualDiagnostic/httpdocs"
</VirtualHost> 

```

---

### Post by tj111 on 2009-10-11
I solved the problem.  The issue was my /home/username directory had permissions of 700, I changed them to 755 and everything works now.
