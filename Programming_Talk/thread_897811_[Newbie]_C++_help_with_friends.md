@dwhitney67: Wouldn't 'protected' be a wiser design?
No, an external class will not have access to protected or private data; only the public data (and methods).

Now if class B inherits from class A, then yes, class B can access the protected data (and methods).

A "friendship" relationship exposes all data/methods of a class, hence the reason it is not kosher to use often.  Probably the best reason to define a friend is when defining/implementing a method that prints debug information about a class's state.  For example:
[PHP]friend std::ostream & operator<<( std::ostream &os, const SomeClass &c )
{
  // friend method has access to private data of object 'c'
  os << c.m_data1 << ", " << c.m_data2;
  return os;
}[/PHP]

### Post by dribeas on 2008-08-22
> **dwhitney67 said:**
> Probably the best reason to define a friend is when defining/implementing a method that prints debug information about a class's state.  For example:
[PHP]friend std::ostream & operator<<( std::ostream &os, const SomeClass &c )
{
  // friend method has access to private data of object 'c'
  os << c.m_data1 << ", " << c.m_data2;
  return os;
}[/PHP]

And even then, providing a virtual (if the class is meant to be inherited) dump method (whether public/protected/private) is usually of help, as it will print at the appropriate level of the hierarchy regardless of the type of the reference.

```

class Inheritable { 
private/protected/public:
   virtual std::ostream& dump( std::ostream & o ) const;

// friend std::ostream& operator<< (std::ostream &, Inheritable const & ); // needed only if private/protected
};
std::ostream & operator << ( std::ostream & o, Inheritable const & );

```

Friends can be wisely used in some other cases. One example of good design with friends is the boost shared_ptr/weak_ptr libraries. Shared_ptr and weak_ptr are very closely coupled and related. They don't form a hierarchy, but they do need access to methods that should not be exposed (should not be public) and neither of the classes is meant to be extended.

Note that the description of the design limits all other possibilities. Only then friendship is appropriate.
