### Post by CptPicard on 2009-08-22
First of all, the "fun-ness" of using some programming language is often underestimated. To me it tells that the language is pleasant to use and you get to apply your mind to the critical parts that the computer will not be able to handle, while making use of the machine to handle those parts that it can. I'm all for "fun" programming, even when some people feel that only programming that feels painful (that is, forces you to spend time on useless, automatable crap) is serious enough ;)


> **uljanow said:**
> That seems to be fast enough since it's not the bottleneck. But still just because youtube uses it, doesn't make it a good choice compared to jboss+tomcat+javaEE.


If YouTube uses it, it's probably quite a good technical choice. The J2EE stack has been awfully over-complex for a long time and has only recently become usable (IMO) with EJB3, annotations and good Netbeans support... yes, if you want a large enterprise system you'll probably want to look there simply because of the libraries and infrastructure, but on the other hand it still doesn't make Java the language any more remarkable than it is. Interestingly Python actually has a lot of language-conceptual stuff in it that Java doesn't, and that will teach the aspiring programmer interesting ideas down the path that would be harder to come by in your usual static-typed OOP language.
