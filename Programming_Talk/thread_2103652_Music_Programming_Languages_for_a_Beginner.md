### Post by gnuman on 2013-01-11
Hi!  I mess around with music and Linux a lot.  There are many options you can try [here]("http://en.wikipedia.org/wiki/List_of_Linux_audio_software") that are not programming languages, but many of these applications are still quite hackable in ways that will give you good skills for more traditional programming.  You learn a lot about an application like Hydrogen when you create your own drum kits from sound samples you record yourself.

On the programming languages front, though, I have enjoyed messing around with some Python audio libraries (and you will certainly be able to do what you had described above).  I don't even remember with libraries I have used, but they were small and very good at what they set out to accomplish.

Also check [Processing]("http://processing.org").  Processing recently went to 2.0, so some of the Audio libraries are still being updated for it, but there is a LOT of quality stuff you can get into very quickly.  Check out the [audio libraries]("http://www.processing.org/reference/libraries/#sound") they have.

On the other hand, if you really want to learn an Audio Programming Language, you could probably spend a lifetime in [Pure Data]("http://en.wikipedia.org/wiki/Pure_Data") or [Nodebox]("http://nodebox.net/"), which feature modular based programming that is a world all it's own!

I'm no expert at this, but I certainly have messed around a LOT with this stuff.  Good Luck! :)
