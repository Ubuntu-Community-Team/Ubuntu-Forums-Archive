### Post by naugiedoggie on 2008-01-24
> **LaRoza said:**
> Python. It has all you want, plus, it is simple and easy to learn (unlike Java)

Hello,

I often see this recommendation -- "it's easy to learn."  Why exactly is that a recommended method for choosing a programming language?  If someone were travelling to Rumania, would you recommend they learn Spanish because "it's easy to learn"?

I recommend that people make the choice based on what they want to do.  In this case, the OP wants a graphical design IDE, with drag/drop and advanced features builtin.  

There are two problems with the python recommendation.  (1) it has a highly idiosyncratic syntax, skill in which is not portable; (2) it's a horrible choice for a programmer whose primary interest is in designing and building GUIs.  And, I say "horrible" because graphical design is not a part of the language, it is something that was grafted on later, using external toolkits.

I personally found Java much easier to learn (coming, as I do, from a background programming in C) and, indeed, failed several attempts to learn python.  Naturally, not having found python "easy to learn," I have a builtin prejudice against this recommendation.  ;-)  

But, even so, I don't find "easy to learn" a compelling argument for *anything*.  Using a hammer is easy, but you don't drive a car like you do a nail.  "If every job starts to look like a nail, you need more than a hammer in your toolbox."

Thanks.
