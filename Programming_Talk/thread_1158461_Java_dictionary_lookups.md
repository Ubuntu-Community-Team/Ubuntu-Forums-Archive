Yes, that's what I was saying. You could write a tool that creates a huge list of put commands which can then be placed in the code. But that's only useful when the text file won't change.

I'd only use this approach (generating code and pasting it in the source) when you're 200% sure the contents won't change. Because if at some point in the future, some key/values need to be added, you'd need to edit the source and recompile it instead of just editting the text file.

Loading the text file when the program starts is probably the simplest approach but the above would work too :)
