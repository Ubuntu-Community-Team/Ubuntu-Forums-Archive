### Post by dribeas on 2008-08-23
> **techmarks said:**
> These become annoying after you use the same application often.
And a good progamme should provide a way to disable such contast pop-ups.


Confirmation messages should be asked only for operations that are not recoverable or the expense of recovery is two high.

If you ask to close an application that takes, say 5 seconds to start again, you should only be confronted with an advisory message if there is data not yet saved. If the user mistakenly pressed exit, he has lost just 5 seconds of his life. I work in a simulation system. Getting back to the same spot you are now may take hours: then it is fine asking to avoid accidentally closing the application.

The main reason for limiting the required user confirmations is not just user disgust, but the fact that if you are asked 100 times and 99 of them are useless, chances are you will just accept without even reading that one time that makes a difference.

   David
