### Post by smartbei on 2009-07-16
I recommend projecteuler.net - though many of the puzzles are math oriented, most can be solved without any background. (I was/am having a lot of fun on these even though my mathematical background is around high-school level).

Besides that, it seems like you should go ahead and start a project. Think of something, be it an application, a script or anything else, and try to program it. Then add features, make a gui, etc. If you need ideas, search google for programming project ideas and find something you like. Or here are some that I think would be fun/instructive to implement:

[list=1]
[*] A bulk image re-sizer. Make it take filenames from the command line, and the target directory and dimensions.
[*] Build a simple chat program - this is good idea if you want to learn about sockets, simple protocols.
[*] Build a collaborative paint program that works between computers. Add options for saving the images, undoing, drawing things other than lines, etc.
[/list]
Good luck!

### Post by smartbei on 2009-07-17
There is a problem with all of the challenges - they are all very defined, so while they are good practice, they do not help you learn to start your own project, design it, etc. The python challenge especially is really random in the things it wants you to do, since its goal is to teach you about some of the different modules that come with python.

that said, I think there is a percieved steep learning curve in the 'jump' between the challenges and the actual projects, one that doesn't really exist. Just start coding what you know, and pretty soon you will get a good idea of what you still need to learn. Then start from scratch :-).

### Post by lisati on 2009-07-17
A lot of programming exercises I've seen over the years don't seem to have much connection with the real world, and are aimed more at illustrating some programming technique. 

One of the more interesting example programs I've seen (in a Pascal text book) that might be good springboard for ideas is one which takes unformatted text and produces a version where the text is fully left- and right-justified. It uses minimal maths, and can get the student thinking about detecting line breaks and paragraph breaks.
