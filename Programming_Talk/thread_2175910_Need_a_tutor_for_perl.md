### Post by drmrgd on 2013-09-23
Beyond picking up Learning Perl as someone else suggested (which is a great suggestion as this is a great book for cutting your teeth on perl), I also recommend soliciting the help of the folks at Perl Monks - [www.perlmonks.org](www.perlmonks.org).  The folks over there are experts at perl and very quick to help.  Be sure to read carefully on how to post your questions as they can be a bit on the picky side and be sure to show that you've tried to work through the problem and are stuck on just a bit of it; they won't just write programs for you.  Also, don't overlook this site as there's quite a few knowledgeable perl folks here that can help you if you get stuck on a problem; I think many of them posted to this very thread :)

Ah - I miss being able to [tell someone to RTFM]("http://ubuntuforums.org/showthread.php?t=2169036").
Thinking back to my first C programming class, I remember being completely overwhelmed by .... everything.  Nothing can replace sitting next to someone who knows what they are doing and watching.  These days, much of that can be replaced by youtube, but 80% of the youtube videos by "experts" are really noobs showing the 1 trick they've learned.  OTOH, Randall Schwartz **is** on Youtube and DOES train in Perl - usually on a cruise ship. ;) So if anyone really wants to learn Perl these days, there are many different ways.
* youtube
* Reviewing other people's code - there must be many millions of Perl scripts on the internet. Average people would freak out if they knew how much perl glued together the internet.  Email, web servers, routers, some databases ... all have some perl running inside. 
* books - my google-fu found this [FREE book]("http://www.perl.org/books/beginning-perl/")
** there are a few best practices books - I've got the O'Reilly one.
** there are a few cookbooks with recipes - I've got the O'Reilly one.
* Perl Dancer (Ruby on Rails clone for Perl) - I've written a few apps with this. It is amazing what can be accomplished in 4 lines - did a 1 table DB editor myself.
* CPAN - extremely large, public repository, written by experts to solve almost any problem - just need to glue the parts together; plus all the code can be viewed, so nothing is hidden.  **Use the SOURCE, Luke!**
* CPAN code reviewing modules - [PerlCritic]("http://search.cpan.org/~thaljef/Perl-Critic-1.118/lib/Perl/Critic.pm") will review your code and make recommendations. Using this module is actually a *Best Practice* recommendation.
* Local PM groups
* PerlMonks
* Onsite / [Offsite training]("http://www.stonehenge.com/merlyn/")
* [YAPC conferences]("http://www.yapc.org/") around the world - the north American one usually happens in June.

Many perl guys are a little crotchety, but there are also some of the kindest people I know too, provided real effort is mad to learn. Ok, so I'm the only crotchet one and EVERYONE else is nice ....  ;)
