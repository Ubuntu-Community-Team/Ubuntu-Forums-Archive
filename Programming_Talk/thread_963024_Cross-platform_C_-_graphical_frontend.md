### Post by WW on 2008-10-29
The first error message tells you one problem:
```

i586-mingw32msvc-gcc: /home/james/.wine/drive_c/Python25/libs/libpython25.a: linker input file unused because linking not done

```
Instead of putting the libpython25.a file in CFLAGS, try putting it in LDFLAGS. So you'll have both CFLAGS and LDFLAGS defined when you run ./configure.

### Post by WW on 2008-10-29
I think you'll need to change the Makefile so that the libpython25.a file is listed after main.o in the command that does the linking.  Try this by hand:
```

$ i586-mingw32msvc-gcc  -g -o MDCpp_client.exe main.o  /home/james/.wine/drive_c/Python25/libs/libpython25.a 

```
I also removed the -I... option; that is not necessary for the linking phase.
