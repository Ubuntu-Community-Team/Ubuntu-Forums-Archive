> **jimi_hendrix said:**
> <OT again> what does that do? </OT>

It blows up the stack! GCC expects the stack to look like this when KernelMain is called:

```
- Return address
- Magic number
- Multiboot pointer
```

But when you jump to the code instead of call, there is no return address, so it gets them all out of order. The magic number becomes the return, the pointer becomes the magic, and whatever is below the bottom of the stack becomes the pointer (it was zero).

Fortunately KernelMain never returns, so it didn't get too much worse.

Have you ever tried [Snusp]("http://esolangs.org/wiki/SNUSP")?

Your little bit of code just reminded me about it. It makes programming more of a game, and is actually quite a lot of fun. Writing the good old 'hello world' has never been so much fun!

EDIT: Just found my version of 'hello world' in Snusp:

  ```

  /==\	hello     
  | //\/\/\/\ world
  | \+++++++/ /\/\/\/\/\
  | /+++++++\ \++/\--/++
  \+\\/\/\/\/ /++.=--\.+
    \\+++++++.\/\<\\\//.
$,===/\\/\.---/.\ | \/-\=\
     \+\+\/+++/>/./  \\-\|
      -\+//   \=/=.---/\-/
      \../ /\/\//\/\
          /---!-.#
          \----/
```
The words 'hello' and 'world' are just Snusp comments. The actual printing of 'hello world' is done by cycling through the ASCII character set using the '+' and '-', and printing is done via the '.'.
