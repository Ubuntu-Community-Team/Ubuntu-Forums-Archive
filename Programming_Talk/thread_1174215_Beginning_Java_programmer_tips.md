### Post by PaulM1985 on 2009-05-30
One of the things I have found quite useful is trying to look at some of the code people put on this forum and look at the problems they are having.

The good thing about Java is that all the pre defined Java classes are explained in the API and by looking at the problems other people have forces you into looking into classes that you may not have considered looking at.

However, it is also very much worth while paying alot of time in class because you may pick up bad habits by self learning that you may not have done learning inline with the rest of your class mates.

Also, think of simple, (I really mean simple) programs that you use and try to replicate them and try to organise your packages well. I fell into the trap of trying to write programs without using classes and doing everything in one very large file because that is how the book I bought was written.

I hope this has helped.  I find it quite difficult to be specific about "the best way" to learn programming,  I can only think of things that I have done wrong in my learning.

Paul

### Post by PaulM1985 on 2009-05-30
How about a GUI version of Towers of Hanoi.  That is quite a popular programming question I found in my Uni work.

Draw spindles and rectangles using the paint(Graphics g) on a JPanel ( so you can use g.drawLine(arguments) to draw the game, and use ActionEvents for clicking on a the first spindle to move the block from and then a click on the spindle to move the block to, with the program testing if the move is valid.

Just an idea. I have found from my studies that simple admin programs, like a Hospital administration tool has shown how important it is to separate functionality of a user interface from the functionality of the core system.  In this way you can make changes to your core system without making changes to your user interface package and visa versa.

Again, I hope these are of some use.  Just things I happened to think of when passing my computer at midnight (England). :-)

Paul

PS
If you havent heard of Towers of Hanoi, you should find it on google pretty easy.

PPS
Actually, if you are a beginner (and I don't want to sound negative as I don't know how much experience you have with JFrames, JPanels, buttons action listeners etc), creating that game may be a bit too much of a step.  It was just a suggestion though.  Your teacher would be a good person to ask.

### Post by opticyclic on 2009-05-30
Find something that you like and try to write a tool for it.
That way you will be interested in the project and it wont be theoretical.
e.g. I like poker, so started to write a tool to look at the stats of my pkr history.
This involved learning about Swing and database connections
If you (or anyone else) wants to get on board, it is at Sourceforge
[http://sourceforge.net/projects/handhistorytool](http://sourceforge.net/projects/handhistorytool)

This book is well worth a read to prevent you from making common errors
[http://books.google.com/books?id=ZZOiqZQIbRMC&printsec=frontcover&dq=effective+java+Joshua+Bloch&ei=odshSseDJ6P2yATN3N2PCA#PPP1,M1](http://books.google.com/books?id=ZZOiqZQIbRMC&printsec=frontcover&dq=effective+java+Joshua+Bloch&ei=odshSseDJ6P2yATN3N2PCA#PPP1,M1)
