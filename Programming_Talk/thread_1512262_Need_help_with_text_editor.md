### Post by PabloH on 2010-06-18
You need to try both and see which you like better. Regardless of which you choose, you should be able to do basic editing in either one. 

This is the subject of endless debates and flamewars. Most of which are good natured, and meant for the comedy value. 

Generally, programmers from east coast schools (MIT) will prefer emacs. System admins and UNIX folks from AT&T and west coast schools will prefer vi and its newer versions.

Usually emacs people open emacs and do everything in emacs. They live out of emacs and even run terminals inside it. Emacs is more of an operating environment rather than a simple text editor. Today we would think of it more as an IDE without all the mouse clicking.

Vi folks usually start and exit vi quickly and do many things in a terminal. Vi was originally meant to be a lean mean text editor. Since it was lean and mean, it was originally limited in features (or bloat). Modern versions of VIM come with plenty of plugins that can be loaded to extend its feature set. Tasks and modes that used to be outside its original domain are now easily performed.

Vi is a "mode" based editor. That means that you enter a mode to accomplish editing tasks. Emacs is "modeless" in the "vi" sense of the word mode. Instead, it requires chorded key combinations to perform editing tasks (an example of a chord is Cntr-X to cut in Windows). Vi avoids the use of chords by using its modes.

Be advised that emacs users need to pay attention to key mapping of chords and to the keyboards they use, or RSI can result. 

When you learn vi, learn the real vi keys-- not the vim remapping of keys that makes it easier for newbies. If you fail to learn the real keys and use the arrow keys, you will be slower and out of luck when you find yourself on an old Sun machine with real vi installed (not vim).

If your friend is really dead serious about an editor fetish, you might be wise to get some new friends with a wider, more mature worldview. It really does not matter what editor you use, so long as it gets the job done and is setup to meet your needs.

Along the same vein, I am very wary of working at any shop that tries to dictate all the tool choices of employees. Especially when they get into choosing IDE's or editors for their employee's. When someone tries to limit your choice of tools to do your job, that is usually a sign that you will be happier elsewhere.

### Post by MattBD on 2010-06-22
Vim is a great editor - I've used it for Python, C, HTML and JavaScript, among others. It's fast and efficient, and it's easy to use it for pretty much any language under the sun. It's definitely worth running through vimtutor and learning it a bit. Try using it exclusively for editing configuration files and you'll soon get the hang of most of it.

Also, once you're using it and have some idea what does what, take the time to add a few entries to your .vimrc file to customise it - these will improve your experience with Vim no end.
