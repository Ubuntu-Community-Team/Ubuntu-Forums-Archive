### Post by nanotube on 2008-04-11
Hey all,

We had a bit of a discussion on this thread:
[http://ubuntuforums.org/showthread.php?t=730773](http://ubuntuforums.org/showthread.php?t=730773)
about UF Programming Talk participants getting together as a team to work on a free software project. 

The first step on getting this rolling is, somewhat obviously, to have a good idea that we can get behind. So let us use this thread to throw some ideas around, and see if we can come up with a project we can rally behind.

Some things to note:

It doesn't have to be a new project - it can be an existing effort that is still at an early development stage.

It doesn't have to be a desktop app - it can be a server app, a CLI app, or a web app (using python and google apps! :) ), or anything else.

Low level things like driver development or kernel hacking would probably not have enough takers (but maybe I'm wrong?)

So go ahead, think about and post some possibilities - the more the better! Even if something seems a little vague, or a little wacky, or seems to present a difficult problem, throw it out there.

### Post by andyrue304 on 2008-04-30
Hiya.

I have an idea for a project that I'd love to see put into practice. I'm only a newbie web designer/developer so would have no idea how to actually do this but think it may be able to be done! N.B. If it has been done, I'd love to know!

So here's my idea (quite long):

Like a lot of web designers I started out by using Dreamweaver (this was pre-linux, obviously). However, I'm now of the opinion (as are a lot of people, I think) that WYSIWYG HTML, CSS etc editors are seriously flawed as they can get behind with current standards. At work, I'm forced to use MS FrontPage, which is a nightmare as it throws in lots of nastys like 5 <font> tags to display what should be done in CSS. Hence I've very often reverted to hand-coding and then checking in a browser.

Something I use quite a lot when I'm researching a new project is the Firefox Web Developer plug-in. This way I can find out "how things were done" and adapt it for use in my own sites. One of the things this does is check the page your viewing for CSS and HTML errors by utilising the Document-type Declaration of the page. You can check any page you view - vey handy.

So, I was wondering if it was possible to have a HTML, CSS Javascript, PHP editor, which has all the powers as any of the editors out there (I'm currently using bluefish) such as auto completion and syntax highlighting. BUT this editor was coded *as plug-in for firefox* and uses the firefox engine for displaying the code.

The benefits of this would be:

1. Always use current standards. Firefox's engine must keep track of the DTDs to display them. Surely then we could make it so the auto-completion is dynamically generaled directly from the DTDs, so no nasties creep in and the editor is constantly up to date. Or at lease the option to update is made easier as Firefox will update itself.

2. No need to build a WYSIWYG - it can use the firefox engine to display what you code by swapping tabs.

3. Built in support for any firefox plug-ins. Use, for example, the pre-coded web-developer tools and others to make our editor light and powerful.

4. Preview will always call the server-script in PHP so the editor will truly be WYSIWYG

And many more I'm sure.

So what do you guys think? Doable? Doable by us?
