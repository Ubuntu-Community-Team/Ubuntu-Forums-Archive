### Post by trent.josephsen on 2012-09-28
I played with Lua a bit for an embedded ARM application once. It seemed ok. Not much stood out to recommend it over Python, which is the only language I would really consider comparing it to. Simple, though -- I can see why it's used for embedded systems. Kind of like an HLL analog to C. I don't think I'd use it for anything that had to run on a PC, though.

As it turned out, the Lua modules I had didn't properly support the chip's SPI interface, so I didn't even finish the controller I was supposed to be programming -- had to switch to C. Or maybe it was just me. In any case, you're unlikely to run into that problem.
