### Post by Geekkit on 2007-09-30
> **Juco said:**
> yeah just read the title lol.  i would use eclipse or nb, but i want a fast running ide, my comp isnt super fast.  thanks!

There's always a trade off, and the trade off is:
[INDENT]
more features == larger memory footprint

less features == smaller memory footprint[/INDENT]

Yes, Java apps do tend to take up more space than native apps - any environment that requires a runtime will cause this to happen (i.e., .NET, Python, PHP, etc.)

If you have 1 GB of memory (or more) then you most likely shouldn't be worrying about this (at least if you're in Linux/Ubuntu).

For me, if I'm creating a quick code example or small "one off" file, I'll use Geany for Java, C, C++ or whatever I'm writing in. If I'm working on something that is more of a project and requires a larger amount of control and containment, I'll use Netbeans since it offers profiling, obfuscation, debugging, documentation, etc.

Hope this helps.
