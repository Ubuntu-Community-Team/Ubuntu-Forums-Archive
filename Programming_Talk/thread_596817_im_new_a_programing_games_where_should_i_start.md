### Post by dumbsnake on 2007-10-30
C++ is a great language to learn, but it is extremely challanging.  You have to be willing to spend a couple years with it, but you won't regret it.  Being as young as you are, it is the perfect time to start coding.

OpenGL is a wonderful library to learn, but you will need some math to get through it that you might not have taken yet.  I'm guessing you are taking something like algebra or prealgebra right now.  To understand OpenGL, you need to understand matrices which you generally don't learn until late in algebra or in geometry or even later.  You may need to pick up a book to help you with the math, but there is no better way to learn.

My only advice other than be patient with yourself, is that you should try and not be too ambitious at first.  Start with the simplest project you can think of.  Then add to it.  I know we all want to create the next greatest thing, but you have so much time to do that.  Don't reach too far and burn yourself out at first.  Goodluck man, PM me if you need any help.

### Post by Wybiral on 2007-11-01
Write your own games: Python
Write professional games: Python THEN C++

Lower level languages like C are rarely used in professional game development (recently) since generic / OO features are practically a must (so languages like C++ and Python are way more common these days).

As a fair warning, if you're planning on larger games like first person shooters / rpgs then a game engine is a MUST. Naturally you can write your own, but with the amount of effort you will spend on that, you will never get to your game. Simply having OpenGL bindings does not qualify a language for game development. Python is especially nice in this area since almost all quality game engines have a Python module out.

Even if you don't plan on larger games like that, use Python. It's great for simple 2d sprite / board games / text games. No need for anything lower level than that. I wouldn't resort to C or C++ until I had already written most of the game in Python and found that I needed a few small chunks of code as a module.
