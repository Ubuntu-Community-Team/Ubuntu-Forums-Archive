I have been developing a computer program's initial stages of design, concept, and function.

The program concept is a "multi-functioning-interactive-shortcut-custom-mapping and navigation-tool". For lack of a better description.

As to my knowledge very few programs of this type exist, which is why I do not know a proper title for it.

Let me share a little about it.

The idea here is that the program's defualt installation is a very simple form containing the following:

The entire object is what I call at this point a Nerve Cell. Nerve because it adapts and evolves to requirements and needs, and cell because it is one unit.

1)**Central hub**: The central hub is the programs control center. It allows the user to add extra cells to cycle through, and other desired options and features.

2)**Main Arms**: The arms adjustable in every way. They can be added or removed, and dragged to custom positions and lengths. The end of the arm has a button, This button has options to work as a link, to perform tasks. The user can add an arm, and choose where he wants it to link. Lets use a folder as an example. The first arm a user adds is linked to "my documents". Once completed, text overlays the button to refer to what it is (my documents folder). The user can rename the button, choose a shape or object for it (provided in the programs textures) aswell as a specific color. 

3) **Branching arms**: These arms are added onto Main arms. The user could add an army for "my music folder", another arm for resume they are working on. etc. Aswell as cusomize those in the way they want.


**The purpose of the program is this:**

The program allows a user to construct a map in a way that best suits them. It allows them to navigate this map(their computer) with sense of direction, color, and shape. (not to mention other possibilities). The user can add an almost unlimited amount of main arms, and branched arms that can toggle in and out of view. On these arms the user can link websites and pages within those sites, Computers on netowrks, Programs, Games, Folders, Songs, Files, Locations, Tasks. Furthermore the user can use the tool to organize directories. Let's say the user desires to organize their desktop. They load the directory of the desktop with the program, and it expands one arm for every file The user can then create Main arm folders, then drag files onto that arm to transfer into that folder, in this sense it does the same action as an operating system provides however, it converts the way the user interacts. (Although there would be a limit to how many items from a directory the program could expand into this visual form, but do keep in mind, the branching arms could be toggled to retract into the button of choice, so that the only branches that are exposed are the ones of choice, and only as expanded as needed.)

The programs further purpose is to save people time and provide a new way to operate your computer using more intuitive functions of the human mind.

I've considered more features to add, such as the ability to simply roll the mouse, to cycle through certain things.. But I will leave that for another time.

The program is intended to have an adjustable overall size. Such that the user could reduce it to a small item in the corner of the screen, or use it in a way that entirely overlays the desktop in a full screen fashion. Again, custom suited for the users needs.


I am an experienced inventor and have a broad knowledge of things, but I lack a lot of programming skills in this department. I really have no clue where to begin.

### Post by arkain101 on 2009-05-10
It is based around extensive research into one of the most effective methods for the human mind to organize data. 

A user interface such as windows xp for example, provides some ability to link and navigate data on the computer, but it becomes limited and defective when the number of links grows. This is when data becomes grid formed, and it is difficult for the user to find a specific thing. For example, 50 shortcuts on your desktop, or 30 items on the start menu's quick launch.

It is not intened to replace the interface of a OS like windows xp, it is intended to provide a user to in a sense design their own interface for their most comman tasks, and in such a way that mouse movement is minimal (quick mapping), and a person can remember up to hundreds of locations with minimal practice. Items develop sense of direction, color, shape, and relationship. Where as with a operating system like xp, it is a mentally struggling task to work with even a dozen locations simultaniously, all the windows look identical, and so do many files, and with certain usages, this grid method of working with data can force people to simplify their tasks (slow down).
