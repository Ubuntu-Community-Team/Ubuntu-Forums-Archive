### Post by dwhitney67 on 2008-09-29
Rather than try to understand your code, I will attempt to explain what strcat() does.

strcat() takes two parameters, one a char *, the other a (const) char *.  You probably already knew that.  It appends the second char * to the first.

strcat() assumes that both of your char * are null-terminated.  Thus, to illustrate:
[php]
#include <string.h>
#include <stdio.h>

int main()
{
  char statement[12] = {0};   // null-terminated string

  strcat( statement, "hello" );   // append "hello"
  strcat( statement, " " );       // append a blank-space
  strcat( statement, "world" );   // append "world"

  printf( "statement = %s\n", statement );

  return 0;
}
[/php]
Until you understand strcat() fully, try to simplify your life and skip the malloc() for memory.  Just use the stack (as shown above).

P.S.  You can determine the end of a null-terminated string using strlen().  This will give you the length of the string, minus the space taken by the null.  Thus strlen("hello") would yield 5, although 6 bytes are required to store the string.

### Post by Andruk Tatum on 2008-10-01
I found the issue, I'm posting it here so anybody else knows what not to do.  As it turns out, it is in none of the code I posted.

The problem is in this code, used to allocate the memory for a string:

[PHP]
if((temp = (char*) malloc(50*sizeof(char))) == NULL) {
	printf("Error allocating memory.\n");
} else {
	temp = "BLAG";
}
[/PHP]

where temp is later changed.  The problem arises because temp appears to be allocated to the heap when in fact it is not.

The line 
```
if((temp = (char*) malloc(50*sizeof(char))) == NULL) {
```
allocates enough memory for temp, but the line 
```
temp = "BLAG";
```
**reassigns** the pointer to the constant string "BLAG".

Then, later on, when temp is concatenated with something, the program segfaults because it is trying to add onto something that should be constant.
