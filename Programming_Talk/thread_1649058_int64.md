### Post by worksofcraft on 2010-12-19
The mantissa of a double precision floating point number has 53 bits of precision in C++ and I need to be able to manipulate it as an integer quantity. It works fine with type "long" on a 64 bit installation but falls over on 32 bit.

in C99 they introduce a type named "long long" which is atleast 64 bits long and would serve my purpose, but it is not in the C++ standard... so loads of warning messages.

in C++0x they will define a type int64 and I'm already using this standard for variadic templates, however even with the -std=c++0x compile time switch it still currently rejects int64.

... so while I would like my C++ software to be portable to 32 bit systems I wondering what be a generic way to implement the 64 bit integer arithmetic?

### Post by MadCow108 on 2010-12-19
g++ can use the C stdint.h with no problem (that is stdint**.h** not cstdint!)
other compilers should be able to do so too.

If not you'll need a bunch of conditional typedefs for the offending compiler (or a ready made header done by the many many others which have this problem e.g: [http://www.azillionmonkeys.com/qed/pstdint.h](http://www.azillionmonkeys.com/qed/pstdint.h))
