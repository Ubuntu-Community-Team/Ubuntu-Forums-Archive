### Post by amiga_os on 2006-11-06
I'm learning Python.  I don't really know how to get started contributing.  I've written some little things, but to start contributing seems a daunting task.  Lot's of people seem to start contributing to FOSS by starting their own project - that's really easy, and there's no "cost" if it goes wrong.

I wondered how everyone else would balance the my need to start with something simple and attainable with the community need for people to get on board with projects already running, rather than spawning another dead source-forge project or keen noob ubuntu personal web-space with yet another lagno clone (or whatever).

How should a would-be contributor best start getting involved?

Troll the forums and see what problems people are having.  If it's something that can be solved programatically, write a program.  If it's something that you had a personal issue with and solved with a program, plug it here and maybe someone else will find it useful.

### Post by _asc_ on 2006-11-07
You could look for projects that support plugins. Then you could extend the main application's functionality with your own plugin.

Since the main application does not depend on your plugin it would have the benefits you described for starting your own project but with limited scale.
