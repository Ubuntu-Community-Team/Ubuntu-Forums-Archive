Let me emphasize this - not to discourage you of course :) In the beginning, all imperative programming language share the same constructs so you've not yet reached a point where you can view the language and judge it fully. C++ can be learned of course and challenges can be fun but they can equally throw people off when they're new in programming land.

Let's take memory management: you need to do this yourself in C++ but in Java this is taken care of by the garbage collector. The difficulty comes when you need to track down a memory leak which is caused by who knows what.. pointer problems, a missing delete statement, etc. It may be better to learn programming in a language like Java ( how to program something, object-oriënted design etc. ) and then move to C++ where you'll need to take care of additional things.

### Post by Wybiral on 2009-02-22
> **Leo Dragonheart said:**
> Everyone keeps saying how hard C++ is what am I missing There has been nothing hard about it so far except for my Idiotic questions about why would you do this and not this instead.

I've been passively observing you learning C++, it's slow going. And you haven't even gotten to the really interesting bits like template metaprogramming (all I've seen you post looks like C with classes).

It's not to say that C++ can't be your first language (you CAN learn it, to some extent, you'll probably have to relearn it later to figure out some of the idioms once you understand programming more), the problem is that all the extra time you're wasting on those details could be spent learning more important concepts, like knowing the difference between an O(n^2) algorithm and an O(lg n) one, learning common algorithms in-general (so you're not constantly reinventing things), and most-of-all... Seeing the real problem you're trying to solve, NOT the extra problem(s) that your language is forcing you to solve (this really doesn't become clear until you've learned a few, but starting with such a complex / broken language is going to make it worse).

### Post by CptPicard on 2009-02-22
> **Leo Dragonheart said:**
> 
1. I like a challenge! The more I challenge my mind the stronger it becomes.

There are conceptual challenges and then just ... challenges from "complexity". In the beginning in particular there more efficient ways to challenge your mind regarding programming, see Wybiral's post.

> 
2. Once I learn C++ I figure I will have learned about 5 or more other languages at the same time. If I can conquer C++ from the start then I should breeze though other languages.

Yes and no. At least if you want your view of programming to have a broad base, you need to understand C++ helps you mostly with imperative static-typed object-oriented languages. Which covers a lot of languages sure, but they are rather similar ones. And that "class" of languages is better learned not in C++ -- you can always learn "how to do it in C++" much faster once you have a strong grasp of what you want to be doing in the first place, and when you have a better idea of the concepts those C++ books you'll be reading are talking about, so you can pick up the technical details.

Also, then there are other paradigms altogether to programming, that yield interesting approaches to problems. The imperative mindset is something you need to partially unlearn there :)

> Like I have said before I have been coding for 5 days now and I am getting along in C++ ok. ( I am not a expert but I am learning ).

In 5 days, you have not formed an adequate opinion unless you're remarkably gifted. ;)

Read this: [http://www.norvig.com/21-days.html](http://www.norvig.com/21-days.html)


> **jimi_hendrix said:**
> ive never used them but arnt there auto_pointers or something for this in C++?

Sure, but that's yet another interesting thing about C++ -- in order to solve a language problem, a "tool" has been invented that pretty much does help solve the problem (C++ is full of "idioms" you just have to know) -- but in order to understand the tool, you still have to understand a lot of the whys and hows and what to do with it. You really need to know C++ first in order to understand and appreciate the auto-pointer.

hehe trying to learn assembly as your first language may seem like a good choice intuitively since it is the most basic, however it is something like trying to learn skiing by taking the double black diamond. 

The fact is most textbooks on assembly require that you know the high-level language of the textbook's choice (usually c or c++.)

I don't know what to recommend to you as your first language, because the first language you learn will seriously bias you in the future. Instead I would like to tell you that you shouldn't learn any particular language until you need it. It should be something of a need to know basis. I think you are off far better studying the theory, then using any programming language to implement any theory you choose to. That will give you lots of motivation to keep learning and also, it will make the experience of learning more profound.
