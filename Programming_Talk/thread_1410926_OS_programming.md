### Post by CptPicard on 2010-02-19
I would suggest that your class is right -- it is better, easier and faster to understand concepts as concepts and not as tangled in code in some specific implementation, in particular in OSes where this stuff can be messy. I tend to believe that even a data structures class doesn't need programming, necessarily... it is a drag on student time that would be better spent comprehending the ideas in the book.

But, you might want to check out something like the source code to Minix... that's how Linus started, I'm not sure if there is something more modern.
