### Post by lomomo on 2006-09-12
the gtkmozembed widget in ruby-gnome is compiled against mozilla in ubuntu. So when someone want's to use a ruby application with gtkmozembed needs to install mozilla too.

As mozilla is not the default browser (and it's not instaled) in ubuntu, shouldn't be better to have the gtkmozembed compiled against firefox instead?

I'm developing a ruby and gtk application with gtkmozembed ([http://www.gnomefiles.org/app.php/Tagore](http://www.gnomefiles.org/app.php/Tagore)) and i think it's a waste of disk space having to install mozilla as i'm not going to use it. Also, gtkmozembed uses the default mozilla theme, which does not integrate well with gnome applications nad looks really ugly!
