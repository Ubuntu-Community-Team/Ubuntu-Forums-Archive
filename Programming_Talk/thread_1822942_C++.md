### Post by shashanksingh on 2011-08-11
Ive done most of my coding in Java and recently doing C++.

I wanted to know why can't I do something like this:

```

class node
{
    int a;
    node next;
};

```

But I can do something like this:
```

class node
{
    int a;
    node * next;
};

```

I used the above part in java to create datastructures.

### Post by Npl on 2011-08-11
node is not a fully defined type when you declare "next". means its size is not know (amongst other things).
now say node has size k. the definition of node says.
sizeof (node) = sizeof(int) + sizeof(node). in other words (assume sizeof(int)= 4):
k = 4 + k. try to find the correct (finite) value for k!
(works with k = infinity)

contrary, sizeof(node *) is just the size of a pointer (4 on 32bit), hence you can calculate the size of the definition.

with java you always have references on Objects (practically special pointers) so it works

### Post by karlson on 2011-08-11
> **shashanksingh said:**
> Ive done most of my coding in Java and recently doing C++.

I wanted to know why can't I do something like this:

```

class node
{
    int a;
    node next;
};

```


The code above means that you are storing a copy of the next node within the current node, which means that you have an infinite recursion in the definition for one and 2 the definition of the class is incomplete so the compiler doesn't know how much storage to allot for the object of the class.

> 
But I can do something like this:
```

class node
{
    int a;
    node * next;
};

```

I used the above part in java to create datastructures.

Here the storage is known as the pointer is always the size of address on the operating system and the only thing that compiler would be required to do is to check the type of the pointer on assignment.
