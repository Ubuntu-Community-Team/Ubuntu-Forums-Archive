[/SIZE]Java apps fall basically in two categories.

1) Web server apps. Here Java programming skills are useful but not essential. But you'll have to get acquainted with a raft of middlewares (Web application servers such as JBoss/WebLogic/WebSphere, and various other stuff often starting with "J": JSP, JFACES, JDBC, Hibernate, Spring. Very boring stuff from a programming standpoint. It's the New Frontier for Cobol programmers (I'm not going to make many friends with that remark :). An eye opening book even shows how the "Enterprise Java Beans" inheirts from the mainframe CICS environment. 

2) Mobile phone apps (Androïd, Symbian <*sigh*>, Blackberry....)

Choose wisely.

This is a far bigger obstacle for more people than any technical hurdle.  Defining a "problem" that you can solve productively using your tool of choice isn't easy -- most programmers have problems assigned to them.

What I have done is to use it to pursue my personal interests.  I wanted to see how far I could take Javascript, and I'm interested in astronomy.  So I wrote a program to simulate gravitational interactions of multiple bodies, where the user could give them some starting criteria.

I am working on learning Python and a GUI, so I'm developing a simulation of biological evolution using Python and WxPython and SQLite3.

I have a Django project in the que that I'll start when this is done, more closely related to my actual profession.

None of it has any immediate commercial value, but all of it is challenging, and should help me accomplish the kind of tasks I might be expected to take on in some nameless future, and I'm having fun at the same time.
