### Post by ThinkBuntu on 2009-06-03
Hm. Java for web development can be an excercise in masochism...if you want to have access to Java libraries, I think that you'd be better off using one of the many JVM-implemented languages, like Clojure, Jython, JRuby, or (my favorite) Rhino for JavaScript.

If you just want to make simple sites, I recommend PHP. People will deride it for its many internal inconsistencies, but at the end of the day, Apache's mod_php is great, and PHP's "everything's a template" style facilitates small ad-hoc websites.

Of course, you can't go wrong with Python. While I think that mod_python can be a little tricky for a beginner, Django, web.py, and other Python frameworks include simple development servers which would get you started.

### Post by PC-XT on 2009-06-05
I got some PHP books from the library, but some were for version 4 and some for 5. I now primarily use [www.php.net](www.php.net), but sometimes also use phpbuilder, w3cschools or about.com. The sites stay on top of the newer developements while books are just static, but the book authors generally put more work into being understandable and helpful to more people. Books will often say "Covers PHP 5" on the cover.

As far as dynamic client-run games, I've used JavaScript, DHTML, Java, a little Flash, and combos of these along with PHP. I prefer Java over Flash, because it is used in other areas of programming as well, so I can share classes. I prefer JavaScript, CSS, and other DHTML when I don't want to have to compile and jar the game to get it working (such as when I want it to be a single file.) Flash can be generated using PHP. Any client code files are readable by the user, if they know how, which is why people try to obfuscate their code. I like to use PHP for any sensitive code, and use the clients primarily just to aide display. You can't always count on how client code runs, depending on the machine, but PHP runs on the same computer you put it on. I use regular HTML as a base, and then add the other stuff.
