### Post by Some Penguin on 2010-01-26
Yup.  If you want to do something in particular, then the choice of language might make a big difference in difficulty.  It's probably possible to write a first-person shooter in Lisp, to use Fortran for generating reports involving fancy regular expressions, or use C for serious servlet-style programming, but these aren't  exactly normal choices to make.  Remember that even though many languages may have similar functionality (in theory), such capabilities may vary in ease of use, and availability of libraries will definitely vary (e.g. as far as I know, there are no GTK bindings for Fortran or Lisp).  So if you don't like reinventing wheels, look around for what code already exists...

Oh, and if you want to someday *work* in the field, you should probably focus on languages that are actually used heavily, rather than those that are merely elegant.  If a place is mostly a Java or C++ shop, knowing Haskell or Standard ML is probably not going to make up for a lack of competence at  Java or C++.
