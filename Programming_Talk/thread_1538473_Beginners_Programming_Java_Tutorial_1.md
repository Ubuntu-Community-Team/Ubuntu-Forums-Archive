### Post by lykeion on 2010-07-25
Here are some suggestions after reading through your first pdf:

1. If you introduce an IDE so early in the tutorial, why don't you also show the benefits of using an IDE, like templates:
'main' + Ctrl+Space
'sys' + Ctrl+Space

2. I guess there are billions of Java tutorials with the following steps: intro > variables > loops > classes & methods > etc
My suggestion is that you have a complete application as the goal of the tutorial, 
and while showing how to write that you introduce the steps one by one. 
This way the reader will be more motivated, instead of YATJT (yet-another-theoretical-java-tutorial) 
the reader get to learn how to write perhaps a cool game in Java and in the process learn the basics of the language.
