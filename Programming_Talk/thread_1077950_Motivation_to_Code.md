### Post by curvedinfinity on 2009-02-22
Basically, the way I have it, coding is problem solving. Problem solving requires creativity. Creativity is directly linked to desire. Desire is based on emotions. Thus, if you aren't happy, you aren't creative, and so on.

I like to think that it is important that you do not associate "work" as "something not fun". The best way I've found to do this is to condition new emotional patterns...
[list]
[*]Schedule some work. "Tomorrow I'm going to code." 
[*]Start your coding when you decided you would. Make sure you are actively solving the problem, and not just blanking out because you don't want to do work right now.
[*]As soon as you become bored (even after 15 minutes), go do the most rewarding experience you can, because you're free to do anything in the world that you want to do now. "I'm done coding, so let me go for a walk in the park."
[*]Every day, start working when you should, and then stop working when you want to, doing something enjoyable.
[*]If you really have to get something done, force yourself to as normal, but any time you're not on a tight deadline, revert to the above.
[/list]

The key is that you load your brain up with the current problem you're solving BEFORE you decide you are done. That loading time is the part that is the hardest to get over (not the actual work that you perform once the problem is all loaded in your brain), so the key is rewarding yourself for doing that.

I've suggested this technique to employees of mine, and it has worked well. The down side is that the conditioning can take a LONG time, like 6 months at the least, and during the first stages, you will be far less productive.

### Post by Greyed on 2009-02-24
Yes, but you cannot get the same benefits from the control system.  This is for two reasons:

1: They are technically binary data as their internal structure is a gzipped directory.
2: OOo XML documents are a single string.  IE, they don't break each element onto its own separate line.

The benefits I was referring two are the ability to cheaply branch **and merge** different lines of code.  The merging part is done by text-based diffs which work on a per-line basis.  Since OOo documents are neither textual by default and when you arm-wrestle it to be textual it is a single line it is impossible to take advantage of the merge process.  Because of this the branch operation is meaningless.
