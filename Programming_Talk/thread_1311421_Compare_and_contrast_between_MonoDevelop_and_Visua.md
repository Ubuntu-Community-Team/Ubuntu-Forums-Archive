Well, at least you are using C#, which Mono is stronger with. Overall, though, I find Monodevelop to be clunkier than Visual Studio.

Part of that, though, is that I'm using VB.NET (actually ASP.NET) here at work and am more experienced with Visual Studio.

One thing you could do is run both Visual Studio and Monodevelop on Windows and toss back-and-forth between them until you determine which one you are more comfortable with, or if there is anything the class demands that Monodevelop cannot handle.
