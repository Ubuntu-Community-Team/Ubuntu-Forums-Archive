### Post by pmasiar on 2008-09-27
> **pp. said:**
> There are some dialects of Pascal which are still being used, and widely used at that. 

Even if there are nowadays not many occasions to really use Pascal, learning a very compact and well designed language can be something of an eye-opener for a hobbyist programmer. For a professional programmer as well, BTW.

IMHO, for hobbyist looking for a simple, compact and well designed language, Python would be much better choice, and one of the reasons is dynamic typing, which makes learning language much simpler.

Pascal might have sense for someone who (after learning Python) wants to learn compact statically typed language, and for some reason C is not an option.

Old languages are old and obsolete for a reason. Ie Object Programming paradigm became popular because it is excellent way to solve problems with assigning meaningfull names to functions in huge codebase, and simplify reusability of the code. Pascal was designed before OOP, and consequently forces obsolete approach to this problem.

If you want to learn old languages, better choice would be old languages what are still excellent solutions in the area they were designed for: Lisp, Forth, Prolog. Learning old language whose usage is in decline IMHO makes little sense, especially for a hobbyist, unless you want to  participate in a project using that language.

That said, Pascal was a decent and popular language back in old days, and if you know Python, learning it would be simple, and quite possibly simpler than other statically typed languages like Java or C.
