### Post by ziekfiguur on 2011-06-05

Not really destroy, to do that you would have to write some specialized software. It will overwrite the complete drive with random data, so you won't be able to boot anymore. If you really want the drive to stop functioning, you would have to do something that makes the reading arm move back and forth a lot, very fast and make the drives spin and brake very fast, untill the hardware wears out.

Yes, check out coreboot. Ubuntu has a package grub-coreboot. I have never tried anything like this, but i think you could just corrupt the binary file that gets uploaded to your bios to break it. However, this will only destroy the motherboard, not the harddrive.

To do the harddrive breaking you would probably have to do some very low level assembly coding to access the hardware directly, instead of using kernel functions.
