### Post by Zugzwang on 2010-03-24
The question that needs to be raised here is: from *what* do you want to generate ready-made code? For example, you cannot expect to input some natural-language description of your problem and get some Java program that does what you want.

If you want to start with some graphical representation of some GUI, there are GUI designers which generate some code for building such a GUI for you. I only know the Java Netbeans designer, which I like quite much. Note that you will of course still add code for the "events", e.g., what happens when the user presses a button.

The are also code generators which generate Java classes from DTD-files which describe the structure of XML documents. This way, you can easily read and write such XML documents in your program.

For database queries, AFAIK, there are also programs in which you can drag-and-drop some queries in a visual way, and then you'll get a SQL command for performing this query.

Then there is ANTLR, which generates lexers/parsers from formal language descriptions.

A comprehensive such list of code generators would probably go on forever. So what precisely are you after?
