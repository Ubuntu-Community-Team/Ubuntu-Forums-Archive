### Post by r-senior on 2012-04-07
I think any aspiring Java programmer should start with simple, but not trivial, programs using a text editor and the command line. When I say simple, but not trivial, I mean programs that use multiple classes in different packages. Also programs that create JAR files and make use of library JAR files. It's good to play with Maven in this scenario too.

Once you understand how the classpath works and how JAR files interact with it, switch to a proper IDE. Not doing so will just hold you back in terms of ease of refactoring and adopting good programming practices. Practices like using beans with getters and setters, programming to interfaces, creating unit test classes (or test-driven developent) with JUnit, using frameworks, etc. These things can be done with a dumb editor but there's a lot of repetitive typing that doesn't teach you anything.
