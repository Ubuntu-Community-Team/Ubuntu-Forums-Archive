The reason HTML and assembly aren't there is because they excluded them. From the FAQ:
> Q: [I]What definition of programming languages has been used?
[/I]
A: A language is considered a programming language if it is Turing complete. **As a  consequence, HTML and XML are not considered programming languages.** This also holds for  data query language SQL. SQL is not a programming language because it is, for instance,  impossible to write an infinite loop in it. On the other hand, SQL extensions PL/SQL and  Transact-SQL are programming languages. ASP and ASP.NET are also not programming languages  because they make use of other languages such as JavaScript and VBScript or .NET compatible  languages. The same is true for frameworks such as Ruby on Rails, ColdFusion, Cocoa, and technologies such as AJAX. **Finally,  we have also excluded assembly languages, although Turing complete, because they have a  very different nature.**

> Anyway, I would never give too much credit to any of these rankings... At the end, what is popularity important for in programming?

Popularity is IMO a two-edged issue. On one hand the entire idea of programming language development and our discussions here about what makes a language different from another one relies on the  assumption that there is something objective in programming languages that "most people" would find desirable. Thus one would assume that popular languages are also "good" languages.

Then there is the annoying fact that unfortunately most coders are average. Thus they will gravitate to average tools which allow them to be productive in a very-average corporate environment. This is why Java and C# are popular. On the other hand the sharpest and most deeply insightful programmers I know tend to have off the beaten path preferences about things...

### Post by nvteighen on 2009-09-13
> **CptPicard said:**
> 
Popularity is IMO a two-edged issue. On one hand the entire idea of programming language development and our discussions here about what makes a language different from another one relies on the  assumption that there is something objective in programming languages that "most people" would find desirable. But the problem is that "popularity" is a pretty vague/vacuous concept that seems to mean "what people prefer most"...Ok, in which situation? For what? Why? Etc. Specially in such a topic like programming, where technical issues are discussed, "popularity" becomes even more irrelevant.
