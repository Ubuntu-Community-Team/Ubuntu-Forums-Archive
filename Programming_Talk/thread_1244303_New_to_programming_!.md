### Post by jacksaff on 2009-08-19
If you're a complete beginner you really want to do something where you can get visible results as fast as possible.

I would start by making a web page in html, clean it up with css and then add some fanciness to it using php.

Then move on to Python and a library like pygame so you can keep yourself amused while you learn. You can make gui programs without really having to know much about programming and you'll learn at your own pace as you start to try do things beyond your current ability.

Starting with C or C++ is likely to be much more frustrating.

Whatever you choose there is likely to be a lot of tutorials online. Buy a book when you know what you want to do and need some specific skills.

There are a bunch of posts on the  same question in this forum so you might also want to search for some other opinions.

### Post by CptPicard on 2009-08-19
> **Drone022 said:**
>  The higher languages go, the less I feel like a programmer for some reason, if I want code that plays a video Id like to see how it all works and work at a pretty low level.

Trust me, it's just a feeling that will pass. ;) There is a period in a programmer's evolution when one wants to dig under the hood (and one should), but later on ... it just becomes drudgery. I would claim that when it comes to the interesting issues of the problem solution itself, higher-level languages actually show you how it all works in a much clearer fashion.

Then there is of course the really interesting thing about HLLs that they really are not reducible to just your function call example... they are not just a wrapper layer that hides stuff. They have language constructs that can be used to produce things that are interesting in their own right and whose creation would be hard in languages that, say, do not support higher-order functions or closures, my two favourite things missing in a typical low-level language.

### Post by TheStatsMan on 2009-08-19
> **CptPicard said:**
>  I would claim that when it comes to the interesting issues of the problem solution itself, higher-level languages actually show you how it all works in a much clearer fashion.


I don't think this is always true. In some of the work I have done with filtering algorithms, I have gained a lot of insight into efficient implementations (in a mathematical sense, i.e. in terms of the computational complexity of the problem) from implementing them in lower level languages. In fact, I would say it has enabled me to solve problems that I otherwise would not have feasibly been able to solve (as a result of the insight I have gained in to the mathematical implementation). Higher level languages are nice in that they bring you to a level of abstraction from the problem that we perhaps operate at in a mathematical sense, but they don't always show you how things work more clearly. One can often come up with a mathematical solution to a problem that would be totally infeasible to implement in practice, so looking at the problem from another viewpoint can be very insightful.  That said I often also use higher level languages to help understand the mechanics of certain algorithms and certain problems and I do understand where you are coming from, but as a whole I would have to disagree.  I am sure this does have a lot to do with the kind of problems one in working on.

Edit: to answer the original question I think I have to agree python is a very good choice, c is an average choice,  c++ is possibly not such a good choice and this is coming from someone that unlike many people on here likes c++.
