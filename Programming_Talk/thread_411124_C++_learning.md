### Post by Wybiral on 2007-04-16
Personally I don't think it's a great idea to start with C++... I learned BASIC first and then C++, and it only took a couple of months since I already had a decent understanding of programming.

You'll get a lot of different responses with a question like this...

There are three approaches really:

1. Start lower, like assembly or C
2. Start higher, like python or ruby
3. Start at C++ and try to dive right in

Personally, I would advise 1 or 2... 3 will be hard to keep up with if you don't have an understanding of 1 or 2.

### Post by Wybiral on 2007-04-16
I think starting with something like python is great, you can dive in a do something right away...

But don't ignore the lowlevel stuff, it's important to understand how your programs work.

I think a good strategy would be to start with python, once you're comfortable with it, try some C. At least learn enough C to write modules for your python programs.

Then if you're comfortable with C, think about learning some assembly, if nothing just for the learning experience of having to understand how your programs work.

After those steps you will either realize that you don't need C++ to do the things you want to do, or you will be comfortable enough with programming to dive right into C++.

C++ is a strange land to just drop yourself off in, it's best to understand the things around it first. Mainly program design, handling data, which you can learn from C... And then you can learn OOP from python.

### Post by Wybiral on 2007-04-17
> **amo-ej1 said:**
> learning C or asm first will only confuse you since the entire object orientation paradigme will be strange to you.

The OOP paradigm is strange to everyone when they start... And C++ doesn't really have that strict of an OOP system, not is it really required.

If you learn assembly, you can see C's structures as chunks of memory in the stack or heap.
Then if you learn C, you can see C++'s classes as extended structures with methods.

I don't think it will confuse you at all if you learn what is really going on from the ground up... In fact, you will have a better understanding of the higher level features because you will understand how they work instead of just accepting that they do work.
