### Post by .Maleficus. on 2009-08-02
I've just got some questions for now, to get an idea of what you're trying to accomplish.

1.  You want this to run in a browser (web browser I assume).  A web browser needs a way to run, presumably on an operating system installed locally.  If I need an OS to launch the browser, why would I open *another* OS to do the same tasks I could do on the host OS (and with more performance)?  Just seems very redundant.

2.  #1 aside, for an OS to be useful it needs applications to run, or it wouldn't do a whole lot.  Are you suggesting we (or your team I guess) rewrite preinstalled Ubuntu applications for your OS?  If not, what exactly are you trying to have this OS do?

3.  You'll find that most F/OSS fanatics are very against Flash (for its proprietary-ness).  What would motivate a potential user to install a plugin for a service they don't like?  Would the open source version of Flash work?  I personally have nothing against Flash, but as I said, many people do.

Also, that's a cool link you provided, thanks :).
