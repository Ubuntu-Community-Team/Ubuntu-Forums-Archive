### Post by Mr. Picklesworth on 2007-10-27
Occasionally, I learn about a new stable release of a multi-purpose IDE that works nicely in GNOME and decide to try it out. However, without fail, these IDEs either deliver little of use or are prone to random crashes. It's trouble enough having *my own* program crashing, but when the IDE does it as well, it's rather annoying.
Something about C/C++ IDEs in GNOME makes it all seem a lost cause. I have yet to find one that really does it for me, and I always go back to Scribes after a few minutes of torture.

The exception to this, for me, has been MonoDevelop. Aside from the GTK# layout tool, it has been relatively crash-free (granted, I don't go to as significant depths in it as others do) and actually gives constant jusitification for its existence. (For example, there is a tool in the context menu to rename a variable across the project). This IDE is still young, so that it seems to work so much more smoothly than its peers is stellar.
Its only drawback has been that it does C# and only C#. I am not a C# person. It seems that the power of that IDE has been noticed, though, as the 0.16 beta has support for C/C++ projects!

I'm looking forward to this IDE evolving into something very widely useful. I already like it, and got into it, way more easily than the other ones. Instead of being a glorified syntax highlighter, MonoDevelop seems to be very aimed towards streamlining project management.
