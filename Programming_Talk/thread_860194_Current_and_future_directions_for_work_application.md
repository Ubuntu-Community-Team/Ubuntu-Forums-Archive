### Post by magicplayr2000 on 2008-07-15
I got thinking today about what direction applications seem to be heading earlier, and what effect it may have on operating system use.  In the past, applications have tended to be run on local machines, which meant that they were dependent on the operating system's API, which is part of the reason that there is a lack of ability to cross over between operating systems easily.    The exception to this was mainframe computing, which seems to have slipped by the wayside in favor of personal computers.  My thought is, with the widespread availability of high-speed internet connections, will we be returning to more mainframe-style computing in the future?  By this, I mean that apps will be more and more focused to web applications, and less to local applications for each user.

What effect, if any, will this have on operating system use?  Once applications are moved to web-servers, they can be essentially used by any OS.  Will there be a spike in Linux users once the tie-downs to the operating system are broken away?  

What effect will this have on the way we use languages or what languages will be commonly used?  I can see this being very bad for compiled languages, and I see languages such as python and ruby picking up the slack.  

I don't want this to become a language war.  I just want to see how everyone thinks the future of application design will look.

### Post by mssever on 2008-07-15
Call me a bit of a skeptic, but I remember that at one time offices were supposed to go paperless. Now that would have been great, but people seem to use more paper now than before. What's with people who regularly print stuff just to read it?

I think that there are some classes of application that will be better on the desktop for quite some time. The first example that comes to mind is OpenOffice. Google Docs doesn't come close to offering what OpenOffice does, and some features would be very difficult to do right in a browser environment, unless the browsers make some serious improvements. And I doubt that Microsoft will be making significant improvements to IE for quite some time. And any webapp that hopes to be successful will have to work with IE.

How about programs like The GIMP and Inkscape? While you could implement a drawing surface in a browser, it'll be a while before such code could run at a decent speed.

Plus, with any webapp that involves creating/laying out documents, you have the problem that there's no standard way to transport fonts seamlessly. There have been attempts, but nothing has caught on. Until something does, there's really no way to make a web document with decent font support without resorting to some sort of brute force hack.

Here's another example. I use Gmail; in my opinion it has the best UI of any email app that I've seen. But if I ever wanted to use encryption or digital signatures, I'd either have to switch to a desktop app or do the work manually. Gmail can't support encryption unless I'm willing to trust Google with my private key, which should never be transmitted over the network.

One more thing to consider. While available bandwidth is increasing, that will only result in more bandwidth being used. So a webapp will still be a fair amount slower than a desktop app. For many purposes, that won't matter. For some, it will.
