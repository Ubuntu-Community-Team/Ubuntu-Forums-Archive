### Post by ofnuts on 2014-05-18
Yes, it was far better than anything else on VM/CMS (Exec2!) when it came out but IMHO it was more verbose than Perl (without having regular expressions). As a VM/SP sysadmin I wrote tens of thousands of lines of Rexx, but I didn't really miss it on Unix platforms I used later on.

Edit: actually I didn't abandon Rexx per se (even if it could have been much better with regular expressions) but VM/CMS: line commands not really designed to interact with a shell, the 'stack' as a data exchange mechanism, the utterly flat file system, EBCDIC (how fun to write C code without braces!), the limitations of the 'transient area". By contrast Unix was so clean and elegant... I may have written some Rexx on OS/2 too.
