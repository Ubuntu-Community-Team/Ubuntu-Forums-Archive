### Post by nvteighen on 2011-11-06
> **archibishop of unix said:**

More job openings (I'm not doing this for a job, BTW, I'm doing this as a hobby), more libraries, etc.


Who told you that and in what context?

C++ *might* be "the real deal" in system programming, application programming, etc. However, when it comes to CGI on the web, Perl, Python, Ruby (and now server-side Javascript) are "the real deal". If you're into embedded programming, C++ is useless because of its tendency of generating big binaries... so C is "the real deal". For Mac OS X, Objective-C is the "official" language... Finally, when it comes to have a C++-like language but you don't want to deal with weird syntax and far more crippled semantics and additionally, you want your code to be cross-platform, the way to go is Java.

No language serves all purposes, none is really "the real thing".

EDIT: The link you were given is for sure the best C++ tutorial there is online. It is one of the few ones that is updated, covers STL and quite good in explaining things.

### Post by gnometorule on 2011-11-07
Back to recommendations...if you're serious about it, get Stroustrap even if it isn't free, and even if it isn't cool. The book quite successfully strives to be the "K/R for C++" (he's the original creator after all) and focuses in great detail on why and when and what for C++ is a good language. In reading it, you learn a lot about programming in general as well, while being walked through many, many pitfalls you find posted as questions here or on sites like stackoverflow.
