### Post by mssever on 2007-09-18
> **CptPicard said:**
> Didn't StarOffice have some kind of pseudo-desktop already?Yeah, I installed StarOffice 5 back in the 90s on my Win95 box. While I appreciated having an alternative to MS Office, the fact that it tried to take over my computer annoyed me enough that I got rid of it. I think that there must have been a reason that Sun (a for-profit company) scrapped the desktop thing.

That isn't to say that an integrated interface that avoided StarOffice's pitfalls couldn't be written. But you'd need to do some serious user testing before spending too much time on such a project--otherwise the time would be wasted. Don't confuse what you want with what the "users" want.

By the way, the term "users" is probably too generic to be useful. Nobody can write a one-size-fits-all interface. Microsoft has tried really hard, but they don't seem to realize that a good command line interface is an essential part of any OS. Also, I'm probably not the only one who considers Windows to be user-unfriendly. Apple makes a high-quality system that a lot of people love. But Mac OS isn't for everyone, either. So you need to define who your users are, and then do a lot of testing with that group before spending too much time on such a gigantic project.
