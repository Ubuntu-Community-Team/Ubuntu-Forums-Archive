### Post by phrostbyte on 2010-03-17
ASP.NET these days could describe many different things. For instance, there is two ASP.NET's the older "WebForms" model, and the newer Model-View-Controller (MVC) model.

ASP.NET *WebForms* development in MonoDevelop is considerably more difficult then in Visual Studio, as MonoDevelop lacks a WYSIWYG interface for it. AFIAK the new version of MonoDevelop adds a WebForms WYSIWYG interface, but it will not be available in Ubuntu until April.

ASP.NET MVC shouldn't be all that different, but ASP.NET MVC workflow is not WYSIWYG, it's all code and HTML. If I was learning ASP.NET these days, I'd skip WebForms entirely and go with MVC. MVC is cleaner architecture, it is inspired from the likes of Django and Ruby on Rails. MVC has better Mono support in general as well.
