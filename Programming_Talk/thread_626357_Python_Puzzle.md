### Post by smartbei on 2007-11-29
Well, I have solved 80 of the challenges so far (username = smartbei :)), using python in probably around 70 of them. The rest of them I did in C/C++, but I am pretty sure they could be done effectively in Python if you think of a better algorithm than the one I used. Python can definately be used for nearly all of the challenges anyway.

I have also found it useful to always program in python first, and in case the algorithm seems like it works well but takes too long on a large data set or range I port it to C/C++ (depending on the challenge (use of stl, etc.)). This is usually not difficult at all.
