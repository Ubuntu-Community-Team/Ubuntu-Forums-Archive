### Post by Tony Flury on 2012-01-14
I am mildly dyslexic, and I have found the biggest issue for me has not been the concepts of software engineering, but the practicalities of things like variable names, and spelling errors in common words (while, repeat etc).

My main coping strategies have been :
1) Using very clearly distinct variable names in my code, and making very clear choices between classes, methods and instance naming.

2) Using syntax sensitive editors, which use colour to highlight syntax constructs, and highlight spelling and other errors.

When I have worked in a team - I always ensured that my code was reviewed by other team members.

I have also found that incremental style techniques work very well - subjecting code to regular unit/module testing is a good way of catching annoying faults which could have a significant impact on a larger body of code. Using OOP type paradigms (clearly defined code/data boundaries etc) help contain errors to small and testable code items.
