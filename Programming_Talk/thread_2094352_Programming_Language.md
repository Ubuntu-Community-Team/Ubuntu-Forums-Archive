### Post by SuperCamel on 2012-12-13
I would suggest beginning to learn C++ before learning python. 

If you learn python first, you'll know how rubbish C++ is and you won't both with it. But C++, as rubbish as it is, is definitely worth learning. It's extremely popular. It may take years to develop with and additional years to compile, but the applications generally run faster than python apps. And you'll also learn more about programming in general, due to the ridiculous amount of features and lack of automatic memory management in C++.  Also, there are some things that C++ can do that python can't. Kernel modules are an example.

### Post by Mikeb85 on 2012-12-13
Java or Scala I think would be worth learning if you already know Java basics (and don't hate it).  

Linux apps seem to be mostly C (GTK+) or C++(QT), with some Python bits.  

Java is great for cross-platform apps, and I use Java apps quite frequently in Ubuntu.  If you want to 'do a bit of everything' then something JVM based (Java, Scala, Groovy, Clojure) is a great choice.  

If you don't care about anyone ever using the program and just want to mess around, have fun, and maybe create something cool for your own benefit, I'd highly recommend **Ruby**.  It's easy to learn, you can create something very quickly, but it does lack a little in the GUI support area (though it's not completely lacking, and you can create stuff).  

For something more esoteric, OCaml or Haskell would be good choices, though I can't promise they'll ever be useful if you want to create a GUI based app.

And as others have said, Python is a very safe bet if you don't know what to learn.  Plus there's plenty of online courses for Python.

### Post by slickymaster on 2012-12-13
As an intermediate language, I would probably advise you Java and C in that order. It is true that Java is much easier than C, but I think that a good programmer has to have good bases of memory management, and it does not get much better than C.
The choice of Java falls more towards being a good language for introduction to object-oriented programming, and for being a good bridge between C and C++.

Finally as a final language C++. It has a superb performance (only overcomed by C), with the advantage of having objects, which makes the perfect match for all kinds of projects.
