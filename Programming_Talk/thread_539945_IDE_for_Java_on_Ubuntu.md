### Post by CptPicard on 2007-09-01
And no, you should not try running some Windows IDEs on top of any emulation layer, it would be just totally painful and as Java is supposed to be cross-platform in itself, its main IDEs that are written in Java run just fine as they are on top of a Linux Java virtual machine.

J# sounds like some Microsoft embrace-extend-extinguish bastardization of Java that attempts to prevent this cross-platform interoperability.... but I'm not sure...

I use both Eclipse and Netbeans. Until recently, Eclipse's basic Java editing, code hints and code refactoring were superior to Netbeans'. Eclipse also has got a huge pile of plugins, which in my opinion are mostly just buggy and broken though, and just force you to reinstall the IDE every now and then.

Netbeans is better integrated, has a good XML editor out of the box which Eclipse doesn't, and in particular works great for web application development, which I have never managed to accomplish in Eclipse as the relevant functionality has just mostly mysteriously died on me most of the times I've attempted...

The Netbeans GUI editor is also great...
