### Post by pmasiar on 2008-05-31
This question is asked often enough to be in FAQ

[http://ubuntuforums.org/showthread.php?t=667422](http://ubuntuforums.org/showthread.php?t=667422)

and [http://ubuntuforums.org/showthread.php?t=796494](http://ubuntuforums.org/showthread.php?t=796494) has 3 threads discussing programming for children.

Summary: GameMaker (for windows) is excellent intro language to program games visually. Other options are Logo and Squeak. Ignore suggestions like C++, posters suggesting that have no clue how to teach smart but non-genius kid programming. Little harder to learn, but more flexible later, is to learn Python (text commandline mode), then learn pygame. And by next summer, our GameBaker (see my sig) might be ready for first beta testers.

### Post by Rasitiln on 2008-05-31
You should make sure he understands what 'game programming' really is first I'd think. You would be surpised how many misconceptions people have regarding just how difficult and time consuming it is. I know in Jr. High I had very limited understanding of what programming was... I thought it was about as hard as using the starcraft map editor. Once your done explaining it, I'd suggest grabbing a few python books for him. Also be sure to explain starting small and that the games he buys are made by very large teams at well payed firms (thats assuming he plays mainstream games like CoD 4 not indie games like puzzle quest)
