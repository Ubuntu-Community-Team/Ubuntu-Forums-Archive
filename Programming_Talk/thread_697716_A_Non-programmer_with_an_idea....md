### Post by pmasiar on 2008-02-20
> **mockdeep said:**
> I'm not sure, but I think I'm detecting a hint of sarcasm here.  Am I to believe that you think a DSL would be a less than efficient means of accomplishing my goals?

Sarcasm is directed to Fbot1. Designing usable DSL, and compiler from it to another language, is not what I would suggest to do for a beginner programmer. I had disagreements with Fbot1 in other threads, too, so...

Just see (or ask around if you don't believe me) how other people value Fbot1's advice.

You will be fine if you keep it simple and don't over-complicate your design. Of course you are aware that this project might take many months or even years to complete? The more interactivity you want, the more programming, and more complicated programming, you need.

Ie for smoothly interactive web-based application you need to know 
- Python (or Ruby)
- web app framework, like Django, Turbogears, or Rails
- SQL database to store data and generate pages
- Object-relation wrapper to access data (part of framework, but not trivial)
- HTML and CSS to make your pages pretty
- javascript + AJAX to make your pages interactive
- make couple of iterations of design, because first couple of attempts will fail.

This is without designing DSL and compiler to actionscript. That would be order of magnitude harder, IMNSHO.
This is also using simple approach and most productive available tools (Python or RoR). Using Java for server could make it even harder.

Don't get discouraged, but be prepared.
