### Post by yaaarrrgg on 2007-03-13
I usually only have one language (with the set of libraries) fresh in my mind.  

Also, I have a basic working knowledge of many common languages (assembly, C, C++, Perl, Python, Java, Javascript, HTML, CSS, MySQL, C#, VB, bash, ... ).  But I don't really try to memorize a language or the details, but I'll read a couple reference books or sources with the attitude "Where can I find 'How to do X'".  I suppose I learn my reference material more than memorizing details of the language. Then, maybe six months down the road, I'll refresh a bit on the language and use it, knowing where to find the details, as I need them.

IMO, syntax is the easiest and most trivial aspect of a language... The larger difference is in the kind of language, like a statically typed language versus a dynamically typed language, or OOP, and the different kinds of solutions that go with these.  The things I keep in mind more are principles of software architecture (or if something seems like an elegant solution, I may adapt it to another language).  This knowledge is largely transferrable from language to language.
