### Post by Krijk on 2008-03-05
I'm trying to automate some system management activities with Bash.

Depending on the OS/application etc a lot of different output-formats are generated. I'll parse the output, but I'm trying to modify these to a 'universal' format.

I run into these issue's with date/time extraction and conversion.

Usually I'll do this by using a - probably much too complex - set of code where I use a sequence of cut's/sed's/awk's/temporary variables/files etc. I'll get there in the end, but it is often a troublesome en time-intensive job.


**Examples:**
1. Convert  **04/03/2008 8:15:00.24**  (DDMMYYYY HH:MM:SSss) to **4 March 2008 8:15**
2. Convert the above to Unixtime
3. Convert **Wed Mar  5 07:51:39 CET 2008** to **GMT**

 Does anybody have some tip's or trick's to achieve this for output generated by different OS's (including Windows) and different time-zone's?

### Post by ghostdog74 on 2008-03-05
with GNU date, you can at least do example one.
```

# date +%Y-%m-%d-%H:%M:%S -d "04/03/2008 8:15:00.24"
2008-04-03-08:15:00

```
you can check the man page of GNU date for more options.

### Post by mssever on 2008-03-05
The closest thing I know of to do what you want is PHP's strtotime() function: [http://us3.php.net/manual/en/function.strtotime.php](http://us3.php.net/manual/en/function.strtotime.php)

It shouldn't be difficult to make yourself a little CLI PHP script that you can call from bash.
