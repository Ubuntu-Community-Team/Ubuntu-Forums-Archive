### Post by leg on 2013-08-15
[Blender]("http://www.blender.org/") probably one of my all time favourite programs. Lots of instructional videos available through the site for all the different features within the program. Includes a built in games engine that you can use through a tool box and also create Python scripts to attach to assets. Allows import/export of many different model types.

One disadvantage to it is that almost everyone I have shown this program before has complained about the difficulty of using the interface. However one of the recent releases was centred on altering it so that it is more like 3dmax in its initial view. Personally I would learn the old way of interacting with it as it is much more productive.

Give it a try.

### Post by leg on 2013-08-15
> **amir2 said:**
> ... Correct me if I am wrong, but most any rendering software use the same model format? Sorry missed this bit. Not really. Most modelling programs will have their own format but also provide conversion utilities. .obj is a fairly basic format supported by just about all modellers. .blend is the format from blender but that will include animations, materials and everything else in your scene. If you want to export a model you would do that through a plugin (which there are lots of)
