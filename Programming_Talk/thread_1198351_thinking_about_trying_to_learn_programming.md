### Post by nvteighen on 2009-06-27
Start with some high-level programming language not one that makes you think about how the computer should work. Examples of high abstraction level languages (short: high-level) are Python, Perl, Ruby, all Lisp dialects, Java... These work by hiding the "mechanical" details like bit pushing and memory management to make you focus on your program and also give you better and more flexible primitive data types to start build on. Low-level languages are used for controlling the computer from its very own core but that means much less flexibility and much more code for very tiny tasks; examples: C, C++, Forth.

High-level languages are able to express mental models much better than low-level ones. The trade-off usually is speed of execution. But, programming is about modelling through a formal language much more than just fiddling with the computer's internals, IMHO. And as they are more expressive, they make you learn how to program more easily... because they are nearer to your mind than a low-level language (which is nearer to the machine).

But, among high level languages, I wouldn't use Java unless you're interested in some job position that needs it... It's because it's a static-typed language; it makes you be aware of the data types explicity... a dynamic-typed language like Python, Perl et al. simplifies that a lot and allows you to do very nice things.

IMO, Python is the best beginner language... It is powerful enough to let you use it even in the future when you're no longer a newbie (unlike BASIC). Of course, after Python, you should learn other languages... a monolingual programmer is always a bad thing! (after having learned Python, C or C++ is a good choice).
