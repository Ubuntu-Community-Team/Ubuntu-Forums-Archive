### Post by jespdj on 2007-10-30
"Which language is better" questions without any specific goals for writing software in mind are pointless. Use the right tool for the job - for some jobs, language X is more suited and for other jobs language Y is more suited.

That said, here's how I think about Python and Ruby:

These two languages look a look like each other at first sight.  As a language, I like Ruby more than Python because Ruby is more object oriented.

In Python, there are some strange inconsistencies. For example, there are a number of global functions like len(s) to find the length of a string. Instead of global functions this should have been a method on the String class. Or look at complex numbers: you do a.real and a.imag to get the real and imaginary parts, but abs(a) to get the magnitude of a number. Why a global function abs? It should have been a.abs instead.

Also, I don't like the way that indentation is significant in Python. IMO white space should not be so significant in a programming language.

On the other hand, it seems that Python is much more used for programming applications on Ubuntu than Ruby is.

Also, AFAIK Ruby programs do not execute very fast compared to other programming languages. The Ruby interpreter just isn't that fast. Probably that doesn't really matter for many applications, though.

Have a look at:
[Programming Language Comparison](http://www.jvoegele.com/software/langcomp.html)
[The Computer Language Benchmarks Game](http://shootout.alioth.debian.org/) - Ruby is pretty low in the ranking
