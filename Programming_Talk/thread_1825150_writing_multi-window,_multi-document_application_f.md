### Post by danh-h on 2011-08-14
What is the best way to write a multi-document, multi-window application on ubuntu (let's say ubuntu 11.04 to be specific)?

If i were to write such a thing on the mac, it's pretty clear what to do: you open up a new "project" with xcode, create some interface templates with the interface builder, write your code, and go into an edit-check-debug cycle.  If you do this, then eventually you'll come up with something that looks native on the mac and interacts well with your users and the other software on the machine.

I would like to do that on a free software platform.  Ubuntu seems like one of the first platforms you'd want to consider for this because obviously the people involved are concerned about the user experience (they've put in a new launcher, polished the standard apps etc).  Also, the immense infrastructure (gcc, emacs, gdb, and lots of other tools) is present and tuned.

So what steps would you take?  (I.e., what are the right libraries or other tools to use?)

I don't really care about the language (although would like to be able to easily include c code in case of any performance bottlenecks).

But i would like to avoid lowest common denominator effects, or creating applications that don't fit in with the ubuntu way.

Thanks in advance for any advice and the reasoning behind it ---- including advice to use platform XYZ instead of ubuntu (as long as XYZ is free software, of course).
