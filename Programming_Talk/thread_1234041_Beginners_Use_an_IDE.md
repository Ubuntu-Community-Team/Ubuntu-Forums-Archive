### Post by lykwydchykyn on 2009-08-07
In both programming and other creative pursuits, I have personally found it most rewarding to learn on simple tools and only switch to more complex tools when I understood the need for them.  This prevents a lot of pointless, distracting knob-turning when you really need to be knuckling down on the basics.

So, I'd say if you have to ask (implying that you don't know why you'd need an IDE) then you should probably stick to a text editor (vim if you like, or whatever is most ergonomic for you).

### Post by CptPicard on 2009-08-07
I would say that especially with a verbose language and large API such as Java you will want to be using IDEs from the start. They help you to commit the rote memorization parts into memory more easily by providing fast enforcement, and make it easier to discover the API as you're hacking. Refactoring and auto-correction tools also aid in learning the proper ways to do the OOP parts...

### Post by nvteighen on 2009-08-17
I'd use an IDE only if you already know what's going under the hood...

Nope, this argument isn't valid to say you first learn a low-level language because they show you what's under the hood.

The issue is the following. IDEs can be useful because they automate extra-programming activities like compiling, project managing, searching documentation, etc... routines that are not part of actually programming but that are present in any programming session. But, the issue is that you should know them, because you won't ever know if the IDE will cover all the possibilities you'll need. You need that knowledge to fully control your programming **session**.

The key word is "session"... While it's acceptable (and the best, IMO), to hide all unneeded computer internal behaivor from your code, your programming, so it gets as nearest as possible to your mental concepts, this is not good when it comes to the managing of how you code... Also keep in mind that not knowing a low-level programming language is also as bad as not knowing how to deal without an IDE...
