### Post by soltanis on 2010-04-24
Getting back on topic, you do not need to "license" your code (by default you own the copyright to it as it is yours). Of course, if you use or link to GPL/LGPL libraries, Stallman might have something to say about that.

You can write an end-user license agreement, but these tend to be over-restrictive. As it stands, your program should be well protected by existing copyright law. You can simply re-iterate those laws by prohibiting things like unauthorized copying, decompiling or disassembly, but be aware that copyright law will not protect you from people disassembling or reverse-engineering your program for the purposes of legitimate interoperability.

Also, if you used someone else's code (which is very common) you need to be aware of how *that* code was licensed. If it is public domain, you cannot claim copyright to it; if it is BSD, you must credit the author and distribute the BSD disclaimer; if it is LGPL or GPL, you *must* license the rest of your source under those licenses (actually, it depends -- there's a certain number of lines involved, something like "more than 10", you should really refer to the license text to sort this out). Other things like Creative Commons or zlib have their own terms but tend to be relatively straightforward.
