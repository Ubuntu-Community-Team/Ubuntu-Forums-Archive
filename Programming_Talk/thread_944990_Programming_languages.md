### Post by ankursethi on 2008-10-13
One of the major goals of Python is to be cross platform. If you look at the list of platforms supported by the standard Python distributions, you'll be pleasantly surprised. It supports systems that you might never even have heard of. If you write your code carefully, it can run on Windows, Linux and the Macintosh unmodified as long as the Python interpreter is installed on the system.

Python supported 64-bit processors even before they were available to the regular consumer. The reason for this is that Python is used extensively for maintainence tasks on many Linux and UNIX servers, many of which ran on 64-bit processors long before your regular desktop OS like Windows. So 64-bit support is no problem.

You don't really need to waste money on books. There's a very nice Python tutorial that covers the entire language concisely on Python.org. Python's documentation is probably the best of all major programming languages I have seen.

Making a syntax error in your code doesn't kill your system. If it were so, developers would need to buy new hardware every few hours. I'm not a professional programmer, but I've been programming for fun for about 2 years. I make syntax errors all the time.

Python is the best language to begin programming. The only other choice would probably be Ruby, but I find Ruby's documentation to be rather lacking.
