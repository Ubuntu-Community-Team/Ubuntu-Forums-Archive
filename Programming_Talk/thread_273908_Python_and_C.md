### Post by JDD on 2006-10-09
Now, i won't say that im an expert programmer, but i have been doing it for a few years now, and in my honest opinion, i would suggest C over python any day, with one exception.

That exception: You plan on programming as a once in a while thing.

Reasons for learning C:

-Low level. Although not as low as assembly, its damn near it, and allows for you to interact with the hardware a little more... intimately. 

-Great concepts/features. These include, _**POINTERS**_, recursion, and memory allocation. Higher level languages don't include these, because some believe them to be a waste of time. I'll agree with them if your planning on programming once in a blue moon. 

-A seemingly endless amount of help. Sure, Python may have a fairly large community, and plenty of tutorials and snippets, but the C community is undoubtedly larger, and C has been around a LOT longer, so its unlikely your going to encounter an obstacle that someone else hasn't already.

-Understanding. This plays in with C being a low level language, if you plan on programming "seriously", i find it nice to understand WHY something works that way, rather then simply accepting the fact that it does... C allows you, rather, makes you understand why, to an extent.

So, when it comes down to it, it depends on wether you plan on pursuing programming seriously, or just dabbling here and there...

My opinion: Go with C.

### Post by kwalo on 2006-10-21
> Recursion, IMHO, is just a horrible, horrible idea and should never have been included in C.
You say so? Try to implement for example Quick Sort  without recursion. Without reursion C would be incomplete. Actually any language without recursion is incomplete. Some recursions can be substituted with iteration, but not all - eg. Hanoi Towers.

Threads are also difficult to maintain. You can have deadlocks, starvations and other problems. Does it mean, that they should not be used? No! There are problems, where threads are necesarry to maintain.

> Memory allocation, is of course, unnecessary in Python.Actually all variables in Python are allocated dynamically on the heap. Opposed to C, or Java, Python does this implicitly. You don't have to say, that you want to allocate memory on stack with malloc, or new. Python handles it automatically. Same is with freeing memory. It is done by garbage collector, when needed. You can still free some unused variables with del keyword.
