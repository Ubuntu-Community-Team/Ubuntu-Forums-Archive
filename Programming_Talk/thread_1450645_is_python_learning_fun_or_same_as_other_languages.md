### Post by CptPicard on 2010-04-09
This is the reason why me and a lot of other people over the years have been pushing Python as a learning language, and then some.

It's very "malleable" and easy to just simply "use"; and I am a big believer in learning from models and through reinforcement. Python allows for exactly this kind of reinforcement easily. You get exposed quickly and easily to all kinds of things, without having to bother with essentially trivialities, which are yet in practice annoying repetitive drudgeries, such as pointer management...

Also, in Java and C++-style languages the "design patterns" issue is a bit too dominant. In python you discover these more naturally and easily; and then in Java/C++ you just "see" them without having to discover them the hard way.

It's exactly the algorithms and such part that you want to be learning; but C++ shouldn't hold you back from that with its details.

### Post by CptPicard on 2010-04-11
While understanding that "fun" is ill-defined and subjective...

> **mmix said:**
> get over it, become fluent in c++.

C++ was the last language I actually became genuinely *fluent* in and that was after like a dozen other languages.

Then again, I would like to claim that a lot of people who claim fluency in C++, are not. If someone gets a more amenable learning curve in some other language they should certainly go for it.

> pointer, address & value in memory hardware.

Not all that amazingly educational in the big picture, but a big slow-down for someone like the OP.

> 
complex algorithms, is composed of simple logic.

Representing complex algorithms in terms of a very simple-logic implementation teaches nothing about the algorithm itself; algorithms often have a fairly abstract mathematical description that is the most informative, and from there, implementing the core point of functional transformations of their input data just gets lost in the noise if implementation language is too low-level.

> 
i recommend to learn java to avoid pointer,
to me, python/golang isn't fun, but,
c/c++/lisp/assembly is.

One of the reasons why both Java and C++ are (IMHO) suboptimal for the learner is that static typing and all the OOP-stuff keeps the beginner from seeing the forest for the trees. Java is just as bad, if not even worse, as C++ in this sense... one has to learn "design patterns" even for simple programs, and that again stops the "steady exposure" that is important in efficient learning.
