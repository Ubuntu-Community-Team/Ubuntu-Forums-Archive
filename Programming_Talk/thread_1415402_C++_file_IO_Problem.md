### Post by dellwood on 2010-02-24
I have a function that reads/searches a file for certain values which it stores in some private class vectors. Problem is, while it agrees that file.eof() is not true in the second while loop, r_line remains empty, so it just skips over the loop and moves on. Here is the code:

```

bool config::read( std::string filename) {
```

### Post by dwhitney67 on 2010-02-24
Why do you need to set the size of the vector?  Your application is clearly an example where you do not.  Reading from the same data twice from the same file is outright silly.

The getline() method returns a reference to the istream (or ifstream in your case); when a istream is queried as a conditional, it is converted to a bool, using the istream's operator bool() method.

So rather than check for eof(), do something line:
```

// open file

// check if file is open... AND do something if not! (other than
// printing a message, you continue on.  Consider returning from
// the function.

std::string r_line;

while (getline(file, r_line))
{
   // parse the r_line

   // insert (ie push_back) results into the vector(s)
}

// at this point you are done with the file.

```

### Post by dribeas on 2010-02-25
Besides the fact that you should use the solution provided by dwhitney67, the reason that your initial code was not working is that when you call '.clear()' you remove all internal error flags, but you are not resetting the pointer to the beginning of the file. The next time you try to read, it will find the end-of-file again and fail the operation. If you needed to restart reading from the beginning you would have to use the ´.seekg´ member functions.
