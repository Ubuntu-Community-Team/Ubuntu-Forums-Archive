### Post by TwoEars on 2011-07-12
It's possible - I did it, and I used only the documentation for the languages, no tutorials. The way I did it was to first create or realise a problem in my head, and then tailor my program to solve it. I awarded myself bonus points for modularity, readability, the number of languages I did it in, and reuse-ability.

I suggest learning a language like Python initially, as certain features will help you along your way, especially with learning algorithms.

### Post by Randy Flagg on 2011-07-13
I had some programming experience from about 15 years ago and I just started learning Python and really feel that its a good language to learn.  A lot of is almost like just writing out normal everyday language.

I have found four things that are helping.

1 - going through dive in to python page by page.

2 - There is an article in there that goes into depth about object oriented programming.

3 - pythonchallenge and the challenges in this forum.  They really make you think about what you learned and try to apply.  To me this is the step before programming for real world functionality.

4 - Google the documentation for syntax, builtin modules, operators etc.
