### Post by nvteighen on 2009-01-26
The C "vs." C++ case is an interesting one where you can see how identical syntax doesn't lead to identical semantics. Moreover, C++ semantics are completely different...

So, I guess it wouldn't matter what you learned before if you are aware of what C++ really is about. That's why I always insist people should also learn the history of programming; so they understand things better. Using the Old and Modern English example: ok, Old English doesn't help you to learn Modern English, but it may help you understand why some stuff occurs. It's not necessary, but it may be worth.

But there are other mistakes too. If you just want OOP, you don't need C++. You can do that in C using structs and some compile-time encapsulation through opaque pointers (you don't get a special syntax, but the concept). Use C++ if you want OOP + polymorphism + generic programming + ... In other words: use <put language> when you need its unique features (but research which its unique features are!).

Another mistake is to think C++ as an extension of C. It was true at the beginning; now it's a doubtful claim... mainly because of how C++ evolved and had its semantics changed.

And I agree: Linus is a bit annoying sometimes...
