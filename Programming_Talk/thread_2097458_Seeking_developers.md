I've been working on a project (titled 'Dough') for quite a while now. It's personal budgeting software, and in my opinion it's turning out pretty well. I've posted about it previously, so I'll just get to the point. I'm seeking any C++ developers who are interested in contributing to this open source project. 

I know that open source projects only attract developers once they have proven themselves worthwhile, and honestly it hasn't gotten many downloads on sourceforge. However, I find it very useful and use it all the time myself so I believe it's got potential. 

The code in this project has been basically split into two parts. The back end and the user interface. They are as separated as possible. The UI uses Gtkmm. The back end contains the code for handling accounts and transactions, loading and saving files and so on. The back end has no dependencies and it's written like a separate API. I've just finished documenting the back end with doxygen so that should be very helpful for anyone who is interested.
