### Post by drmrgd on 2013-12-05
Seems to me that a lot of people are gravitating toward Python and Java these days.  For many, I think the native OO approach that Python has is what draws people in (plus people seem to hate the 'line noise' of a language like Perl).  Those weren't concerns of mine (and I actually like the 'line noise of Perl personally :) ), plus I feel like Perl is more complimentary to Linux and other Linux tools (like AWK from which perl was sort of derived).  So, for me, who mainly does text processing like tasks (complex data structures and such) and some sys admin like stuff, Perl was a clear choice.  

I don't think you can go wrong either way, and since you seem to like to explore languages, I think it might be beneficial to learn both at some point.  I've actually (when I can find some free time here and there) have started learning Python to compliment Perl, which took me about 4 months to feel very comfortable with (to the point that I can pretty much code most things that I need to do with minimal effort).

I will also say that if you choose to start with Perl, Learning Perl is a great book and I really, really enjoyed it.  I've worked on a couple other languages since and I don't think any of the other prototypical books for those languages come close to being as well written as that book.  I didn't like Intermediate Perl as much; I really only found the first half of the book useful, and there seemed to be a lot of mistakes, typos, and whatnot in it.  But, another book - which may overlap with Learning Perl more than you'd like, but to me was great at filling in some gaps - is "[Effective Perl Programming]("http://www.amazon.com/Effective-Perl-Programming-Idiomatic-Development/dp/0321496949/ref=sr_1_3?ie=UTF8&qid=1386295269&sr=8-3&keywords=brian+d+foy")".  But between Learning Perl and Programming Perl, that should really cover you well, I think.
