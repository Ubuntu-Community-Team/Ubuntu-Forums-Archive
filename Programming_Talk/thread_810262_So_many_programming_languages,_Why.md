### Post by Quikee on 2008-05-28
If you want to be maximum flexible in a certain "domain" it is beneficial to have a language specifically tailored for that use (and may in time exceed the needs of the domain). 

Such languages are for example PHP for server-side html "postprocessor", bash for system shell scripting, javascript for browser-side scripting, SQL for database querying and management,...

The other is the common reason when someone is not happy with already existing language in a certain domain and creates its own similar language which fits his needs better (CPython vs. IronPython vs. Jython - almost identical, but running on different environments with some environment specifics).

### Post by pmasiar on 2008-05-28
Languages (and IT technologies) are incompatible **by design**. If vendor can rope you in into some technology, you will learn term and think in terminology of a particular vendor - you will not be able to even understand similar technology from different vendor. And to switch is extra effort - simpler is just upgrade. And you are locked in :-)

Ie why java has servlets and servlet containers instead of web applications? Why SQL did not standardized indexing, and even JOIN and INSERT? Why C# and not Java? etc.
