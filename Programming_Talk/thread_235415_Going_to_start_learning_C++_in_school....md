### Post by jwenting on 2006-08-14
> **Daverz said:**
> I had a C++ course in the late 90s that was the usual "C++ and OOP" course, which I think is the wrong way to go about it.  C++ is not a good language to learn OOP with, nor do I think OOP should be emphasized over other techniques. 

That depends on the teacher and the main subject of the course.
If it's mainly an OO course the language is pretty much irrelevant and the students should be expected to pick it up quickly enough.
If it's mainly a C++ course the OO element can be downplayed.

In both cases of course a dedicated course on the other element is called for afterwards.

OO is a vital element of modern programming practice, it should certainly form an integral part of the curiculum of any programming course using an OO language.
The problem with many 1990s programming courses (which you and I both took) is that this wasn't at the time realised and you were taught C++ with the exact same paradigms used previously to teach C or Modula.
The OO part was just thrown in on the side as a nice little extra feature, rather than a core component.
In fact, when I was taught C++ they didn't even mention the STL, and we were told to skip that part of the book because it was irrelevant.

The fallacy of that system didn't become apparent until years later when OO started to rapidly replace procedural programming as the main system.


> **Daverz said:**
> 
If you're lucky, you're instructor will feel the same way.  Otherwise, you might try the book **Accelerated C++**.

That's certainly a good book, but lacking in its treatment of OO fundamentals and core language features in preference to rigidly using STL for everything.
