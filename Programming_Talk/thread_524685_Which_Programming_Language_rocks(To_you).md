### Post by pmasiar on 2007-08-14
Seems to me that most people know only one or few languages: something deplorable, like BASIC (hehe I have flame resistant underwear :-) ), and then then invest huge amount of time  learning something complicated, like Java or C++, and are afraid to invest same time to learn something different, so they love their java and stick with it :-)

I was lucky to get good CompSci education (one of my favorite courses was comparing many languages - one a week), and was raised on old languages, many now obsolete, so i was forced to learn couple more, and it is easier to compare now. And also i know now that syntax is the easy part - real bugger is learning the library. (and because dynamic languages can have simpler API, *libraries* for dynamic languages are the real time-saver)

Great thing in language is garbage collection (and you can have GC without OOP) - which is maybe 80% of the reason why Java is more popular for people raised on C/C++. Static typing and compilation gives programmer false belief that her program is "checked by compiler for errors" and "faster". And as Graham explains in his [Beating the averages](http://www.paulgraham.com/avg.html), average programmer does not miss advanced features ( see "The Blub Paradox"), because her brains thinks without them - they look "weird" and un-necessary.

Another experience which will help to shape your language preference is to have maintain 1M lines of code written over last 10 years by someone else... :-)

OK now I prefer Python because is dynamically typed, most readable (and easy to maintain) language with excellent library support. 

But for a language which I really like as *the* coolest idea I would vote [Forth](http://en.wikipedia.org/wiki/Forth_%28programming_language%29). It is brilliant in sense how minimal it is, and how it is designed as mind-enhancer for smart programmers: it doesn't do any handholding, does not protect programmer from many errors which competent programmer should not do, and as a result allows competent programmer create incredible things in a very small place. Forth accomplishes it by being different from any other language: in interpreted (or better: incremental compiler), but *incredibly* optimized: function call takes just 1 CPU instruction! All languages are stack engines deep inside, but work hard to hide it from user. Forth does not,  just the opposite: it allows programmer to manipulate stack directly, or even perform binary patch of already compiled function! It is high-level language, or better even: language to design your own language, customized to solve exactly the kind of problem you are solving. You can even change the interpreter which interprets input line. But you can seamlessly reprogram any definition in Assembly, anytime. It is definitely *NOT* beginner friendly: is is expert-friendly :-) If you know, what you are doing, you can create ie. Pascal compiler (including P-code interpreter, lexical parser, and assembler) in just 11 pages of code. 

So yes, Forth rocks. It is still used by device drivers, because how extremely compact and fast code it generates, but it is not very popular - and my guess is, it is because not enough people are smart enough to use it :-)

BTW: "free" as in "with dynamic (latent) typing", in case anyone missed it, of course. *NOT* in sense of "free/open source license", even if Python is free in that sense too.
