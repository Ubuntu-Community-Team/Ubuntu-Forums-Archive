Variety is good. It's one of the reasons that there are so many Linux distros. A 'good' movie editor to you may not be a 'good' movie editor to someone else. One person may say, "features is most important, and performance is not", while another may say, "I want my app to be fast, I'm sick of waiting for my movie editor to load". Tada, two movie editors.

Plus, it encourages them both to grow. Movie Editor A team wants more people to use their editor, because then they get more developers helping them with it. Movie Editor B team is the same. Thus, they both try to make their Movie Editor as good as possible to get the most developers.

And finally you ask, "Where's Ubuntu and collaboration?" Well, plus what I said above, Ubuntu is not the people that decides who writes what software. Ubuntu decides what gets put in Ubuntu and how the basics of the system works, not the features of a particular piece of software.

### Post by soltanis on 2009-07-09
I once got that bright idea (Hey, I'm going to write a game engine! How hard can it be?) long ago...

So anyway, I agree with you that most open-source engines straight up suck and are totally blown away by commercial engines. Open source is still rendering like its 2002 and then you see something like Crysis -- it's enough to make you want to switch to Windows (except...no).

Probably the #1 problem is proliferation of games that look like Quake. I don't know if this is a Linux gamers thing or what, but seriously I'm sick of having weird looking rocket-shotguns and running at the speed of sound -- how about a squad based shooter for a change?

Anyway I can tell you from experience that you are going to have a hell of a hard time, especially alone, especially if you've never worked game engines before (and it sounds like you haven't so this will be one hell of a learning experience for you). I don't want to discourage you -- rather, I want to encourage you to start off by seeing what you can do by playing around with existing engines.

I recommend Sauerbraten; in my opinion, this is the most advanced open source engine produced, even though ironically its intent is not at all to create eye-candy. It also has some pretty hackable source code. See if you can't do something cool (like add more advanced shader effects) using that engine (and please contribute your changes!) so that you get a feel for how to do things before you dive into your own codebase.

> **Ratscallion said:**

At the risk of going off on a tangent, not necessarily; C++ is not a strict superset of C, and especially if the header uses questionable typing (unlikely, but possible), C++ will refuse to compile it. That is unless you use some extern "C" { } crap (in which case the C++ compiler will use a real C compiler to compile the code).
