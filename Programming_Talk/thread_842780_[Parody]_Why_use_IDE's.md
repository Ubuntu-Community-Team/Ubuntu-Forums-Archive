### Post by wdaniels on 2008-06-29
There's not very much in most IDEs that I find useful. I don't like things such as templates, "wizards", class builders, GUI designers etc. I just don't like the code they generate, and I don't find that they save more than a few seconds time-wise. For very large projects IDEs can help to keep track of things, but that depends on the language and how well the codebase is structured to some extent.

Auto-completion? Can be a helpful mind-jogger but doesn't usually save any time exactly. Revision control? No faster or easier than the command line or nautilus scripts. XML editor? That could be useful for certain kinds of large project, but it doesn't really need to be in the IDE. Refactoring tools? Yes, they can be helpful occasionally on a large codebase, but not really day-to-day work. Standard headers? All but the simplest text editors can handle that but it doesn't really save any effort - I usually just copy and paste from elsewhere anyway. Code quality reports? Waste of time in my opinion. Unit tests and coverage? If you need this I guess it's handy to be in the IDE, but command line is easy enough. Deployment tools? The only ones I've used I didn't like. Object/Class/Assembly browsers I do like, but I prefer to use proper sources of documentation if available.

What other kinds of tools do you get in IDEs? I forget since I haven't really bothered looking at any for a while...

I like the idea of having everything in one place in an IDE, but I simply can't stand how slow, bloated, cluttered and often unstable they tend to be (especially Eclipse). Anything heavier than gedit just annoys me. I've never been able to put my finger on precisely the reasons why, but I always seem to be a lot more productive when using a lightweight text editor. I'm sure I don't just imagine that. I expect I'd quite enjoy using a terminal editor such as Vim but I never really found a good enough reason to bother learning it (yet)...

### Post by pmasiar on 2008-06-29
Once I used IntelliJ briefly and liked one feature: when you edited variable name, it asked if you want to  rename that variable. Handy. Saves 2 seconds every rename, and you can use it without thinking about it.

MultiEdit has handy feature: when you click on variable name, it selects it. Then, if you press Ctrl and click on selection, it will find that variable down, if on Shift, finds it up. And ME also select whole lines if you select more than one line, regardless where you start (all other editors select  stream of chars, you need to jump home to select whole lines).

ME templates also expanded on space, not Ctrl-Space like Eclipse - so again you do not have think about it.

In Eclipse, I like when you click on tab header, it toggles between fullscreen zoom and view with sidebars/properties. 

In SPE, I like sidebar with code outline (entries for every function in that file, allows quick navigation). SPE also supports code tags, which are displayed as entries in outline - kind of permanent bookmarks. Again, quicker code navigation. There are many other handy tricks, I would love them all in one editor. :-)
