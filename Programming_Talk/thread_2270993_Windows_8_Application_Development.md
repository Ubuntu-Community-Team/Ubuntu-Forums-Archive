### Post by ofnuts on 2015-03-27
Install a virtual machine in your Linux. It doesn't need to be that big since it's going to run the bare essentials plus whatever is needed to develop. Performance is about as good as running directly on the metal. If you are using VirtualBox, the "shared folders" gives the VM access to your native file systems with decent performance (so you can still use your favourite Linux tools on your source code, for instance).

This is what I've been doing for the last 3 years. It even has the advantage that when I switch to a new machine, I can just import the VM file and don't need to re-install the whole tool chain...
