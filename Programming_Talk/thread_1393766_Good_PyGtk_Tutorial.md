### Post by Kegusa on 2010-01-29
I found when starting out with pygtk following the huge tutorial that pygtk has was a really good starter. I followed it, made some own mind experiments and tinkered and tweaked everything that the tutorial told me alongside with looking into the documentation for pygtk. I find the shorter tutorials to be lacking a bit of the hows and whys of doing things. 

Also try to do things in your own way when following the tutorial, write down ideas you get when looking at a particular feature and try to make it yourself by using what you already know. (hint, there's almost always things that the tutorial is missing so you have to turn to the documentation and further reading for solutions)

I made some question games, calculators, script-starters and sql tools when reading the tutorial so go ahead and try to make whatever alongside the tutorial.

I didnt have the luxury to print out the mammoth PDF file so I had it in its own window alongside my editors. Go get the gargantuan PDF of 412 pages here: [http://www.pygtk.org/tutorial.html](http://www.pygtk.org/tutorial.html)

### Post by nvteighen on 2010-01-30
Hmm... there's also a quite nice application for GNOME called Devhelper which serves as a documentation browser. Install that and also the tutorial from the repos (devhelper and python-gtk2-tutorial packages, respectively) and you'll have a searchable and easy-to-handle version of the tutorial.

Devhelper works with several other APIs like the C GTK+, Rythmbox, ATK, Pango, etc.
