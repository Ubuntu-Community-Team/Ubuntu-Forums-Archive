### Post by jespdj on 2008-04-08
I started in 1984 when I was 13 with the Commodore 64 :)  programming in BASIC and soon after that in assembly language. I did a lot of "dabbling" since then and also wrote larger programs.

I guess that if you want to write a larger program for yourself, you first have to have an idea of what you want to make. Then start programming and stick with it. If there's something that you don't know how to do, then find it out; do research, read documentation, follow tutorials, experiment, and use your new knowledge in your program. Experimenting is very important when learning how to program.

In the business world, things ofcourse are very different than when you're just hacking a program together for yourself. I'm a professional software developer now (working mainly in Java). Any serious software development project starts with a major amount of planning, gathering requirements, writing documents with specifications, etc. and the programming itself is only a small part of the whole project.

### Post by wrightee on 2008-04-08
I started when I was 9 by reading a book called "Fred Learns About Computers" that besides having lots of uninteresting stuff about vats of mercury had a few chapters about control flow, variables and the like.  For two years I was a sad little nerd writing programs on bits of paper until the ZX81 was invented.  It was about a year before I made "the leap" and wrote a "useful" program - at least I thought it was (it was to play Yahzee against the machine, complete with ascii rolling dice!).  

The phase change came because of a goal - I liked Yahzee and I wanted to see if the computer liked it too... That forced me to up my game and learn what I needed (associative arrays, as they were called..).

Thinking about it, the same is true now.  Each time I learn a new language, I think of a piece of software I'd like to have and code it. Coding challenges and the like are fun exercises now and again, and very useful for interviewing new coders for my work, but unless I really want either a) the program I'm writing or b) the satisfaction of knowing I made the machine do X and that's cool...it's all a bit of an uphill struggle.

So my advice - have a coding goal, run at it! (er, no pun intended)

This one works:
[PHP]
import random

flips = 100
headcount = 0
tailcount = 0

raw_input("Press ENT to flip the coin 100 times.")

while flips > 0:
    coin = random.randrange(2)
    flips -= 1
    if coin == 0:
        headcount += 1
        print "The coin landed on...Heads!"
    elif coin == 1:
        tailcount += 1
        print "The coin landed on...Tails!"

print "You have flipped the coin 100 times."
print "\n\nThe coin landed on heads", headcount, "times."
print "\nThe coin landed on tails", tailcount, "times."

raw_input("Press ENT to exit.") 
[/PHP]

You should have put "coin = random.randrange(2) " in the while loop. Your program threw the coin once. Furthermore it is "raw_input" and not "input_raw".

I hope it helps

EDIT: I would also make the loop a bit simpler:

[PHP]
import random

flips = 100
headcount = 0
tailcount = 0

raw_input("Press ENT to flip the coin 100 times.")

while flips > 0:
    flips -= 1
    if random.randrange(2) == 0:
        headcount += 1
        print "The coin landed on...Heads!"
    else:
        tailcount += 1
        print "The coin landed on...Tails!"

print "You have flipped the coin 100 times."
print "\n\nThe coin landed on heads", headcount, "times."
print "\nThe coin landed on tails", tailcount, "times."

raw_input("Press ENT to exit.") 
[/PHP]
