### Post by digby on 2006-06-29
There are several ways to answer this question, and most of them hinge on what exactly you want to do with programming.  If you're more specific, we can be of better help, but if you're just genuinely curious about programming and don't have a specific end in mind, a good starting point would be to learn python.

You can get a good start on that with [this tutorial]("http://honors.montana.edu/%7Ejjc/easytut/easytut/").

Failing that, you might consider something like scheme with [this text]("http://www.htdp.org/") from MIT.  Scheme isn't used as commonly, but that book will supposedly give you a good foundation in the important conceptual areas to make it easier to branch out from there. 

I'd started trying to learn both, but that's neither easy nor (from what others have told me) recommended.  So I've decided to stick with python for now.  That decision was mostly motivated by the fact that I have a friend that knows a fair bit of python and almost nothing of sheme.

### Post by ltmon on 2006-06-30
There are two main schools of thought:

1. Start with the lowest level (i.e. most difficult) programming language so that you know how everything works under the hood.  Only then move onto something higher level and more productive.  So you would start by learning assemby language or C/C++ and then move on to Python/Ruby/C#/Java etc.

2. Start at a high level because it's a gentler introduction.  Move on to advanced things when you need it.  So start with Ruby/Python and move on to Java/C# and then to C/C++.

I know plenty of people at my work that have learnt to program both ways, and none is better than the other.  Pick what appeals to your way of learning.

p.s. My favourite language is Ruby.  Here is an *awesome* tutorial: [http://pine.fm/LearnToProgram/](http://pine.fm/LearnToProgram/)
