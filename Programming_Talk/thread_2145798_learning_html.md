### Post by King Dude on 2013-05-16
The languages truly best to learn for web development (server side or otherwise) are PHP, HTML, CSS, JavaScript, and perhaps XML. There is also C#, Erlang, and Java, but I think of those as optional.

EDIT:

I take back what I said about Bash, it is used in web development- but not in the coding of a web page. Bash would primarily be useful on the server end since you do not have a GUI, assuming you're running a Linux distro that does not come pre-packaged with a GUI.

### Post by epicoder on 2013-05-16
I would recommend you avoid W3Schools, see my signature on why. Here are some better resources:
[QUOTE=w3fools.com]

[LIST]
[*]                       [WebPlatform.org]("http://webplatform.org")                       is a good collection of all sorts of resources. 
[*]                       [Opera Web Standards Curriculum]("http://dev.opera.com/articles/view/1-introduction-to-the-web-standards-cur/#toc")                       covers the basics of web standards-based design in HTML and CSS. 
[*]                       [SitePoint]("http://reference.sitepoint.com/")  is a pretty good reference for HTML, CSS and JavaScript. Their  documentation always mentions feature support across different browsers,  and describes known browser bugs. 
[*]                       The W3C, itself, has a wiki-based general [Learn]("http://www.w3.org/html/wiki/Learn") page as well as an [HTML element reference]("http://www.w3.org/community/webed/wiki/HTML/Elements"). 
[*]                                                  [The MDN (Mozilla's Developer Network)]("https://developer.mozilla.org/en-US/docs") takes over at intermediate CSS and                           covers JavaScript better than anyone. 
[/LIST]
[/QUOTE]

If you're just trying to learn html, don't bother with php/python and the like. Just open a text editor and try some things out:
