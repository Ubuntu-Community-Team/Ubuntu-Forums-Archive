### Post by LaRoza on 2008-01-08
Often, posters want to buy books on programming. I decided to make a thread encompassing our recommendations would be a good thing, instead of randomly responding to queries.

I am a big advocate of the free online resources, and usually only use books for references. 

Please post books that can be **bought** that you would recommend someone buy. If possible, post a link to an image of the book.

Give a short review on it also.

###################################################

[The C Programming Language](http://en.wikipedia.org/wiki/The_C_Programming_Language_(book)") also called **K&R** -- This book is the number one book for C, although it is not best for people starting out in programming. The new edition covers the Ansii Standard, this edition is not in the wikipedia article I linked to. 

ISBN-13 978-0131103627
ISBN-10 0131103628

[DOM Scripting]("http://www.amazon.com/DOM-Scripting-Design-JavaScript-Document/dp/1590595335") -- This book is a very good book for learning how to use ECMAScript wisely in Web pages. It doesn't cover the language itself, only the basics and the use of the DOM to make clean, reusable, and sensible client side scripts.

ISBN-13: 978-1590595336
ISBN-10: 1590595335


[O'Reilly Publishing]("http://www.oreilly.com/") -- Many books from this publishing company are worth buying.

### Post by CptPicard on 2008-01-08
Enlightening text. I'm out of school already and I find myself re-reading this every now and then and it just gets better -- I didn't have enough time back then to really dig deeper, the class was short and there was no time:

[Structure and Interpretation of Computer Programs]("http://mitpress.mit.edu/sicp/")

[Sipser's Introduction to The Theory of Computation]("http://www-math.mit.edu/~sipser/book.html") got me through the "theory of computation" first graduate course at my uni. I wasn't yet quite as comfortable with the formalisms of CS as I am today, so I struggled a bit and was actually quite nervous going to the final exam, but I aced it. :) Nowadays I prefer reading the lecture notes which were too terse back then though... the book's strong point is that it's not as verbose as the Cormen, Leiserson et al book mentioned above, though. It has avoided the bloating disease that infects too many textbooks without sacrificing examples and readability.

### Post by elithrar on 2008-02-27
[Learning Ruby](http://www.oreilly.com/catalog/9780596529864/index.html) from O'Reilly is very good - it's quite thin, but when you consider that Ruby itself is a very condensed language it works well. Some parts can be skipped if you're a proficient programmer, but it does introduce you to Ruby's syntax, some OO concepts and how Ruby handles loops/flow control/hashes and data types.

[Ruby Cookbook](http://www.oreilly.com/catalog/rubyckbk/?CMP=AFC-ak_book&ATT=Ruby+Cookbook) is another O'Reilly one. As **LaRoza** said, *most* (not all!) of the O'Reilly books are decent. The Cookbook is pretty much a resource for code-snippets and for system admin tasks in Ruby, which is what makes it so useful. It's actually sitting on my desk right now - the problem/solution/discussion format works very well for what it is intended to be. Code snippets are generally basic, but cover alternate solutions and frankly that's all you need to build on and integrate into your own scripts.

I've also got [Learning Python](http://www.oreilly.com/catalog/9780596513986/index.html) & [Programming Python](http://www.oreilly.com/catalog/python3/index.html#top) waiting in the wings, but I've only read reviews and flicked through them briefly.

Most of the Apress books are pretty decent too, though I wasn't a fan of the "Ruby for System Administration" title they pushed out.

[IMG]http://ecx.images-amazon.com/images/I/515GH1J2JJL._AA240_.jpg[/IMG]

**Linux Programming by Example**, by Kurt Wall
Copyright © 2000 by Que® Corporation
International Standard Book Number: 0-7897-2215-1

Synopsis: Introduces programmers familiar with C to the GNU/Linux programming environment.  I consider this book, or something similar to it, essential to developing on the GNU/Linux platform.  Not to be confused with another, newer book titled *Linux Programming by Example: The Fundamentals* which I have not yet been able to peruse.




[IMG]http://ecx.images-amazon.com/images/I/51HJF%2B%2BjseL._AA240_SH20_.jpg[/IMG]

**The Art of UNIX Programming**, by Eric Steven Raymond
Copyright © 2004 by Pearson Education, Inc.
International Standard Book Number: 0-13-142901-9

Synopsis: The Art of UNIX Programming covers the essential paradigms of the UNIX programming model, its evolution and incorporatiion into other operating systems, and its philosophy.  It's goal is not to teach programming as much as the ideas behind how programming has been done.




[IMG]http://ecx.images-amazon.com/images/I/41WV4RS0TNL._AA240_SH20_.jpg[/IMG]

**The C Programming Language, 2nd Ed.**
by Brian W. Kernighan and Dennis M. Ritchie
Copyright © 1988 Prentice Hall
International Standard Book Number: 0-13-110370-9

Synopsis: Although an aging text, it remains canonical to many programmers around the world.  There are very few that could not benefit from this book.




[IMG]http://ecx.images-amazon.com/images/I/519G7HET7WL._AA240_SH20_.jpg[/IMG]

**Professional Assembly Language**, by Richard Blum
Copyright © 2005 by Wiley Publishing, Inc.
International Standard Book Number:  0-7645-7901-0

Synopsis:  I consider this text optional.  However, in my experience, I have not seen a better treatment of GNU assembler.  Most beneficial to a programming *pedant* who, like myself, wants to get as close to the hardware as possible.




[IMG]http://ecx.images-amazon.com/images/I/41TJYIXYsdL._AA240_SH20_.jpg[/IMG]

**Programmer's Guide to nCurses**, by Dan Gookin
Copyright © 2007 by Wiley Publishing, Inc.
International Standard Book Number: 978-0-470-10759-1

Synopsis: Excellent reference for programmer's familiar with C who wish to write advanced character-cell user interfaces.




[IMG]http://ecx.images-amazon.com/images/I/31jMR9ltFHL._AA240_SH20_.jpg[/IMG]

**Practical C Programming, 3rd Edition**, by Steve Oualline
Copyright © 1997 O'Reilly Media, Inc.
International Standard Book Number: 1-5659-2306-5

Synopsis:  An indispensable guide to C programming with easy-to-use charts, tables, and look-up cheats.




[IMG]http://ecx.images-amazon.com/images/I/51YY701AZ0L._AA240_SH20_.jpg[/IMG]

**Beginning C: From Novice to Professional, 4th Ed.**, by Ivor Horton
Copyright © 2006 by Ivor Horton
International Standard Book Number: 1-59059-735-4

Synopsis:  This is the ideal beginner's book to programming with C.  It starts with the simplest concepts, leading the reader incrementally into more advanced topics.  Combining this book with *The C Programming Language, 2nd Ed.* would create the perfect programming starter pack.




[IMG]http://g-ecx.images-amazon.com/images/G/01/nav2/dp/no-image-avail-img-map._V46862177_AA192_.gif[/IMG]

**Sam's Advanced C**, by Peter D. Hipson
Copyright © 1992 Sam's Publishing
International Standard Book Number: 0-672-30168-7

Synopsis:  Another text covering topics of various skill levels in the C programming language.  Despite its title, it is not my opinion that this is more "advanced" then other texts.  This book was not my favourite, but I included it anyway since it does effectively provide an alternative point-of-view to programming with C.  I wouldn't recommend getting this before any of the others I've mentioned (indeed, if you can even find a copy).
