### Post by k2t0f12d on 2008-02-25
I have *C++ Programming - Program Design Including Data Structures, 3rd Ed.* by D.S. Malik.  The book isn't bad *per se*, just not great.  I definitely wouldn't keep it in the same shelf as K&R or Stroustrup.  I found that concepts of computer programming were *easier* to absorb through practicing straight C, and shed a lot of light on the syntax of C++ that was initially unclear to me.  If you aren't in a hurry to immediately produce working code, I recommend getting

[LIST]
*The C Programming Language, Second Edition* by Brian W. Kernighan and Dennis M. Ritchie
[/LIST]
and then compliment that book with two or three others such as
[LIST]
*C In a Nutshell* by Tony Crawford and Peter Prinz
[*]*Beginning C From Novice to Professional, 4th Ed.* by Ivor Horton
[*]*Practical C Programming, 3rd Ed.* by Steve Oualline
[/LIST]
It doesn't matter which you use with K&R, so long as they aren't truely awful.  At the very least you should have one reference manual and one instructional text.  The more books that you have the more you can cross reference and query the thoughts of multiple minds on troubling concepts.  You will probably find that you develop preferences and tend to grab the text that you find most helpful (if in doubt K&R are *always* right :rolleyes:).  After C, you can repeat the process for C++ with Stroustup and two to three other books.

Beware of texts that start talking heavily about software engineering *unless that is part of your learning objective.*  Most instructional texts are going to include *some* practical software engineering discussion, simply to familiarize the reader with the development cycle, debugging, etc.  However, if you aren't actually reading and writing code *at least* by chapter two, you probably have a software engineering book masquerading as a programming text, such as,
[LIST]
*C++ Plus Data Structures, 3rd Ed.* by Nell Dale
[/LIST]
a book which I haven't ever gotten far enough into to discover any programming advice.  Which isn't to say that the book *doesn't* have any useful advice on programming, only that I already have others that got me to the things I needed to see more quickly.
