### Post by chris1974 on 2009-02-07
I have been building websites as a hobby for a few years, and I am thinking about building a web app with Ruby on Rails.  But one think keeps nagging me: I don't think that html or xhtml is the best tool to build a user interface.

html was originally designed to describe text based documents, it has tags to describe such as paragraphs and headings.  This seems inappropriate for describing a user interface.

With the current proliferation of software as a service, I am surprised that technologies such as XUL have not gained acceptance.  Most people still use html to build their web UI despite better tools.

Do you think people will ever stop using html to build UIs?

### Post by Tomosaur on 2009-02-07
> **chris1974 said:**
> 
Do you think people will ever stop using html to build UIs?

I would say 'yes', but not for a long time yet. When the 'web-application' thing really starts to take off, I would imagine the demand for something better than HTML would allow something new to emerge. I would say .html pages would still be used, but for their intended purpose (marking up information) - while the overall user interface will be using Technology X to do the job. Kind of like how websites now mostly use templates and call the information to display as and when it's needed, but where the template is not HTML based.

Of course, you would also need everybody to upgrade their browsers - which will take a LONG time.
