### Post by matthew.ball on 2010-04-01
To start off with, try [GNU nano]("http://www.nano-editor.org/") with [GNU Screen]("http://www.gnu.org/software/screen/") - Have one terminal screen editing source code in GNU nano, and a terminal screen for compiling source files with g++.

This sort of leads straight into either vim or emacs. After a while, as the complexity of your projects grow you'll be editing multiple files at once, and this is exactly what the buffers in vim and emacs offer.

I could go on, but I don't know how interested you would be in the idea of terminal editing.

Edit: I guess it would be worthwhile to check out [the GNU debugger]("http://www.gnu.org/software/gdb/") and [GNU make]("http://www.gnu.org/software/make/")
