### Post by rnodal on 2008-09-05
Hello all:

I would like to introduce my project. SourceForge unix name: "easyge", my code name for the project "EGE" and the full name is "Easy Game Engine". Note "easy" is relative ;). As my graduation project I decided to do a 2D game engine so this project was born from that. The current "preview" release code named "Graduation" is the result of the three months that I spent working in my graduation project. It was programmed in C++ using SDL and OpenGL. During the development I did not have access to a Mac so only Linux and Windows builds are available. The current release features 4 demos and one small (really small) game. The release also contains CodeBlocks projects for easy editing and rebuilding. Since I'm officially done with my graduation project I now moving on with the development of the project into something bigger and better. These are my ultimate/current/future goals that I would like to accomplish with this project:

Ultimate goals:
* Free/Open source. (DONE)
* Multi-platform. (Linux, Windows, Mac)(No Mac support yet :()
* As much compatibility as possible across Linux distributions (This is a big one)
* Easy to use and still be powerfull.(Do things through C++, scripts or tools but never limited to one choice. You decide which way to go)
* A lot of tools to make development very fast.(Font tool for bitmap fonts, GUI editor, particle system editor and more.)
* Game independent. (Make any game you want. 2D/3D)
* AI, Physics, Networking

Current goals:
* Clean/improve current code.
* Fix current bugs (Add new ones(just kidding :)))
* Add features that I was not able to add due to lack of time.
* Create font editor, GUI editor, Particle system editor and texture editor.
* Turn the engine into a very good/lots of feature/stable/easy to use 2D engine.
* Create more than 4 demos and more sample games.
* Write some tutorials and have some basic API documentation at least.
* Script support. This would allow you to write certain part of your game/application using some sort of scripting language. But you will always have the ability to code everything in C++ if you want to :).  

Future goals:
* Move into the 3D part of it.
* Release more sample games.
* Improve documentation.
* Networking support.
* Anything else that comes up.


Pretty much the idea is to concentrate in the 2D part of the game engine so once that part is done the engine can be used to create 2D games. Once I'm satisfied with that part then move into the 3D part of it. And then just fix problems, add features and tools. It case you were wondering if you could help, YES you can. It is simple as this: use it. Then report back, how hard was it for you to accomplish your task? Did something work different than expected. Is there anything you would like the engine to have? Did you find a mistake? Any feed back would be worth. It's really hard for me to develop the engine and at the same time develop demos to test the engine. I can and I like doing it but it is very time consuming. So testing the engine and providing feed back is where I need help the most. Thanks for reading and looking forward your feed back.

Some notes:
The API style of the engine was/is heavily influenced by [http://hge.relishgames.com/](http://hge.relishgames.com/) .

Last but not least, thanks for your time. The link to the project where you can download the "Graduation" release is [http://www.sf.net/projects/easyge](http://www.sf.net/projects/easyge) and the link in my signature is the link the the project website(which needs an update)
