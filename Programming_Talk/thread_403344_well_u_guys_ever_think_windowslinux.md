You are missing how operating system works. There can be only one kernel, managing hardware resources. Either Windows or Linux (or X), but not both.

I recall one project which runs *modified* Linux kernel as task under Windows, but it is not stock Linux kernel. And Windows would be responsible for security even of Linux tasks, because all runs under Windows.

Mainframes had for long time virtualization  systems allowing to run multiple OSes. See [http://en.wikipedia.org/wiki/Hypervisor](http://en.wikipedia.org/wiki/Hypervisor) - Linux is getting ready, but Windows is not interested. Whole point of Windows platform lock-in is to be the only platform used and *not* be compatible with other platforms. Work to be compatible with others, if you have 80-90% of the market, is to help others to catch up, so from point of company-almost monopoly it is counter-productive.

Of course customers ask for it, and some company started developing it, so MSFT bought it before it came to market :-) check [MS Virtual Server](http://en.wikipedia.org/wiki/Microsoft_Virtual_Server) (I hate then MS gives their products generic names - you cannot search for it in google, and I am sure it is intentional). One of many similar products is also [VMvare](http://en.wikipedia.org/wiki/VMware).
