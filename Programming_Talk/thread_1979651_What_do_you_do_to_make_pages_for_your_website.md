### Post by lykwydchykyn on 2012-05-24
> **RadicaX said:**
> @snows & fallenshadow.

I figured any WYSIWYG would make pretty messy code, but is it faster to compose the basic website that way, then go over it with something to help you find all the mistakes?

Or do you save time just writing the code by hand and the such?

Seems to me it defeats the purpose of using a WYSIWYG if you're going to subsequently clean up the code.  I guess it depends on how you intend to clean up the code.

For me, "clean code" ideally means that the HTML is purely semantic and all presentation is defined by a separate CSS file.

If I'm writing a static HTML page that's mostly content (say, a blog post), my preferred method is to use a simple markup language (RST, Markdown, etc) and just convert it to html with something like pandoc.
