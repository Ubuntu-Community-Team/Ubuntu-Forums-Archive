### Post by frogotronic on 2011-02-07
Hi,

I'd like to fix a major usability bug in LibreOffice 3.3:

When using Impress and an external monitor with the presenter console installed, displaying an embedded sound object (like a movie) causes sound to come from BOTH the display image/slide AND the preview (presenter-console) slide.

This creates a horrible echo, rendering the presenter-console unusable.

This is a SHOWSTOPPER bug for anyone in the professional arena, teaching,
presenting, meeting, talks, etc.  More often than not, people find out about this bug *during* the presentation.

I'd like to fix it, but I'll need help.  Basically, the problem is that a movie started of the main screen starts and then there is a slight delay while the movie on the presenter-console (p-c) screen starts.  As no-one is viewing the p-c screen except the speaker, no harm done.  BUT the sound gets piped from the main screen and the p-c screen and hence the problem.  What I need to do is to figure out how to suppress all sound in the p-c screen as per the presenter-console that came with OpenOffice.org 3.1.

Here's the link to the extension: [http://extensions.services.openoffice.org/project/presenter-screen](http://extensions.services.openoffice.org/project/presenter-screen)

Here's the link to the source code: [http://svn.services.openoffice.org/ooo/trunk/sdext/source/presenter/](http://svn.services.openoffice.org/ooo/trunk/sdext/source/presenter/)
