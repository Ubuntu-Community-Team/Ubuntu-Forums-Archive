### Post by Tomosaur on 2008-09-10
Depends what you mean by 'language'.

Syntax and such shouldn't take you all that long really, and once you're familiar with the C/C++-like languages, you can pretty much move between them comfortably with only a few differences in syntax.

The real time-consumer, I've found, is learning the tools at your disposal, and the libraries available to you.

I'm currently going through the interview process for a development job - they company seems to want to hire me (2nd interview, im the only candidate who has made it that far :P ), but it's a .Net development job using C#. I'm fine with C# syntax (occasionally need a bit of refreshing, but that's no big deal), especially coming from a primarily Java-based background from university and such. The main issue I have to deal with is learning the .Net platform and libraries, and the whole job kind of hinges on that. I have to do a presentation next week for the second part of the interview, hopefully I'll be ok but a fair bit of revision is going to be required. I'm ok with the 'technology' they want me to talk about, but it's the .Net implementation I need to get familiar with.

So for me, anyway, it depends what you're talking about. Java and the JDK I'm ok with. C# I'm ok with, .Net framework im still learning.

### Post by Paul Miller on 2008-09-11
The other answers in this thread contain much truth, but I'd like to add a little of my own spin to the question.

I feel one has reached the point you're referring to for some language X when one can answer the question "Do I know how to accomplish nontrivial task Y in language X" in the affirmative a good deal of the time.  For example, in Python, I feel like I've reached this point.  So, I suppose I could say I'm "done" with learning Python and move on to, say, Scheme or Haskell; or, I could look for a job programming Python (I'm currently a grad student); or, I could learn one or several of the web frameworks available for Python; or, I could do what I'm currently doing and write a textbook on algorithms using Python; and so on.  

Really, the only concrete guideline I can give is this: what are your personal goals are in learning the language?  If you feel as if you've achieved them, then, I'd suggest moving on to another language.  If you'd like to get an even deeper understanding of your language, move on to something else involving the same language.

In my particular case, I feel as if I've learned Python well enough that I have something to teach other people.  So, that's what I've decided to do.
