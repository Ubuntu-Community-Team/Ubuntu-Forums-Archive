### Post by RIchard James13 on 2008-01-23
I think when you call a function that requires self as a parameter the system takes care of it so your code should look like.
[PHP]class Regex(token):
    def __init__(self, parser, parent):
        token.__init__( 'Regular Expression', parser)
[/PHP]

Unless of course the token class requires a reference back to the class it is being called from but I don't think this is the case because

Regex is called with token
Regex calls token

Therefore Regex has a token. A token most likely does not need a Regex.

When you are overriding a method from superclass (like in this case Regex.__init__ overrides token.__init__) you have to pass self as an parameter.

try:

[PHP]class A(object):
	def __init__(self, parameter):
		print "class A init with", parameter

class B(A):
	def __init__(self):
		A.__init__(self, "parameter")
		print "class B init"

B()[/PHP]

and change A.__init__(self, "parameter") to A.__init__("parameter") and you'll see it won't even work ;)

An alternative to this type of overriding is to call super(B, self).__init__("parameter")
