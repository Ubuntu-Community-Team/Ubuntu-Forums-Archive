### Post by Zugzwang on 2010-03-24
These are the steps to follow:
[LIST=1]
[*]Look at the documentation of the database you are using. Find out how it interfaces with the rest of the world and pick a proper way to achieve what you want.
[*]Check that you still think that you need a special program for that task and that the ones already existing are really insufficient for the task.
[*]Pick a suitable programming language. Suitability is defined wrt. what you know and the easy of implementing that particular task in it.
[*]Learn enough of the language to be able to start programming.
[*]Program it. If you don't know how, that you didn't follow the previous steps thoroughly enough.
[/LIST]

If you have further questions, please make sure that they are presented in an understandable way. For example, your request "... new database in postgis like we create new tables in the same way." does not make too much sense. You want to write a program to achieve something with databases that it already does with tables? That is logically unsound, so you probably mean something else.
