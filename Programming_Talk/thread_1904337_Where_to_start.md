### Post by HotCupOfJava on 2012-01-04
For starters, you just need to get an idea about what programming is like. There are a number of different languages you could get your feet wet with. A number of people have already suggested Python, and that is probably a good choice. You can learn about control structures, program flow, logical operations, and collections (the meat and potatoes of programming) without having to worry about a fussy compiler which is likely to frustrate a beginner. You also have the ability to try each line of your new code in an interpreter so you can see right away how it might behave. You will learn right away about proper code indentation for readability. Ruby would also be a very good choice for many of the same reasons. 

Once that is all said and done though, you will want to experience how a strongly typed, compiled language such as Java or C/C++ behaves. There are a large number of languages out there with syntax very similar to these as well. And while many Python and Ruby enthusiasts find them distasteful for very good reasons, understanding how these languages work will be invaluable to you.

If you haven't been completely scared away from programming at this point (and you still have that "want to learn" enthusiasm), you might get a bit more exotic and study one of the lesser-used but still quite powerful languages such as Lisp or Prolog. They represent very different ways of thinking about programming, but they can do some very amazing things.

### Post by N1GHTS on 2012-01-06
When I was young, I wanted to make video games. The internet was not really available to me so all I had as a model was the Nintendo Entertainment System. In an effort to make a game, since I didn't have a computer (I didn't have $5,000 to spare), I built my own from junk yard parts. Found this cool LCD matrix display, figured out how to make the lights tun on and off, and made a keyboard with aluminum foil and wires.. my CPU was an APPLE 6800(? cant remember the number exactly) cpu. Since I had no internet, I called up for them to mail me the datasheet. I also got a book on assembly language and digital electronic fundamentals. 

That was a great start for me. I finally got a hand-me-down 80286 IBM and became experienced in BBS's (the internet's grandfather) and learned to code in BASIC and eventually C, then C++. At this point in my life I know countless languages and know how to apply them in specific situations in my professional career.

In my opinion, to learn to program by choosing a first choosing a programming language is counter productive. It makes more sense to me to first learn how flip-flops work, how to clock a network of gates and buses, how microcode works with stacks and registers, and how to manage interrupt requests before you get into high level programming languages. It makes programming less magical, more functional.

Programming is about making your hardware do stuff, and usually you need to know what you want to accomplish before you ask what you need to make it happen. Knowing what you need takes insight and in turn good research. My professional advice is do not marry a programming language: be promiscuous. And once and a while you may even need to invent your own language to get a job done right (I've made many).
