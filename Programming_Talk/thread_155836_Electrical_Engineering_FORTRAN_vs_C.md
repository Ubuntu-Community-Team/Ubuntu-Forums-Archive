### Post by ekarni on 2006-04-05
Aero eng grad student here.  In my experience in industry (2.5 years of co-ops and internships), I've yet to run across a C program that I didn't write.  One place had a lot of old (1970's - early 80's) Fortran77 codes that were still in active use and development.  Knowing Fortran made life much easier there.  Otherwise, everything has been C++ or Matlab.  

A few thoughts:
-If you're expecting to work with large and/or old codes, go Fortran.  There are still plenty of senior people out there that use it for new programs, and it is still important for supercomputer apps and the like.
-If you're expecting to work with hardware (microcontrollers esp) and/or hardware drivers, definately learn C
-If you're expecting to work with modern software, find a way to take an object-oriented language like C++ or Java.  

Parting shot:  programming is programming.  When you're learning, it's the concepts that are critical to master, not the language.  You can always program with the reference guide on your lap or open in a browser (though they don't like that when you're taking tests!).  Once you've got the ideas down, jumping around between languages really is not too bad.  Object-oriented vs. top-down has been a slightly bigger jump for me, but still manageable.  Learn well, because as an engineer, you'll be programming all through school and very likely beyond.

### Post by rplantz on 2006-04-06
I got my BSEE in 1962. First job was designing a horizon sensor for the Gemini space craft. I used FORTRAN for circuit analysis in the main control loop.

Went back to grad school and used DEC's FOCAL (very much like BASIC) to write programs for analysing my data.

Decided that I really like programming. Worked several places using assembly language -- Data General Nova/Eclipse and IBM Series 1 minicomputers.

The last real EE job I had was in 1984. I wrote a log function in assembly language that could be called from C. It was for a spectrum analyser.

I think that C would put you in a position to work on embedded systems. FORTRAN would allow you to work with the legacy code that's out there. It sorta depends on what you prefer, which I realize may be difficult to decide at this point in your life.

What would look best on your resume is to take both classes. The second one should be easier and should reinforce the concepts you learned in the first one. Perhaps you could audit the second one.
