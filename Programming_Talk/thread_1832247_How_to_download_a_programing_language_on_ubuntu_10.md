### Post by nvteighen on 2011-08-24
But, do you understand what a programming language is, right? A programming language is a formal language that allows symbolic representation of a solution for some computable problem... In simpler terms: it's a language that you learn to "talk" about programs... You can't download a language, you download a program that interprets code written in a language!

A completely different thing is what you use to write code. A text editor like gEdit works for a beginner, IMO, but there are a couple of options. There are also so-called "IDEs", which are programs that integrate the interpreter or compiler with a text editor and allow you to have everything in "one place".

### Post by nvteighen on 2011-08-25
The biggest advantage of the terminal is that allows people to give advice and help more clearly...

Also, I'm amazed: everyone has missed the most important step: build-essential.

0. Ok, open a Terminal.

1. Type:
```

sudo apt-get update

```

This updates the software repository.

2. When it finished, type:
```

sudo apt-get install build-essential

```

For some reason, that package isn't installed by default in a Debian or Ubuntu desktop installation. The GNU C and C++ compilers are installed by default, but the needed headers and other stuff aren't, so without build-essential, you can't do anything at all.

It is likely that build-essential was installed by installing Eclipse or some of the others IDEs suggested to you. But we're not sure of that, so explicitly asking for it to be installed will solve our doubts ;)

3. Now, onto some testing. Seriously, I highly recommend you the gEdit + Terminal way because you should know the basics about compiling. It's funny: when it comes to programming languages, I'm a huge proponent of "first learn the higher-level, then the internal"; but when it comes to programming tools, I'm completely the opposite and usually try to disuade people of going the IDE-path when learning... :| Anyway, do what you want, but for the purposes of this post, please follow my way, so that we can actually test your programming environment.

4. Open gEdit and write this silly C++ program:
```

#include <iostream>

int main()
{
    std::cout << "Hi! I'm written in C++..." << std::endl;
    std::cout << "...and that's so sad that I wanna die" << std::endl;

    return 0;
}

```

Save it as *silly.cpp*. By the way, please notice that in the bottom panel you can choose gEdit's syntax highlighting; choose "C++" and see how nicely keywords are now recognized and the code becomes a lot more readable.

5. Open a Terminal, cd to the directory where silly.cpp is saved.

6. Now, type this:
```

g++ -o silly silly.cpp -Wall

```

This should be your C++ programming mantra. No matter if you automate it via Makefiles or using an IDE, that's all it takes to compile a module into an executable program.

The -o flag is used to set the oroduced executable's filename. If you don't set it, the executable will be named a.out by default, for historical reasons.

The -Wall flag activates all standard warnings. Please, always remember this: warnings are your friends. They may be annoying, but they point out problems that you should resolve because they would induce undefined behavior. Undefined behavior is the worst thing in programming: it means that you actually don't know what the program will do... it *may* or *may not* do what you expect, randomly.

There are other flags, but I guess you don't need them now.

7. To execute the program, which we have called *silly*:
```

./silly

```

8. Now a final kind advice. As I said, when it comes to programming languages, I usually suggest understanding what programming is by using a higher-level language that allows you to forget about platform-specific issues that are actually more about how our computers are built than about learning how to think when programming. I'd suggest you Python, Perl, Ruby, Javascript, some Lisp, etc. (Python would be fine, because you wouldn't need to install anythig, as you've been told).

Of course, you're free to choose whatever language you like more. And if you think C++ is fine for you, I can't forbid it. But what I do sincerely encourage you to do is that in the future you learn as most programming languages as possible. None programming language is the ideal and each one has its "place"; the more you know, the more your horizon will expand... It's like visiting different countries and different cultures: all programming languages sort of "impose" a certain mindset because of their built-in concepts. (You'll also notice that this will allow you to learn different languages much more easily).

You're just 12, you've got plenty of time to learn lots and lots of things :D

Good luck!

From the Printer Menu (Select) Print to File -> (Tick) PDF -> [Under Name] (Type) firstprogram.pdf -> [Under Save in folder] (Select) Desktop -> (Click) Print
-----------------------------------------------------------

After you have done those steps you will have a file, firstprogram.pdf on your desktop.  Click to bring it up.
