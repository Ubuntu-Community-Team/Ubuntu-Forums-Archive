### Post by pedrotuga on 2008-11-21
I agree with some opinions already stated.

To be honest I don't think gedit is non-professional or limited in any way. In fact it has a very powerful plugin support, using various plugins you can be really productive with gedit.
Check out the plugins menu. You can also google more esoteric plugins that are not officially supported.
Personally I use the code snippets like a maniac, it's great.

Other killer features: side panel with tree filesystem navigation and remote filesystem support.

> Most of the serious Linux programmers I know use either vim or emacs
No offense, but I really dislike this kind of statement and consider it to be a bit of a prejudice. What do you base that on? I mean, what makes vim or emacs users more serious programmers than others?

Personally I use vim and emacs on a daily basis, but the editor I still use the most is gedit. I believe that's a matter of preference.

Anyway, getting back to the main topic... more bloated IDEs do have some advantages:
-they write all the build stuff for you
-you can read documentation as you type
-some come with other stuff like visual form designers, uml stuf, etc. that will write a lot of code for you. Don't expect miracles though.

The only IDE I use is netbeans and i love it, but I only use it for java stuff. It has support for PHP, wich I still write a lot of, but i still prefer gedit for that purpose.

Bottom line, I am one of those guys that believes IDEs are overrated.
Hope I don't get stamped as a zealot.

### Post by Greyed on 2008-11-22
Your question isn't answerable.  IDE... for what language?

Remember, IDE stands for "Integrated Development Environment".  We cannot intelligently give you opinions on what the better/best IDE is until you tell us what it is supposed to be inetegrating *with*.

Eclipse******* or KDevelop for Python?   Not when compared to WingIDE.  On the other hand someone using WingIDE for C++ work would be out in the cold.

Personally I prefer vim for most of my work, WingIDE when I really need to get shoulder deep in the code, fall back on Komodo Edit in VI mode in places where I need more than vim but cannot install WingIDE and prefer Mercurial over SVN for my revision control.  However I am a semi-professional Python programmer who finds the distributed model of development natural.  I'm pretty sure you needs and tastes differ.  ;)


*******  Yes, I know there's a Python plugin for Eclipse.  But it doesn't do nearly as much as WingIDE does.  :P
