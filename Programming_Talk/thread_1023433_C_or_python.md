### Post by Grant A. on 2008-12-27
I would recommend learning Python, then C, and then finally C++. Python is much simpler than C, and C is more difficult than Python, but less difficult than C++, which is the most difficult of the 3.


I would personally recommend this Python book:

[Python Programming for the Absolute Beginner, Second Edition, by Michael Dawson]("http://www.amazon.com/Python-Programming-Absolute-Beginner-Second/dp/1598631128/ref=pd_bbs_sr_1?ie=UTF8&s=books&qid=1230436789&sr=8-1")


This is the C book I am currently learning from:

[C Programming for the Absolute Beginner, Second Edition, by Michael Vine]("http://www.amazon.com/C-Programming-Absolute-Beginner-Second/dp/1598634801/ref=sr_1_1?ie=UTF8&s=books&qid=1230436869&sr=1-1")

You might need Windows for the Python book in the latter stages, but the C book is fully UNIX/Linux compatible.

### Post by scooper on 2008-12-28
Definitely Python.   C/C++ were my career foundation languages, but I wouldn't touch them now if I didn't have to. :)

The problem with C/C++ for learning is that they provide many more opportunities to make mistakes, ones that are often hard to track down.  While such challenges can be "character-building" after acquiring skills, they are more likely to frustrate a beginner.

Ruby's also a fine language.  Java is good in its own way, but the libraries tend to be over-engineered and complex.  You can get lost in a thick tangle of objects and design patterns.  Nevertheless it can be simple and clean if you don't stray too far at first.

Good luck!
