### Post by Zugzwang on 2010-10-21
Two folders in a deb are absolutely no problem. You just follow some basic tutorial on .deb making and you'll see that that functionality comes without further consideration anyway (see, e.g., here: [http://www.ibm.com/developerworks/linux/library/l-debpkg.html](http://www.ibm.com/developerworks/linux/library/l-debpkg.html) - in the final step, you pack a whole file system hierarchy into your .deb package).

*But* if you want to extract something to the current user's home directory when installing the .deb file, then there is something conceptually fundamentally wrong with what you are trying to do - .deb files always contain system-wide stuff. Initial configuration files or the such should automatically be created at the first start of your program - how do you know that the current user is the one that wants to use your software? (Also, when installing a .deb file, the "current user" is typically "root", so you could only check who is logged in that the time of installation, but as already said, there's something wrong with what you are trying to do)
