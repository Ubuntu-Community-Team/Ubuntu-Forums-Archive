### Post by spupy on 2009-09-07
I want to set-up my website so that I can post simple content on it. Comments and similar stuff would be nice, but I can live without such functionality. My primary need is to easily put stuff online on my website. Said "stuff" is text without much formatting, almost no images, some links.
Also, I have my own server, so there are no hosting limitations.

As I see it, I have the following choices, with their pros and cons:
_**I. CMS (Joomla, Drupal, WordPress, etc...)**_
  **+** The bigger part of the work is installing.
  **+** Not much coding (if any at all).
  **-** Need to learn how to customize things ( I don't want some standard template everyone uses).
  **-** Waaaay over the top, too feature-rich for what I need.
_**II. Wiki (mediaWiki)**_
  **+** Easy installation.
  **+** Pages easy to create & edit.
  **-** Again, customizations look limited for me.
  **-** I don't need the multi-user features of a wiki.
_**III. Code something myself (Django).**_ Right now my mind is set on this. I have basic Django knowledge so I will be able to code the website I need in day or two, then spend some time doing the design.
  **+** Will be exactly as I want it, no extra cruft.
  **+** Customization is totally in my hands.
  **+** Will learn new stuff.
  **-** Requires coding and design (not really that much of a problem)
  **-** Somewhat harder installation.

Do you have any advises? Thanks for reading!
