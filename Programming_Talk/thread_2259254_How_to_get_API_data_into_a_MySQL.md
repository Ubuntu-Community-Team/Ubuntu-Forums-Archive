### Post by TheFu on 2015-01-03
Are you a C programmer?  If not, it will take about 6 months to get to the point where you can begin thinking about this problem and able to use the C API for MySQL.

Scripting is a better way - has NOTHING to do with the editor.  Scripts are programs that people write. You can write one too. mysql has APIs in many different languages - C, python, Ruby, Perl, and probably a few others. The C API is probably the least likely one you want to use.  If you say the exact API you are trying to use on the remote system (not the DB), perhaps there is a better way to do what you want?  APIs are highly specific to the program they are designed to work with, not a generic thing at all.

So ... basically I letting you know that you need lots of knowledge to do what you want. If you aren't a programmer, expect 2-3 months of work learning that to get to the point to start working on this.  Python is likely the best language to start with.  Programming isn't usually hard, it just takes practice and effort.
