### Post by k64 on 2010-01-30
There aren't that many **default** UI changes as of this moment. However, there will be later on. Case in point: RGBA gamma. Currently, you have to hack GTK+ to enable RGBA. In Lucid Lynx, it will be enabled by default.

If you're wondering, RGBA is window border transparency. You need Compiz to enable it plus a GTK+ library to get the full effect. However, the Lucid Lynx team is hacking GTK+ for you so you don't have to. It may be in Alpha 3, but Beta 1 is a more likely release date. That's when most of the UI changes typically occur.

### Post by Mr. Picklesworth on 2010-01-31
> **zekopeko said:**
> No it doesn't. Vista and 7 blur transparent parts so that it doesn't look like crap.

Thankfully for us all, that screenshot is just a screenshot that [OMG! Ubuntu! posted]("http://www.omgubuntu.co.uk/2009/12/lucid-to-get-aero-style-rgba.html") which has been taken out of context on these here forums regardless of its caption. (Although, in fairness, the article itself was pretty inaccurate). GTK+ has supported RGBA for a while, but it requires both the theme engine to support it (Murrine does; that out of context screenshot is an example of the Murrine theme in action) and each individual application to request it. For example, the system monitor enables RGBA for no apparent reason, so if you're using Impression (the single best GTK theme ever!) you'll notice it's semi-transparent.

As it is, themes do not do this remotely tastefully, and I don't think they're trying to. Basically everything that could possibly be transparent is made transparent - usually too much so, to boot, so it's difficult to read text. It's really just to say "hah, it works!"
It helps to enable Compiz's blur plugin (if you're using Compiz; I was somehow talked into it lately), since it magically blurs behind transparent parts of windows.

Now, the client-side window decorations patch:
Really cool idea to handle window decorations in each client application (via GTK+), instead of having the window manager do it. Window decorations are things like the title bar, close button and border (with resizing handle). That is all.
Windows does this. We can do it better. Enables lots of cool stuff: more sophisticated GTK themes, easier themeing, rounded corners on menus (maybe), giving windows some control over their title bars (though the amount of control we want to give them is up for debate). Reduces scope of window manager to something more in the realm of *managing* windows.

In order to do stuff like smooth corners, this demands enabling RGBA by default in each client (remember that it's currently disabled unless explicitly requested). A theme engine must also explicitly support client side window decorations and RGBA for this to work. With this all being expected behaviour at some point down the line, theme makers will be be putting much more thought into the alpha channel than they seem to at the moment.

Long story short, by default this will never look anything like that screenshot of Quod Libet, and in fact the RGBA part is more a side-effect than a direct ambition. Also, enabling RGBA in the toolkit level does not stop gnome-panel from doing fake transparency and making people mad at their computers.

Unfortunately it doesn't look like this patch will be landing in Lucid, but there is definitely lots of interest in it.
