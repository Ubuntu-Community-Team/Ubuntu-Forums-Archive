> I wonder how many command line only apps there are installed by default that never get used. No one seems to mention them.

Probably very few. Just because you may not type a command at a terminal window does not mean the command does not get used. Linux, like Unix, uses hundreds of command line programs behind the scenes to accomplish what the user sees and experiences. 

I bet most of the GUI programs you run every day are really just interfaces to various command line programs. I know Brasero is one of those programs. Brasero relies on a handful of command line programs that work behind the gui to accomplish what the user wants. Get rid of wodim or cdrdao because most users do not type those commands at a terminal window everyday and you will have a whole lot of unhappy campers that thought they were removing useless fluff that nobody needed.

Ubuntu, like those dummies series books, may have outgrown its usefulness for more experienced users. If you are a user complaining that Ubuntu is getting too bloated and is forcing you to install software you no longer want or use, then it may be time for you to move to another distribution that is better suited to your tastes. 

Ubuntu is geared towards new users and if you no longer fit the description of new user, then move to a distribution that targets your own level of expertise. 

However, Ubuntu is not just for new users and allows the user to customize the installation. So, as someone who is no longer a new user and does not need the hand-holding of Ubuntu, either shut-up and customize it the way you want and leave the default install to target new users, or move on to another distribution. :-)
