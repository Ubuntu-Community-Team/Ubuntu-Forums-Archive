### Post by dino99 on 2010-04-27
here is how to do:

what you need:

- prepare and organize your hdd(s): with windows you have to check hdd health and defrag, in case resize the partition. you can use free software if you need : [http://partedmagic.com/](http://partedmagic.com/)

- on the space not used by windows, you need to create several partitions for ease with linux:
 1) a main system partition named root: select / when you format it and select it as the bootable one, choose ext3 or ext4 as file system (about 12 Go is enough)
 2) a swap partition: select swap to format it (about twice your ram)
 3) a private partition for your work named /home with ext3 or ext4 choice.

to install lucid you need the "alternate" cd and make the choice, when asked, to select and format yourself (instead of the automatic installer) to apply 1) 2) 3) above

the errors you described previously was due to this important step, take time to format step by step with caution and validate each step one by one. Then installation can start, when it ask for bootloader (grub), install it on MBR of the main partition / (root),take care to not install it onto windows partition.

when installation is ended, reboot. You will see grub menu normally as you have a multi-boot. If you dont see it, system will boot with lucid. Grub menu, if hidden, can be shown if you press "shift" key at end of bios process.
To rebuild grub menu and have windows in it, into console, run:
sudo grub-mkconfig, then, update-grub

of course /home need room to store your data, the more the better, but you can let say only use 30 Go for example and the space not used now can be formated later to test a new distro or have a backup partition or else.
