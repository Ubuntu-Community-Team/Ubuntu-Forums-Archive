### Post by kansasnoob on 2010-03-30
Well looking at this:

> sda1: __________________________________________________ _______________________

File system: ntfs
[B]Boot sector type: Grub 2
Boot sector info: Grub 2 is installed in the boot sector of sda1[/B] and
looks at sector 404906155 of the same hard drive for
core.img, but core.img can not be found at this
location. No errors found in the Boot Parameter Block.
Operating System: Windows Vista
Boot files/dirs: /bootmgr /Boot/BCD /Windows/System32/winload.exe
/ubuntu/winboot/wubildr.mbr /ubuntu/winboot/wubildr

Makes me think Vista has a boot sector problem. Grub should almost never be installed to a partition rather than the mbr of a drive, certainly NOT to a Windows partition!

So look what Meierfra wrote here (I know he's referring to Win 7 but same applies to Vista):

[http://ubuntuforums.org/showpost.php?p=8929988&postcount=357](http://ubuntuforums.org/showpost.php?p=8929988&postcount=357)

Basically the same as here:

[http://support.microsoft.com/kb/927392](http://support.microsoft.com/kb/927392)

Once you've repaired the Vista bootsector you'll probably need to restore grub2 to the mbr of sda by booting your Ubuntu Live CD, choosing to Try without changes, then go to terminal and run the following:

```
sudo mount /dev/sda5 /mnt && sudo mount --bind /dev /mnt/dev &&sudo mount --bind /proc /mnt/proc && sudo chroot /mnt

```

```
grub-install /dev/sda
```

If that shows any errors:

```
grub-install --recheck /dev/sda
```

Then just:

```
exit
```

```
sudo umount /mnt/dev && sudo umount /mnt/proc && sudo umount /mnt
```

Then reboot and once you're booted into Ubuntu again run:

```
sudo update-grub
```

Wait for it to say done. Hopefully then you'll be able to boot both Windows and Ubuntu.
