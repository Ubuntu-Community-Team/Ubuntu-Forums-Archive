### Post by nanog on 2010-03-03
It certainly sounds like a bug. Have you tried running sudo update-grub (via chroot). I think that the earlier comment about device.map is a possibility. If you have a device mapped to a partition instead of the whole drive that would cause your problem. 

For raid1 /boot/grub/device.map should look something like this:

```

(hd0)   /dev/sda
(hd1)   /dev/sdb

```

Adjust drive names based on the output of ```
sudo fdsik -l
```.
