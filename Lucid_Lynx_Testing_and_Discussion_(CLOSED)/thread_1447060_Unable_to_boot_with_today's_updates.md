Run: nano /var/log/apt/history.log

Scroll to the bottom and you should see all the changes just made by the last update. The version number in parentheses shows the old version first and then the version it was replaced with.

It may be possible to remove the newer packages installed and then re-install the older packages.

What is forcing you to the recovery console? Is the screen garbled before you can even get to the log-in screen? If you can get to the log-in, can you load Failsafe Gnome?

One of the packages I got about two hours after you posted (which doesn't affect me because I have an ATI IGP) is nvidia-common. If your screen is getting garbled when gdm loads, that would be my first suspicion. The other packages that get my attention are the ones relating to plymouth. I haven't tried a re-boot since.

Have you tried looking for this problem on [launchpad]("https://launchpad.net/ubuntu")? Since you have access to the list of all packages updated last time, it should be fairly straightforward to search on each package to see if anyone else has reported something similar.

Apparently messing with Plymouth can trigger such a cascade now. [It's discussed in this thread]("http://ubuntuforums.org/showpost.php?p=9082096&postcount=604"). Run:

apt-get install ubuntu-desktop

Try that from the single user prompt. That's a metapackage that will pull in a bunch of stuff that will get you a gnome desktop so you can install anything that didn't get pulled in later from synaptic.

If you prefer using KDE, then run:

apt-get install kubuntu-desktop

Then run nano, gedit or kate to check /var/log/apt/history.log. Install what is still missing from the GUI. There may not be anything much left to worry about unless it's the proper configuration for the graphics driver.
