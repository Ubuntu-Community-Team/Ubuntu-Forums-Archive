### Post by kurros on 2010-03-29
I understand what you are asking for but the Gnome project's MO for years has been to deliver a consistent desktop experience. This is why the [Human Interface Guidelines]("http://library.gnome.org/devel/hig-book/stable/input-keyboard.html.en") define standard keyboard shortcut requirements such as Ctrl+Q to quit an application and Ctrl+W to close the window.

Additional keyboard shortcuts that Metacity et al. provides in the Keyboard Shortcuts control panel (such as Alt+F4) should not affect or steal the application's keyboard shortcuts.

I'm sure you understand that OS X works a bit differently due to its system menu bar and closing windows not actually killing the application. That usage pattern is not really translatable to the Gnome environment.
