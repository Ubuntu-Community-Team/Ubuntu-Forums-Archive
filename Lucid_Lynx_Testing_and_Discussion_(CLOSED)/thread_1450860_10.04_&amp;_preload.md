### Post by andrewabc on 2010-04-09
preload has not been fixed to work with ubuntu (didn't work in 9.10 either).

Yes it may slow down boot, but your most frequently used apps will start faster as they will be in ram on startup.

[recent thread about it](http://ubuntuforums.org/showthread.php?t=1434838)

[https://bugs.launchpad.net/ubuntu/+source/preload/+bug/299361](https://bugs.launchpad.net/ubuntu/+source/preload/+bug/299361)

since you have it installed and running, type
sudo tail -f /var/log/preload.log

and see what shows up. If bunch of errors then it still hasn't been fixed for ubuntu.

You have 2gb ram which is good for preload (1gb ram wouldn't help much since ubuntu+apps itself would use most of that). But you have SSD which should have low seek/latency timing, and maybe faster reads, than HDD which makes preload not as useful. Hard drives in netbooks are slow! thus loading into ram on startup would help start apps faster.
