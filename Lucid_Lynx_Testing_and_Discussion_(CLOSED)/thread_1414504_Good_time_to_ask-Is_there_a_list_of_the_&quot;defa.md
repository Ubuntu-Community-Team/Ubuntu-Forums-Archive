### Post by psyke83 on 2010-02-23
> **emarkay said:**

Not only so we all know a "Starting point", but it's been a while since I have seen one of these; would be nice to know what's "fluff" and what's essential.

One way to know what packages are installed by default would be to look at the metapackage "ubuntu-desktop", and see the related packages that are associated to it.

[http://packages.ubuntu.com/lucid/ubuntu-desktop](http://packages.ubuntu.com/lucid/ubuntu-desktop)

There's no direct upstream link to each piece of packaged software (which is not "equipment" - nor "manufactured", by the way ;)). However, Firefox can automate this quite easily when you highlight the text of the package name or description, right-click and choose "Search Google for <highlighted text>".

One last thought:

If you intend to test Lucid properly (that is, contribute in constructive ways, such as filing bug reports), in my opinion, there is an implicit agreement to test the system as it is presented in its default form. Removing packages that *you* think are unnecessary may have unintended consequences.

Here's an example - if you remove PulseAudio, don't file a bug report stating that the volume indicator applet no longer works correctly. Why? The applet may depend on the presence of PulseAudio to work, and filing a bug may waste developer/bug triager's resources, especially if you forget to mention in your bug report that you manually removed the PulseAudio packages.

Yes, there is potentially value in identifying problems as a result of certain programs removed. Another counter-argument is that you are removing yourself from the testing pool for the "fluff" software you remove, but that's not really a big deal.

There's something else you should keep in mind: during development, the metapackages (such as ubuntu-desktop) are in constant flux. If you remove a package marked as a depends (as opposed to a recommends), it will cause the metapackage to be removed from your system. Removing the metapackage won't break your system, but it does mean that your system will not install new packages that are assigned as depends or recommends to newer versions of the metapackage. In other words, your installed set of programs may not be consistent with everybody else's.
