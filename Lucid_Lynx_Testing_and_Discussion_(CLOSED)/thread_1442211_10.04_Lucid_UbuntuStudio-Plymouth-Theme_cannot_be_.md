### Post by Azd325 on 2010-03-30
Hey,
i had the same  bug and i have a solution!

You must delete the ubuntustudio plymouth files in /var/lib/dpkg/info/

```
cd /var/lib/dpkg/info/
```
```
ls | grep plymouth
```
Output:
```
libplymouth2.list
libplymouth2.md5sums
libplymouth2.postinst
libplymouth2.postrm
libplymouth2.shlibs
libplymouth2.symbols
libplymouth-dev.list
libplymouth-dev.md5sums
lubuntu-plymouth-theme.list
lubuntu-plymouth-theme.md5sums
lubuntu-plymouth-theme.postinst
lubuntu-plymouth-theme.postrm
plymouth.conffiles
plymouth.list
plymouth.md5sums
plymouth.postinst
plymouth.postrm
plymouth.preinst
plymouth-theme-ubuntu-logo.list
plymouth-theme-ubuntu-logo.md5sums
plymouth-theme-ubuntu-logo.postinst
plymouth-theme-ubuntu-logo.postrm
plymouth-theme-ubuntu-text.list
plymouth-theme-ubuntu-text.md5sums
plymouth-theme-ubuntu-text.postinst
plymouth-theme-ubuntu-text.postrm
plymouth-x11.list
plymouth-x11.md5sums
ubuntustudio-plymouth-theme.list
ubuntustudio-plymouth-theme.md5sums

ubuntustudio-plymouth-theme.postrm

```
then i delete the plymouth files of the ubuntustudio package
```
sudo rm ubuntustudio-plymouth-theme.*
```

Finisch! and it was able to update xD

I'm happy!

### Post by H13N.H3N on 2010-04-01
indeed i found the solution and is quite easy, just go to /lib/plymouth/themes and copy an existent theme, rename it as 'ubuntustudio-logo' and the same for the file that folder is storing to 'ubuntustudio-logo.plymouth' this way the fake file will be recognized as a valid theme and you'll work with no problems, i made this and i have no more errors.

edit:
 in your case something is quite different, i see you have "/usr/sbin/plymouth-set-default-theme" failed to remove, but in '/usr/sbin' those files does not exist, all of them are in /var/bin/plymouth-"-".* as far as i saw on my system, maybe you'll need to copy one of those files that are detected as missing in the directory that dpkg are trying to find it
