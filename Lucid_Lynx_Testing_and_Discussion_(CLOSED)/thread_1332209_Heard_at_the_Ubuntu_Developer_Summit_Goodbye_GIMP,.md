In trying to determine the cause of this disparity of **27.4Mb**, I examined the upstream [Debian package]("http://packages.debian.org/squeeze/gimp-data") and find that the Debian package includes the */usr/share/locale* directory which is not present in the Ubuntu package. I suspect it is more than just mere coincidence that the size of this */usr/share/locale* directory is precisely 27.4 megabytes. It would appear that the Ubuntu maintainers removed 27 megabytes worth of files from the Debian package, yet failed to consider the effect this might have on the resultant installed size of the package.

Perhaps this issue should be revisited employing *valid* data else Ubuntu will find that removing GIMP should not provide the benefits expected.

### Post by 23meg on 2009-11-24
> **saulgoode said:**
> Then I suspect that the Ubuntu developers have made an egregious error in determining the uncompressed disk usage of GIMP. 

The package information page for the [gimp-data package]("http://ns2.canonical.com/karmic/graphics/gimp-data") specifies that the "Installed size" is **33.9Mb**; yet when I extract the package contents and uncompress the archives, the resultant size is **6.5Mb**.

In trying to determine the cause of this disparity of **27.4Mb**, I examined the upstream [Debian package]("http://packages.debian.org/squeeze/gimp-data") and find that the Debian package includes the */usr/share/locale* directory which is not present in the Ubuntu package. I suspect it is more than just mere coincidence that the size of this */usr/share/locale* directory is precisely 27.4 megabytes. It would appear that the Ubuntu maintainers removed 27 megabytes worth of files from the Debian package, yet failed to consider the effect this might have on the resultant installed size of the package.

Perhaps this issue should be revisited employing *valid* data else Ubuntu will find that removing GIMP should not provide the benefits expected.

That disparity has to do with the build system calculating the package size *before* stripping out the translations, rather than afterwards. It's documented in [bug #451764]("https://bugs.launchpad.net/ubuntu/+source/pkgbinarymangler/+bug/451764").
