### Post by darkod on 2010-02-28
I'm not sure whether that can be the issue, but your grub.cfg file is beyond 137GB

167.6GB: boot/grub/grub.cfg

Some BIOSes can't boot beyond 137GB. The ubuntu root is /dev/sda1 and is at the start of the disk but somehow the particular boot file ended up all the way at 167GB mark.

If you want to eliminate this possibility, redo a manual install with /boot partition at start of the disk.
