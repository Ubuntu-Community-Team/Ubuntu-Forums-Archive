### Post by Pauly BC on 2010-02-16
I like Gnome DE because it is simple and fast by default, though easily modified using things like Conkey and Docky and Gnome-do to suit your needs. I don't like Gnome because it is a little bit old in default layout design and the integration of graphics effects is buggy for ATI owners. Looking at the early articles and screenshots it looks like Gnome 3.0 will be very nice.

I like KDE because it is fast and modern by default and still easily modified to suit your taste. I have not really tried out its ATI integration yet, but just booting off the CD it is better than Gnome. I have not tried 4.4 yet but the early reviews are all very positive.

I like Enlightenment DR17 (MoonOS) because it is eye candy to the limit. I have a hint of distrust for software crafted in Cambodia and downloaded from China, especially after last month's wide scale internet attacks. Plus it is a bear to learn your way around because the menu structure is a little scattered for my taste.

Unfortunately every time I install one version of Ubuntu and try to use the repositories and PPA's to install alternate DE's everything becomes unstable. Network, sound, graphics, dependencies, they all go wrong when I try to implement second or third DE's. Just browse through the forum and you will see I am not alone with this issue.

For the Lucid Lynx LTS version, can you release a distro of Ubuntu that uses a nice GDM at login where you can select Gnome, KDE, LXDE or Enlightenment for that session? Or alternatively, can you structure the repositories in a way that I can install Ubuntu with the default Gnome and easily add other DE's without losing stability? It is certainly not a hard drive space issue as my 500 GB can handle the extra load. If it is a server bandwidth issue, then do it via torrent or mirrors and repositories.
