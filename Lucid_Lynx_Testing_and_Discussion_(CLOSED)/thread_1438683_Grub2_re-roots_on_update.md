### Post by oldfred on 2010-03-25
These are not my comments but one's I copies from various posts where posters had issues with grub2 installing in the wrong drive.

Important for external removeable drives: Fixed 2010-02-04 for Lucid
[https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/496435](https://bugs.launchpad.net/ubuntu/+source/grub2/+bug/496435)
reinstalls grub and allows choice of which drive to install to. Choose boot drive if not sda
sudo dpkg-reconfigure grub-pc
I finally found out how to prevent the MBR to be overwritten.
I ran dpkg-reconfigure grub-pc and deselected /dev/sda which meant that the values field for grub-pc/install_devices in /var/cache/debconf/config.dat is now empty. Then nothing is written to the MBR or the boot sector when grub-pc gets updated. Occasionally I should probably rerun manually grub-install /dev/sda2 after grub-pc package updates to keep stage1 install in sync.

### Post by Ibidem on 2010-03-25
Quoting grub.cfg:
#
# It is automatically generated by /usr/sbin/grub-mkconfig using templates
# from /etc/grub.d and settings from /etc/default/grub
#

I suspect that you have two installs of GRUB2 in parallel; at update they probably refresh the MBR, pointing to the booted install (as the config files do).  It would take a great deal of monkey-work to make the two use the same configuration.  If you were to add another boot option (say, the current kernel with "single"), that should only show up in one version of grub2.  Upgrade the other, and it will disappear.

When grub.cfg is generated, the MBR is refreshed.
@Trev293: You can install "grub"--the system sorts stuff out  to a degree.
(That's grub Legacy)
