### Post by hedehodo on 2009-12-20
I think the word "linux" should be dropped out of the name as the software or the apps shouldnt be os dependant for this project to have a greater future. the collection of these apps maybe used by anyone having a macos x or windows in terms of freedom. however ofcourse the priority should be linux first, as linux lacks what the other oses have :)

Multimedia Studio
Creativity Studio
Design Studio

we can be inspired by openoffice.org idea

Open Design Studio (suggesting the free apps instead of using the word linux)

Open Creative Tools
Open Creativity Collection
Open Media Studio
Open Multimedia Studio
Open Media Tools

well I guess there can be a lot of names when you try to list them :)

but my main suggestion would be to make really simple icons and pictograms for the tools and plugins. i dont quite like the icons to be all colorful with reflections all over them. everything should be grayscale except the work we are working on. the app we are working on should make the design we are doing look good. it should be this simple sleek yet elegant frame. so the icons can be like the new icons designed for ubuntu's new osd panel pop up. or the mac os x sound or bluetooth icon. simple yet powerful (the hardest thing to do)I am not suggesting we should copy the style of something i am just pointing out how those examples work. I wouldnt want a copycat of another software as this new exciting project. 

there should also be something like adobe bridge which would really make the connection between those software.

one other important issue is that the integration with the system. the theme used in the system affects the way the design looks a lot. for example if all the icons are designed to look white on top of a dark gray background and the user has a system with a theme having the window color set as bright gray then the icons will not be visible enough or at least they will not look as intended. I dont know any technical backbone packages used in linux for window display purposes so i may say something that is impossible but it would be great to have an embedded window color information even a window type information. also the new fluent or the ribbon user interface used in some windows apps is imho successful. [Microsoft Office]("http://office.microsoft.com/en-us/products/HA101679411033.aspx")

of course if used wisely :) cause i dont really like to use microsoft office. but including the title bar in the interface increases the space to work on the screen which is very important for a designer. which can also be used as a tab system to switch in between different apps of this collection without disturbing the unified look and feel of apps as a whole.


well sorry for this very long post but ideas are never over when it comes to how to make linux better :)

cheers..
