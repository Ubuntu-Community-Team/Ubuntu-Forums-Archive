Here's one...

Kaprekar&#8217;s Constant.

Take any four-digit number except an integral multiple of 1111 (i.e., don&#8217;t take one of the nine numbers with four identical digits). Rearrange the digits of your number to form the largest and smallest strings possible. That is, write down the largest permutation of the number, the smallest permutation (allowing initial zeros as digits), and subtract. Apply this same process to the difference just obtained. Within the total of seven steps, you always reach 6174. At that point, further iteration with 6174 is pointless: 7641&#8211;1467 = 6174.

Example: Start with 8028. The largest permutation is 8820, the smallest is 0288, and the difference is 8532. Repeat with 8532 to calculate 8532&#8211;2358 = 6174. Your own example may take more steps, but you will reach 6174.

### Post by beercz on 2008-07-05
Here's another one.

Using mathematical operator and ***exactly*** four 4s it is possible to make a sequence of integers from 0 upwards.

The only rule is that you ***must*** use all four 4s (not 3, or 5).  If you use the square root operator you can assume that only the positive value is used.  e.g. the square root of 4 (SQRT(4)) is positive 2 (not negative 2).  The normal mathematical precedence applies.

Here are a few examples to get started, starting at 0:

0=4+4-4-4
1=4/4 * 4/4
2=4/4 + 4/4
3=(SQRT(4)+SQRT(4))-4/4
4=(SQRT(4)+SQRT(4))*4/4
5=(SQRT(4)+SQRT(4))+4/4
6=(4+SQRT(4))*4/4
7=(4+SQRT(4))+4/4
8=SQRT(4)+SQRT(4)+SQRT(4)+SQRT(4)
9=SQRT(4)*4+4/4
10=4+4+4-SQRT(4)

One can continue the sequence and get some really high values.  However, one or two of the values can be difficult to arrive at.

Okay I did a little search for the answer to this one... found this... it doesn't answer how multiplying even number with an even number equals an odd number, yet it does address the basic concept of the proofs and disproofs behind odds and evens. Here it is... (found in an excerpt of "Key Maths" by David Baker)...

An even number is a whole number that will divide exactly by 2. In other words, an even number has a factor of 2. If n is a whole number, then 2n must be even since it has a factor of 2. This is a useful result because if we want to write down a number that must be even then we can write 2n.

Every odd number can be written as an (even number) + 1 = 2n + 1.

You can use these basic results to prove more complex results about odd and even numbers.

Example
Prove that the sum of an even number and an odd number is always an odd number.
Some examples are 2 + 3 = 5 and 14 + 7 = 21 but to show that the rule is always true you need to use algebra.
Any even number can be written as 2a (where a is a whole number).
Any odd number can be written as 2b+1 (where b is a whole number).
Any even number plus an odd number can now be written as
2a + 2b + 1 = 2(a + b) + 1
But a + b is a whole number so 2(a + b) + 1 must be an odd number.

Here's one...

Kaprekar&#8217;s Constant.

Take any four-digit number except an integral multiple of 1111 (i.e., don&#8217;t take one of the nine numbers with four identical digits). Rearrange the digits of your number to form the largest and smallest strings possible. That is, write down the largest permutation of the number, the smallest permutation (allowing initial zeros as digits), and subtract. Apply this same process to the difference just obtained. Within the total of seven steps, you always reach 6174. At that point, further iteration with 6174 is pointless: 7641&#8211;1467 = 6174.

Example: Start with 8028. The largest permutation is 8820, the smallest is 0288, and the difference is 8532. Repeat with 8532 to calculate 8532&#8211;2358 = 6174. Your own example may take more steps, but you will reach 6174.
